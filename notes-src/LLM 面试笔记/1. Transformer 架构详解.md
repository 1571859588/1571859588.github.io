## 1. 整体架构

Transformer 由 **Encoder** 和 **Decoder** 两部分堆叠而成。原始论文中各有 $N=6$ 层。

核心组件包括：

- **Multi-Head Self-Attention**：让每个 token 关注序列中所有其他 token
- **Feed-Forward Network (FFN)**：位置独立的两层 MLP
- **Layer Normalization**：稳定训练，加速收敛
- **Residual Connections**：缓解深层网络的梯度消失

## 2. Self-Attention 机制

Self-Attention 的核心公式：

\[
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
\]

其中 $Q$（Query）、$K$（Key）、$V$（Value）都是从输入 $X$ 通过线性变换得到的：

\[
Q = XW^Q, \quad K = XW^K, \quad V = XW^V
\]

### 2.1 为什么除以 $\sqrt{d_k}$？

当 $d_k$ 较大时，$QK^T$ 的点积值会变得很大，导致 softmax 梯度进入饱和区。除以 $\sqrt{d_k}$ 将方差控制为 1，使梯度保持健康。

## 3. Multi-Head Attention

多个注意力头并行计算，每个头关注不同的表示子空间：

\[
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O
\]

\[
\text{head}_i = \text{Attention}(QW^Q_i, KW^K_i, VW^V_i)
\]

### 3.1 为什么需要 Multi-Head？

- **不同子空间**：不同头可以关注不同位置/关系（语法、语义、位置等）
- **证据**：论文中头数 $h=8$，$d_k = d_v = d_{\text{model}} / h = 64$

## 4. Position-wise FFN

每个位置独立应用相同的两层全连接：

\[
\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2
\]

内部维度通常设为 $d_{ff} = 4 \times d_{\text{model}} = 2048$。

> FFN 充当了**键值存储（key-value memory）**的角色：Attention 负责 token 间的信息交换，FFN 负责每个 token 内部的非线性变换和记忆存储。

## 5. 位置编码

由于 Self-Attention 是排列不变的，需要注入位置信息。原始论文使用正弦位置编码：

\[
PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)
\]

\[
PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)
\]
