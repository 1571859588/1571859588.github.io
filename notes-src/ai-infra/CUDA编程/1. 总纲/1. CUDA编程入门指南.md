## 三条主线贯穿全程

阅读所有笔记时，请时刻记住这三条主线：

### 主线 1：Memory-Bound vs Compute-Bound

```
                算术强度（FLOP/Byte）
                       │
                       ▼
   低（< 平衡点）              高（> 平衡点）
   ↓                            ↓
Memory-Bound                  Compute-Bound
代表：Reduce、Softmax          代表：GEMM
优化：减少内存访问             优化：提升算术强度
      提高带宽利用率                 增加 Tiling 复用
工具：ncu 看 SOL Memory        工具：ncu 看 SOL Compute
```

### 主线 2：数据搬运层次

```
Global Memory ──Block 协作加载──► Shared Memory ──Thread 加载──► Register
   (慢，几百 cycles)              (中，几十 cycles)            (快，1 cycle)
                          Tiling                     寄存器外积
```

所有优化算法都是在这条链上做文章：减少 Global 访问、增加 Shared/Register 复用。

### 主线 3：抽象层次与开发效率

```
手写 CUDA ──── 性能 100%，开发 1x
   ↑
Triton    ──── 性能 80-95%，开发 5-10x
   ↑
torch.compile ─ 性能 70-90%，开发 10x+（零代码改动）
   ↑
PyTorch eager ─ 性能 50-70%，开发 20x
```

**核心权衡**：越往下性能越好但开发越慢。新硬件（TMA、Async Copy）总是先在底层暴露，编译器滞后。
