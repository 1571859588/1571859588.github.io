## 一句话结论

Softmax 是典型 Memory-Bound 算子（算术强度低），优化核心是**减少全局内存访问次数**和**提升带宽利用率**。从 V0 单线程处理一行的 5% 带宽，到 V4 两遍融合的 75% 带宽，四步优化覆盖了 Block 级并行规约、Warp Shuffle 寄存器级规约、float4 向量化加载、Pass 融合四大套路。进一步突破 75% 需要 Online Softmax 算法（下一篇主题）。

## 核心概念（新人友好讲解）

### 1. Softmax 是什么（生活类比）

**类比**：期末考试改卷。你有一堆学生的原始分（任意实数，可能很高也可能很低），想把他们转成"占比"——高分者占比大、低分者占比小、所有占比加起来恰好等于 1。Softmax 就是干这件事的函数，把一组任意实数"压缩"成一个概率分布。

**数学定义**：对输入向量 $\mathbf{x} = (x_0, x_1, \ldots, x_{N-1})$：

$$
\text{Softmax}(x_i) = \frac{e^{x_i}}{\sum_{j=0}^{N-1} e^{x_j}}
$$

分子是当前元素的指数，分母是所有元素指数之和。在 Transformer Self-Attention 中，Softmax 作用于 $QK^T$ 的每一行，把注意力分数转成归一化权重。序列长度 $N$ 通常 2048 / 4096 / 8192 甚至 128K，Softmax 效率直接影响模型整体性能。

### 2. 数值溢出问题（为什么需要 Safe Softmax）

Softmax 公式里有 $e^{x_i}$，这在浮点数下极易溢出：

| 数据类型 | 最大可表示值 | 对应 $e^{x}$ 的 $x$ 上限 | 超过后 |
|---------|-------------|------------------------|--------|
| FP32 | $3.4 \times 10^{38}$ | $x > 88.7$ 时 $e^x$ 溢出 | 变 `+inf` |
| FP16 | $6.55 \times 10^4$ | $x > 11.09$ 时 $e^x$ 溢出 | 变 `+inf` |
| 下溢 | — | $x = -100$ 时 $e^x \approx 0$ | 分母为 0 → `NaN` |

在 Attention 中 $QK^T$ 的值很容易超过 88（大 $d_k$ 不缩放时），朴素实现直接崩溃。

### 3. 减最大值技巧（Safe Softmax 核心）

**数学证明**：对每个元素减同一个常数 $c$，Softmax 输出不变：

$$
\frac{e^{x_i - c}}{\sum_j e^{x_j - c}} = \frac{e^{x_i} \cdot e^{-c}}{\sum_j e^{x_j} \cdot e^{-c}} = \frac{e^{x_i}}{\sum_j e^{x_j}}
$$

分子分母的 $e^{-c}$ 约掉了。选 $c = \max(\mathbf{x})$ 后，所有指数参数 $\leq 0$，结果落在 $(0, 1]$ 之间，彻底消除上溢。这就是 **Safe Softmax**。

### 4. 三遍扫描（Three-Pass）流程

Safe Softmax 自然分解为三次扫描：

```
┌─────────────────────────────────────────────────────────┐
│  Pass 1: 求行最大值   m = max(x₀, x₁, ..., x_{N-1})     │
│            ↓                                            │
│  Pass 2: 指数求和     d = Σ exp(xⱼ - m)                 │
│            ↓                                            │
│  Pass 3: 归一化       yᵢ = exp(xᵢ - m) / d              │
└─────────────────────────────────────────────────────────┘
```

| 遍次 | 操作 | 数学表达 |
|------|------|---------|
| 第 1 遍 | 求行最大值 | $m = \max_j(x_j)$ |
| 第 2 遍 | 指数求和 | $d = \sum_j e^{x_j - m}$ |
| 第 3 遍 | 归一化 | $y_i = e^{x_i - m} / d$ |

每遍都要读一遍整行数据，总共 $3N$ 次全局内存读取。Softmax 算术强度极低（每次读取对应 1-2 次浮点运算），是典型的 **Memory-Bound**，优化目标是**减少内存访问**和**提升带宽利用率**。

### 5. 关键概念速查表（新人必看）

| 概念/术语 | 通俗解释 | 类比 |
|---------|---------|------|
| **Softmax** | 把任意实数转成概率分布（和为 1） | 改卷后算占比 |
| **Safe Softmax** | 减最大值后再算 exp，防溢出 | 改卷前先减去最高分 |
| **max-subtract trick** | $e^{x_i - m}$ 替代 $e^{x_i}$ | 同上 |
| **expf** | CUDA 的单精度指数函数 `expf(x)` | 数学库函数，比 `exp` 快 |
| **fmaxf** | 单精度取最大值，比 `fmax` 快 | — |
| **三遍扫描** | max → sum → normalize，读 3 遍输入 | 翻 3 次花名册 |
| **两遍扫描** | 融合 exp-sum 与 normalize，读 2 遍 | 翻 2 次花名册 |
| **Block 级归约** | 一个 Block 协作处理一行，SMEM 做规约 | 一组人协作改一行 |
| **Warp Shuffle 归约** | `__shfl_down_sync` 寄存器间交换数据 | 同一小组内传纸条 |
| **float4 向量化** | 一条指令加载 4 个 float（16B） | 一次搬 4 箱货 |
| **Pass 融合** | 把两遍扫描合成一遍，减少一次读取 | 一次性做完两步 |
| **Memory-Bound** | 算术强度低，瓶颈在内存带宽 | 搬运工而非计算员 |

### V0：朴素实现（单线程处理一行）

最简单的并行化：每个线程独立处理一行，串行完成 max / exp-sum / normalize 三步。行与行之间天然并行。

```cuda
// V0: 朴素实现，一个线程处理一行
__global__ void softmax_v0(float* input, float* output, int M, int N) {
    int row = blockIdx.x * blockDim.x + threadIdx.x;
    if (row >= M) return;

    float* x = input  + row * N;
    float* y = output + row * N;

    // Pass 1: 求最大值
    float max_val = -INFINITY;
    for (int i = 0; i < N; i++) {
        max_val = fmaxf(max_val, x[i]);
    }

    // Pass 2: 指数求和
    float sum = 0.0f;
    for (int i = 0; i < N; i++) {
        sum += expf(x[i] - max_val);
    }

    // Pass 3: 归一化
    float inv_sum = 1.0f / sum;
    for (int i = 0; i < N; i++) {
        y[i] = expf(x[i] - max_val) * inv_sum;
    }
}
```

**三大性能问题**：
1. **并行度不足**：一行 $N=4096$ 个元素只用 1 个线程串行处理，GPU 数千 CUDA Core 大量空闲
2. **非合并访存**：同一 Warp 内 32 线程处理 32 个不同行，内循环访问地址相差 $N \times 4$ 字节，严重非合并
3. **冗余计算**：`expf(x[i] - max_val)` 在 Pass 2 和 Pass 3 重复计算两次

**实测带宽利用率：约 5%（~102 GB/s）**——能跑通但不可用。

### V1：Safe Softmax 三遍扫描（Block 级并行规约）

**改进思路**：让一个 Block 内多个线程协作处理同一行。每线程负责一段数据，通过 Shared Memory 做并行规约求 max 和 sum。相当于把 Softmax 拆成"两次 Reduce（max-reduce + sum-reduce）+ 一次 element-wise（normalize）"。

```cuda
// V1: 一个 Block 处理一行，Block 内并行规约
__global__ void softmax_v1(float* input, float* output, int M, int N) {
    extern __shared__ float smem[];  // 用于规约

    int row = blockIdx.x;
    int tid = threadIdx.x;

    float* x = input  + row * N;
    float* y = output + row * N;

    // Pass 1: 并行求最大值
    float max_val = -INFINITY;
    for (int i = tid; i < N; i += blockDim.x) {
        max_val = fmaxf(max_val, x[i]);
    }
    smem[tid] = max_val;
    __syncthreads();

    // Shared Memory 规约求全局最大值
    for (int s = blockDim.x / 2; s > 0; s >>= 1) {
        if (tid < s) {
            smem[tid] = fmaxf(smem[tid], smem[tid + s]);
        }
        __syncthreads();
    }
    max_val = smem[0];
    __syncthreads();

    // Pass 2: 并行求指数和
    float sum = 0.0f;
    for (int i = tid; i < N; i += blockDim.x) {
        sum += expf(x[i] - max_val);
    }
    smem[tid] = sum;
    __syncthreads();

    // Shared Memory 规约求总和
    for (int s = blockDim.x / 2; s > 0; s >>= 1) {
        if (tid < s) {
            smem[tid] += smem[tid + s];
        }
        __syncthreads();
    }
    sum = smem[0];
    __syncthreads();

    // Pass 3: 归一化写出
    float inv_sum = 1.0f / sum;
    for (int i = tid; i < N; i += blockDim.x) {
        y[i] = expf(x[i] - max_val) * inv_sum;
    }
}
```

**调用方式**：

```cuda
int block_size = 256;
int smem_size = block_size * sizeof(float);
softmax_v1<<<M, block_size, smem_size>>>(d_input, d_output, M, N);
```

**对比 V0 的提升**：
- 并行度：1 线程/行 → 256 线程/行，每线程只处理 $N/256 = 16$ 个元素
- 合并访存：同 Block 相邻线程访问行内相邻地址（stride=1），完美合并

**实测带宽利用率：约 35%（~714 GB/s）**，相比 V0 提升约 7 倍。

### V2：Warp Shuffle 优化规约

**改进思路**：V1 规约完全依赖 Shared Memory，路径偏长（寄存器→SMEM→寄存器），每轮还要 `__syncthreads()`。改用"Warp 内 Shuffle + Warp 间 Shared Memory"两级结构：Warp 内用 `__shfl_down_sync` 寄存器直通，Warp 间用 32 元素的 SMEM 数组做最终规约。

```cuda
// Warp 内规约：Max
__device__ float warpReduceMax(float val) {
    for (int offset = 16; offset > 0; offset >>= 1) {
        val = fmaxf(val, __shfl_down_sync(0xffffffff, val, offset));
    }
    return val;
}

// Warp 内规约：Sum
__device__ float warpReduceSum(float val) {
    for (int offset = 16; offset > 0; offset >>= 1) {
        val += __shfl_down_sync(0xffffffff, val, offset);
    }
    return val;
}

// 两级规约：Block 内 Max
__device__ float blockReduceMaxShuffle(float val) {
    __shared__ float warp_max[32];

    int lane = threadIdx.x % 32;
    int wid  = threadIdx.x / 32;

    val = warpReduceMax(val);

    if (lane == 0) warp_max[wid] = val;
    __syncthreads();

    int num_warps = blockDim.x / 32;
    val = (lane < num_warps) ? warp_max[lane] : -INFINITY;
    if (wid == 0) val = warpReduceMax(val);

    // 广播结果给所有线程
    __shared__ float block_result;
    if (threadIdx.x == 0) block_result = val;
    __syncthreads();
    return block_result;
}

// 两级规约：Block 内 Sum
__device__ float blockReduceSumShuffle(float val) {
    __shared__ float warp_sum[32];

    int lane = threadIdx.x % 32;
    int wid  = threadIdx.x / 32;

    val = warpReduceSum(val);

    if (lane == 0) warp_sum[wid] = val;
    __syncthreads();

    int num_warps = blockDim.x / 32;
    val = (lane < num_warps) ? warp_sum[lane] : 0.0f;
    if (wid == 0) val = warpReduceSum(val);

    __shared__ float block_result;
    if (threadIdx.x == 0) block_result = val;
    __syncthreads();
    return block_result;
}

// V2: Warp Shuffle 两级规约
__global__ void softmax_v2(float* input, float* output, int M, int N) {
    int row = blockIdx.x;
    int tid = threadIdx.x;

    float* x = input  + row * N;
    float* y = output + row * N;

    // Pass 1: 求行最大值
    float local_max = -INFINITY;
    for (int i = tid; i < N; i += blockDim.x) {
        local_max = fmaxf(local_max, x[i]);
    }
    float max_val = blockReduceMaxShuffle(local_max);

    // Pass 2: 求指数和
    float local_sum = 0.0f;
    for (int i = tid; i < N; i += blockDim.x) {
        local_sum += expf(x[i] - max_val);
    }
    float sum = blockReduceSumShuffle(local_sum);

    // Pass 3: 归一化
    float inv_sum = 1.0f / sum;
    for (int i = tid; i < N; i += blockDim.x) {
        y[i] = expf(x[i] - max_val) * inv_sum;
    }
}
```

**为什么 Warp Shuffle 更快**：

| 对比维度 | Shared Memory 规约 | Warp Shuffle 规约 |
|---------|-------------------|-------------------|
| 数据路径 | 寄存器 → SMEM → 寄存器 | 寄存器 → 寄存器 |
| Bank Conflict | 可能存在 | 不存在 |
| 同步开销 | 每轮 `__syncthreads()` | Warp 内天然同步 |
| 延迟 | ~20-30 cycles | ~5 cycles（约 1/4~1/6） |

> 💡 Shuffle 延迟约为 SMEM 的 1/4~1/6，且无 `__syncthreads()`。规约后期线程数少时，这种节省尤为显著。

**实测带宽利用率：约 52%（~1060 GB/s）**，相比 V1 提升约 1.5 倍。

### V3：向量化加载 float4

**改进思路**：V2 瓶颈转移到全局内存加载。每循环每线程只加载 1 个 float（4B），指令调度开销占比偏高。用 `float4` 一次加载 16B（4 个 float），减少 4 倍循环迭代，提升内存事务利用率，增加指令级并行（ILP）。

```cuda
// V3: float4 向量化加载
__global__ void softmax_v3(float* input, float* output, int M, int N) {
    int row = blockIdx.x;
    int tid = threadIdx.x;

    float* x = input  + row * N;
    float* y = output + row * N;

    // 向量化指针
    float4* x4 = reinterpret_cast<float4*>(x);
    float4* y4 = reinterpret_cast<float4*>(y);
    int N4 = N / 4;  // float4 元素数

    // Pass 1: 向量化求最大值
    float local_max = -INFINITY;
    for (int i = tid; i < N4; i += blockDim.x) {
        float4 data = x4[i];
        local_max = fmaxf(local_max, fmaxf(fmaxf(data.x, data.y),
                                            fmaxf(data.z, data.w)));
    }
    // 处理尾部元素（N 不是 4 的倍数时）
    for (int i = N4 * 4 + tid; i < N; i += blockDim.x) {
        local_max = fmaxf(local_max, x[i]);
    }
    float max_val = blockReduceMaxShuffle(local_max);

    // Pass 2: 向量化求指数和
    float local_sum = 0.0f;
    for (int i = tid; i < N4; i += blockDim.x) {
        float4 data = x4[i];
        local_sum += expf(data.x - max_val) + expf(data.y - max_val)
                   + expf(data.z - max_val) + expf(data.w - max_val);
    }
    for (int i = N4 * 4 + tid; i < N; i += blockDim.x) {
        local_sum += expf(x[i] - max_val);
    }
    float sum = blockReduceSumShuffle(local_sum);

    // Pass 3: 向量化归一化
    float inv_sum = 1.0f / sum;
    for (int i = tid; i < N4; i += blockDim.x) {
        float4 data = x4[i];
        float4 result;
        result.x = expf(data.x - max_val) * inv_sum;
        result.y = expf(data.y - max_val) * inv_sum;
        result.z = expf(data.z - max_val) * inv_sum;
        result.w = expf(data.w - max_val) * inv_sum;
        y4[i] = result;
    }
    for (int i = N4 * 4 + tid; i < N; i += blockDim.x) {
        y[i] = expf(x[i] - max_val) * inv_sum;
    }
}
```

> ⚠️ float4 要求 16 字节对齐。`cudaMalloc` 自动 256 字节对齐，满足要求；同时确保 $N$ 是 4 的倍数（否则用尾部循环兜底）。

**实测带宽利用率：约 65%（~1325 GB/s）**，相比 V2 提升约 1.25 倍。

### V4：两遍融合 Kernel

**改进思路**：三遍扫描读了 3 次输入。把 Pass 2（exp-sum）和 Pass 3（normalize）部分融合——第 2 遍中每线程把自己负责的 $e^{x_i - m}$ 暂存到 Shared Memory，求出全局 $d$ 后立即从 SMEM 读取并写出归一化结果，避免第 3 次全局读取。

```cuda
// V4: 两遍 Kernel（Pass 1 求 max，Pass 2 融合 exp-sum + normalize）
// 需要额外 Shared Memory 暂存 exp 值
__global__ void softmax_v4(float* input, float* output, int M, int N) {
    extern __shared__ float smem[];  // 大小 = N（暂存 exp 值）

    int row = blockIdx.x;
    int tid = threadIdx.x;

    float* x = input  + row * N;
    float* y = output + row * N;

    float4* x4 = reinterpret_cast<float4*>(x);
    int N4 = N / 4;

    // Pass 1: 求行最大值（向量化 + Warp Shuffle）
    float local_max = -INFINITY;
    for (int i = tid; i < N4; i += blockDim.x) {
        float4 data = x4[i];
        local_max = fmaxf(local_max, fmaxf(fmaxf(data.x, data.y),
                                            fmaxf(data.z, data.w)));
    }
    for (int i = N4 * 4 + tid; i < N; i += blockDim.x) {
        local_max = fmaxf(local_max, x[i]);
    }
    float max_val = blockReduceMaxShuffle(local_max);

    // Pass 2: 计算 exp 并暂存，同时累加求 sum
    float local_sum = 0.0f;
    for (int i = tid; i < N; i += blockDim.x) {
        float exp_val = expf(x[i] - max_val);
        smem[i] = exp_val;  // 暂存到 Shared Memory
        local_sum += exp_val;
    }
    float sum = blockReduceSumShuffle(local_sum);

    // 直接从 Shared Memory 读取 exp 值并归一化写出
    float inv_sum = 1.0f / sum;
    for (int i = tid; i < N; i += blockDim.x) {
        y[i] = smem[i] * inv_sum;
    }
}
```

**适用场景限制**：

> ⚠️ V4 需要 $N \times 4$ 字节 Shared Memory 暂存 exp 值。$N=4096$ 时需 16KB（A100 每 SM 164KB SMEM，够用）；$N \geq 32768$ 时 SMEM 装不下，需改用 Online Softmax（下一篇）。

| 适用场景 | 推荐方案 |
|---------|---------|
| $N \leq 4096$（SMEM 放得下） | V4 两遍融合 |
| $N > 4096$（SMEM 不够） | V3 三遍扫描，或 Online Softmax |

**实测带宽利用率：约 75%（~1529 GB/s）**（$N=4096$），相比 V3 提升约 1.15 倍。

## 面试回答（可直接口述的 30-60 秒版本）

**问：请描述 CUDA Softmax 算子的优化过程。**

答：Softmax 是 Memory-Bound 算子，优化核心是减少内存访问和提升带宽利用率。首先数值上要用 Safe Softmax——先减最大值再算 exp，防止 $e^x$ 溢出，这天然形成三遍扫描：求 max、求 sum、归一化。V0 单线程处理一行只有 5% 带宽，问题在并行度低和非合并访存；V1 让一个 Block 256 个线程协作处理一行，用 Shared Memory 做 max-reduce 和 sum-reduce，带宽升到 35%；V2 把规约改成 Warp Shuffle 两级结构，`__shfl_down_sync` 寄存器直通无 Bank Conflict、无 `__syncthreads()`，延迟只有 SMEM 的 1/4，带宽升到 52%；V3 用 float4 向量化加载，一次 16B 减少指令数，带宽升到 65%；V4 把 Pass 2 和 Pass 3 融合，把 exp 值暂存到 SMEM 避免第 3 次全局读取，带宽升到 75%。再往上需要 Online Softmax 从算法层面把三遍降成两遍，那是 FlashAttention 的数学基础。

## 深入追问（可能被追问的点）

- **Q1：减最大值为什么不改变 Softmax 结果？**
  A：分子分母同时乘 $e^{-c}$，约掉了。数学上 $\frac{e^{x_i - c}}{\sum_j e^{x_j - c}} = \frac{e^{x_i} \cdot e^{-c}}{\sum_j e^{x_j} \cdot e^{-c}} = \frac{e^{x_i}}{\sum_j e^{x_j}}$。选 $c = \max(\mathbf{x})$ 后所有指数参数 $\leq 0$，结果落在 $(0, 1]$，防溢出。

- **Q2：FP16 下 Safe Softmax 还有问题吗？**
  A：减最大值后指数参数 $\leq 0$，$e^{x_i - m} \in (0, 1]$，单次不会溢出。但求和时如果 $N$ 很大，$\sum e^{x_i - m}$ 可能超过 FP16 的 65504 上限。工程上常用 FP32 累加、最后转 FP16，或在 warp 内用 FP32 做 reduce。

- **Q3：V0 的非合并访存为什么严重？**
  A：同一 Warp 的 32 线程处理 32 个不同行，内循环里它们访问的地址相差 $N \times 4$ 字节。GPU 一次内存事务加载 128B 连续段，这种跨步访问让每条指令触发 32 个独立事务，带宽利用率跌到几十分之一。

- **Q4：V2 两级规约为什么 Warp 间还要用 SMEM？**
  A：Warp Shuffle 只能在 Warp 内（32 线程）交换数据，跨 Warp 不行。所以每个 Warp 先在内部 Shuffle 出 1 个结果，再把 8 个 Warp（blockDim=256）的结果写到 32 元素的 SMEM 数组，由 Warp 0 做最终规约。

- **Q5：V4 把 exp 暂存到 SMEM，为什么不能直接存寄存器？**
  A：每线程要处理 $N/\text{blockDim} = 16$ 个元素，存 16 个 float 需要 16 个寄存器。看上去不多，但归约阶段也需要寄存器，加上控制流和中间变量，容易触发寄存器溢出（spill 到 Local Memory）。SMEM 容量大（164KB/SM）、寻址灵活，更适合暂存整行数据。真正用寄存器缓存的方案是 Online Softmax V3（下篇）。

- **Q6：V4 的 Shared Memory 需求怎么算？**
  A：$N \times 4$ 字节。$N=4096$ 时需 16KB。A100 每 SM 164KB，按 2 Block/SM 驻留算可用 82KB，16KB 完全够。但 $N=32768$ 时需 128KB，单 Block 就吃掉大部分 SMEM，Occupancy 严重下降，得换 Online Softmax。

- **Q7：为什么不用 `exp` 而用 `expf`？**
  A：`exp` 是双精度（double）版本，GPU 上双精度性能远低于单精度（A100 FP32:FP64 = 2:1，消费卡更低）。`expf` 是单精度版本，配合 `fmaxf`、`fminf` 等 `f` 后缀函数，全程 FP32 计算。

- **Q8：V4 融合 Pass 2 和 Pass 3，为什么 Pass 1 不能也融合？**
  A：Pass 2 的 `expf(x[i] - max_val)` 必须先知道全局 max，而 max 要等 Pass 1 全部读完才能算出来。这就是三遍扫描的本质矛盾——Pass 2 依赖 Pass 1 的结果。要打破这个矛盾需要 Online Softmax 的递推技巧（边找 max 边算 sum），下篇详解。

## 易混淆点对比（表格）

| 易混概念 | 区别 | 记忆技巧 |
|---------|------|----------|
| 朴素 Softmax vs Safe Softmax | 前者直接 $e^{x_i}$ 会溢出，后者减 max 防溢出 | Safe = 减 max |
| 三遍 vs 两遍 vs 一遍扫描 | 3 次读输入 / 2 次 / 1 次 | 数字 = 读取次数 |
| Shared Memory 规约 vs Warp Shuffle 规约 | 前者经 SMEM 有 Bank Conflict，后者寄存器直通 | Shuffle 更快 |
| V1 SMEM 规约 vs V2 两级 Shuffle 规约 | V1 全程 SMEM，V2 先 Warp 内 Shuffle 再 Warp 间 SMEM | V2 两级 |
| V3 float4 vs V0 float | 16B/指令 vs 4B/指令，指令数减 75% | float4 = 4 倍 |
| V3 三遍 vs V4 两遍 | V3 读 3 次输入，V4 用 SMEM 暂存 exp 只读 2 次 | V4 少读 1 次 |
| expf vs exp | 单精度 vs 双精度，GPU 上 FP32 快得多 | 带 f 后缀 |
| fmaxf vs fmax | 单精度取 max vs 双精度 | 带 f 后缀 |
| Pass 融合 vs Online Softmax | V4 用 SMEM 暂存实现融合，Online 用递推算法实现融合 | V4 缓存，Online 递推 |

## 常见报错速查表

| ❌ 报错/问题 | 原因 | ✅ 解决 |
|-------------|------|--------|
| 结果出现 NaN | 没减最大值，`expf(100)` 溢出成 inf，inf/inf = NaN | 必须先 Pass 1 求 max 再做 Safe Softmax |
| 结果全为 0 | 分母 sum = 0（输入全 -INFINITY 或全极小值） | 检查输入数据，或加 epsilon 兜底 |
| V1 结果偶发错误 | 规约循环缺 `__syncthreads()` | 每轮归约后都要 `__syncthreads()` |
| V1/V2 性能不及预期 | `blockDim.x` 不是 32 的倍数 | 用 128 或 256，保证整数个 Warp |
| V3 float4 段错误 | 指针未 16B 对齐 | 用 `cudaMalloc`（自动 256B 对齐），勿用 `malloc` |
| V3 结果错误 | $N$ 不是 4 的倍数但没处理尾部 | 加 `for (i = N4*4 + tid; i < N; ...)` 兜底 |
| V4 SMEM 不足 | $N$ 太大，$N \times 4$ 超过 SMEM 限制 | $N > 4096$ 时改用 V3 或 Online Softmax |
| `__shfl_down_sync` 报错 | mask 不是 `0xffffffff` 或 Warp 不完整 | 用 `0xffffffff` 全线程参与 |
| `expf` 速度慢 | 误用 `exp`（双精度） | 改用 `expf`，确保全程 FP32 |
| Launch failed | smem_size 超过默认 48KB | 用 `cudaFuncSetAttribute` 动态共享内存 |

> 💡 动态共享内存（`extern __shared__`）默认上限 48KB，超过需手动开启动态调度：
> ```cuda
> cudaFuncSetAttribute(softmax_v4, cudaFuncAttributeMaxDynamicSharedMemorySize, N * sizeof(float));
> ```
