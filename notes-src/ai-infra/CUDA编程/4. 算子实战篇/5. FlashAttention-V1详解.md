## 一句话结论

FlashAttention V1 用 **Tiling（分块）+ Online Softmax（边扫边修正）** 把 Attention 的中间矩阵 $S=QK^T$、$P=\text{softmax}(S)$ 从不落 HBM，全程在 SRAM 中流转，**精确（无近似）** 地把 HBM 访问量从 $O(N^2)$ 降到 $O(N^2d^2/M)$，让长序列从 OOM 变成可训练。

## 背景与定位

- **这篇讲什么**：标准 Attention 为什么 Memory-Bound；FlashAttention V1 的两大核心技术（Tiling + Online Softmax）；前向完整流程；反向重计算策略；IO 复杂度推导；Triton 简化实现。
- **为什么重要**：FlashAttention 是当今大模型训练/推理的"地基"——LLaMA、GPT-4、Qwen 等几乎都在用。它也是 Flash Decoding、PagedAttention 等现代推理优化的前置知识。不理解 FA，几乎等于不理解现代 Attention 实现。
- **前置知识**：
  - 标准 Attention 公式 $\text{Attention}(Q,K,V)=\text{softmax}(QK^T/\sqrt d)V$
  - CUDA 内存层次：HBM（显存，大但慢）vs SRAM（共享内存，小但快）
  - Tiling 分块思想（参考 `2. CUDA-GEMM算子性能优化`）

## 核心概念（新人友好讲解）

### 1. 标准 Attention 公式回顾

$$
\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt d}\right)V
$$

记 $S=QK^T/\sqrt d$、$P=\text{softmax}(S)$、$O=PV$。其中 $Q,K,V\in\mathbb R^{N\times d}$，$N$ 是序列长度，$d$ 是 head dimension（如 LLaMA-7B 的 $d=128$）。

**生活类比**：Q 是"提问者"，K 是"关键词"，V 是"答案内容"。Attention = 用提问者与每个关键词的相似度（$QK^T$）作为权重，加权平均所有答案内容（$\text{softmax}\cdot V$）。$\sqrt d$ 是为了消除维度带来的尺度偏大（点积随 $d$ 增大而增大）。

### 2. 标准 Attention 的瓶颈（新人必懂）

| 📊 指标 | 📝 数值 | 💡 说明 |
|---------|---------|---------|
| 中间矩阵 $S, P$ 大小 | $O(N^2)$ | 各需 $N\times N$ 存储 |
| HBM 读写量 | $O(N^2)$ | 反复读写 $N\times N$ 矩阵 |
| 计算量 | $O(N^2 d)$ | 两次矩阵乘 |
| 算术强度 | $\approx O(d)$ | $d$ 通常只有 64/128，算术强度低 → Memory-Bound |

**关键问题**：$S$ 和 $P$ 大小为 $N\times N$，必须落 HBM。当 $N=4096, d=128$ 时，单个 head 的 $S$ 矩阵就要 $4096\times 4096\times 2=32$ MB（FP16）。

**生活类比**：厨师（GPU 计算单元）做菜很快，但食材（数据）从仓库（HBM）搬到厨房（SRAM）太慢，厨师大部分时间在等食材——这就是 Memory-Bound。标准 Attention 的瓶颈不是算不过来，而是搬不过来。

### 3. 核心思想：IO-Awareness

![FlashAttention 核心思想](https://raw.githubusercontent.com/caomaolufei/AIInfraGuide/main/AIInfraGuide/images/flashattentionv1-0.png)

FlashAttention 的核心洞察：**Attention 是 Memory-Bound 操作，优化关键不在减少 FLOPs，而在减少 HBM 访问次数。**

传统稀疏 Attention 路线是"少算"（减少 FLOPs），FlashAttention 路线是"少搬"（减少 IO），结果后者既快又精确。

### 4. 两个核心技术

| 技术 | 解决什么问题 | 类比 |
|------|------------|------|
| **Tiling（分块）** | 不生成完整 $N\times N$ 矩阵，每次只在 SRAM 中算一小块 | 把大菜一次端不完，分批做小份 |
| **Online Softmax** | 分块时 softmax 依赖全局 max/sum，怎么办？边扫边修正 | 每来一组学生成绩就立刻更新班级排名统计 |

**关键点**：两个技术缺一不可。Tiling 解决"如何不存 $N\times N$"，Online Softmax 解决"分块时 softmax 怎么算"。

### 5. Online Softmax 递推（新人重点）

**标准 Softmax 三遍扫描**（输入 $x\in\mathbb R^N$）：
1. 求全局最大值 $m=\max_i x_i$（数值稳定性）
2. 求归一化分母 $\ell=\sum_j e^{x_j-m}$
3. 计算输出 $\text{softmax}(x_i)=e^{x_i-m}/\ell$

三遍依赖串行，与分块矛盾。

**Online Softmax**：维护"可修正"的局部统计量，新数据块到来时递推更新。

设已处理前 $j-1$ 块，维护局部最大值 $m^{(j-1)}$ 和局部指数和 $\ell^{(j-1)}$。第 $j$ 块 $x^{(j)}$ 到来时：

$$
m^{(j)}=\max(m^{(j-1)},\max(x^{(j)}))
$$

$$
\ell^{(j)}=\ell^{(j-1)}\cdot e^{m^{(j-1)}-m^{(j)}}+\sum_i e^{x_i^{(j)}-m^{(j)}}
$$

**正确性来源**：恒等式 $e^{x_i-m_{\text{new}}}=e^{x_i-m_{\text{old}}}\cdot e^{m_{\text{old}}-m_{\text{new}}}$。每当最大值更新，旧指数项乘以修正因子 $e^{m_{\text{old}}-m_{\text{new}}}$ 即可。

⚠️ **重要**：Online Softmax 是**精确**的，不是近似！结果与标准 Softmax 数学上完全等价（除浮点舍入差异）。

### 6. Tiling 分块策略

![Tiling 分块计算策略](https://raw.githubusercontent.com/caomaolufei/AIInfraGuide/main/AIInfraGuide/images/flashattentionv1-1.png)

将 $Q,K,V$ 沿序列维度分块：
- $Q$ 分成 $T_r=\lceil N/B_r\rceil$ 块，每块 $B_r$ 行
- $K,V$ 分成 $T_c=\lceil N/B_c\rceil$ 块，每块 $B_c$ 行

块大小由 SRAM 容量 $M$（元素个数）和 head 维度 $d$ 决定：

$$
B_c=\left\lceil\frac{M}{4d}\right\rceil,\quad B_r=\min\left(\left\lceil\frac{M}{4d}\right\rceil,d\right)
$$

**为什么是 $M/(4d)$？** SRAM 同时驻留 $Q_i, K_j, V_j, O_i$ 四块 $\approx B\times d$ 张量，共 $4Bd$ 元素，要求 $\le M$，解出 $B\le M/(4d)$。

**为什么 $B_r$ 还要和 $d$ 取 $\min$？** 修正系数 $e^{m^{(j-1)}-m^{(j)}}$ 是逐行标量，原论文简化分析约束 $B_r\le d$。

### 7. SRAM 使用规划

| 📊 数据 | 大小 | 📝 用途 |
|---------|------|---------|
| $Q_i$ 块 | $B_r\times d$ | 当前 Query 块 |
| $K_j$ 块 | $B_c\times d$ | 当前 Key 块 |
| $V_j$ 块 | $B_c\times d$ | 当前 Value 块 |
| $S_{ij}$ 块 | $B_r\times B_c$ | 局部注意力分数 |
| $O_i$ 块 | $B_r\times d$ | 累积输出块 |
| $m_i, \ell_i$ | $2B_r$ | 行最大值、行指数和（每行一个标量） |

总 SRAM 需求约 $(2B_r+2B_c)\times d+B_r\times B_c+2B_r$，主导项是 $(2B_r+2B_c)\times d$。

### 8. V1 循环结构（外 K/V，内 Q）

```
外循环: for j = 1 to T_c (遍历 K/V 块)
    从 HBM 加载 K_j, V_j 到 SRAM
    内循环: for i = 1 to T_r (遍历 Q 块)
        从 HBM 加载 Q_i, O_i, m_i, l_i 到 SRAM
        计算局部注意力分数 S_ij = Q_i * K_j^T
        更新统计量和输出
        将更新后的 O_i, m_i, l_i 写回 HBM
```

**设计含义**：每个 K/V 块只从 HBM 加载一次，但 Q 和 O 会被 $T_c$ 次外循环反复读写。这是 V1 的 IO 瓶颈——V2 正是为此改进。

### 9. 架构图（ASCII）

```
                ┌─────────────────── HBM（显存，大但慢）───────────────────┐
                │   Q [B,H,N,d]   K [B,H,N,d]   V [B,H,N,d]               │
                │   O [B,H,N,d]   M [B,H,N]    L [B,H,N]   ← 辅助统计量   │
                └────────────┬───────────────────────────────────────────┘
                             │ 分块加载（Tiling）
                             ▼
        ┌─────────── SRAM（共享内存，小但快，~100KB/SM）──────────┐
        │   Q_i [B_r,d]   K_j [B_c,d]   V_j [B_c,d]              │
        │   S_ij [B_r,B_c]   O_i [B_r,d]   m_i, l_i [B_r]        │
        │                                                         │
        │   ① S_ij = Q_i @ K_j^T / sqrt(d)                       │
        │   ② m_new = max(m_i, rowmax(S_ij))                     │
        │   ③ alpha = exp(m_i - m_new)                            │
        │   ④ l_new = alpha * l_i + rowsum(exp(S_ij - m_new))    │
        │   ⑤ O_new = (alpha*l_i/l_new)*O_i + (exp(...)/l_new)@V_j│
        └─────────────────────────┬───────────────────────────────┘
                                  │ 写回 O_i, m_i, l_i（仅 V1 反复读写）
                                  ▼
                              回到 HBM
```

### 10. 关键术语速查表

| 参数/术语 | 通俗解释 | 类比 |
|----------|---------|------|
| $N$ | 序列长度（token 数） | 一篇文章的字数 |
| $d$ | head dimension，每个 token 的特征维度 | 每个字用多少维向量表示 |
| $B_r, B_c$ | Q 块、K/V 块的行数 | 一次处理几个 token |
| $M$ | SRAM 容量（元素个数） | 厨房案板大小 |
| HBM | 显存，大（80GB）但慢（~2TB/s） | 仓库 |
| SRAM | 共享内存，小（~100KB/SM）但快（~19TB/s） | 厨房案板 |
| Tiling | 把大矩阵切成小块算 | 大菜分批做 |
| Online Softmax | 边算边修正 max/sum，避免两趟扫描 | 边收卷子边算排名 |
| $m_i, \ell_i$ | 每行的局部最大值、指数和 | 班级当前最高分、分数累加 |
| logsumexp | $\log(\sum e^{x_i-m})+m$，融合存储 | 把 max 和 sum 压成一个数 |

### 1. 张量形状贯穿全过程

输入 $Q, K, V$ 都是 4 维张量，前两维 batch 与 head（作为"批次"维度原样传递），最后两维做矩阵运算：

```bash
Q: [B, H, N, D]
K: [B, H, N, D]   →  K^T (最后两维转置): [B, H, D, N]
V: [B, H, N, D]

Step 1  注意力分数：
  S = Q @ K^T / sqrt(D)
    = [B, H, N, D] @ [B, H, D, N]
    = [B, H, N, N]

Step 2  Softmax（沿最后一维 N，逐行归一化）：
  P = softmax(S, dim=-1)
    形状: [B, H, N, N]

Step 3  输出：
  O = P @ V
    = [B, H, N, N] @ [B, H, N, D]
    = [B, H, N, D]   （与 Q 同形）
```

辅助张量（FlashAttention 专属）：

- `M: [B, H, N]` —— 每行的最大值（数值稳定 + Online Softmax 修正）
- `L: [B, H, N]` —— 每行的指数和（softmax 分母）

### 2. 前向传播完整伪代码（一字不改）

```python
# FlashAttention V1 前向传播伪代码
def flash_attention_forward(Q, K, V, B_r, B_c):
    N, d = Q.shape
    O = zeros(N, d)
    m = full(N, -inf)  # 行最大值
    l = zeros(N)        # 行指数和

    # 外循环：遍历 K/V 块
    for j in range(0, N, B_c):
        K_j = K[j:j+B_c]  # 从 HBM 加载
        V_j = V[j:j+B_c]  # 从 HBM 加载

        # 内循环：遍历 Q 块
        for i in range(0, N, B_r):
            Q_i = Q[i:i+B_r]          # 从 HBM 加载
            O_i = O[i:i+B_r]          # 从 HBM 加载
            m_i = m[i:i+B_r]          # 从 HBM 加载
            l_i = l[i:i+B_r]          # 从 HBM 加载

            # 计算局部注意力分数
            S_ij = Q_i @ K_j.T / sqrt(d)  # [B_r, B_c]

            # 更新统计量
            m_new = max(m_i, rowmax(S_ij))
            l_new = l_i * exp(m_i - m_new) + rowsum(exp(S_ij - m_new))

            # 更新输出
            O_new = O_i * (l_i * exp(m_i - m_new) / l_new).unsqueeze(1)
            O_new += (exp(S_ij - m_new) / l_new.unsqueeze(1)) @ V_j

            # 写回 HBM
            O[i:i+B_r] = O_new
            m[i:i+B_r] = m_new
            l[i:i+B_r] = l_new

    return O
```

### 3. 五步更新公式（以第 $i$ 个 Q 块、第 $j$ 个 K/V 块为例）

**Step 1**：计算局部注意力分数

$$
S_{ij} = Q_i K_j^\top / \sqrt{d} \in \mathbb{R}^{B_r \times B_c}
$$

**Step 2**：计算当前块的行最大值

$$
\tilde{m}_{ij} = \text{rowmax}(S_{ij}) \in \mathbb{R}^{B_r}
$$

**Step 3**：更新全局行最大值

$$
m_i^{\text{new}} = \max(m_i, \tilde{m}_{ij})
$$

**Step 4**：计算修正因子并更新指数和

$$
\alpha = e^{m_i - m_i^{\text{new}}}
$$

$$
l_i^{\text{new}} = \alpha \cdot l_i + \text{rowsum}\left(e^{S_{ij} - m_i^{\text{new}}}\right)
$$

**Step 5**：更新输出

$$
O_i^{\text{new}} = \frac{\alpha \cdot l_i}{l_i^{\text{new}}} \cdot O_i + \frac{1}{l_i^{\text{new}}} \cdot e^{S_{ij} - m_i^{\text{new}}} \cdot V_j
$$

### 4. Triton 简化实现（一字不改）

```python
import math
import torch
import triton
import triton.language as tl

@triton.jit
def flash_attn_v1_kernel(
    Q, K, V, O,    # [B, H, N, D]
    M, L,          # 行最大值与行指数和
    stride_qb, stride_qh, stride_qn, stride_qd,
    stride_kb, stride_kh, stride_kn, stride_kd,
    stride_vb, stride_vh, stride_vn, stride_vd,
    stride_ob, stride_oh, stride_on, stride_od,
    stride_mb, stride_mh,
    N,
    D: tl.constexpr,        # 注意力头维度（head dimension），即每个 token 的特征长度，用在 scale = 1/sqrt(D)
    BLOCK_M: tl.constexpr,  # Q 块行数 B_r
    BLOCK_N: tl.constexpr,  # K/V 块行数 B_c
    BLOCK_D: tl.constexpr,  # 头维度上加载/计算的块宽度，当D本身就是2的幂时，BLOCK_D==D；否则 BLOCK_D > D，多出来的位置用 mask 屏蔽掉
):
    # ========== V1 并行划分：grid = (batch, head) ==========
    # 每个 program 串行扫完整个 (batch, head) 的所有 K/V 块和 Q 块
    batch_id = tl.program_id(0)
    head_id = tl.program_id(1)

    # 当前 (batch, head) 在各张量上的起始偏移
    qkvo_bh_q = batch_id * stride_qb + head_id * stride_qh
    qkvo_bh_k = batch_id * stride_kb + head_id * stride_kh
    qkvo_bh_v = batch_id * stride_vb + head_id * stride_vh
    qkvo_bh_o = batch_id * stride_ob + head_id * stride_oh
    ml_bh = batch_id * stride_mb + head_id * stride_mh

    d_range = tl.arange(0, BLOCK_D)
    scale = 1.0 / math.sqrt(D)

    # ============ 外循环：遍历 K/V 块（V1 关键设计） ============
    for kv_start in range(0, N, BLOCK_N):
        kv_range = kv_start + tl.arange(0, BLOCK_N)
        kv_mask = kv_range < N

        # K_j, V_j 一次加载，常驻寄存器/SRAM，被所有 Q 块复用
        k = tl.load(K + qkvo_bh_k + kv_range[:, None] * stride_kn + d_range[None, :] * stride_kd,
                    mask=kv_mask[:, None] & (d_range[None, :] < D), other=0.0)
        v = tl.load(V + qkvo_bh_v + kv_range[:, None] * stride_vn + d_range[None, :] * stride_vd,
                    mask=kv_mask[:, None] & (d_range[None, :] < D), other=0.0)

        # ============ 内循环：遍历 Q 块 ============
        for q_start in range(0, N, BLOCK_M):
            q_range = q_start + tl.arange(0, BLOCK_M)
            q_mask = q_range < N

            # 加载 Q_i, O_i, m_i, l_i —— V1 中每个外循环都要从 HBM 重新读
            q = tl.load(Q + qkvo_bh_q + q_range[:, None] * stride_qn + d_range[None, :] * stride_qd,
                        mask=q_mask[:, None] & (d_range[None, :] < D), other=0.0)
            o_i = tl.load(O + qkvo_bh_o + q_range[:, None] * stride_on + d_range[None, :] * stride_od,
                          mask=q_mask[:, None] & (d_range[None, :] < D), other=0.0).to(tl.float32)
            m_i = tl.load(M + ml_bh + q_range, mask=q_mask, other=float('-inf'))
            l_i = tl.load(L + ml_bh + q_range, mask=q_mask, other=0.0)

            # S_ij = Q_i @ K_j^T / sqrt(d)
            s = tl.dot(q, tl.trans(k)) * scale
            # 屏蔽 K/V 越界列，防止假 0 污染 softmax 分母
            s = tl.where(kv_mask[None, :], s, float('-inf'))

            # Online Softmax 更新
            m_new = tl.maximum(m_i, tl.max(s, axis=1))
            alpha = tl.exp(m_i - m_new)
            p = tl.exp(s - m_new[:, None])
            l_new = alpha * l_i + tl.sum(p, axis=1)

            # 输出修正 + 累加（与 5.1 Step 5 公式一致）
            o_new = o_i * (alpha * l_i / l_new)[:, None] \
                    + tl.dot(p.to(v.dtype), v) / l_new[:, None]

            # 把更新后的 O_i, m_i, l_i 写回 HBM，供下次外循环使用
            tl.store(O + qkvo_bh_o + q_range[:, None] * stride_on + d_range[None, :] * stride_od,
                     o_new.to(O.dtype.element_ty),
                     mask=q_mask[:, None] & (d_range[None, :] < D))
            tl.store(M + ml_bh + q_range, m_new, mask=q_mask)
            tl.store(L + ml_bh + q_range, l_new, mask=q_mask)

def flash_attn_v1(Q, K, V, BLOCK_M=64, BLOCK_N=64):
    # Q, K, V: [B, H, N, D]
    B, H, N, D = Q.shape
    O = torch.zeros_like(Q)
    M = torch.full((B, H, N), float('-inf'), device=Q.device, dtype=torch.float32)
    L = torch.zeros((B, H, N), device=Q.device, dtype=torch.float32)

    BLOCK_D = triton.next_power_of_2(D)
    grid = (B, H)  # V1 并行：仅在 batch×head 维度上
    flash_attn_v1_kernel[grid](
        Q, K, V, O, M, L,
        Q.stride(0), Q.stride(1), Q.stride(2), Q.stride(3),
        K.stride(0), K.stride(1), K.stride(2), K.stride(3),
        V.stride(0), V.stride(1), V.stride(2), V.stride(3),
        O.stride(0), O.stride(1), O.stride(2), O.stride(3),
        M.stride(0), M.stride(1),
        N, D=D, BLOCK_M=BLOCK_M, BLOCK_N=BLOCK_N, BLOCK_D=BLOCK_D,
    )
    return O
```

**代码要点回顾**：

1. **V1 并行划分**：grid 为 `(batch, head)`，K/V 在外、Q 在内——每个 K/V 块只从 HBM 加载一次，但 Q/O/m/l 会被 $T_c$ 次外循环反复读写
2. **辅助张量 M, L**：因为不同外循环迭代之间需要共享 Q 的统计量 $m_i, l_i$，必须在 HBM 中开辟 `[B, H, N]` 的张量持久化
3. **K/V 越界屏蔽**（`tl.where(kv_mask, s, -inf)`）是正确性的关键，遗漏会让 softmax 分母被假 0 污染
4. **`m, l` 初值** 分别为 `-inf` 和 `0`，配合 `alpha = exp(-inf - m_new) = 0`，让首次迭代自然退化为正常累加
5. **V1 的局限**：`(batch, head)` 通常远小于 GPU 上的 SM 数（典型 A100 有 108 个 SM，而 batch×head 在推理时常只有几十），SM 利用率不足；V2 把 Q 提到 grid 上并行，正是为了解决这一点

### 5. Kernel 设计要点

> 📌 **关于并行划分（grid 与 block 设计）**：
> - **grid（program 粒度）=（B, H）**：每个 program 负责一个 `(batch, head)` 切片下完整的注意力计算，program 之间完全独立、无通信。
> - **block 内部线程**：在一个 program 内部，Triton 会把 `tl.dot`、`tl.max`、`tl.sum` 等向量化操作自动拆分到一个 thread block 的若干 warp 上。块大小由 `BLOCK_M`（Q 的行数 $B_r$）、`BLOCK_N`（K/V 的行数 $B_c$）、`BLOCK_D`（D 维向量化宽度）共同决定。
> - **program 内部串行**：外循环遍历 K/V 块、内循环遍历 Q 块
>
> 由于不同 Q 块在不同外循环迭代之间共享统计量 $m_i, l_i$ 与累积输出 $O_i$，单个 program 的寄存器装不下"所有 Q 块"的 m/l/O，必须借助 HBM 的辅助张量 `M, L` 在迭代之间持久化（这正是 V1 IO 较多的根本原因——V2 通过交换循环顺序，把 Q 提到 grid 上并行，让 m/l/O 全程留在寄存器中，从而消除了这部分 HBM 访问）。
>
> ⚠️ **V1 并行度的天然瓶颈**：grid 大小 $B \times H$ 在推理场景常常只有几十（如 batch=1、heads=32 → 仅 32 个 program），而现代 GPU 通常有 80~140 个 SM（A100=108，H100=132），SM 利用率只有约 30%。这是 V1 的主要短板，也是 V2 必然出现的动因。

1. **Thread Block 映射**：V1 中每个 Thread Block 对应外循环的一个 K/V 块（即一个 $j$），内部遍历所有 Q 块
2. **Shared Memory 分配**：为 $K_j$、$V_j$（外循环常驻）以及 $Q_i$、$S_{ij}$ 预分配 SRAM
3. **寄存器使用**：统计量 $m_i$、$l_i$ 和累积输出 $O_i$ 尽量放在寄存器中
4. **循环结构**：外循环加载 K/V 块，内部完成 GEMM 和 Softmax 更新

### 8. 反向传播与重计算

标准 Attention 反向需要中间矩阵 $S$ 和 $P$（$N\times N$）：

$$
dV = P^\top dO
$$

$$
dP = dO \cdot V^\top
$$

$$
dS = P \odot \left(dP - \text{rowsum}(dP \odot P) \cdot \mathbf{1}^\top\right)
$$

FlashAttention 利用恒等式 $\text{rowsum}(dP\odot P)=\text{rowsum}(dO\odot O)$（记作 $D_i$），避免显式构造完整 $dP$。

**重计算策略**：不保存 $S, P$，反向时用保存的 $m, l$ 重新加载 $Q_i, K_j$ 计算 $S_{ij}$，再恢复 $P_{ij}$：

$$
P_{ij} = \text{diag}(l_i)^{-1} \cdot e^{S_{ij} - m_i}
$$

| ✅ 收益 | ❌ 代价 |
|---------|---------|
| 显存从 $O(N^2)$ 降到 $O(N)$ | 额外的 FLOP 开销（约 50% 前向计算量） |
| HBM 访问大幅减少 | SRAM 需要同时存放更多数据 |
| 可以训练更长序列 | 反向传播核函数更复杂 |

由于 Attention 是 Memory-Bound 的，额外计算被 HBM 访问的减少所"隐藏"，实际 wall-clock 时间反而更短。

## 面试回答（可直接口述的 30-60 秒版本）

**问：FlashAttention V1 是怎么加速 Attention 的？**

答：核心是两点——Tiling 和 Online Softmax。标准 Attention 的瓶颈不在算力，而在 HBM 读写：中间矩阵 $S=QK^T$ 和 $P=\text{softmax}(S)$ 大小 $N\times N$，必须落 HBM，读写量 $O(N^2)$，序列一长就 OOM 也算不动。FlashAttention 把 $Q, K, V$ 切成小块，每次只加载一小块到 SRAM 计算，中间 $S, P$ 全程不落 HBM。但分块后 softmax 没法先扫一遍求全局 max——所以用 Online Softmax：维护一个局部 max 和局部 sum，每来一个新块就用 $e^{m_{\text{old}}-m_{\text{new}}}$ 修正之前的统计量，最后结果和标准 softmax 完全等价、没有任何近似。最终 HBM 访问量从 $O(N^2)$ 降到 $O(N^2d^2/M)$（$M$ 是 SRAM 大小，$d\ll N$ 时大幅减小），且因为是精确算法不损失精度。反向传播时不存 $S, P$，用保存的 $m, l$ 重算，省显存换少量计算，因为 Memory-Bound 所以 wall-clock 反而更短。

## 深入追问（可能被追问的点）

- **Q1: Online Softmax 凭什么和标准 Softmax 等价？**
  A: 关键恒等式 $e^{x_i-m_{\text{new}}}=e^{x_i-m_{\text{old}}}\cdot e^{m_{\text{old}}-m_{\text{new}}}$。当新块带来更大的 max，旧的指数项乘以修正因子 $e^{m_{\text{old}}-m_{\text{new}}}$ 即可换算到新基准下。整个过程是代数恒等式，无近似。

- **Q2: 为什么块大小是 $M/(4d)$？**
  A: SRAM 要同时装 $Q_i, K_j, V_j, O_i$ 四块 $\approx B\times d$ 的张量，共 $4Bd\le M$，解出 $B\le M/(4d)$。

- **Q3: FlashAttention 的 IO 复杂度 $O(N^2d^2/M)$ 怎么推出来？**
  A: 由 $B_c=O(M/d)$，外循环次数 $T_c=N/B_c=O(Nd/M)$。每次外循环要遍历完整的 $Q, O$（$O(Nd)$ 元素），所以 $Q, O$ 总访问量 $T_c\cdot Nd=O(N^2d^2/M)$。$K, V$ 整体只读一次 $O(Nd)$，相对可忽略。

- **Q4: 为什么说 FlashAttention 是"渐近最优"的？**
  A: 论文证明任何精确 Attention 算法的 HBM 访问下界为 $\Omega(N^2d^2/M)$，FlashAttention 达到这个下界。

- **Q5: 重计算不会让反向更慢吗？**
  A: 不一定。Attention 是 Memory-Bound，反向时省下了 $O(N^2)$ 的 $S, P$ 读写，多出的 FLOPs 被隐藏在 HBM 等待里。实测 wall-clock 反而更短。

- **Q6: V1 在长序列推理时为什么 SM 利用率低？**
  A: grid 只有 $(B, H)$，推理 batch=1、heads=32 时只有 32 个 program，A100 有 108 个 SM，利用率约 30%。这是 V2 必然出现的动因。

- **Q7: 为什么 V1 选外循环 K/V 而不是 Q？**
  A: 源于 online softmax 的累积逻辑：每扫完一列 K/V 块就更新一次 O 的归一化，自然把 K/V 放外循环。代价是 O 要反复读写。V2 发现让 Q 只加载一次更优。

- **Q8: `m, l` 初值为什么是 $-\infty$ 和 $0$？**
  A: 配合 $\alpha=e^{m_{\text{old}}-m_{\text{new}}}=e^{-\infty-m_{\text{new}}}=0$，让首次迭代自然退化为正常累加（旧贡献被乘以 0 抹掉），无需特判。

## 易混淆点对比

| 易混概念 | 区别 | 记忆技巧 |
|---------|------|----------|
| Memory-Bound vs Compute-Bound | 前者瓶颈在 HBM 带宽，后者在算力 | 标准 Attention 是前者，GEMM 是后者 |
| 标准 Softmax vs Online Softmax | 前者三遍扫描，后者递推修正一遍扫描 | Online = 边扫边修 |
| Tiling vs 稀疏 Attention | 前者只改数据流向不改数学，后者改数学（近似） | Tiling 精确，稀疏近似 |
| $O(N^2)$ vs $O(Nd^2/M)$ | 前者标准 Attention，后者 FlashAttention | $d\ll N$ 时差异巨大 |
| 重计算 vs 保存中间值 | 前者省显存多计算，后者省计算多显存 | FA 选前者 |
| $m_i$ vs $\ell_i$ | 前者行最大值，后者行指数和 | max 是分子修正，sum 是分母 |
| FlashAttention V1 vs V2 | V1 外循环 K/V、grid=(B,H)；V2 外循环 Q、grid=(B,H,N/B_r) | V2 把 Q 提到外循环并行 |
| $\text{softmax}(x)$ vs $\text{softmax}(x)\cdot V$ | 前者是权重，后者是输出 | FA 把两者融合 |
| HBM vs SRAM | 前者 80GB/2TB/s，后者 ~100KB/19TB/s | 大慢 vs 小快 |

## 常见报错速查表

| ❌ 报错/问题 | 原因 | ✅ 解决 |
|-------------|------|--------|
| Triton 编译报 `AttributeError: module 'triton' has no attribute 'jit'` | Triton 版本过旧或与 PyTorch 不兼容 | 升级 `pip install -U triton`，匹配 PyTorch 版本 |
| 输出 NaN | 忘记减最大值，$e^x$ 溢出 | 检查所有 `tl.exp` 输入是否已减去 `m_new` |
| 输出全 0 或全错 | K/V 越界 mask 遗漏，假 0 污染 softmax 分母 | 加 `tl.where(kv_mask, s, -inf)` |
| 首次迭代结果异常 | `m, l` 初值不对 | `m=−inf, l=0`，配合 `alpha=exp(−inf−m_new)=0` |
| OOM | 块大小过大，超过 SRAM | 减小 `BLOCK_M, BLOCK_N`，参考 $4Bd\le M$ |
| 与 PyTorch `F.scaled_dot_product_attention` 对不上 | 缺少 `/sqrt(d)` scale，或最后未除以 $\ell$ | 检查 `scale = 1/sqrt(D)`，检查最终 `O = O_tilde / l` |
| 性能远低于预期 | grid 太小 SM 利用率低（V1 推理常见） | 这是 V1 固有问题，需用 V2 |
| `tl.dot` 报 shape 错误 | 输入非 2D 或维度不匹配 | 确保传给 `tl.dot` 的是 2D 张量，形状对齐 |
| 反向传播显存没省下来 | 框架仍保存 $S, P$ | 用 `torch.utils.checkpoint` 或 FA 官方实现 |
| 性能剖析时 `ncu` 报权限错误 | Nsight Compute 默认需 root 才能采集硬件指标 | 用 `--target-processes all` 或仅采软件指标 |
