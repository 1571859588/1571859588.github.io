## 三条主线贯穿全程

阅读所有笔记时，请时刻记住这三条主线：

### 主线 1：算力 vs 带宽（Roofline 平衡点）

```
              算术强度（FLOP/Byte）
                     │
                     ▼
 低（< 平衡点）            高（> 平衡点）
 ↓                        ↓
Memory-Bound            Compute-Bound
代表：Softmax、Attention   代表：GEMM
优化：减少显存访问         优化：提升算术强度
      FlashAttention           Tiling 复用
```

每代架构都在拉高平衡点（加算力 + 加带宽），但算力增长快于带宽，所以越来越多的算子变成 Memory-Bound。

### 主线 2：精度演进（Tensor Core 的故事）

```
FP32 → FP16(Volta) → INT8(Turing) → TF32/BF16(Ampere) → FP8(Hopper) → FP4(Blackwell)
 32bit   16bit        8bit          19bit/16bit          8bit           4bit
```

每压缩一级精度，理论算力翻倍。但精度降低带来数值稳定性问题，所以每代都有配套方案（AMP loss scaling → Transformer Engine 自动精度调度）。

### 主线 3：单卡 → 多卡 → 集群

```
单卡 HBM          多卡 NVLink/NVSwitch        跨节点 InfiniBand
 8 TB/s             900~1800 GB/s               50~400 GB/s
 │                  │                           │
 └── 机内最快       └── 机内 GPU 互联            └── 机间通信（瓶颈所在）
```

通信带宽从机内到机间骤降一个数量级，这是分布式训练设计的核心约束。

## 代码环境

- **硬件**：任意 NVIDIA GPU（V100 / A100 / H100 / RTX 30xx / RTX 40xx 均可）
- **软件**：nvidia-smi（驱动自带）、PyTorch 2.x、（可选）TensorRT/trtexec
- **sudo 权限**：本系列假设用户无 sudo 权限
  - 除 `nvidia-smi mig -cgi`（创建 MIG 实例）⚠️ 需 sudo 外，其余命令均 ✅ 无需 sudo
  - 详见 `../索引.md` 的「代码环境」段和各篇的命令权限标注
- **WSL 注意**：`nvidia-smi topo -m` / `nvlink -s` 在 WSL 中可能输出受限，建议在真实 Linux GPU 服务器上运行
