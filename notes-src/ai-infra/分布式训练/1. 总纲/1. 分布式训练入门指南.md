# 分布式训练入门指南

> **一句话结论**：分布式训练解决两个独立问题——显存"装不下"（切分状态）和时间"跑不完"（复制并行）；五大策略各有分工，按"够用就好、不够再加维度"的决策树逐步升级到 3D 并行。

## 三条主线

### 主线一：显存账本（"钱花在哪了"）

训练一个大模型，显存开销远不止参数本身。以 BF16 + AdamW 为例：

```
┌──────────────────────────────────────────────────────────┐
│              7B 模型静态显存账本（共 ~112 GB）             │
├──────────────────┬──────────┬────────────────────────────┤
│ 优化器状态 84GB  │ 75%      │ FP32主权重 + 一阶动量 + 二阶动量 │
│ BF16 梯度  14GB  │ 12.5%    │ 反向传播产出                │
│ BF16 参数   14GB │ 12.5%    │ 前向/反向工作副本            │
└──────────────────┴──────────┴────────────────────────────┘
```

- 7B 模型仅静态显存就 ~112 GB，单张 A100 40G 远远装不下
- 优化器状态占 75%，是 ZeRO 优先切分的对象
- 还没算激活值（随 batch/序列长度增长）

### 主线二：五大并行策略（"怎么切"）

| 策略 | 切什么 | 主要通信 | 解决的问题 |
|------|--------|---------|-----------|
| DP/DDP | 切数据 | 梯度 AllReduce | 跑不完（加速） |
| ZeRO/FSDP | 切优化器/梯度/参数 | ReduceScatter + AllGather | 装不下（省显存） |
| TP | 切单层矩阵运算 | 层内 AllReduce | 单层太大 |
| PP | 切网络层（按深度） | 段间传激活值 | 层数太多 |
| SP/CP | 沿序列维度切 | AllGather/All-to-All | 序列太长 |

### 主线三：带宽决定布局（"谁放机内、谁跨机"）

```
带宽从高到低：NVLink(数百GB/s) > InfiniBand(~50GB/s) > 以太网
通信从密到疏：TP > CP/SP > PP > DP

  ┌─────────────────────────────────┐
  │         单机 8 卡（NVLink）       │
  │    TP（通信最密，必须机内）        │
  │    ┌──────────────────────┐     │
  │    │  机内 FSDP/ZeRO       │     │
  │    └──────────────────────┘     │
  └─────────┬───────────────────────┘
            │ InfiniBand（跨机）
  ┌─────────▼───────────────────────┐
  │    PP（传激活值，可跨机）          │
  │    DP（同步梯度，可跨机）          │
  └─────────────────────────────────┘
```

## 环境说明

### 硬件环境

- **用户环境**：Ubuntu Linux 服务器，**8 张 A100 40G**，**无 sudo 权限**
- 单机 8 卡通过 NVLink/NVSwitch 互联，机内带宽数百 GB/s
- A100 40G 每卡 40GB HBM2e 显存

### 软件环境搭建

```bash
# ✅ 无需 sudo：用 conda 创建环境
conda create -n dist_train python=3.10 -y
conda activate dist_train

# ✅ 无需 sudo：安装 PyTorch（带 CUDA 支持）
# 以 CUDA 12.1 为例，根据你的驱动版本选择
conda install pytorch torchvision torchaudio pytorch-cuda=12.1 -c pytorch -c nvidia -y

# ✅ 无需 sudo：验证安装
python -c "import torch; print(f'PyTorch: {torch.__version__}'); print(f'CUDA available: {torch.cuda.is_available()}'); print(f'GPU count: {torch.cuda.device_count()}')"

# ✅ 无需 sudo：查看 GPU 拓扑
nvidia-smi topo -m
```

### 项目文件结构

```
dist_train_project/
├── train_ddp.py          # DDP 训练脚本
├── train_fsdp.py         # FSDP 训练脚本
├── collective_demo.py    # 集合通信原语演示
├── optimizer_demo.py     # 优化器显存分析演示
├── data/                 # 数据目录
├── checkpoints/          # 模型保存目录
└── logs/                 # 日志目录
```

### sudo 权限速查

| 操作 | 是否需要 sudo | 说明 |
|------|:---:|------|
| `nvidia-smi` | ✅ 无需 | 查看 GPU 状态、拓扑 |
| `torchrun` | ✅ 无需 | 启动分布式训练 |
| `conda install` / `pip install` | ✅ 无需 | 安装 Python 包 |
| NCCL 环境变量（`export NCCL_DEBUG=INFO`） | ✅ 无需 | 调试通信 |
| `ibstat` / `ibv_devinfo` | ⚠️ 视环境 | 查看 IB 网卡，通常系统已装则 ✅ |
| GPUDirect RDMA | ⚠️ 需 root | 无 sudo 不可用，影响跨机通信性能 |
| `/dev/shm` 扩容 | ⚠️ 需 root | 容器内需 `--shm-size`，无 sudo 则用 `NCCL_SHM_DISABLE=1` 绕过 |
| 安装系统级 IB 驱动 | ⚠️ 需 sudo | 通常管理员已装好 |

## 核心概念速查

| 概念 | 一句话解释 | 详见 |
|------|-----------|------|
| rank | 进程的全局编号（0 ~ world_size-1） | 1.2 环境搭建 |
| local_rank | 进程在本机内的编号（决定绑哪块 GPU） | 1.2 环境搭建 |
| world_size | 参与训练的总进程数 | 1.2 环境搭建 |
| AllReduce | 所有卡数据规约后人人有份 | 2.1 通信原语 |
| Ring AllReduce | 环形传递，通信量与卡数无关 | 2.1 通信原语 |
| Bucket | DDP 把梯度分桶，实现通信与计算重叠 | 4.1 数据并行 |
| ZeRO | 逐层切分优化器/梯度/参数，消除冗余 | 5.1 ZeRO |
| Column/Row Parallel | 张量并行的两种矩阵切分方式 | 6.1 张量并行 |
| Pipeline Bubble | 流水线并行中 GPU 空闲的时间段 | 7.1 流水线并行 |
| Activation Checkpointing | 只存部分激活，反向时重算 | 8.1 显存优化 |
| Expert Parallelism | 不同 Expert 放不同 GPU | 10.1 MoE |
| 3D 并行 | TP × PP × DP 组合 | 11.1 混合并行 |
