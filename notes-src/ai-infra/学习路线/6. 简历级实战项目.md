## 二、项目一：单机私有化大模型部署项目（初级岗必备）

### 2.1 项目描述

在单台 A10-24G GPU 云服务器上，基于 Ubuntu 22.04，使用 Docker + vLLM 部署 4bit-AWQ 量化后的 Qwen-14B 大模型；封装 OpenAI 标准 API；配置 Nginx 反向代理 + SSL 实现 HTTPS 域名访问；systemd 实现服务异常自愈；接入 Prometheus 监控 GPU 显存、接口 QPS；对接 Milvus 向量库实现 RAG 文档问答。

### 2.2 技术栈

```
Ubuntu 22.04
  ├── NVIDIA Driver 535
  ├── CUDA 12.1 + cuDNN
  ├── nvidia-docker2
  ├── Docker
  ├── vLLM（推理框架）
  ├── AWQ 4-bit 量化
  ├── Nginx（反向代理 + SSL）
  ├── Systemd（服务保活）
  ├── Prometheus + Grafana（监控）
  └── Milvus（RAG 向量库）
```

### 2.3 完整部署流程

#### 第 1 步：环境准备

```bash
# Ubuntu 22.04 + NVIDIA 驱动 + CUDA 12.1
# 参考 阶段三 笔记

nvidia-smi                    # 确认 GPU 可用
nvcc -V                       # 确认 CUDA 12.1
docker run --rm --gpus all nvidia/cuda:12.1-base nvidia-smi   # 确认容器可用 GPU
```

#### 第 2 步：下载并量化模型

```bash
# 下载 Qwen-14B
huggingface-cli download Qwen/Qwen1.5-14B --local-dir /models/Qwen-14B

# AWQ 4-bit 量化
pip install autoawq
python -c "
from awq import AutoAWQForCausalLM
model = AutoAWQForCausalLM.from_pretrained('/models/Qwen-14B')
model.quantize({'zero_point': True, 'q_group_size': 128, 'w_bit': 4})
model.save_quantized('/models/Qwen-14B-AWQ')
"

# 验证大小（28GB → 10GB）
du -sh /models/Qwen-14B /models/Qwen-14B-AWQ
```

#### 第 3 步：vLLM 部署 + systemd 保活

```bash
# 创建启动脚本
cat > /root/start_vllm.sh << 'EOF'
#!/bin/bash
vllm serve /models/Qwen-14B-AWQ \
  --quantization awq \
  --dtype float16 \
  --max-model-len 4096 \
  --port 8000 \
  --host 0.0.0.0
EOF
chmod +x /root/start_vllm.sh

# systemd 服务
cat > /etc/systemd/system/vllm.service << 'EOF'
[Unit]
Description=vLLM Inference Service
After=network.target

[Service]
User=root
ExecStart=/bin/bash /root/start_vllm.sh
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 启动
systemctl daemon-reload
systemctl start vllm
systemctl enable vllm

# 验证
curl http://localhost:8000/v1/models
```

#### 第 4 步：Nginx 反向代理 + SSL

```bash
sudo apt install nginx certbot python3-certbot-nginx -y

cat > /etc/nginx/conf.d/vllm.conf << 'EOF'
server {
    listen 80;
    server_name api.your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;            # 流式输出
        proxy_read_timeout 300s;
    }
}
EOF

# 申请 SSL 证书
sudo certbot --nginx -d api.your-domain.com

# 重载
nginx -t && systemctl reload nginx
```

#### 第 5 步：Prometheus + Grafana 监控

```bash
# 安装 DCGM Exporter（GPU 指标）
docker run -d --name dcgm-exporter \
  --gpus all \
  -p 9400:9400 \
  nvcr.io/nvidia/k8s/dcgm-exporter:latest

# 安装 Prometheus
docker run -d --name prometheus \
  -p 9090:9090 \
  -v /etc/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml \
  prom/prometheus

# prometheus.yml
cat > /etc/prometheus/prometheus.yml << 'EOF'
global:
  scrape_interval: 15s
scrape_configs:
  - job_name: 'dcgm'
    static_configs:
      - targets: ['localhost:9400']
  - job_name: 'vllm'
    static_configs:
      - targets: ['localhost:8000/metrics']
EOF

# 安装 Grafana
docker run -d --name grafana \
  -p 3000:3000 \
  grafana/grafana
# 导入仪表盘 ID: 12239
```

#### 第 6 步：Milvus + RAG

## 三、项目二：3 节点 K8s 分布式 AI 推理集群（中级岗，25K+ 必备）

### 3.1 项目描述

搭建 3 节点 K8s 集群（1 台 Master 管理节点、2 台 GPU 工作节点）；部署 NVIDIA GPU-Operator 实现 GPU 资源调度；利用 Deployment 管理 vLLM 推理服务，实现流量上涨自动扩容、流量降低自动缩容；Ingress-Nginx 实现域名路由；ArgoCD 实现模型版本 CI/CD 灰度发布；搭建全集群 GPU 监控 + 邮件告警。

### 3.2 技术栈

```
3 节点 K8s 集群
  ├── kubeadm（集群搭建）
  ├── NVIDIA GPU-Operator（GPU 调度）
  ├── vLLM（推理服务）
  ├── HPA（自动扩缩容）
  ├── Ingress-Nginx（域名路由）
  ├── ArgoCD（CI/CD 灰度发布）
  ├── Prometheus + Grafana（全集群监控）
  └── Alertmanager（邮件告警）
```

### 3.3 完整部署流程

#### 第 1 步：搭建 3 节点 K8s 集群

```bash
# 所有节点执行
sudo swapoff -a
sudo sed -i '/swap/d' /etc/fstab
# 安装 containerd + kubeadm + kubelet + kubectl（见阶段四）

# Master 节点
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config

# 生成加入命令
kubeadm token create --print-join-command
# 输出类似：kubeadm join 192.168.1.10:6443 --token xxx --discovery-token-ca-cert-hash xxx

# Worker 节点（2 台 GPU 机器）
sudo kubeadm join 192.168.1.10:6443 --token xxx --discovery-token-ca-cert-hash xxx

# Master 验证
kubectl get nodes
# 3 个节点都是 Ready
```

#### 第 2 步：部署 GPU-Operator

```bash
helm repo add nvidia https://nvidia.github.io/gpu-operator
helm install gpu-operator nvidia/gpu-operator -n gpu-operator --create-namespace

# 验证 GPU 节点
kubectl get nodes -o custom-columns="NAME:.metadata.name,GPU:.status.capacity.nvidia\.com/gpu"
# 两个 worker 节点应该显示 GPU 数量
```

#### 第 3 步：部署 vLLM + HPA 自动扩缩容

```yaml
# vllm-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: vllm
  namespace: ai
spec:
  replicas: 2
  selector:
    matchLabels:
      app: vllm
  template:
    metadata:
      labels:
        app: vllm
    spec:
      containers:
      - name: vllm
        image: vllm/vllm:latest
        args: ["--model", "/models/Qwen-14B-AWQ", "--quantization", "awq"]
        resources:
          limits:
            nvidia.com/gpu: 1
          requests:
            nvidia.com/gpu: 1
---
apiVersion: v1
kind: Service
metadata:
  name: vllm-svc
  namespace: ai
spec:
  selector:
    app: vllm
  ports:
  - port: 8000
---
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: vllm-hpa
  namespace: ai
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: vllm
  minReplicas: 2
  maxReplicas: 6
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```

#### 第 4 步：Ingress-Nginx 域名路由

```bash
helm install ingress-nginx ingress-nginx/ingress-nginx -n ingress-nginx --create-namespace
```

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: vllm-ingress
  namespace: ai
spec:
  rules:
  - host: api.your-domain.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: vllm-svc
            port:
              number: 8000
```

## 四、面试深挖准备

### 4.1 项目一常见追问

| 追问 | 回答要点 |
|------|----------|
| AWQ 量化原理？ | 基于激活值感知，保留重要权重，4-bit 压缩 |
| 28GB 怎么算的？ | 14B 参数 × 2 字节(FP16) = 28GB |
| 15 并发怎么测的？ | vLLM benchmark_serving 工具，100 请求 / 10 QPS |
| systemd 崩溃重启延迟？ | RestartSec=10，10 秒后重启 |
| RAG 检索准确率？ | top-k=3，余弦相似度，准确率约 85% |

### 4.2 项目二常见追问

| 追问 | 回答要点 |
|------|----------|
| GPU-Operator 做了什么？ | 自动装驱动、注册 nvidia.com/gpu 资源 |
| HPA 扩容太慢怎么办？ | 预测式扩容、减小阈值、预热 Pod |
| 灰度发布怎么回滚？ | ArgoCD 一键回滚到上一个版本 |
| 跨节点 GPU 通信？ | 同 Pod 内用 NVLink，跨节点不推荐 |
