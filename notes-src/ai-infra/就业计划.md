### 5.2 训练 Infra 原理调试类

| 题目 | 关键点 |
|------|--------|
| DeepSpeed ZeRO-1/2/3 vs FSDP vs Megatron 3D tradeoff？ | ZeRO-3 内存节省最大但通信高；FSDP PyTorch 原生；Megatron 3D 平衡 |
| 大模型训练 OOM/低 MFU/loss spike 诊断流程？ | Nsight/PyTorch Profiler 查 bound；NCCL 日志查通信 |
| BF16 vs FP16 区别？ | BF16 动态范围大（指数 8bit vs 5bit），减少 underflow/overflow |
| GQA/MQA vs MHA？KV Cache 节省与质量 tradeoff？ | GQA 减少 KV 头数节省显存 |
| FlashAttention v2/v3 改进点？ | 减少 HBM 读写、tiling 优化、Tensor Core |
| MoE 训练通信瓶颈？ | All-to-All/EP/TP 优化，Top-K 路由 + auxiliary loss |

### 5.3 推理 Infra 优化类

| 题目 | 关键点 |
|------|--------|
| KV Cache 导致推理成本 10x 高于预期，如何修复？ | PagedAttention（块表虚拟分页）+ Radix Tree 前缀复用 + 连续批处理 + 量化 KV Cache |
| Speculative Decoding 实现细节？ | 小模型并行生成，大模型验证+修正；加速 1.5-2.5x |
| 量化 Llama 模型技巧与精度 tradeoff？ | GPTQ/AWQ + FP8/INT4；加速 2-4x vs 精度损失 <1-2% |
| vLLM/Triton/TensorRT-LLM 对比？ | 各有优劣，看场景 |

### 5.4 CUDA Kernel 底层优化类

| 题目 | 关键点 |
|------|--------|
| GEMM/matmul Kernel 优化？ | tiling、WMMA、Tensor Core |
| softmax/layer norm/SwiGLU 高效 Kernel？ | 融合算子减少访存 |
| Roofline 模型分析 Kernel bound？ | 计算/内存/通信判断 |
| bank conflict、shared memory、occupancy 优化？ | GPU 内存层次优化 |
