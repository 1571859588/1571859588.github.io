## 一句话结论

**这一章讲的是"模型最后吐出来的一串分数怎么变成概率、怎么和正确答案比差距"。Softmax 把任意实数分数归一成"加起来等于 1"的概率分布；交叉熵是"用模型分布去编码真实分布平均要花多少比特"，是分类任务最常用的损失；KL 散度是"两个分布差多远"的不对称尺子，交叉熵 = 熵 + KL；LogSumExp 是把 Softmax 算得既不溢出又能求导的工程钥匙；温度参数控制分布尖锐还是平坦，是采样的"水龙头"；Top-k / Top-p 是在不改变相对大小关系的前提下，只从一小撮候选里抽样，避免长尾噪声。** 你需要记住九件事：
1. **Logits 不是概率**：它只是模型打的"分数"，可以是任意实数（含负数），加起来也不等于 1
2. **Softmax = "把分数转成百分比"**：用指数放大差距、再除以总和归一，保证结果都在 0~1 之间且和为 1
3. **数值稳定的 Softmax 必须先减最大值**：因为 $e^{1000}$ 在浮点数里会溢出成 inf，分子分母同除 $e^m$ 不改变结果
4. **LogSumExp 是 softmax 的对数版"防溢出三件套"**：$\log \operatorname{softmax}(\mathbf{z})_i = z_i - \operatorname{LSE}(\mathbf{z})$，所有现代框架都这么算
5. **温度 T 越小分布越尖**：$T \to 0$ 退化成 argmax（贪心），$T \to \infty$ 退化成均匀分布（瞎猜）
6. **交叉熵 $H(\mathbf{q}, \mathbf{p}) = -\sum_i q_i \log p_i$**：当真实分布 q 是 one-hot 时，只剩 $-\log p_y$，这就是分类任务的损失
7. **信息熵 $H(\mathbf{p})$ 是"不确定性的大小"**：抛硬币约 1 bit，掷骰子约 2.58 bit，越不确定熵越大
8. **KL 散度不对称**：$D_{KL}(\mathbf{q} \lVert \mathbf{p}) \ne D_{KL}(\mathbf{p} \lVert \mathbf{q})$，所以"用 p 近似 q"和"用 q 近似 p"代价不同
9. **$\operatorname{PPL} = e^{\text{平均交叉熵}}$**：困惑度，越大越"困惑"，是语言模型汇报的常用指标

## 5.1 Logits 不是概率

### 直觉

想象你在参加一个"认动物"比赛，模型看一张图，对"猫、狗、鸟"三类各打一个分：

| 类别 | 猫 | 狗 | 鸟 |
|------|----|----|----|
| 模型打分 (logits) | 2.5 | 1.0 | 0.3 |

这些分数叫 **logits**，它有几个特点：

1. **可以是任意实数**：正的、负的、零都行，比如 $[-1.2, 0, 3.4]$ 也合法
2. **加起来不等于 1**：2.5 + 1.0 + 0.3 = 3.8，不是 1
3. **大小只反映"相对优劣"**：2.5 > 1.0 > 0.3，所以模型认为"猫最像"

但它**不是概率**——概率必须满足"每个 $\ge 0$ 且总和 = 1"。所以我们不能直接说"模型有 2.5 的概率是猫"，这话没意义。

### 为什么要 Softmax？

我们想要一个函数，把 logits 这串"任意分数"变成"加起来等于 1 的概率"。要求：
- 保留相对大小关系（分大的概率也大）
- 放大差距（让"明显大"的更突出，避免 [2.5, 2.4, 2.3] 看起来差不多）
- 全为正（概率不能负）

**指数函数 $e^x$ 完美满足**：$e^x$ 永远为正、单调递增（保大小关系）、还能放大差距（$e^{2.5} \approx 12.18$，$e^1 \approx 2.72$，差距从 1.5 倍放大到了 4.5 倍）。

### 定义

给定 logits 向量 $\mathbf{z} = (z_1, z_2, \ldots, z_V)$，**Softmax** 把它变成概率分布 $\mathbf{p} = (p_1, p_2, \ldots, p_V)$：

$$
p_i = \operatorname{softmax}(\mathbf{z})_i = \frac{e^{z_i}}{\sum_{j=1}^{V} e^{z_j}}
$$

验证三条性质：
1. **非负**：$e^x > 0$ 恒成立，所以 $p_i > 0$
2. **归一**：$\sum p_i = \sum (e^{z_i} / \sum e^{z_j}) = (\sum e^{z_i}) / (\sum e^{z_j}) = 1$
3. **保序**：$z_i > z_j \Rightarrow e^{z_i} > e^{z_j} \Rightarrow p_i > p_j$

### 重要性质：对统一平移不敏感

**定理**：$\operatorname{softmax}(\mathbf{z} + c) = \operatorname{softmax}(\mathbf{z})$，其中 c 是任意常数（给每个分量都加同一个 c）。

**证明（不跳步）**：

$$
\begin{aligned}
\operatorname{softmax}(z_i + c)
&= \frac{e^{z_i + c}}{\sum_j e^{z_j + c}} \\
&= \frac{e^{z_i} \cdot e^c}{\sum_j (e^{z_j} \cdot e^c)} \quad \text{（指数法则 } e^{a+b} = e^a \cdot e^b\text{）} \\
&= \frac{e^{z_i} \cdot e^c}{e^c \cdot \sum_j e^{z_j}} \quad \text{（把 } e^c \text{ 从求和里提出来）} \\
&= \frac{e^{z_i}}{\sum_j e^{z_j}} \quad \text{（分子分母约掉 } e^c\text{）} \\
&= \operatorname{softmax}(z_i)
\end{aligned}
$$

**这个性质是后面"减最大值保稳定"的数学根据**：因为减去同一个常数不改变 softmax 结果，所以我们敢放心地减去 $\max(\mathbf{z})$。

### NumPy 实现（朴素版，先不防溢出）

```python
import numpy as np

def naive_softmax(z):
    """朴素 softmax，不要在生产环境用，会溢出"""
    exps = np.exp(z)
    return exps / np.sum(exps)

z = np.array([2.5, 1.0, 0.3])
p = naive_softmax(z)
print("logits  :", z)              # [2.5 1.  0.3]
print("softmax :", np.round(p, 4)) # [0.714 0.158 0.128]
print("sum     :", np.sum(p))      # 1.0  完美归一
```

### PyTorch 用法

```python
import torch
import torch.nn.functional as F

z = torch.tensor([2.5, 1.0, 0.3])
p = F.softmax(z, dim=-1)
print(p)  # tensor([0.7140, 0.1581, 0.1279])
```

> 注意：**生产中几乎不会单独调用 `F.softmax`**。因为后面要接 log 算交叉熵，分两步会损失精度。直接用 `F.log_softmax` 或 `F.cross_entropy`，背后就是下一节的 LogSumExp。

## 5.2 数值稳定的 Softmax

### 直觉：为什么会溢出？

计算机里的浮点数能表示的范围是有限的。**双精度浮点数最大约 $1.8 \times 10^{308}$，超过就变成 `inf`（无穷大）**。

指数函数涨得特别快：
- $e^1 = 2.718$
- $e^{10} \approx 22026$
- $e^{100} \approx 2.7 \times 10^{43}$
- $e^{1000} \approx 1.97 \times 10^{434}$ → **直接溢出成 `inf`**

更阴险的是：**就算最终结果是有限值，中间步骤也可能溢出**。看这个例子：

```python
>>> import numpy as np
>>> z = np.array([1000, 1001, 1002])
>>> np.exp(z)
__main__:1: RuntimeWarning: overflow encountered in exp
array([inf, inf, inf])
>>> np.exp(z) / np.sum(np.exp(z))
array([nan, nan, nan])   # inf / inf = nan，全废了
```

但其实三个 logits 只差 1、2，softmax 应该给出大致 [0.09, 0.24, 0.67] 这种很正常的概率。问题出在中间步骤。

### 解决办法：减去最大值

**核心思路**：根据 5.1 证明的"对统一平移不敏感"性质，给所有 logits 同时减去它们的最大值 $m = \max(\mathbf{z})$，结果不变。但减完后最大的 logits 变成 0，$e^0 = 1$，绝对不会溢出。

**稳定形式**：

$$
p_i = \frac{e^{z_i - m}}{\sum_j e^{z_j - m}}, \quad m = \max_j z_j
$$

### 推导（为什么减最大值不改变结果？）

令 $m = \max(z_j)$，给每个 z 都减 m（相当于 5.1 里的常数 $c = -m$）：

$$
\begin{aligned}
\frac{e^{z_i - m}}{\sum_j e^{z_j - m}}
&= \frac{e^{z_i} \cdot e^{-m}}{\sum_j (e^{z_j} \cdot e^{-m})} \quad \text{（指数法则）}\\
&= \frac{e^{z_i} \cdot e^{-m}}{e^{-m} \cdot \sum_j e^{z_j}} \quad \text{（提取公因式）}\\
&= \frac{e^{z_i}}{\sum_j e^{z_j}} \quad \text{（约掉 } e^{-m}\text{）}\\
&= \operatorname{softmax}(z_i)
\end{aligned}
$$

**结果完全一样，但数值上安全**：因为 $z_i - m \le 0$ 恒成立（m 是最大值），所以 $e^{z_i-m} \le e^0 = 1$，绝对不会溢出。最小的值（原本最小的 logits）会变成很小的负数，$e^{\text{很大的负数}}$ 趋近 0，但不会出问题。

### 减最大值的另一个好处：防"下溢"

不光上溢（inf），还有"下溢"——很小的正数被计算机当 0 处理。如果所有 $e^{z_i}$ 都下溢成 0，分母就是 0，出现 0/0 = nan。减最大值后，至少 $e^{m-m} = e^0 = 1$ 这一项保底，分母至少是 1，绝不会变成 0。

### NumPy 实现（稳定版）

```python
import numpy as np

def stable_softmax(z):
    """数值稳定的 softmax，生产可用"""
    m = np.max(z)                    # 1) 求最大值
    exps = np.exp(z - m)             # 2) 减最大值后再 exp（关键！）
    return exps / np.sum(exps)       # 3) 归一

# 测试：朴素版会 nan 的例子
z = np.array([1000.0, 1001.0, 1002.0])
print(stable_softmax(z))   # [0.09003057 0.24472847 0.66524096]  正常出结果
```

### PyTorch 内部就是这么做的

```python
import torch
import torch.nn.functional as F

z = torch.tensor([1000.0, 1001.0, 1002.0])
print(F.softmax(z, dim=-1))
# tensor([0.0900, 0.2447, 0.6652])
```

PyTorch / TensorFlow / JAX 的 `softmax` 函数**内部都默认减最大值**，所以你直接调 API 永远是稳定的。但如果你手写（比如论文复现、自定义 CUDA kernel），一定要记得这一步。

### 一句话记忆

> **"先减 max，再 exp，再归一"——这三步是 softmax 的工程标配。**

## 5.3 LogSumExp（LSE）

### 直觉

训练时我们经常需要 **log softmax**（因为交叉熵里要算 $\log p_i$）。如果先算 softmax 再取 log，会有两个问题：

1. **数值精度损失**：softmax 输出在 $(0, 1)$，$p_i$ 很小时 $\log p_i$ 是个大负数，中间 $p_i$ 这一步损失精度
2. **可能 log(0)**：如果 $p_i$ 下溢成 0，$\log(0) = -\infty$，直接炸

更好的办法是**一步到位**：直接算 $\log(\sum e^{z_j})$，而且要稳定。这个东西就叫 **LogSumExp**（LSE）。

### 定义

$$
\operatorname{LSE}(\mathbf{z}) = \log \sum_j e^{z_j} = m + \log \sum_j e^{z_j - m}, \quad m = \max_j z_j
$$

**推导（不跳步，证明这两种写法相等）**：

$$
\begin{aligned}
\log \sum_j e^{z_j}
&= \log \sum_j \left( e^{z_j} \cdot 1 \right) \\
&= \log \sum_j \left( e^{z_j} \cdot \frac{e^m}{e^m} \right) \quad \text{（乘 } e^m / e^m = 1\text{，不改变值）}\\
&= \log \sum_j \left( e^{z_j - m} \cdot e^m \right) \\
&= \log \left( e^m \cdot \sum_j e^{z_j - m} \right) \quad \text{（把 } e^m \text{ 提到求和外面）}\\
&= \log e^m + \log \sum_j e^{z_j - m} \quad \text{（用 log(ab) = log a + log b）}\\
&= m + \log \sum_j e^{z_j - m} \quad \text{（log } e^m = m\text{）}
\end{aligned}
$$

第二项里的 $z_j - m \le 0$，所以 $e^{z_j-m} \le 1$，求和不会溢出，log 也安全。

### log softmax 用 LSE 表达

**定理**：$\log \operatorname{softmax}(\mathbf{z})_i = z_i - \operatorname{LSE}(\mathbf{z})$

**证明**：

$$
\begin{aligned}
\log \operatorname{softmax}(\mathbf{z})_i
&= \log \frac{e^{z_i}}{\sum_j e^{z_j}} \quad \text{（softmax 定义）}\\
&= \log e^{z_i} - \log \sum_j e^{z_j} \quad \text{（log(a/b) = log a − log b）}\\
&= z_i - \operatorname{LSE}(\mathbf{z}) \quad \text{（左边 } \log e^{z_i} = z_i\text{，右边就是 LSE 定义）}
\end{aligned}
$$

**意义**：log softmax 变成了"原 logits 减一个标量"，超简单，且全程稳定。

### LSE 是 softmax 的"光滑版 max"

观察：当 z 里某个分量远大于其他时（比如 $\mathbf{z} = [10, 1, 0]$），$\operatorname{LSE}(\mathbf{z}) \approx 10 = \max(\mathbf{z})$。这是因为 $e^{10}$ 远大于 $e^1$、$e^0$，求和被 $e^{10}$ 主导。

更精确地：

$$
\max(\mathbf{z}) \le \operatorname{LSE}(\mathbf{z}) \le \max(\mathbf{z}) + \log n
$$

- 左边不等式：LSE 至少要包含 max 项本身
- 右边不等式：最多 n 个相等的大值相加，取 log 加上 log n

所以 LSE 是 max 函数的"光滑上界"——它把不可导的 max 变成了可导的函数。这个性质在变分推断、注意力机制里都有用。

### NumPy 实现 + 与朴素版精度对比

```python
import numpy as np

def logsumexp_stable(z):
    """稳定的 LogSumExp"""
    m = np.max(z)
    return m + np.log(np.sum(np.exp(z - m)))

def log_softmax_naive(z):
    """朴素版：先 softmax 再 log（不推荐）"""
    p = np.exp(z) / np.sum(np.exp(z))
    return np.log(p)

def log_softmax_stable(z):
    """稳定版：用 LSE"""
    return z - logsumexp_stable(z)

# 测试 1：普通情况，两者应该接近
z1 = np.array([2.5, 1.0, 0.3])
print("普通情况：")
print("  naive :", log_softmax_naive(z1))     # [-0.846 -2.346 -3.046]
print("  stable:", log_softmax_stable(z1))    # [-0.846 -2.346 -3.046]
print("  diff  :", np.max(np.abs(log_softmax_naive(z1) - log_softmax_stable(z1))))

# 测试 2：大数情况，朴素版直接挂
z2 = np.array([1000.0, 1001.0, 1002.0])
print("\n大数情况（朴素版会 nan）：")
try:
    print("  naive :", log_softmax_naive(z2))  # [nan nan nan]
except:
    pass
print("  stable:", log_softmax_stable(z2))    # [-1.105 -0.105  0.895]

# 测试 3：极小概率，朴素版精度丢失
z3 = np.array([50.0, 0.0, 0.0])  # 第一个概率压倒性大
print("\n极小概率情况：")
print("  naive :", log_softmax_naive(z3))     # [-0.       -50.       -50.      ]  第一个被截断成 0
print("  stable:", log_softmax_stable(z3))    # [-1.9e-22 -50.       -50.      ]  保留微小差异
```

**精度差异说明**：
- 大数情况：朴素版中间 `np.exp(1000)` 溢出成 inf，然后 inf/inf = nan
- 极小概率：朴素版先算 softmax 得到 [1, 0, 0]（后两个被下溢成 0），取 log 得到 $[0, -\infty, -\infty]$；稳定版保留了真实的 $[-1.9 \times 10^{-22}, -50, -50]$

### PyTorch 用法

```python
import torch
import torch.nn.functional as F

z = torch.tensor([2.5, 1.0, 0.3])

# 1) LogSumExp
print(torch.logsumexp(z, dim=-1))           # tensor(3.3460)

# 2) log softmax（推荐用这个，而不是先 softmax 再 log）
print(F.log_softmax(z, dim=-1))
# tensor([-0.8460, -2.3460, -3.0460])

# 3) 直接算交叉熵（内部用 log_softmax，最稳）
target = torch.tensor(0)  # 正确类别是第 0 类
print(F.cross_entropy(z.unsqueeze(0), target.unsqueeze(0)))
# tensor(0.8460)  等于 -log_softmax(z)[0]
```

### 一句话记忆

> **"要 log softmax，就用 $\mathbf{z} - \operatorname{LSE}(\mathbf{z})$，绝不要先 softmax 再 log。"**

## 5.4 温度参数（Temperature）

### 直觉

Softmax 输出的分布有"尖锐"和"平坦"之分。看两组 logits：

- $\mathbf{z} = [10, 1, 0]$：softmax $\approx [0.9999, 0.0001, 0.0000]$，**很尖**——几乎全押在第一个
- $\mathbf{z} = [1.0, 0.9, 0.8]$：softmax $\approx [0.367, 0.332, 0.301]$，**很平**——三个差不多

"尖"意味着模型很自信，"平"意味着模型在犹豫。**温度参数 T 就是一个旋钮，让我们在不改 logits 相对大小的情况下，调节这个尖平程度**。

类比：**调水龙头控制水花大小**。
- T 小 → 水流集中成一股（尖锐，自信）
- T 大 → 水花散开一片（平坦，犹豫）

### 定义

引入温度 $T > 0$，把 softmax 改成：

$$
p_i(T) = \frac{e^{z_i / T}}{\sum_j e^{z_j / T}}
$$

**T 的作用是把所有 logits 同时除以 T**。因为 softmax 对统一平移不敏感，但对统一缩放**敏感**：

- $T = 1$：原样不变
- $T < 1$：$z_i/T$ 比 $z_i$ 更"拉开"，差距被放大 → 分布更尖
- $T > 1$：$z_i/T$ 比 $z_i$ 更"压缩"，差距被缩小 → 分布更平

### 极限情况

**$T \to 0^+$（趋近 0）**：

$z_i/T \to \pm\infty$，最大的那个 $z_k/T \to +\infty$ 远大于其他，$e^{z_k/T}$ 主导求和，所以 $p_k \to 1$，其他 $\to 0$。这就是 **argmax**（贪心选最大）。

$$
\lim_{T \to 0^+} \operatorname{softmax}(\mathbf{z}/T)_i = \begin{cases} 1 & \text{if } i = \arg\max \mathbf{z} \\ 0 & \text{otherwise} \end{cases}
$$

**$T \to +\infty$**：

$z_i/T \to 0$，所有 $e^{z_i/T} \to e^0 = 1$，$p_i \to 1/n$。退化成**均匀分布**（瞎猜）。

$$
\lim_{T \to +\infty} \operatorname{softmax}(\mathbf{z}/T)_i = \frac{1}{n}
$$

### 不同 T 值的分布对比表

取 logits z = [2.5, 1.0, 0.3]（n=3），看 T 变化时三个概率怎么变：

| T | $p_1$（猫） | $p_2$（狗） | $p_3$（鸟） | max 概率 | 分布形态 | 直觉 |
|---|----------|----------|----------|----------|----------|------|
| **0.5** | 0.8811 | 0.1106 | 0.0083 | 0.8811 | 非常尖 | "几乎肯定是猫" |
| **0.8** | 0.7773 | 0.1902 | 0.0325 | 0.7773 | 较尖 | "大概率是猫" |
| **1.0** | 0.7140 | 0.1581 | 0.1279 | 0.7140 | 基准 | "倾向猫但不确定" |
| **1.5** | 0.6321 | 0.2567 | 0.1112 | 0.6321 | 略平 | "猫狗都可能" |
| **2.0** | 0.5764 | 0.3002 | 0.1234 | 0.5764 | 更平 | "猫略优，狗紧追" |
| **5.0** | 0.4249 | 0.3468 | 0.2283 | 0.4249 | 很平 | "三个都差不多" |
| **10** | 0.3810 | 0.3536 | 0.2654 | 0.3810 | 接近均匀 | "基本瞎猜" |
| **$\infty$** | 0.3333 | 0.3333 | 0.3333 | 0.3333 | 完全均匀 | "纯随机" |
| **$\to 0$** | 1.0000 | 0.0000 | 0.0000 | 1.0000 | argmax | "绝对肯定" |

**怎么读这张表**：
- T 从大往小调：最大概率从 0.333 → 0.381 → 0.425 → … → 0.714 → 0.881 → 1.000，**单调上升**，分布越来越尖
- 三个概率的"差距"也越来越大：T=10 时 0.381 vs 0.265（差 0.12），T=0.5 时 0.881 vs 0.008（差 0.87）
- T=1 是"原汁原味"的 softmax，$T<1$ 是"放大镜"，$T>1$ 是"模糊镜"

### NumPy 代码：温度扫描

```python
import numpy as np

def softmax_with_temperature(z, T):
    """带温度的稳定 softmax"""
    z_T = z / T
    m = np.max(z_T)
    exps = np.exp(z_T - m)
    return exps / np.sum(exps)

z = np.array([2.5, 1.0, 0.3])
print(f"{'T':>6} | {'p1':>8} {'p2':>8} {'p3':>8} | {'max':>8}")
print("-" * 50)
for T in [0.5, 0.8, 1.0, 1.5, 2.0, 5.0, 10.0]:
    p = softmax_with_temperature(z, T)
    print(f"{T:>6.2f} | {p[0]:>8.4f} {p[1]:>8.4f} {p[2]:>8.4f} | {p.max():>8.4f}")
```

### PyTorch 用法


### 应用场景

- **知识蒸馏**：教师网络用高温 T_soft（如 T=4）产生"软标签"，让学生网络学"暗知识"（不是只有猫对，狗也有一点点像）
- **LLM 采样**：低 T 让模型更确定（适合代码生成），高 T 让模型更有创意（适合写诗），通常 $T \in [0.7, 1.3]$
- **RL 中的探索**：策略网络的 logits 除以 T，T 大时动作分布更平均，鼓励探索

### 一句话记忆

> **"温度 T 是分布的'尖平旋钮'：T 小越尖越贪心，T 大越平越随机。"**

## 5.5 交叉熵（Cross-Entropy）

### 直觉

**交叉熵衡量"用模型分布 p 去编码真实分布 q，平均每个事件要花多少比特"**。

类比：你写一本"猫狗鸟识别手册"，要给每种动物分配码长。理想做法是按真实出现频率分配（频繁的给短码，罕见的给长码）。但如果你**以为的频率（模型 p）和真实频率（数据 q）不一致**，你的码本就不是最优的，平均码长就会比理论最小值长。

这个"用 p 编 q 平均要花的码长"就是交叉熵 $H(\mathbf{q}, \mathbf{p})$。p 越接近 q，$H(\mathbf{q}, \mathbf{p})$ 越小；$\mathbf{p} = \mathbf{q}$ 时达到最小值 $H(\mathbf{q})$。

### 定义

给定真实分布 $\mathbf{q} = (q_1, \ldots, q_V)$ 和模型分布 $\mathbf{p} = (p_1, \ldots, p_V)$，**交叉熵**：

$$
H(\mathbf{q}, \mathbf{p}) = -\sum_i q_i \log p_i
$$

注意：
- log 里的 $p_i$ 是模型预测概率（要学习优化它）
- 前面 $q_i$ 是真实分布的权重（数据集给的，固定）
- 底数通常取 e（自然对数），单位是 nat；取 2 时单位是 bit。机器学习里默认 ln。

### One-hot 简化推导（重点！不跳步）

**场景**：分类任务中，真实标签通常用 one-hot 编码。比如 3 类分类、正确答案是第 0 类（猫），则 q = (1, 0, 0)。

**目标**：把 $H(\mathbf{q}, \mathbf{p})$ 在 q 是 one-hot 时化简成只剩一项。

**推导（每一步都写出来）**：

假设正确类别是 y（索引），则 q 是 one-hot：$q_y = 1$，其他 $q_i = 0$ ($i \ne y$)。

$$
\begin{aligned}
H(\mathbf{q}, \mathbf{p}) &= -\sum_i q_i \log p_i \quad \text{（交叉熵定义）}\\
&= -\big( q_1 \log p_1 + q_2 \log p_2 + \cdots + q_y \log p_y + \cdots + q_V \log p_V \big) \quad \text{（展开求和）}\\
&= -\big( 0 \cdot \log p_1 + 0 \cdot \log p_2 + \cdots + 1 \cdot \log p_y + \cdots + 0 \cdot \log p_V \big) \\
&\quad \text{（代入 one-hot：} q_i = 0 \text{ 对 } i \ne y,\ q_y = 1\text{）}\\
&= -\big( 0 + 0 + \cdots + \log p_y + \cdots + 0 \big) \quad \text{（0 乘任何数都是 0）}\\
&= -\log p_y \quad \text{（只剩一项）}
\end{aligned}
$$

**最终结果**：

$$
\boxed{\mathcal{L} = -\log p_y}
$$

**这就是分类任务最常用的损失函数：负对数似然（NLL）**。$p_y$ 是模型给"正确类别"分配的概率，我们希望它越大越好，所以 $-\log p_y$ 越小越好。

### 为什么是 $-\log$？直觉

- 模型预测对（$p_y \to 1$）：$-\log(1) = 0$，损失接近 0，不惩罚
- 模型预测错（$p_y \to 0$）：$-\log(0) = +\infty$，损失爆炸，狠狠惩罚
- 中间值：$-\log(0.5) \approx 0.693$，$-\log(0.1) \approx 2.303$，$p_y$ 越小惩罚越大

```
损失曲线 L = -log(p_y)
  L
  ↑
3 ┤                          ×  (p=0.05, L=3.00)
  │                      ×
2 ┤                  ×       (p=0.14, L=2.00)
  │              ×
1 ┤          ×               (p=0.37, L=1.00)
  │      ×
0 ┤  ×──────────────────  (p=1.00, L=0.00)
  └──┬──┬──┬──┬──┬──┬──→ p_y
     0  .2 .4 .6 .8  1
```

### NumPy 实现

```python
import numpy as np

def cross_entropy_onehot(p, y):
    """
    p : 模型预测的概率向量 (n,)
    y : 正确类别的索引（整数）
    """
    eps = 1e-12  # 防 log(0)
    return -np.log(p[y] + eps)

def cross_entropy_general(q, p):
    """一般形式：q 和 p 都是分布"""
    eps = 1e-12
    return -np.sum(q * np.log(p + eps))

# 模型预测
p = np.array([0.7, 0.2, 0.1])  # 模型说"70% 猫, 20% 狗, 10% 鸟"
y = 0                           # 真实是猫

# one-hot 形式
print("one-hot 简化:", cross_entropy_onehot(p, y))   # 0.3567

# 一般形式（验证两者相等）
q = np.array([1.0, 0.0, 0.0])
print("一般形式     :", cross_entropy_general(q, p)) # 0.3567

# 模型预测错时损失爆涨
p_wrong = np.array([0.1, 0.2, 0.7])  # 模型错以为是鸟
print("预测错时     :", cross_entropy_onehot(p_wrong, y))  # 2.3026
```

### 直接从 logits 算（推荐）

实际工程里**不会**先 softmax 再 log，而是直接用 logits 算（内部用 LogSumExp 保稳定）：

```python
import numpy as np

def cross_entropy_from_logits(z, y):
    """z: logits, y: 正确类别索引"""
    m = np.max(z)
    log_sum_exp = m + np.log(np.sum(np.exp(z - m)))
    return -z[y] + log_sum_exp   # = -(z_y - LSE) = -log_softmax(z)[y]

z = np.array([2.5, 1.0, 0.3])
y = 0
print("从 logits 算:", cross_entropy_from_logits(z, y))  # 0.8460
```

### PyTorch 用法

```python
import torch
import torch.nn.functional as F

# 方式 1: F.cross_entropy（最推荐，自动处理稳定性和 one-hot）
logits = torch.tensor([[2.5, 1.0, 0.3]])  # shape (batch=1, classes=3)
target = torch.tensor([0])                  # 正确类别索引
loss = F.cross_entropy(logits, target)
print(loss)  # tensor(0.8460)

# 方式 2: 手动算（验证理解）
log_softmax = F.log_softmax(logits, dim=-1)
print(-log_softmax[0, 0])  # tensor(0.8460)

# 方式 3: NLLLoss + log_softmax（等价）

print(loss)  # tensor(0.8460)
```

> **"分类损失 = $-\log$(模型给正确类别打的概率)。one-hot 把求和压缩成一项，剩下的就是 NLL。"**

## 5.6 信息熵（Information Entropy）

### 直觉

**信息熵衡量"一个分布本身有多不确定"**。

- **抛硬币（均匀）**：50% 正面 50% 反面，你完全猜不到结果 → 不确定性最大
- **抛硬币（$p=0.99$ 正面）**：几乎肯定是正面 → 不确定性很小
- **掷骰子（均匀）**：6 个面等概率 → 比抛硬币更不确定（选项更多）

熵就是这个"不确定性大小"的数值化度量。

### 直觉对比：抛硬币 vs 掷骰子

**抛均匀硬币** p = (0.5, 0.5)：

$$
H = -(0.5 \log_2 0.5 + 0.5 \log_2 0.5) = -\log_2 0.5 = \log_2 2 = 1 \text{ bit}
$$

意思是：平均要 1 个二进制位才能编码结果（正/反）。

**掷均匀骰子** p = (1/6, 1/6, 1/6, 1/6, 1/6, 1/6)：

$$
H = -\sum_{i=1}^{6} \frac{1}{6} \log_2 \frac{1}{6} = -\log_2 \frac{1}{6} = \log_2 6 \approx 2.585 \text{ bit}
$$

意思是：平均要约 2.585 个位才能编码结果（因为 $2^2 = 4 < 6 < 8 = 2^3$，所以需要 3 位，但平均下 来 2.585 位够用，理论下界）。

**对比**：骰子的熵（2.585）> 硬币的熵（1.0），说明骰子更不确定——这和直觉一致。

**关键洞察**：熵只取决于**分布本身**，和"用什么编码"无关。它是分布的固有属性。

### 定义

给定分布 $\mathbf{p} = (p_1, \ldots, p_V)$，**信息熵**：

$$
H(\mathbf{p}) = -\sum_i p_i \log p_i
$$

**几点说明**：
- log 底数：取 2 单位是 bit，取 e 单位是 nat，机器学习里默认 e（nat）
- 约定 $0 \cdot \log 0 = 0$（极限意义下 $\lim_{x \to 0^+} x \log x = 0$）
- 熵 $\ge 0$（因为 $p_i \le 1$，$\log p_i \le 0$，前面加负号变非负）

### 熵的几个性质

1. **非负**：$H(\mathbf{p}) \ge 0$
2. **最大值**：均匀分布时最大，$H_{\max} = \log n$（n 是类别数）
3. **最小值**：one-hot 分布时为 0（完全确定，没有不确定性）
4. **对称性**：$H(\mathbf{p})$ 只看概率值，不看顺序——$(0.5, 0.3, 0.2)$ 和 $(0.2, 0.5, 0.3)$ 熵一样

### 推导：为什么最大熵是 log n？

用拉格朗日乘子法（这里只给思路，不深入）。在约束 $\sum p_i = 1$ 下最大化 $H(\mathbf{p}) = -\sum_i p_i \log p_i$：

对每个 $p_i$ 求偏导并令其等于 0：

$$
\frac{\partial}{\partial p_i} \left( -\sum_i p_i \log p_i + \lambda (\sum p_i - 1) \right) = -(\log p_i + 1) + \lambda = 0
$$

解得 $\log p_i = \lambda - 1$，即所有 $p_i$ 相等。结合 $\sum p_i = 1$ 得 $p_i = 1/n$。此时：

$$
H_{\max} = -\sum_{i=1}^n \frac{1}{n} \log \frac{1}{n} = -\log \frac{1}{n} = \log n
$$

### NumPy 代码：几种分布的熵对比

```python
import numpy as np

def entropy(p, base=np.e):
    """计算信息熵"""
    p = np.array(p, dtype=np.float64)
    p = p[p > 0]  # 滤掉 0，避免 log(0)
    return -np.sum(p * np.log(p)) / np.log(base)  # 换底公式

# 1) 确定分布（one-hot）：熵 = 0
print("one-hot (1, 0, 0):", entropy([1.0, 0.0, 0.0]))                # 0.0

# 2) 均匀硬币：熵 = ln 2 ≈ 0.693 nat
print("均匀硬币 (0.5, 0.5):", entropy([0.5, 0.5]))                   # 0.693 nat

# 3) 偏置硬币：熵更小
print("偏置硬币 (0.99, 0.01):", entropy([0.99, 0.01]))               # 0.056 nat

# 4) 均匀三分类：熵 = ln 3 ≈ 1.099
print("均匀三分类 (1/3, 1/3, 1/3):", entropy([1/3, 1/3, 1/3]))       # 1.099 nat

# 5) 均匀骰子：熵 = ln 6 ≈ 1.792 nat
print("均匀骰子:", entropy([1/6]*6))                                  # 1.792 nat

# 6) 不均匀分布
print("不均匀 (0.7, 0.2, 0.1):", entropy([0.7, 0.2, 0.1]))           # 0.802 nat

# 用 bit 单位
print("\n用 bit 单位：")
print("均匀硬币:", entropy([0.5, 0.5], base=2))   # 1.0 bit
print("均匀骰子:", entropy([1/6]*6, base=2))      # 2.585 bit
```

### PyTorch 用法

```python
import torch

def entropy_torch(p, dim=-1):
    """计算熵（输入是概率分布）"""
    p = p.clamp(min=1e-12)  # 防 log(0)
    return -(p * p.log()).sum(dim=dim)

p = torch.tensor([0.5, 0.5])
print(entropy_torch(p))  # tensor(0.6931)

# 从 logits 算熵（先用 softmax 转概率，再算熵）

```

## 5.7 KL 散度（Kullback-Leibler Divergence）

### 直觉

**KL 散度衡量"两个分布 p 和 q 差多远"**，但**它不是距离**——因为它不对称。

类比：你想从北京去上海，可以坐飞机（快但贵）或火车（慢但便宜）。"飞机替代火车"和"火车替代飞机"的代价不同——前者省钱省时间但需多花钱买票，后者省钱但要多花一天时间。**"用 p 替代 q"和"用 q 替代 p"代价不对称**，就像 KL 散度。

### 定义

给定两个分布 $\mathbf{q}$ 和 $\mathbf{p}$（都是概率向量），**KL 散度**（也叫相对熵）：

$$
D_{KL}(\mathbf{q} \lVert \mathbf{p}) = \sum_i q_i \log \frac{q_i}{p_i}
$$

连续版本：$D_{KL}(\mathbf{q} \lVert \mathbf{p}) = \int q(x) \log(q(x)/p(x)) dx$

**几点关键性质**：
1. **非负**：$D_{KL}(\mathbf{q} \lVert \mathbf{p}) \ge 0$（吉布斯不等式保证）
2. **$D_{KL}(\mathbf{q} \| \mathbf{p}) = 0$ 当且仅当 $\mathbf{q} = \mathbf{p}$**（完全相同）
3. **不对称**：$D_{KL}(\mathbf{q} \lVert \mathbf{p}) \ne D_{KL}(\mathbf{p} \lVert \mathbf{q})$，一般情况差很多
4. **不是真正的距离**：因为不对称，不满足度量空间的对称性公理

### 推导：$D_{KL} \ge 0$（吉布斯不等式）

利用 $-\log x$ 是凸函数 + Jensen 不等式：

$$
\begin{aligned}
D_{KL}(\mathbf{q} \lVert \mathbf{p}) &= \sum_i q_i \log \frac{q_i}{p_i} \\
&= -\sum_i q_i \log \frac{p_i}{q_i} \quad \text{（} \log(a/b) = -\log(b/a)\text{）} \\
&\ge -\log \left( \sum_i q_i \cdot \frac{p_i}{q_i} \right) \quad \text{（Jensen 不等式，} -\log \text{ 是凸函数）}\\
&= -\log \left( \sum_i p_i \right) \quad \text{（} q_i \text{ 约掉）}\\
&= -\log 1 \quad \text{（p 是分布，和为 1）}\\
&= 0
\end{aligned}
$$

等号成立当且仅当 p_i/q_i 对所有 i 都相等，即 p = q。

### 交叉熵 = 熵 + KL（重要恒等式）

**定理**：$H(\mathbf{q}, \mathbf{p}) = H(\mathbf{q}) + D_{KL}(\mathbf{q} \lVert \mathbf{p})$

**证明（不跳步）**：

$$
\begin{aligned}
H(\mathbf{q}, \mathbf{p}) &= -\sum_i q_i \log p_i \quad \text{（交叉熵定义）}\\
&= -\sum_i q_i \log \left( p_i \cdot \frac{q_i}{q_i} \right) \quad \text{（乘 } q_i/q_i = 1\text{，不改变值）}\\
&= -\sum_i q_i \left( \log q_i + \log \frac{p_i}{q_i} \right) \quad \text{（} \log(ab) = \log a + \log b\text{）}\\
&= -\sum_i q_i \log q_i - \sum_i q_i \log \frac{p_i}{q_i} \quad \text{（拆成两个求和）}\\
&= -\sum_i q_i \log q_i + \sum_i q_i \log \frac{q_i}{p_i} \quad \text{（第二项 } \log(p_i/q_i) = -\log(q_i/p_i)\text{）}\\
&= H(\mathbf{q}) + D_{KL}(\mathbf{q} \lVert \mathbf{p}) \quad \text{（第一项是熵，第二项是 KL）}
\end{aligned}
$$

**这个等式极其重要**：

$$
\boxed{H(\mathbf{q}, \mathbf{p}) = H(\mathbf{q}) + D_{KL}(\mathbf{q} \lVert \mathbf{p})}
$$

**意义**：
- 训练时 q 是固定的（数据集决定），所以 $H(\mathbf{q})$ 是常数
- 最小化交叉熵 $H(\mathbf{q}, \mathbf{p})$ $\Leftrightarrow$ 最小化 KL 散度 $D_{KL}(\mathbf{q} \| \mathbf{p})$ $\Leftrightarrow$ 让 p 趋近 q
- 这就是"最小化交叉熵 = 最大似然估计 = 让模型分布逼近数据分布"的数学根据

### KL 不对称：具体数值例子

取两个分布：

- q = (0.7, 0.2, 0.1)（真实分布，主要在第 0 类）
- p = (0.4, 0.4, 0.2)（模型分布，更均匀）

**计算 $D_{KL}(\mathbf{q} \| \mathbf{p})$**（用 q 当基准，看 q 偏离 p 多少）：

$$
\begin{aligned}
D_{KL}(\mathbf{q} \lVert \mathbf{p}) &= \sum_i q_i \log \frac{q_i}{p_i} \\
&= 0.7 \log\frac{0.7}{0.4} + 0.2 \log\frac{0.2}{0.4} + 0.1 \log\frac{0.1}{0.2} \\
&= 0.7 \log 1.75 + 0.2 \log 0.5 + 0.1 \log 0.5 \\
&= 0.7 \times 0.5596 + 0.2 \times (-0.6931) + 0.1 \times (-0.6931) \\
&= 0.3917 - 0.1386 - 0.0693 \\
&= 0.1838 \text{ nat}
\end{aligned}
$$

**计算 $D_{KL}(\mathbf{p} \| \mathbf{q})$**（用 p 当基准，看 p 偏离 q 多少）：

$$
\begin{aligned}
D_{KL}(\mathbf{p} \lVert \mathbf{q}) &= \sum_i p_i \log \frac{p_i}{q_i} \\
&= 0.4 \log\frac{0.4}{0.7} + 0.4 \log\frac{0.4}{0.2} + 0.2 \log\frac{0.2}{0.1} \\
&= 0.4 \log\frac{4}{7} + 0.4 \log 2 + 0.2 \log 2 \\
&= 0.4 \times (-0.5596) + 0.4 \times 0.6931 + 0.2 \times 0.6931 \\
&= -0.2238 + 0.2773 + 0.1386 \\
&= 0.1921 \text{ nat}
\end{aligned}
$$

**对比**：$D_{KL}(\mathbf{q} \lVert \mathbf{p}) = 0.1838 \ne D_{KL}(\mathbf{p} \lVert \mathbf{q}) = 0.1921$，**确实不对称**。差距看起来不大，但这是因为两个分布还比较接近；如果差距大，两个方向的 KL 会差几个数量级。

### 两个方向的 KL 直觉

**前向 KL（$D_{KL}(\mathbf{q} \lVert \mathbf{p})$，q 在分子）**：
- $q_i = 0$ 的项贡献为 0（$0 \cdot \log 0 = 0$），不关心
- $q_i$ 大的项权重大，要求 p 在这些位置不能太小，否则 $\log(q_i/p_i)$ 会很大
- 行为：**"覆盖"模式**——p 必须覆盖 q 的所有高质量区域，否则狠狠惩罚
- 应用：最大似然估计（让模型覆盖数据分布）

**反向 KL（$D_{KL}(\mathbf{p} \lVert \mathbf{q})$，p 在分子）**：
- $p_i$ 大的位置要求 q 也不能太小
- p 倾向于集中在 q 高的地方，避开 q 低的地方
- 行为：**"寻峰"模式**——p 喜欢躲到 q 的某个峰下，哪怕漏掉其他峰
- 应用：变分推断（VAE 用反向 KL，会出现"模式坍缩"）

### NumPy 实现：对称性验证

```python
import numpy as np

def kl_divergence(q, p):
    """计算 D_KL(q || p)"""
    q = np.array(q, dtype=np.float64)
    p = np.array(p, dtype=np.float64)
    eps = 1e-12
    mask = q > 0  # q=0 的项贡献为 0
    return np.sum(q[mask] * np.log(q[mask] / (p[mask] + eps)))

q = np.array([0.7, 0.2, 0.1])
p = np.array([0.4, 0.4, 0.2])

kl_qp = kl_divergence(q, p)  # q || p
kl_pq = kl_divergence(p, q)  # p || q

print(f"D_KL(q||p) = {kl_qp:.4f} nat")  # 0.1838
print(f"D_KL(p||q) = {kl_pq:.4f} nat")  # 0.1921
print(f"两者差异: {abs(kl_qp - kl_pq):.4f}")  # 0.0083
print(f"是否相等: {np.isclose(kl_qp, kl_pq)}")  # False

# 验证恒等式 H(q,p) = H(q) + D_KL(q||p)
def entropy(p):
    p = np.array(p)
    p = p[p > 0]
    return -np.sum(p * np.log(p))

def cross_entropy(q, p):
    q = np.array(q)
    p = np.array(p)
    eps = 1e-12
    return -np.sum(q * np.log(p + eps))

H_q = entropy(q)
H_qp = cross_entropy(q, p)
print(f"\n验证 H(q,p) = H(q) + D_KL(q||p):")
print(f"  H(q)        = {H_q:.4f}")
print(f"  D_KL(q||p)  = {kl_qp:.4f}")
print(f"  H(q)+KL     = {H_q + kl_qp:.4f}")
print(f"  H(q,p)      = {H_qp:.4f}")  # 应该相等
```

### PyTorch 用法

```python
import torch
import torch.nn.functional as F

# PyTorch 的 F.kl_div 是 D_KL(input || target)，注意参数顺序
# input 和 target 都要求是 log probabilities
log_p = torch.log(torch.tensor([0.4, 0.4, 0.2]))
q = torch.tensor([0.7, 0.2, 0.1])

# F.kl_div 计算 sum( target * (log(target) - input) ) = D_KL(target || input)
# 等价于 D_KL(q || p)
kl = F.kl_div(log_p, q, reduction='sum')
print(kl)  # tensor(0.1838)
```

> **坑点**：`F.kl_div` 的第一个参数是 **log 概率**，第二个是原始概率，计算的是 $D_{KL}(\text{第二参数} \lVert \text{第一参数})$。这个 API 设计反直觉，用错会得到反向 KL。

### KL 散度的应用

- **VAE**：损失 = 重构损失 + $D_{KL}(q(z|x) \lVert p(z))$，让编码分布接近先验
- **知识蒸馏**：学生学教师的软标签，最小化 $D_{KL}(\text{学生} \lVert \text{教师})$
- **变分推断**：用简单分布 q 近似复杂后验 p，最小化 $D_{KL}(\mathbf{q} \lVert \mathbf{p})$
- **强化学习**：TRPO/PPO 用 $D_{KL}(\text{新策略} \lVert \text{旧策略})$ 限制更新步长

### 一句话记忆

> **"KL 不对称！$H(\mathbf{q}, \mathbf{p}) = H(\mathbf{q}) + D_{KL}(\mathbf{q} \lVert \mathbf{p})$，最小化交叉熵 = 最小化 KL = 让模型逼近数据。"**

## 5.8 Perplexity（PPL，困惑度）

### 直觉

**困惑度（Perplexity, PPL）是语言模型最常用的评估指标，表示"模型对一段文本平均有多困惑"**。

类比：你参加一个填空题考试，每道题有 V 个选项（V 是词表大小）。
- 完美模型（每题都 100% 猜对）：PPL = 1，毫无困惑
- 瞎猜模型（每个选项等概率）：PPL = V，完全困惑
- 实际模型：PPL 介于 1 和 V 之间，越小越好

PPL 可以理解成"模型在每一步平均在多少个词之间犹豫"。

### 定义

给定一段长度为 N 的文本，模型在每个位置给出概率 p(xₜ | x_<t)，则**困惑度**：

$$
\operatorname{PPL} = \exp\left( -\frac{1}{N} \sum_{t=1}^{N} \log p(x_t | x_{<t}) \right) = \exp(\bar{L})
$$

其中 L̄ 是平均交叉熵损失（每个 token 的平均负对数似然）。

**推导（不跳步）**：

1. 模型对整段文本的对数似然：

$$
\log p(x_{1:N}) = \sum_{t=1}^{N} \log p(x_t | x_{<t})
$$

2. 平均每个 token 的对数似然：

$$
\frac{1}{N} \log p(x_{1:N}) = \frac{1}{N} \sum_{t=1}^{N} \log p(x_t | x_{<t}) = -\bar{L}
$$

3. 几何平均概率：

$$
\left( \prod_{t=1}^{N} p(x_t | x_{<t}) \right)^{1/N} = \exp\left( \frac{1}{N} \log p(x_{1:N}) \right) = \exp(-\bar{L})
$$

4. 困惑度是这个几何平均的倒数（"越困惑"="概率越低"）：

$$
\operatorname{PPL} = \frac{1}{\exp(-\bar{L})} = \exp(\bar{L})
$$

**关键关系**：

$$
\boxed{\operatorname{PPL} = e^{\bar{L}}}
$$

### 数值感觉

| 平均损失 $\bar{L}$ | $\operatorname{PPL} = e^{\bar{L}}$ | 含义 |
|------------|-----------|------|
| 0.0 | 1.0 | 完美预测（每词概率都是 1） |
| 0.5 | 1.65 | 很好 |
| 1.0 | 2.72 | 不错 |
| 2.0 | 7.39 | 一般 |
| 5.0 | 148.4 | 较差 |
| 10.0 | 22026 | 很差 |
| $\log(V)$ | V | 瞎猜水平 |

参考：现代 LLM 在英文文本上 PPL 大约 1.5~3，早期 n-gram 模型 PPL 在 50~200。

### NumPy 实现

```python
import numpy as np

def perplexity(neg_log_probs):
    """
    neg_log_probs: 每个 token 的负对数似然，shape (N,)
    """
    avg_loss = np.mean(neg_log_probs)
    return np.exp(avg_loss)

# 模拟一段文本的 5 个 token
# 模型给每个正确 token 的负对数概率
neg_log_probs = np.array([0.5, 1.2, 0.8, 2.0, 1.5])
ppl = perplexity(neg_log_probs)
print(f"平均损失: {neg_log_probs.mean():.4f}")
print(f"PPL: {ppl:.4f}")  # 3.4848

# 直觉：模型平均在 ~3.5 个词之间犹豫

# 对比：完美模型
print(f"完美 PPL: {perplexity(np.array([0.0]*5))}")  # 1.0

# 对比：瞎猜模型（10 个词的词表，每词概率 0.1，NLL = -log(0.1) = 2.3026）
print(f"瞎猜 PPL (V=10): {perplexity(np.array([np.log(10)]*5))}")  # 10.0
```

## 5.9 Top-k 与 Top-p 采样

### 直觉

LLM 生成文本时，每一步要在词表（V 可能 5 万~10 万）里选下一个 token。怎么选？

- **Greedy（贪心）**：永远选概率最大的。问题：生成重复、机械
- **纯随机采样**：按整个分布随机抽。问题：长尾里有很多奇怪 token，会乱
- **Top-k**：只在概率最高的 k 个里抽（k=50 是常见值）。**固定数量**
- **Top-p（nucleus）**：只抽"累计概率达到 p"的最小一批 token（p=0.9 是常见值）。**动态数量**

类比：你在图书馆找书看下一本。
- Greedy：永远借借阅次数最多的那本——无聊
- 纯随机：闭眼随便抽——可能抽到《母猪产后护理》
- Top-k：只在前 k 名热销书里挑——稳定但有时局限
- Top-p：只在"前几名加起来覆盖 90% 销量"的书里挑——动态适应，热销集中时少挑几本，热销分散时多挑几本

### Greedy（贪心解码）

**算法**：每步 argmax，选概率最大的 token。

```python
import numpy as np

def greedy_sample(logits):
    """贪心：永远选最大"""
    return np.argmax(logits)

logits = np.array([2.5, 1.0, 0.3, 0.1, -1.0])
print(greedy_sample(logits))  # 0
```

### Top-k 采样

**算法**：
1. 找出 logits 中最大的 k 个
2. 把这 k 个之外的 logits 设成 $-\infty$（softmax 后概率变 0）
3. 在这 k 个里按 softmax 概率随机抽一个

**为什么设成 $-\infty$ 而不是直接删？**：保留向量长度，方便向量化计算。

```python
import numpy as np

def top_k_sampling(logits, k=2, temperature=1.0):
    """
    logits: shape (V,)
    k: 保留的最大数量
    """
    logits = logits / temperature
    
    # 1) 找 top-k 的索引
    top_k_idx = np.argsort(logits)[-k:]  # 升序排，取最后 k 个
    
    # 2) 构造 masked logits：非 top-k 设成 -inf
    masked = np.full_like(logits, -np.inf)
    masked[top_k_idx] = logits[top_k_idx]
    
    # 3) softmax（自动归一，-inf 对应项变 0）
    m = np.max(masked)  # 防 overflow，max 一定是有限值
    exps = np.exp(masked - m)
    exps[masked == -np.inf] = 0  # 显式处理 inf - inf 的 nan
    probs = exps / np.sum(exps)
    
    # 4) 按概率采样
    return np.random.choice(len(logits), p=probs)

np.random.seed(42)
logits = np.array([2.5, 1.0, 0.3, 0.1, -1.0])
print("Top-k=2 采样 10 次:")
for _ in range(10):
    print(top_k_sampling(logits, k=2), end=" ")
# 永远只会是 0 或 1（前两个）
```

### Top-p 采样（Nucleus Sampling）

**算法**：
1. 按概率从大到小排序
2. 累加概率，找到第一个使累计概率 $\ge p$ 的位置
3. 保留这一位及之前的所有 token，其他设成 $-\infty$
4. 在保留的 token 里按归一化概率采样

**和 Top-k 的区别**：k 是固定数量，p 是固定累计概率阈值。当分布很集中（一个 token 占 90%）时，top-p 可能只保留 1 个；当分布很平时，top-p 可能保留几十个。

```python
import numpy as np

def top_p_sampling(logits, p=0.9, temperature=1.0):
    """
    logits: shape (V,)
    p: 累计概率阈值
    """
    logits = logits / temperature
    
    # 1) softmax 得到原始概率
    m = np.max(logits)
    exps = np.exp(logits - m)
    probs = exps / np.sum(exps)
    
    # 2) 按概率降序排序
    sorted_idx = np.argsort(probs)[::-1]  # 降序
    sorted_probs = probs[sorted_idx]
    
    # 3) 累计概率
    cumulative = np.cumsum(sorted_probs)
    
    # 4) 找到第一个累计 ≥ p 的位置（保留它和它之前的）
    # 用 searchsorted 找第一个 ≥ p 的索引
    cutoff = np.searchsorted(cumulative, p) + 1  # +1 因为要包含达到 p 的那一位
    cutoff = min(cutoff, len(logits))
    
    # 5) 保留前 cutoff 个，其他设成 0
    keep_idx = sorted_idx[:cutoff]
    masked_probs = np.zeros_like(probs)
    masked_probs[keep_idx] = probs[keep_idx]
    masked_probs = masked_probs / np.sum(masked_probs)  # 重新归一
    
    return np.random.choice(len(logits), p=masked_probs)

np.random.seed(42)
logits = np.array([2.5, 1.0, 0.3, 0.1, -1.0])
print("Top-p=0.9 采样 10 次:")
for _ in range(10):
    print(top_p_sampling(logits, p=0.9), end=" ")
```

### 三种采样对比演示

```python
import numpy as np

def stable_softmax(z):
    m = np.max(z)
    exps = np.exp(z - m)
    return exps / np.sum(exps)

# 假设词表 8 个 token
logits = np.array([2.5, 1.0, 0.3, 0.1, -0.5, -1.0, -2.0, -3.0])
probs = stable_softmax(logits)

print("原始分布:")
for i, p in enumerate(probs):

print(f"  累计 top 概率:")
sorted_p = np.sort(probs)[::-1]
cum = np.cumsum(sorted_p)
for i, (p, c) in enumerate(zip(sorted_p, cum)):
    print(f"    排名{i+1}: p={p:.4f}, 累计={c:.4f}")

print("\n三种采样各 20 次:")
np.random.seed(0)
print("Greedy:", [int(np.argmax(logits))]*20)

```

**输出解读**：
- Greedy 永远是 0
- Top-k=3 只在 {0, 1, 2} 中选
- Top-p=0.9 因为 top 1 已经 0.71，top 2 累计 0.87，top 3 累计 0.98，所以保留前 3 个，结果和 Top-k=3 类似
- 但如果分布更平（比如 logits 都接近），Top-k 还是固定 3 个，Top-p 会保留更多

### PyTorch 用法

PyTorch 没有直接的 top-k 采样函数，但可以用 `torch.multinomial` 配合 mask 实现。HuggingFace transformers 提供了现成的：

```python
import torch
from torch.distributions import Categorical

# 通用采样函数
def sample_from_logits(logits, top_k=0, top_p=1.0, temperature=1.0):
    """
    logits: shape (V,)
    top_k=0 表示不限制；top_p=1.0 表示不限制
    """
    logits = logits / temperature
    
    # Top-k 过滤
    if top_k > 0:
        top_k = min(top_k, logits.size(-1))
        kth_value = torch.topk(logits, top_k)[0][..., -1, None]
        indices_to_remove = logits < kth_value
        logits[indices_to_remove] = float('-inf')
    
    # Top-p 过滤
    if top_p < 1.0:
        sorted_logits, sorted_indices = torch.sort(logits, descending=True)
        cumulative_probs = torch.cumsum(torch.softmax(sorted_logits, dim=-1), dim=-1)
        # 找到第一个累计 > top_p 的位置
        sorted_indices_to_remove = cumulative_probs > top_p
        # 至少保留第一个（防止全 remove）
        sorted_indices_to_remove[..., 0] = False
        # 把 remove 的位置 logits 设成 -inf
        indices_to_remove = sorted_indices_to_remove.scatter(
            -1, sorted_indices, sorted_indices_to_remove
        )
        logits[indices_to_remove] = float('-inf')
    
    # 采样
    probs = torch.softmax(logits, dim=-1)
    return torch.multinomial(probs, num_samples=1)

# 测试
torch.manual_seed(0)
logits = torch.tensor([2.5, 1.0, 0.3, 0.1, -0.5, -1.0, -2.0, -3.0])
print("Greedy (T→0):", sample_from_logits(logits.clone(), temperature=1e-5).item())
print("Top-k=3:", [sample_from_logits(logits.clone(), top_k=3).item() for _ in range(5)])
print("Top-p=0.9:", [sample_from_logits(logits.clone(), top_p=0.9).item() for _ in range(5)])
```

### 应用场景

| 采样方法 | 适用场景 | 优点 | 缺点 |
|---------|---------|------|------|
| **Greedy** | 代码生成、数学解题、翻译 | 确定性、稳定 | 容易重复、缺乏创意 |
| **Top-k** | 通用文本生成 | 平衡多样性和质量 | k 是固定的，不适配不同分布 |
| **Top-p** | 创意写作、对话 | 动态适应分布形态 | 偶尔保留过多噪声 |
| **Beam Search** | 翻译、摘要 | 综合多步 | 不是采样，是搜索；缺多样性 |

工程经验：通常 **temperature + top-p** 组合用得多（如 T=0.7, p=0.9）。Top-k 在 LLM 里 k=50 是经典值，但前沿模型越来越倾向用 top-p。

### 一句话记忆

> **"Top-k 固定数量，Top-p 固定累计概率。Top-p 更聪明，会自适应分布形态。"**

## 面试回答（3题）

### Q1：为什么 Softmax 要减最大值？请推导一下。

**答**：为了数值稳定。Softmax 公式里有 $e^{z_i}$，当 $z_i$ 较大（比如 1000）时，$e^{1000}$ 会超过浮点数表示范围，溢出成 inf，导致最终结果是 nan。

解决办法是利用 Softmax 的"对统一平移不敏感"性质——给所有 $z_i$ 同时减去最大值 $m = \max(\mathbf{z})$，结果不变。推导如下：

$$
\frac{e^{z_i - m}}{\sum_j e^{z_j - m}} = \frac{e^{z_i} \cdot e^{-m}}{e^{-m} \cdot \sum_j e^{z_j}} = \frac{e^{z_i}}{\sum_j e^{z_j}}
$$

减去最大值后，最大的 $z_i - m = 0$，$e^0 = 1$，绝对不会溢出；其他 $z_j - m \le 0$，$e^{\text{非正数}} \in (0, 1]$，也安全。这是 PyTorch / TensorFlow 等框架的标准做法。

### Q2：交叉熵和 KL 散度的关系？为什么分类任务用交叉熵不用 KL？

**答**：两者关系是 **$H(\mathbf{q}, \mathbf{p}) = H(\mathbf{q}) + D_{KL}(\mathbf{q} \lVert \mathbf{p})$**，其中 $H(\mathbf{q})$ 是真实分布的熵，$D_{KL}(\mathbf{q} \lVert \mathbf{p})$ 是 KL 散度。

证明：$H(\mathbf{q},\mathbf{p}) = -\sum_i q_i \log p_i = -\sum_i q_i \log(q_i \cdot p_i/q_i) = -\sum_i q_i \log q_i - \sum_i q_i \log(p_i/q_i) = -\sum_i q_i \log q_i + \sum_i q_i \log(q_i/p_i) = H(\mathbf{q}) + D_{KL}(\mathbf{q} \lVert \mathbf{p})$。

**为什么用交叉熵**：在训练时，真实分布 q 是固定的（由数据集决定），所以 $H(\mathbf{q})$ 是常数。最小化交叉熵 $H(\mathbf{q}, \mathbf{p})$ 等价于最小化 $D_{KL}(\mathbf{q} \lVert \mathbf{p})$，两者优化目标完全一致。但交叉熵计算更直接——one-hot 情况下直接等于 $-\log p_y$，不用先算 $H(\mathbf{q})$，实现更简单。另外，当 q 是 one-hot 时 $H(\mathbf{q}) = 0$，此时 $H(\mathbf{q}, \mathbf{p}) = D_{KL}(\mathbf{q} \lVert \mathbf{p})$，两者数值也相等。

### Q3：Top-k 和 Top-p 采样有什么区别？分别适合什么场景？

**答**：

- **Top-k**：保留概率最高的 k 个 token，固定数量。简单直接，但 k 是超参数，对不同分布形态可能不合适——分布集中时 k=50 太多（保留太多噪声），分布平坦时 k=50 又太少（错过合理候选）。
- **Top-p（nucleus）**：保留累计概率达到 p 的最小 token 集合，动态数量。能自适应分布形态：分布集中时只保留 1-2 个，分布平坦时保留几十个。

**适合场景**：
- Top-k：通用场景，k=50 是经典默认值
- Top-p：更现代的方法，p=0.9 是常用值。HuggingFace transformers 默认推荐 top-p
- 实际工程中常组合使用：先 top-k 再 top-p，再加温度调节

补充：Greedy（argmax）适合需要确定性的任务（代码生成、数学解题），但容易陷入重复；纯随机采样多样性过高，会输出乱码；Top-k/Top-p 是质量和多样性的折中。

## 易混淆点表

| 易混淆概念 | 区别 | 记忆口诀 |
|-----------|------|---------|
| **Logits vs 概率** | Logits 是任意实数，加起来$\ne 1$；概率在 $[0,1]$，和为 1 | "logits 是分数，softmax 后才是概率" |
| **Softmax vs Sigmoid** | Softmax 多分类归一（多类和为1）；Sigmoid 二分类独立（每类独立 0~1） | "softmax 互斥，sigmoid 独立" |
| **交叉熵 $H(\mathbf{q},\mathbf{p})$ vs 熵 $H(\mathbf{p})$** | $H(\mathbf{q},\mathbf{p})$ 是两个分布的事；$H(\mathbf{p})$ 是一个分布的事 | "熵自带 = 单身；交叉熵 = 两个人" |
| **交叉熵 vs KL 散度** | 交叉熵 = 熵 + KL；训练时 q 固定，二者优化等价 | "KL 是交叉熵扣掉熵的净距离" |
| **$D_{KL}(\mathbf{q}\lVert\mathbf{p})$ vs $D_{KL}(\mathbf{p}\lVert\mathbf{q})$** | 不对称！前者"覆盖模式"，后者"寻峰模式" | "分子在前 = 谁是基准" |
| **Top-k vs Top-p** | k 是固定数量，p 是固定累计概率阈值 | "k 数量，p 概率" |
| **Greedy vs argmax** | 完全相同——Greedy 就是每步 argmax | "Greedy = 步步 argmax" |
| **log p vs logit** | log p 是 log 概率（$\le 0$）；logit 是 $\log(p/(1-p))$，二分类原始分数 | "logit 是分数（任意值），log p 是已归一" |
| **NLL vs CrossEntropy** | one-hot 时数值相等；NLL 是 $-\log p_y$，CrossEntropy 是 $-\sum q \log p$ | "one-hot 下 NLL = CrossEntropy" |
| **PPL vs Loss** | $\operatorname{PPL} = e^{\text{Loss}}$；PPL 在 $[1, V]$，Loss 在 $[0, +\infty)$ | "PPL 是 loss 的指数化" |
| **温度 T vs Top-k/p** | T 调节整体尖锐度（连续），Top-k/p 截断尾部（离散） | "T 是水龙头，Top 是筛子" |
| **LogSumExp vs max** | LSE 是 max 的光滑上界，可导；max 不可导 | "LSE = smooth max" |
| **F.cross_entropy 输入** | 接受 logits（不是概率）+ 类别索引（不是 one-hot） | "进 logits 出 loss，标签是索引" |
| **F.kl_div 参数顺序** | 第一参数是 log_p，第二参数是 q，计算 $D_{KL}(\mathbf{q}\lVert\mathbf{p})$ | "第二个参数是真分布" |

## 本章公式速查表

| 概念 | 公式 |
|------|------|
| Softmax | $p_i = e^{z_i} / \sum e^{z_j}$ |
| 平移不变 | $\operatorname{softmax}(\mathbf{z}+c) = \operatorname{softmax}(\mathbf{z})$ |
| 稳定 Softmax | $p_i = e^{z_i-m} / \sum e^{z_j-m},\ m=\max(\mathbf{z})$ |
| LogSumExp | $\operatorname{LSE}(\mathbf{z}) = m + \log \sum e^{z_j-m}$ |
| log softmax | $\log \operatorname{softmax}(\mathbf{z})_i = z_i - \operatorname{LSE}(\mathbf{z})$ |
| 温度 softmax | $p_i(T) = e^{z_i/T} / \sum e^{z_j/T}$ |
| 交叉熵 | $H(\mathbf{q},\mathbf{p}) = -\sum_i q_i \log p_i$ |
| one-hot 交叉熵 | $\mathcal{L} = -\log p_y$ |
| 信息熵 | $H(\mathbf{p}) = -\sum_i p_i \log p_i$ |
| KL 散度 | $D_{KL}(\mathbf{q}\lVert\mathbf{p}) = \sum_i q_i \log(q_i/p_i)$ |
| 恒等式 | $H(\mathbf{q},\mathbf{p}) = H(\mathbf{q}) + D_{KL}(\mathbf{q}\lVert\mathbf{p})$ |
| PPL | $\operatorname{PPL} = e^{\bar{L}},\ \bar{L} = \text{平均交叉熵}$ |
| LSE 上界 | $\max(\mathbf{z}) \le \operatorname{LSE}(\mathbf{z}) \le \max(\mathbf{z}) + \log n$ |
