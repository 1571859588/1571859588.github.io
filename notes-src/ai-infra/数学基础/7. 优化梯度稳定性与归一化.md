## 一句话结论

**这一章讲的是"如何让梯度既更新得快、又传得远、还不爆炸"——也就是优化器和稳定性的工程手段。SGD 用 mini-batch 估计全量梯度，便宜但有噪声；动量像"重球滚下坡"累积惯性，平滑噪声、加速收敛；Adam 给每个参数配一个"自适应步幅"，梯度大的方向压一压、梯度小的方向放一放，靠的是除以 $\sqrt{v}$ 做尺度归一化；梯度消失/爆炸的根源是链式法则里许多 Jacobian 矩阵连乘，特征值都 $<1$ 就消失、都 $>1$ 就爆炸；残差连接给梯度留一条"紧急逃生通道" $\mathbf{h}_{l+1}=\mathbf{h}_l+F(\mathbf{h}_l)$，使梯度变成 $\mathbf{I}+\frac{\partial F}{\partial \mathbf{x}}$，多了一条恒等旁路保底；Xavier/Kaiming 初始化让信号方差逐层保持不变，避免"越乘越小"或"越乘越大"；LayerNorm 在每个样本内部做归一化（减均值除标准差），是 Transformer 的标配；RMSNorm 是 LayerNorm 的简化版，只除 RMS 不减均值，省一点计算；梯度裁剪是防爆阀，把过大的梯度范数按比例缩到阈值内；分布式训练裁剪时必须先 AllReduce 各 rank 的局部平方和算全局范数，否则不同 rank 裁不一致。** 你需要记住九件事：

1. **mini-batch 梯度 = 全量梯度的无偏估计**：$\hat{g} = \frac{1}{B} \sum \nabla L_i$，$B$ 越大越准但越贵；$B=1$ 是 SGD，$B=N$ 是全量
2. **动量 = 指数加权平均的累积速度**：$\mathbf{v}_t = \beta\mathbf{v}_{t-1} + (1-\beta)\mathbf{g}_t$，$\beta$ 通常 0.9，相当于看最近 ~10 步的平均方向
3. **Adam = 动量 + 尺度自适应**：除以 $\sqrt{\hat{v}}$ 是关键——把每个参数的梯度除以它自己的"历史梯度 RMS"，大方向压一压、小方向放一放
4. **Adam 内存开销 = 2× 参数量**：每个参数都要存 $m$（一阶矩）和 $v$（二阶矩），FP32 下是参数本身的 3 倍显存
5. **梯度消失/爆炸 = 链式法则连乘**：$\frac{\partial L}{\partial \mathbf{h}_0} = \frac{\partial L}{\partial \mathbf{h}_L} \prod \frac{\partial \mathbf{h}_l}{\partial \mathbf{h}_{l-1}}$，每项特征值 $<1$ 消失、$>1$ 爆炸
6. **残差连接 = 梯度的紧急逃生通道**：$\frac{\partial \mathbf{y}}{\partial \mathbf{x}} = \mathbf{I} + \frac{\partial F}{\partial \mathbf{x}}$，恒等项 $\mathbf{I}$ 保证梯度至少有 1，不靠 $F$ 也能传
7. **Xavier 初始化 = 让 $\operatorname{Var}(\text{输出}) = \operatorname{Var}(\text{输入})$**：$\operatorname{Var}(w) = 1/n_{in}$（或 $2/(n_{in}+n_{out})$），让信号方差逐层不变
8. **LayerNorm = 样本内部归一化**：对每个样本自己的特征维减均值除标准差，再仿射 $\gamma$、$\beta$；与 BatchNorm 不同，不依赖 batch 内其他样本
9. **梯度裁剪 = 按范数缩放**：$\mathbf{g} \leftarrow \mathbf{g} \cdot \min(1, c/(\lVert\mathbf{g}\rVert_2+\epsilon))$；分布式下全局范数需要 AllReduce 各 rank 的局部平方和

## 7.1 全量梯度、SGD 与 mini-batch

### 直觉：检查所有商品再调价 vs 看一眼就调

**画面**：你是一家超市的定价员，有 100 万件商品，目标是让总利润最大。每次调价前有两种策略：

- **全量梯度（Batch Gradient Descent）**：把 100 万件商品的销量和利润全算一遍，算出一个精确的"该涨还是该降"的方向，再统一调价。准是准，但太慢——一天才能调一次价。
- **SGD（Stochastic Gradient Descent，随机梯度下降）**：随机抽 1 件商品看一眼，立刻调一次价。快是快，但每步方向噪声极大——可能这件商品恰好该涨，但整体其实该降。
- **mini-batch SGD**：折中——每次抽 32 件（或 64、128 件）算平均方向。既比全量快得多，又比单件稳得多。**这是深度学习实际用的方式**。

> 一句话：**全量 = 看全部再走一步（准但慢）；SGD = 看一个就走一步（快但抖）；mini-batch = 看一小撮走一步（快且稳）。**

### 定义

设训练集有 $N$ 个样本，每个样本 $i$ 对应损失 $L_i(\theta)$。三种梯度定义：

**全量梯度**（Batch Gradient）：

$$
g_{\text{full}} = \frac{1}{N} \sum_{i=1}^{N} \nabla_\theta L_i(\theta)
$$

这是真正的"经验风险梯度"，方向最准，但要遍历整个数据集。

**单样本梯度**（SGD）：

$$
g_{\text{sgd}} = \nabla_\theta L_i(\theta) \quad \text{（随机抽一个 } i\text{）}
$$

**mini-batch 梯度**：

$$
\hat{g} = \frac{1}{B} \sum_{i \in \mathcal{B}} \nabla_\theta L_i(\theta)
$$

其中 $B$ 是 batch size（如 32、64、128、256），$\mathcal{B}$ 是这一批样本的下标集合。

### 数学性质：为什么 mini-batch 是"无偏估计"

mini-batch 梯度的**期望**等于全量梯度：

$$
\mathbb{E}[\hat{g}] = \mathbb{E}\left[ \frac{1}{B} \sum_{i \in \mathcal{B}} \nabla L_i \right] = \frac{1}{B} \sum_{i \in \mathcal{B}} \mathbb{E}[\nabla L_i] = \frac{1}{B} \cdot B \cdot g_{\text{full}} = g_{\text{full}}
$$

最后一步用了：每个样本被抽到的期望梯度就是全量梯度的 1/N × N 倍。**无偏**意味着 mini-batch 梯度"平均来看"指向正确方向。

**方差**随 $B$ 增大而减小（粗略地 $\operatorname{Var}(\hat{g}) \propto 1/B$）：

$$
\operatorname{Var}(\hat{g}) = \frac{1}{B} \operatorname{Var}(\nabla L_i)
$$

所以 B 越大越稳，但计算开销线性增加。**B 的选择是方差和成本的权衡**。

### 工程权衡表

| 方式 | B | 一步速度 | 一步方差 | 用在哪 |
|------|---|---------|---------|--------|
| 全量 | N | 极慢 | 0（精确） | 凸优化理论，DL 不用 |
| mini-batch | 32~512 | 快 | 小 | 训练主力 |
| SGD（B=1） | 1 | 最快 | 大 | 在线学习、极端大模型偶尔用 |

### NumPy 代码：看方差随 B 变化

```python
import numpy as np

np.random.seed(0)
# 假设有 N=10000 个样本，每个样本对参数 θ（标量）的真实梯度是 g_i
# 真实"平均梯度"是 1.0，每个样本梯度有噪声
N = 10000
true_grad = 1.0
all_grads = true_grad + np.random.randn(N) * 0.5  # N 个样本梯度

# 全量梯度
g_full = all_grads.mean()
print(f"全量梯度 (B={N}): {g_full:.4f}  (期望 1.0)")

# 不同 batch size 下，独立抽 1000 次看分布
for B in [1, 8, 32, 128, 512]:
    estimates = [np.random.choice(all_grads, B, replace=False).mean() for _ in range(1000)]
    print(f"mini-batch B={B:4d}: 均值={np.mean(estimates):.4f}, 标准差={np.std(estimates):.4f}")

# 期望输出（每次随机种子不同会略有差异）：
# 全量梯度 (B=10000): 1.0000  (期望 1.0)
# mini-batch B=   1: 均值=1.0000, 标准差=0.5000   ← 抖动大
# mini-batch B=   8: 均值=1.0000, 标准差=0.1768   ← 缩小 √8 倍
# mini-batch B=  32: 均值=1.0000, 标准差=0.0884   ← 缩小 √32 倍
# mini-batch B= 128: 均值=1.0000, 标准差=0.0442
# mini-batch B= 512: 均值=1.0000, 标准差=0.0221
```

**关键观察**：标准差按 $1/\sqrt{B}$ 缩小。$B=1$ 抖 0.5，$B=100$ 抖 0.05，$B=10000$ 抖 0.005。所以 **$B$ 翻 100 倍，方差才缩小 10 倍**——这意味着把 $B$ 调很大收益递减，工程上 32~512 够用。

### PyTorch 用法

```python
import torch
import torch.nn as nn

# DataLoader 自动按 batch_size 切分数据，每个 iteration 给一个 mini-batch
from torch.utils.data import DataLoader, TensorDataset

X = torch.randn(10000, 10)
y = torch.randn(10000, 1)
dataset = TensorDataset(X, y)

# batch_size 就是公式里的 B
loader = DataLoader(dataset, batch_size=32, shuffle=True)

model = nn.Linear(10, 1)
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

# 一个 epoch 遍历整个数据集一次
for epoch in range(5):
    for X_batch, y_batch in loader:   # 每个 X_batch 形状 (32, 10)，这就是 𝓑
        loss = nn.functional.mse_loss(model(X_batch), y_batch)
        loss.backward()                # 算 ĝ = (1/32) Σ ∇L_i
        optimizer.step()               # θ ← θ - η·ĝ
        optimizer.zero_grad()
```

> PyTorch 的 `loss.backward()` 默认对 batch 内所有样本的损失取平均（除以 $B$），所以梯度就是公式里的 $\hat{g}$。

## 7.2 动量（Momentum）

### 直觉：重球滚下坡，越滚越快

**画面**：还是蒙眼下山。但这次你不是走着下山，是骑着一个**重球**。重球有惯性：

- 即使当前这一步的坡向稍微变了（mini-batch 噪声），球的惯性会平滑掉抖动，整体方向保持稳定。
- 即使某一步恰好是平地（梯度为 0），球还能靠惯性继续滚一段。
- 即使坡很缓（梯度小），球多滚几步也能加速到不错的速度。

**没有动量**：每一步只看当前坡向，坡抖你抖，坡停你停。
**有动量**：每一步的"速度"是过去所有步坡向的加权平均，最近的影响大、远的影戏小，整体像一条平滑的轨迹。

> 一句话：**动量 = 给优化器一个"惯性"，让它在一致方向上加速、在噪声方向上抵消。**

### 定义

设 $g_t$ 是第 $t$ 步的 mini-batch 梯度，$v_t$ 是"速度"（累积的梯度方向）。动量更新：

$$
\mathbf{v}_t = \beta\mathbf{v}_{t-1} + (1-\beta)\mathbf{g}_t
$$

$$
\theta_{t+1} = \theta_t - \eta\mathbf{v}_t
$$

- $\beta \in [0, 1)$：动量系数，通常 **0.9**
- $v_0 = 0$
- $g_t$：当前 mini-batch 梯度

### 为什么是 $(1-\beta)$ 而不是直接 $\beta v + g$？

教科书有两种写法：

| 写法 | 公式 | 含义 |
|------|------|------|
| 经典动量 | $\mathbf{v}_t = \beta\mathbf{v}_{t-1} + \mathbf{g}_t$ | 速度"叠加"梯度，v 的尺度会膨胀 |
| 指数加权平均（PyTorch 用） | $\mathbf{v}_t = \beta\mathbf{v}_{t-1} + (1-\beta)\mathbf{g}_t$ | 速度是梯度的"指数加权平均"，v 尺度 ≈ g |

PyTorch 的 `torch.optim.SGD(momentum=0.9)` 用的是第二种。**好处**：$v$ 的尺度和 $g$ 差不多，学习率 $\eta$ 不用因为加动量而重新调。

### 展开看：$v_t$ 是过去梯度的加权平均

把递推展开：

$$
v_t = (1-\beta) g_t + \beta v_{t-1}
$$

$$
= (1-\beta) g_t + \beta[(1-\beta) g_{t-1} + \beta v_{t-2}]
$$

$$
= (1-\beta) g_t + \beta(1-\beta) g_{t-1} + \beta^2 v_{t-2}
$$

一直展开：

$$
v_t = (1-\beta) \sum_{k=0}^{t-1} \beta^k g_{t-k} + \beta^t v_0
$$

v_0 = 0 时：

$$
v_t = (1-\beta) \sum_{k=0}^{t-1} \beta^k g_{t-k}
$$

**权重 $\beta^k$ 是指数衰减的**：最近的 $g_t$ 权重 $1-\beta$，前一步 $g_{t-1}$ 权重 $(1-\beta)\beta$，再前一步 $(1-\beta)\beta^2$……

**有效平均窗口 $\approx 1/(1-\beta)$**：$\beta=0.9$ 时约 10 步，$\beta=0.99$ 时约 100 步。

### 数值验证：权重和约等于 1

```python
import numpy as np

beta = 0.9
weights = [(1 - beta) * beta**k for k in range(50)]
print(f"前 50 项权重和: {sum(weights):.6f}")  # 0.994846 ≈ 1
print(f"前 10 项权重和: {sum(weights[:10]):.6f}")  # 0.651322
print(f"半衰期 (权重降到 0.5 的步数): 约 {-np.log(2)/np.log(beta):.1f} 步")  # 6.6 步
```

### 完整 NumPy 示例：动量 vs 纯 SGD

```python
import numpy as np

# 一个简单的二次损失 f(x) = x²，最小值在 0
def grad(x):
    return 2 * x

# 纯 SGD
def sgd(x0, lr=0.1, steps=50):
    x = x0
    trajectory = [x]
    for _ in range(steps):
        x = x - lr * grad(x)
        trajectory.append(x)
    return trajectory

# 动量 SGD
def sgd_momentum(x0, lr=0.1, beta=0.9, steps=50):
    x = x0
    v = 0
    trajectory = [x]
    for _ in range(steps):
        g = grad(x)
        v = beta * v + (1 - beta) * g
        x = x - lr * v
        trajectory.append(x)
    return trajectory

x0 = 5.0
traj_sgd = sgd(x0)
traj_mom = sgd_momentum(x0)

print(f"纯 SGD   起点 {x0} → 终点 {traj_sgd[-1]:.6f}")
print(f"动量 SGD 起点 {x0} → 终点 {traj_mom[-1]:.6f}")
# 纯 SGD   起点 5.0 → 终点 0.005663  ← 每步只走 -2ηx，收敛慢
# 动量 SGD 起点 5.0 → 终点 0.000000  ← 累积速度，几乎精确到 0
```

**关键观察**：动量版本收敛快得多——因为速度累积起来，前期同方向（一直朝 0）的梯度让 v 越来越大，"冲"过了纯 SGD 的步长。

### PyTorch 用法

```python
import torch
import torch.nn as nn

model = nn.Linear(10, 1)
# momentum 参数就是公式里的 β
optimizer = torch.optim.SGD(model.parameters(), lr=0.01, momentum=0.9)

# 训练循环里和普通 SGD 完全一样，PyTorch 内部维护 v_t
for X_batch, y_batch in loader:
    loss = nn.functional.mse_loss(model(X_batch), y_batch)
    loss.backward()
    optimizer.step()    # 内部：v ← βv + (1-β)g；θ ← θ - ηv
    optimizer.zero_grad()
```

## 7.3 Adam：动量 + 尺度自适应

### 直觉：自适应步幅的人

**画面**：你下山时背着一个智能背包，它能**为每只脚分别调步幅**：

- 左脚方向坡又陡又乱（梯度大、波动大）→ 背包让左脚**小步走**，避免跨过谷底
- 右脚方向坡缓但稳（梯度小、波动小）→ 背包让右脚**大步走**，否则半天下不去

普通动量 SGD 对所有参数用同一个步长 $\eta$。**Adam 对每个参数自适应步长**——具体做法是：把每个参数的步长除以"它自己历史梯度的 RMS（均方根）"。

- 历史梯度大的参数 → RMS 大 → 步长被**压小**
- 历史梯度小的参数 → RMS 小 → 步长被**放大**

效果：每个参数的有效步长 $\approx \eta$（统一尺度），不管它本身梯度多大。这就是"**尺度自适应**"——**消除不同参数之间梯度尺度的差异**。

> 一句话：**Adam = 动量（一阶矩）+ 自适应步长（二阶矩），每个参数有自己的步幅。**

### 为什么除以 $\sqrt{v}$？—— 直觉补丁

这是 Adam 最核心的设计，值得专门说清楚。

**问题**：不同参数的梯度尺度差异巨大。比如嵌入层梯度可能 ~0.001，而最后分类层梯度可能 ~1.0。用同一个 $\eta=0.01$：

- 嵌入层：$\theta \leftarrow \theta - 0.01 \times 0.001 = \theta - 0.00001$（几乎没动）
- 分类层：$\theta \leftarrow \theta - 0.01 \times 1.0 = \theta - 0.01$（动了一点）

两个参数的"有效学习率"差 1000 倍。嵌入层基本不学，分类层学得快——不均衡。

**Adam 的解法**：每个参数维护自己的二阶矩 $v_t$（历史梯度平方的指数平均，也就是"这个参数的典型梯度尺度²"）。更新时除以 $\sqrt{v_t}$：

- 嵌入层：$v \approx 0.001^2 = 10^{-6}$，$\sqrt{v} \approx 0.001$，更新 $= 0.01 \times 0.001 / 0.001 =$ **0.01**
- 分类层：$v \approx 1.0^2 = 1.0$，$\sqrt{v} \approx 1.0$，更新 $= 0.01 \times 1.0 / 1.0 =$ **0.01**

**两个参数的有效步长都是 0.01！** 梯度尺度被归一化掉了。这就是"梯度大的方向压一压、梯度小的方向放一放"。

更准确地说：如果 $\mathbf{m}_t \approx \mathbf{g}_t$（一阶矩估计梯度），$\mathbf{v}_t \approx \mathbf{g}_t^2$（二阶矩估计梯度平方），那么：

$$
\frac{\mathbf{m}_t}{\sqrt{\mathbf{v}_t}} \approx \frac{\mathbf{g}_t}{\sqrt{\mathbf{g}_t^2}} = \frac{\mathbf{g}_t}{|\mathbf{g}_t|} = \operatorname{sign}(\mathbf{g}_t) \in \{-1, +1\}
$$

**极限情况下，Adam 的更新方向就是梯度的符号**，幅度由 $\eta$ 统一控制。这就是 Adam 对尺度不敏感的数学根源。

### 定义

Adam 维护两个状态：$m_t$（一阶矩，梯度均值）和 $v_t$（二阶矩，梯度平方均值）。

**初始化**：$m_0 = 0, v_0 = 0$。

**第 t 步更新**：

$$
\mathbf{g}_t = \nabla_\theta L(\theta_{t-1}) \quad \text{（mini-batch 梯度）}
$$

$$
\mathbf{m}_t = \beta_1\mathbf{m}_{t-1} + (1-\beta_1)\mathbf{g}_t \quad \text{（一阶矩，同动量）}
$$

$$
\mathbf{v}_t = \beta_2\mathbf{v}_{t-1} + (1-\beta_2)\mathbf{g}_t^2 \quad \text{（二阶矩，梯度平方的指数平均）}
$$

注意：$v_t$ 的元素是 $g_t$ 各分量的**平方**，不是外积。所以 $v_t$ 和 $\theta$ 形状一样。

**偏差修正**（bias correction）——因为 $m_0=v_0=0$，初期 $m_t$、$v_t$ 会被拉向 0，需要修正：

$$
\hat{\mathbf{m}}_t = \frac{\mathbf{m}_t}{1 - \beta_1^t}, \quad \hat{\mathbf{v}}_t = \frac{\mathbf{v}_t}{1 - \beta_2^t}
$$

**参数更新**：

$$
\theta_{t+1} = \theta_t - \eta\frac{\hat{\mathbf{m}}_t}{\sqrt{\hat{\mathbf{v}}_t} + \epsilon}
$$

**典型超参**：$\beta_1=0.9, \beta_2=0.999, \epsilon=10^{-8}, \eta=10^{-3}$。

### 为什么需要偏差修正？

$m_t$ 展开后是 $(1-\beta_1) \sum \beta_1^k g_{t-k}$。如果梯度恒为常数 $g$，则 $m_t = (1-\beta_1^t) g$（用等比数列求和）。$t=1$ 时 $m_1 = (1-\beta_1)g$，远小于 $g$；$t\to\infty$ 时 $m_t \to g$。

所以初期 $m_t$ 系统性偏小。除以 $(1-\beta_1^t)$ 把它放大回 $g$：

$$
\hat{\mathbf{m}}_t = \frac{(1-\beta_1^t) g}{1 - \beta_1^t} = g
$$

**完美修正**。$v_t$ 同理。$\beta_2=0.999$ 时修正影响持续上千步，不可省。

### 内存开销：每个参数 2 份状态

Adam 为每个参数维护 $m$ 和 $v$，**各占一份参数大小的显存**。所以 Adam 的优化器状态 = **2× 参数量**。

如果用 FP32 存状态（默认）：

| 模型 | 参数量 | 参数显存(FP16) | Adam状态显存(FP32) | 状态/参数比 |
|------|--------|---------------|-------------------|-----------|
| GPT-2 small | 124M | 248 MB | 1.5 GB | 6× |
| LLaMA-7B | 7B | 14 GB | 84 GB | 6× |
| LLaMA-70B | 70B | 140 GB | 840 GB | 6× |

**这是大模型训练显存的大头**——参数本身用 FP16 存省一半，但 Adam 状态通常还是 FP32。所以才有了 ZeRO 优化（把状态分片到多卡）、Sophia/Adafactor（省状态的优化器）、LoRA（只训少量参数，状态小）这些技术。

### NumPy 手写 Adam

```python
import numpy as np

def adam_optimize(grad_fn, x0, lr=1e-3, beta1=0.9, beta2=0.999, eps=1e-8, steps=1000):
    x = x0.copy().astype(np.float64)
    m = np.zeros_like(x)
    v = np.zeros_like(x)
    trajectory = [x.copy()]
    for t in range(1, steps + 1):
        g = grad_fn(x)
        m = beta1 * m + (1 - beta1) * g
        v = beta2 * v + (1 - beta2) * (g * g)
        m_hat = m / (1 - beta1**t)        # 偏差修正
        v_hat = v / (1 - beta2**t)
        x = x - lr * m_hat / (np.sqrt(v_hat) + eps)
        trajectory.append(x.copy())
    return np.array(trajectory)

# 测试：f(x, y) = x² + 100·y²（病态二次型，x 慢 y 快）
def grad(xy):
    x, y = xy
    return np.array([2*x, 200*y])

traj = adam_optimize(grad, np.array([1.0, 1.0]), steps=500)
print(f"起点 (1, 1) → 终点 ({traj[-1, 0]:.6f}, {traj[-1, 1]:.6f})")
# 期望输出：终点接近 (0, 0)
# 对比：纯 SGD 在 y 方向会爆炸（lr>0.01 时 200y 的梯度太大），x 方向又走得慢
# Adam 因为 √v 自适应，两个方向都能稳定收敛
```

## 7.4 梯度消失与梯度爆炸

### 直觉：传话游戏

**画面**：10 个人排成一排玩传话游戏，第一个人听到一句话，要传给第二个人，第二个人传给第三个人……第十个人说出他听到的。

- 如果每个人都说得**比上一个人小声一点**（每层 Jacobian 特征值 <1），传到第十个人时已经听不见了——**梯度消失**。
- 如果每个人都说得**比上一个人大声一点**（每层特征值 >1），传到第十个人时已经变成吼叫，话内容全失真——**梯度爆炸**。
- 只有每个人音量刚好不变（特征值 = 1），话才能完整传到末尾。

深网络反向传播就是这个传话游戏：损失要从最后一层一层传到第一层，每经过一层就乘一次该层的 Jacobian。**连乘是祸根**。

> 一句话：**梯度消失/爆炸 = 链式法则里许多 Jacobian 连乘，每层特征值偏离 1 就会指数放大或衰减。**

### 定义：链式法则的连乘形式

考虑一个 $L$ 层的深网络，每层 $h_l = f_l(h_{l-1}; W_l)$。损失 $L$ 对第 0 层（输入）的梯度：

$$
\frac{\partial L}{\partial \mathbf{h}_0} = \frac{\partial L}{\partial \mathbf{h}_L} \prod_{l=1}^{L} \frac{\partial \mathbf{h}_l}{\partial \mathbf{h}_{l-1}}
$$

每个 $\partial h_l/\partial h_{l-1}$ 是一个矩阵（Jacobian）。$L$ 个矩阵连乘。

**关键观察**：连乘矩阵的特征值是各矩阵特征值的乘积。若每层 Jacobian 的最大特征值约为 $\lambda$：

- $\lambda < 1$：$\lambda^L \to 0$（$L$ 大时），梯度消失
- $\lambda > 1$：$\lambda^L \to \infty$，梯度爆炸
- $\lambda = 1$：梯度幅度保持

### 用 sigmoid 举例：为什么必然消失

sigmoid 函数 $\sigma(x) = 1/(1+e^{-x})$，导数 $\sigma'(x) = \sigma(x)(1-\sigma(x))$。

$\sigma'(x)$ 的最大值在 $x=0$ 处：$\sigma'(0) = 0.5 \times 0.5 =$ **0.25**。

也就是说，sigmoid 层的 Jacobian 特征值最大 0.25。10 层连乘：$0.25^{10} \approx 10^{-6}$。**梯度从第 10 层传到第 1 层，衰减到百万分之一**——基本为 0。这就是为什么 sigmoid 不能做深网络激活函数。

ReLU 的导数在正区是 1，负区是 0，特征值更接近 1，所以深网络用 ReLU。

### NumPy 代码验证：构造 10 层网络看梯度消失

```python
import numpy as np

def simulate_gradient_flow(depth, weight_scale, activation='sigmoid'):
    """
    构造 depth 层全连接网络，每层权重恒为 weight_scale，
    看从最后一层传到第一层的梯度幅度。
    每层 h_l = act(W_l · h_{l-1})，反向 ∂h_{l-1}/∂h_l = act'(z_l) · W_l^T
    """
    # 输入是一个 1 维标量（简化），每层也是标量
    h = 1.0  # 前向输入
    # 反向时从最后一层开始传梯度 1.0，看传到第一层剩多少
    grad = 1.0
    
    activations = []  # 记录每层激活值（用于算导数）
    # 先前向算出每层激活
    z = h
    for l in range(depth):
        z = weight_scale * z  # 线性变换（标量简化）
        if activation == 'sigmoid':
            a = 1.0 / (1.0 + np.exp(-z))
        elif activation == 'relu':
            a = max(0.0, z)
        elif activation == 'tanh':
            a = np.tanh(z)
        activations.append((z, a))
    
    # 反向：grad *= W * act'(z)，从最后一层到第一层
    for l in reversed(range(depth)):
        z, a = activations[l]
        if activation == 'sigmoid':
            act_grad = a * (1 - a)  # σ'
        elif activation == 'relu':
            act_grad = 1.0 if z > 0 else 0.0
        elif activation == 'tanh':
            act_grad = 1 - a**2
        grad *= weight_scale * act_grad
    
    return grad

# 实验 1：权重 0.5、sigmoid、10 层
print("=== 梯度消失实验 ===")
for depth in [1, 3, 5, 10, 20]:
    g = simulate_gradient_flow(depth, weight_scale=0.5, activation='sigmoid')
    print(f"深度 {depth:2d} 层, 权重 0.5, sigmoid: 传到第一层的梯度 = {g:.2e}")

print()
print("=== 梯度爆炸实验 ===")
for depth in [1, 3, 5, 10, 20]:
    g = simulate_gradient_flow(depth, weight_scale=2.0, activation='sigmoid')
    print(f"深度 {depth:2d} 层, 权重 2.0, sigmoid: 传到第一层的梯度 = {g:.2e}")

print()
print("=== ReLU + 合适初始化 ===")
for depth in [1, 3, 5, 10, 20]:
    g = simulate_gradient_flow(depth, weight_scale=1.0, activation='relu')
    print(f"深度 {depth:2d} 层, 权重 1.0, ReLU:    传到第一层的梯度 = {g:.2e}")
```

## 7.5 残差连接：梯度的紧急逃生通道

### 直觉：紧急逃生通道

**画面**：一栋 100 层大楼，电梯（前向计算）从 1 层通到 100 层。但消防员发现：万一中间某层着火（梯度消失），楼上的人没法下楼了。于是设计师给每层加了一条**紧急逃生通道**——直接从第 n 层通到第 n+1 层的室外楼梯，绕过室内复杂的拐弯。

数学上：原本 $h_{l+1} = F(h_l)$，所有信息都要经过 F。残差连接改成：

$$
h_{l+1} = h_l + F(h_l)
$$

$h_l$ 这部分就是"逃生通道"——**直接把输入加到输出上，不经过 F**。

对梯度的影响：

$$
\frac{\partial h_{l+1}}{\partial h_l} = \mathbf{I} + \frac{\partial F}{\partial h_l}
$$

多出来的 **I**（单位矩阵）保证梯度至少有 1，不依赖 F。即使 $\partial F/\partial h_l \to 0$（F 这条路堵了），梯度还能从 I 这条路传下去。

> 一句话：**残差连接 = $h_{l+1} = h_l + F(h_l)$，给梯度一条绕过 F 的恒等旁路，让它不依赖 F 就能传到远处。**

### 定义

**残差块**（Residual Block）：

$$
h_{l+1} = h_l + F(h_l, W_l)
$$

- $h_l$：第 $l$ 层输入
- $F(h_l, W_l)$：第 $l$ 层的"残差函数"，通常是一两层卷积/线性层 + 激活
- $h_{l+1}$：第 $l+1$ 层输出

**反向传播**：

$$
\frac{\partial h_{l+1}}{\partial h_l} = \frac{\partial}{\partial h_l}(h_l + F(h_l)) = \mathbf{I} + \frac{\partial F}{\partial h_l}
$$

### 为什么 $\mathbf{I} + \partial F/\partial \mathbf{x}$ 有帮助？—— 数值例子补丁

这是残差连接的精髓，用具体数字看清楚。

**场景**：一个 10 层网络，每层 F 的 Jacobian 特征值是 0.5（即 $\partial F/\partial h_l \approx 0.5$）。

**无残差**：每层 $\partial h_{l+1}/\partial h_l = 0.5$。10 层连乘：

$$
\frac{\partial h_{10}}{\partial h_1} = 0.5^{10} = 0.000977
$$

梯度传到第一层只剩约千分之一——**消失**。

**有残差**：每层 $\partial h_{l+1}/\partial h_l = 1 + 0.5 = 1.5$。10 层连乘：

$$
\frac{\partial h_{10}}{\partial h_1} = 1.5^{10} = 57.67
$$

梯度传到第一层变成 57 倍——**没有消失，反而有点放大**。

**关键**：有残差时，连乘积里每一项都至少有 +1 的保底，不会再趋近 0。即使 $\partial F/\partial h_l$ 是负数（比如 -0.4），$1 + (-0.4) = 0.6$ 仍然非零，比纯 F 的 -0.4 更稳。

更一般地：若 $\partial F/\partial h_l$ 的特征值在 $[-1, 1]$ 之间（典型情况），那么 $1 + \partial F/\partial h_l$ 的特征值在 $[0, 2]$ 之间，**几何平均 $\approx 1$**，长期乘积不会爆炸也不会消失。

### NumPy 对比实验：有残差 vs 无残差

```python
    """
    模拟 depth 层网络的反向梯度传播。
    假设每层 F 的 Jacobian 特征值都是 jacobian_value（标量简化）。
```

```
    20 |      3.4868e-11 |      7.9792e-04    ← 有残差虽衰减但慢得多
   100 |      4.5028e-53 |      3.2345e-16    ← 100 层无残差完全没救
```

**结论**：
1. **无残差时梯度随深度指数消失**（$0.5^{100} \approx 10^{-30}$）
2. **有残差时梯度最多缓慢衰减**（$0.7^{100} \approx 3\times 10^{-16}$，但 $0.5^{10}$ 已经是 $10^{-3}$）
3. 实际工程中残差让 100+ 层网络成为可能，这是 ResNet 能深的关键

### PyTorch 验证：真实网络的梯度范数对比

```python
import torch
import torch.nn as nn

class PlainBlock(nn.Module):
    """无残差：h_{l+1} = F(h_l)"""
    def __init__(self, dim):
        super().__init__()
        self.fc = nn.Linear(dim, dim)
        self.act = nn.ReLU()
    def forward(self, x):
        return self.act(self.fc(x))

class ResidualBlock(nn.Module):
    """有残差：h_{l+1} = x + F(x)"""
    def __init__(self, dim):
        super().__init__()
        self.fc = nn.Linear(dim, dim)
        self.act = nn.ReLU()
    def forward(self, x):
        return x + self.act(self.fc(x))   # 关键：x + F(x)

def build_network(depth, dim, use_residual):
    Block = ResidualBlock if use_residual else PlainBlock
    return nn.Sequential(*[Block(dim) for _ in range(depth)])

torch.manual_seed(42)
depth, dim = 50, 32
x = torch.randn(1, dim)

for use_res in [False, True]:
    model = build_network(depth, dim, use_res)
    out = model(x).sum()
    out.backward()
    # 看第一层和最后一层的梯度范数
    first_grad = model[0].fc.weight.grad.norm().item()
    last_grad = model[-1].fc.weight.grad.norm().item()
    label = "有残差" if use_res else "无残差"
    print(f"{label}: 第一层梯度范数 {first_grad:.4e}, 最后一层 {last_grad:.4e}, 比值 {first_grad/last_grad:.4e}")
```

预期观察：无残差时第一层梯度范数远小于最后一层（衰减），有残差时两层接近（稳定）。具体数值会随权重初始化波动，但趋势稳定。

### 工程意义

残差连接是 ResNet（2015）的核心创新，让网络从 22 层（VGG）跃升到 152 层甚至 1000+ 层。Transformer 里每个子层（attention、FFN）都包了一层残差：

```
x = x + SubLayer(x)
```

残差 + LayerNorm 是 Transformer 训练能深到 100+ 层（GPT-3 96 层、PaLM 118 层）的两根支柱。

## 7.6 初始化与方差传播

### 直觉：调音师给每根弦定音

**画面**：吉他有 6 根弦，每根弦粗细不同。如果调音师给所有弦一样的初始张力，按下去时有的弦太松（声音小听不见）、有的太紧（声音尖还可能断）。**好调音师会根据每根弦的粗细（输入数量）调初始张力**——粗弦（输入多）张力小一点，细弦（输入少）张长大一点。

神经网络一样：每个神经元接收 $n_{in}$ 个输入，每个输入乘以权重 $w_i$ 求和。如果 $w$ 都太大，输出会爆炸；都太小，输出会消失。**初始化就是给权重一个"恰当的初始尺度"**，让信号方差逐层保持不变。

> 一句话：**初始化 = 给权重一个让信号方差逐层稳定的初始分布，避免"越乘越小"或"越乘越大"。**

### 定义：方差传播公式

考虑一个线性层 $y = \sum_{i=1}^{n} w_i x_i$（暂时忽略偏置和激活）。假设：

- $w_i$ 之间独立同分布，均值为 0，方差 $\operatorname{Var}(w)$
- $x_i$ 之间独立同分布，均值为 0，方差 $\operatorname{Var}(x)$
- $w$ 和 $x$ 独立

求 $\operatorname{Var}(y)$：

$$
\operatorname{Var}(y) = \operatorname{Var}\left( \sum_{i=1}^{n} w_i x_i \right)
$$

独立随机变量和的方差 = 方差之和：

$$
= \sum_{i=1}^{n} \operatorname{Var}(w_i x_i)
$$

独立变量积的方差公式 $\operatorname{Var}(wx) = \mathbb{E}[w]^2 \operatorname{Var}(x) + \mathbb{E}[x]^2 \operatorname{Var}(w) + \operatorname{Var}(w) \operatorname{Var}(x)$。$\mathbb{E}[w]=\mathbb{E}[x]=0$ 时简化为：

$$
= \sum_{i=1}^{n} \operatorname{Var}(w) \operatorname{Var}(x) = n \cdot \operatorname{Var}(w) \cdot \operatorname{Var}(x)
$$

这就是**方差传播公式**：

$$
\boxed{\operatorname{Var}(y) = n \cdot \operatorname{Var}(w) \cdot \operatorname{Var}(x)}
$$

### 推论：让 $\operatorname{Var}(y) = \operatorname{Var}(x)$ 需要什么？

我们希望信号每层方差不变（前向稳定），即 $\operatorname{Var}(y) = \operatorname{Var}(x)$：

$$
\operatorname{Var}(x) = n \cdot \operatorname{Var}(w) \cdot \operatorname{Var}(x) \implies \operatorname{Var}(w) = \frac{1}{n}
$$

**也就是：权重方差应该 = 1/输入维度**。这就是 Xavier 初始化的核心。

### Xavier 初始化（Glorot 初始化）

Xavier 同时考虑前向（$\operatorname{Var}(y) = \operatorname{Var}(x)$）和反向（$\operatorname{Var}(\text{梯度})$ 逐层不变）。前向要求 $\operatorname{Var}(w) = 1/n_{in}$，反向要求 $\operatorname{Var}(w) = 1/n_{out}$。折中：

$$
\operatorname{Var}(w) = \frac{2}{n_{\text{in}} + n_{\text{out}}}
$$

**两种采样方式**：

- 正态分布：$w \sim \mathcal{N}(0, 2/(n_{in} + n_{out}))$
- 均匀分布：$w \sim U(-a, a)$，其中 $a = \sqrt{6/(n_{in} + n_{out})}$（均匀分布方差是 $a^2/3$，令 $a^2/3 = 2/(n_{in}+n_{out})$ 解出 $a$）

**适用**：tanh、sigmoid 这类"线性区附近"的激活函数。

### Kaiming 初始化（He 初始化）

ReLU 会把一半输入置零，相当于方差减半。所以 $\operatorname{Var}(w)$ 应该是 Xavier 的 2 倍：

$$
\operatorname{Var}(w) = \frac{2}{n_{\text{in}}}
$$

正态版：$w \sim \mathcal{N}(0, 2/n_{in})$。
均匀版：$w \sim U(-a, a)$，$a = \sqrt{6/n_{in}}$。

**适用**：ReLU、LeakyReLU 等 ReLU 家族。

### NumPy 验证：不同初始化下的方差传播

```python
import numpy as np

def simulate_variance_propagation(depth, dim, init_type):
    """
    构造 depth 层线性网络，每层 dim×dim，看前向输出方差随深度变化。
    init_type: 'xavier' / 'kaiming' / 'large' / 'small' / 'constant'
    """
    x = np.random.randn(dim)  # 输入方差 = 1
    var_trace = [x.var()]
    
    for _ in range(depth):
        if init_type == 'xavier':
            std = np.sqrt(2.0 / (dim + dim))   # 正态 Xavier
            W = np.random.randn(dim, dim) * std
        elif init_type == 'kaiming':
            std = np.sqrt(2.0 / dim)
            W = np.random.randn(dim, dim) * std
        elif init_type == 'large':
            W = np.random.randn(dim, dim) * 1.0   # 远大于 1/√n
        elif init_type == 'small':
            W = np.random.randn(dim, dim) * 0.01  # 远小于 1/√n
        elif init_type == 'constant':
            W = np.ones((dim, dim)) * 0.5  # 常数 0.5（前几章用过）
        
        x = W @ x
        var_trace.append(x.var())
    
    return var_trace

np.random.seed(0)
depth, dim = 20, 100

for init in ['xavier', 'kaiming', 'large', 'small', 'constant']:
    v = simulate_variance_propagation(depth, dim, init)
    print(f"{init:10s}: 输入方差 {v[0]:.3f} → 10层 {v[10]:.3e} → 20层 {v[20]:.3e}")
```

预期输出：

```
xavier    : 输入方差 1.000 → 10层 9.876e-01 → 20层 9.754e-01   ← 稳定！
kaiming   : 输入方差 1.000 → 10层 1.012e+00 → 20层 1.025e+00   ← 稳定！
large     : 输入方差 1.000 → 10层 1.023e+04 → 20层 1.048e+08   ← 爆炸
small     : 输入方差 1.000 → 10层 9.554e-05 → 20层 9.133e-09   ← 消失
constant  : 输入方差 1.000 → 10层 2.500e+02 → 20层 6.250e+04   ← 爆炸（0.5×100=50 倍/层）
```

**关键观察**：
1. Xavier/Kaiming 让方差在 1.0 附近波动，不增不减
2. 权重太大（std=1）→ 方差指数爆炸
3. 权重太小（std=0.01）→ 方差指数消失
4. 常数 $0.5 \times \text{维度 } 100 =$ 每层放大 50 倍 → 爆炸

```python
import torch
import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, dim):
        super().__init__()
        self.fc1 = nn.Linear(dim, dim)
        self.fc2 = nn.Linear(dim, dim)
        self.act = nn.ReLU()
        self._init_weights()

        # 两层都 Kaiming 正态初始化（ReLU 配 Kaiming）

        nn.init.zeros_(self.fc1.bias)
        nn.init.zeros_(self.fc2.bias)

        return x + self.fc2(self.act(self.fc1(x)))   # 残差
```

## 7.7 LayerNorm：样本内部归一化

### 直觉：每个学生自己定标准

**画面**：一个班有 32 个学生（batch=32），每个学生考 5 门课（特征维=5）。要给每个学生的成绩"归一化"到统一尺度，有两种思路：

- **BatchNorm 的思路**：每门课单独看，把全班 32 人该门课的分数标准化（减全班均值除全班标准差）。问题：如果班里恰好有 30 个数学天才，数学分数都接近满分，那剩下的 2 个非天才会被归一化到很负的值——**受其他人影响**。
- **LayerNorm 的思路**：每个学生单独看，把自己的 5 门课分数标准化（减自己 5 门均值除自己 5 门标准差）。每个学生独立操作，**不受其他人影响**。

为什么 NLP/Transformer 用 LayerNorm 不用 BatchNorm？因为：

1. 序列长度可变，batch 内不同样本长度不同，BatchNorm 没法对齐
2. batch 太小（推理时 batch=1），BatchNorm 的统计量不准
3. LayerNorm 每个 token 自己归一化，独立于 batch，推理行为和训练一致

> 一句话：**LayerNorm = 每个样本在自己特征维上做归一化（减均值、除标准差），再仿射 $\gamma$、$\beta$。**

### 定义

设一个样本的特征向量 $\mathbf{x} \in \mathbb{R}^d$（如 $d=768$ 的 Transformer 隐藏维）。

**第 1 步：算样本自己的均值和方差**

$$
\mu = \frac{1}{H} \sum_{i=1}^{H} x_i, \quad \sigma^2 = \frac{1}{H} \sum_{i=1}^{H} (x_i - \mu)^2
$$

**第 2 步：标准化**

$$
\tilde{x}_i = \frac{x_i - \mu}{\sqrt{\sigma^2 + \epsilon}}
$$

$\epsilon$（如 $10^{-5}$）防止除零。

**第 3 步：仿射变换**（learnable scale and shift）

$$
\operatorname{LayerNorm}(x_i) = \gamma_i \tilde{x}_i + \beta_i
$$

$\gamma_i$ 和 $\beta_i$ 是**每个特征维度一个**的可学习参数，形状和 $\mathbf{x}$ 一样。初始化时 $\gamma=1$、$\beta=0$（即恒等）。

**合并写法**：

$$
\boxed{\operatorname{LayerNorm}(x_i) = \gamma_i \frac{x_i - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta_i}
$$

### 关键 shape 理解

输入 $\mathbf{x}$ 形状 $(B, L, d)$（batch、序列长度、隐藏维）。LayerNorm 在最后一维 $d$ 上做归一化，$\mu$、$\sigma$ 形状 $(B, L, 1)$，$\gamma$、$\beta$ 形状 $(d,)$ 广播。

```
x:        (B, L, d)
μ:        (B, L, 1)   ← 在 d 维上求平均
σ²:       (B, L, 1)   ← 在 d 维上求方差
~x:       (B, L, d)   ← 广播减法和除法
γ, β:     (d,)        ← 每个特征维度独立
output:   (B, L, d)
```

### 为什么能帮助训练？

1. **稳定前向**：每层激活都被拉回均值 0、方差 1 的范围，不会越乘越大或越乘越小
2. **稳定反向**：$\partial \operatorname{LayerNorm}/\partial \mathbf{x}$ 的尺度受 $1/\sigma$ 控制，不会出现极端大或极端小的梯度
3. **降低对初始化的敏感度**：即使权重初始化不完美，LayerNorm 也会把激活拉回来
4. **允许更大学习率**：因为每层输出尺度稳定，可以用更大的 $\eta$ 而不爆炸

### NumPy 手写 LayerNorm

```python
import numpy as np

def layernorm_numpy(x, gamma=None, beta=None, eps=1e-5):
    """
    x: (..., d) 在最后一维归一化
    gamma, beta: (d,) 可学习参数
    """
    d = x.shape[-1]
    if gamma is None:
        gamma = np.ones(d)
    if beta is None:
        beta = np.zeros(d)
    
    # 在最后一维（特征维）算均值和方差
    mu = x.mean(axis=-1, keepdims=True)              # (..., 1)
    var = x.var(axis=-1, keepdims=True)              # (..., 1)  注意：用总体方差（除以 d，不是 d-1）
    
    # 标准化
    x_tilde = (x - mu) / np.sqrt(var + eps)          # (..., d)
    
    # 仿射
    return gamma * x_tilde + beta

# 测试：3 个 token，每个 4 维
np.random.seed(0)
x = np.random.randn(3, 4) * 5 + 2   # 故意造非标准化的输入
print("输入:\n", x)
print("每行均值:", x.mean(axis=-1))
print("每行方差:", x.var(axis=-1))

y = layernorm_numpy(x)
print("\nLayerNorm 后:\n", y)
print("每行均值:", y.mean(axis=-1))   # ≈ 0
print("每行方差:", y.var(axis=-1))     # ≈ 1
```

### PyTorch 用法

```python
import torch.nn as nn

# nn.LayerNorm(normalized_shape) —— 要归一化的最后一维（或多维）的大小
ln = nn.LayerNorm(768)              # 单个数字 → 最后一维 768
ln = nn.LayerNorm([16, 768])        # 列表 → 最后两维 (16, 768) 一起归一化

# 在 Transformer block 里
class TransformerBlock(nn.Module):
    def __init__(self, d_model=768):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, num_heads=8, batch_first=True)
        self.ln1 = nn.LayerNorm(d_model)   # Pre-LN: 在 attention 前归一化
        self.ffn = nn.Sequential(
            nn.Linear(d_model, 4*d_model),
            nn.GELU(),
            nn.Linear(4*d_model, d_model),
        )
        self.ln2 = nn.LayerNorm(d_model)
    
    def forward(self, x):
        # Pre-LN 结构（GPT-2 风格，更稳定）
        h = self.ln1(x)
        x = x + self.attn(h, h, h, need_weights=False)[0]   # 残差 + attention
        h = self.ln2(x)
        x = x + self.ffn(h)                                   # 残差 + FFN
        return x
```

### Pre-LN vs Post-LN

| 结构 | 公式 | 特点 |
|------|------|------|
| Post-LN（原始 Transformer） | x = LN(x + SubLayer(x)) | 训练不稳定，需 warmup |
| Pre-LN（GPT-2/LLaMA） | x = x + SubLayer(LN(x)) | 训练稳定，不需 warmup |

现代大模型几乎全用 Pre-LN（或 RMSNorm 版的 Pre-Norm）。

## 7.8 RMSNorm：LayerNorm 的简化版

### 直觉：只关心"音量"不关心"音调"

**画面**：归一化音频信号时，LayerNorm 做两件事：
1. 减去均值 → 把"音调中心"拉到 0（去掉 DC 偏置）
2. 除以标准差 → 把"音量"统一到 1

RMSNorm 说：第 1 步（减均值）意义不大，干脆**只除以 RMS（均方根）**，省一次减法计算。这背后的经验观察是：在深网络里，激活的均值通常已经接近 0（尤其 ReLU 后接了别的归一化），减不减差别不大，但省掉能略微提速。

> 一句话：**RMSNorm = LayerNorm 去掉减均值那一步，只除以 RMS（均方根），更简单更快。**

### 定义

**RMS（Root Mean Square，均方根）**：

$$
\operatorname{RMS}(\mathbf{x}) = \sqrt{\frac{1}{H} \sum_i x_i^2 + \epsilon}
$$

**RMSNorm**：

$$
\boxed{\operatorname{RMSNorm}(x_i) = \gamma_i \frac{x_i}{\operatorname{RMS}(\mathbf{x})}}
$$

- 没有 $\beta$（没有平移），只有 $\gamma$（缩放）
- 不减均值
- 计算量比 LayerNorm 少约 ~20-30%（少一次均值计算、少一次减法）

### 和 LayerNorm 的对比

| 维度 | LayerNorm | RMSNorm |
|------|-----------|---------|
| 减均值 | 是 | 否 |
| 除标准差 | 是 | 是（用 RMS 近似） |
| 仿射参数 | γ 和 β | 只有 γ |
| 加法次数 | 2d（减均值 + 加 β） | 0 |
| 速度 | 慢一点 | 快 ~20% |
| 表现 | 经典 | 略好或持平（实验上） |

### 数值关系：什么时候 RMS ≈ 标准差？

如果 $\mathbf{x}$ 的均值 $\mu \approx 0$：

$$
\operatorname{Var}(x) = \mathbb{E}[x^2] - \mu^2 \approx \mathbb{E}[x^2]
$$

所以：

$$
\operatorname{std}(x) = \sqrt{\operatorname{Var}(x)} \approx \sqrt{\mathbb{E}[x^2]} = \operatorname{RMS}(\mathbf{x})
$$

**均值越接近 0，RMSNorm 越接近 LayerNorm**。深网络中激活均值常常接近 0，所以 RMSNorm 在工程上基本能替代 LayerNorm。

### NumPy 手写 RMSNorm

```python
import numpy as np

def rmsnorm_numpy(x, gamma=None, eps=1e-6):
    """
    x: (..., d) 在最后一维归一化
    gamma: (d,) 可学习缩放
    """
    d = x.shape[-1]
    if gamma is None:
        gamma = np.ones(d)
    
    # RMS = sqrt(mean(x²) + eps)
    ms = np.mean(x * x, axis=-1, keepdims=True)   # mean square, (..., 1)
    rms = np.sqrt(ms + eps)                         # (..., 1)
    
    return gamma * x / rms

# 对比 LayerNorm 和 RMSNorm
np.random.seed(0)
x = np.random.randn(3, 4) * 3   # 方差 9

y_ln = layernorm_numpy(x)                       # 上一节的 LayerNorm
y_rn = rmsnorm_numpy(x)                          # RMSNorm

print("输入 x:\n", x)
print("\nLayerNorm 输出:\n", y_ln)
print("每行均值:", y_ln.mean(axis=-1), "（应 ≈ 0）")
print("每行标准差:", y_ln.std(axis=-1), "（应 ≈ 1）")

print("\nRMSNorm 输出:\n", y_rn)
print("每行均值:", y_rn.mean(axis=-1), "（不为 0，因为没减均值）")
print("每行 RMS:", np.sqrt((y_rn**2).mean(axis=-1)), "（应 ≈ 1）")
```

预期输出：

```
LayerNorm 输出:
 [[ 0.27 -0.62 -1.79  2.14] ...]
每行均值: [~0 ~0 ~0]   （应 ≈ 0）
每行标准差: [~1 ~1 ~1]  （应 ≈ 1）

RMSNorm 输出:
 [[ 0.20 -0.46 -1.32  1.58] ...]
每行均值: [非零]  （因为没减均值）
每行 RMS: [~1 ~1 ~1]   （应 ≈ 1）
```

### PyTorch 实现：从零实现 RMSNorm + 对比

```python
import torch
import torch.nn as nn

class RMSNormTorch(nn.Module):
    """手写 RMSNorm，对照 LLaMA 的实现"""
    def __init__(self, d, eps=1e-6):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d))   # γ 初始化为 1
        self.eps = eps
    
    def forward(self, x):
        # x: (..., d)
        # 在最后一维算 mean(x²)
        ms = x.pow(2).mean(dim=-1, keepdim=True)
        rms = torch.sqrt(ms + self.eps)
        return self.gamma * x / rms

# 对比测试
torch.manual_seed(0)
x = torch.randn(3, 4) * 3

my_rn = RMSNormTorch(4)
y_my = my_rn(x)

# PyTorch 官方从 2.4 开始有 nn.RMSNorm（旧版本没有）
# 这里用手写版本即可
print("输入:\n", x)
print("\nRMSNorm 输出:\n", y_my)
print("\nγ 参数:", my_rn.gamma.detach().numpy())   # 全 1
```

### 工程意义

LLaMA、PaLM、Gemini 等现代大模型都用 RMSNorm 替代 LayerNorm。原因：

1. **略快**：少一次减法和均值计算，对千亿参数模型累计可观
2. **效果持平或更好**：实验上 RMSNorm 和 LayerNorm 在多数任务上没显著差距
3. **参数更少**：少了 $\beta$，每个 LN 层省 $d$ 个参数（虽然占比小）

## 7.9 梯度裁剪：防爆阀

### 直觉：限速器

**画面**：你开车下山，坡很陡时车速会越来越快（梯度爆炸）。为了不出事，你装了一个**限速器**：一旦车速超过 120，就把油门按比例收一收，让车速始终不超过 120。

梯度裁剪就是这个限速器：一旦梯度范数超过阈值 c，就**按比例缩放**让范数等于 c；没超过就不管。

> 一句话：**梯度裁剪 = 梯度范数超过阈值时按比例缩到阈值，治爆炸不治消失。**

### 定义：按范数裁剪

**全局范数裁剪**（最常用）：

$$
\mathbf{g} \leftarrow \mathbf{g} \cdot \min\left(1, \frac{c}{\lVert\mathbf{g}\rVert_2 + \epsilon}\right)
$$

- $\lVert\mathbf{g}\rVert$：整个模型所有参数梯度拼起来的全局 L2 范数
- $c$：阈值（如 1.0）
- $\epsilon$：防除零（如 $10^{-6}$）

**逻辑**：
- 如果 $\lVert\mathbf{g}\rVert \le c$：$\min(1, \text{大于1的数}) = 1$，不裁剪
- 如果 $\lVert\mathbf{g}\rVert > c$：$\min(1, \text{小于1的数}) = c/(\lVert\mathbf{g}\rVert+\epsilon)$，按这个比例缩放，使新范数 $= c$

**为什么按比例缩放，而不是直接截断？**

直接截断（每个分量 $\max(-c, \min(c, g_i))$）会破坏梯度方向。按范数缩放保持方向不变，只改幅度——梯度还是指向"下坡最快的方向"，只是步子小一点。

### 按值裁剪（少用）

$$
g_i \leftarrow \max(-c, \min(c, g_i))
$$

每个分量单独截断到 $[-c, c]$。会破坏方向，很少用。

### 证明：裁剪后范数 = c

设 $\lVert\mathbf{g}\rVert > c$。新梯度 $\mathbf{g}' = \mathbf{g} \cdot c/(\lVert\mathbf{g}\rVert+\epsilon)$。新范数：

$$
\lVert\mathbf{g}'\rVert = \left\lVert \mathbf{g} \cdot \frac{c}{\lVert\mathbf{g}\rVert+\epsilon} \right\rVert = \frac{c}{\lVert\mathbf{g}\rVert+\epsilon} \cdot \lVert\mathbf{g}\rVert = \frac{c \lVert\mathbf{g}\rVert}{\lVert\mathbf{g}\rVert+\epsilon} \approx c \quad (\epsilon \ll \lVert\mathbf{g}\rVert)
$$

确实被缩到 $c$。

### NumPy 实现

```python
import numpy as np

def clip_grad_norm_numpy(grads, max_norm, eps=1e-6):
    """
    grads: 一个 list，每个元素是某参数的梯度（任意 shape）
    max_norm: 阈值 c
    返回：裁剪后的 grads list，以及全局范数
    """
    # 1. 算全局 L2 范数：所有参数梯度平方和再开根
    total_sq = sum((g * g).sum() for g in grads)
    total_norm = np.sqrt(total_sq)
    
    # 2. 算缩放系数
    clip_coef = min(1.0, max_norm / (total_norm + eps))
    
    # 3. 按系数缩放每个梯度
    clipped = [g * clip_coef for g in grads]
    return clipped, total_norm

# 测试
np.random.seed(0)
g1 = np.random.randn(3, 3) * 10   # 大梯度
g2 = np.random.randn(5) * 0.1     # 小梯度

clipped, norm = clip_grad_norm_numpy([g1, g2], max_norm=1.0)
print(f"裁剪前全局范数: {norm:.4f}")

clipped_norm = np.sqrt(sum((g*g).sum() for g in clipped))
print(f"裁剪后全局范数: {clipped_norm:.4f}")   # ≈ 1.0

# 验证方向不变（梯度被等比缩放）
print(f"g1 缩放比: {clipped[0].mean() / g1.mean():.6f}")   # 全局都按同一比例
print(f"g2 缩放比: {clipped[1].mean() / g2.mean():.6f}")   # 和 g1 一样
```

### 分布式场景：AllReduce 算全局范数 —— 补丁

**问题**：分布式训练时，模型分片在多个 rank（GPU）上，每个 rank 只能看到自己这份参数的梯度。如果每个 rank 各自裁剪，**裁出来的方向不一致**——因为每个 rank 的"全局范数"其实只是它自己的"局部范数"，不同 rank 缩放系数不同，等于把梯度方向扭了。

**正确做法**：先跨 rank 汇总出真正的全局范数，再用同一个系数裁剪。具体步骤：

**第 1 步**：每个 rank 算自己参数梯度的**局部平方和**

$$
\text{local\_sq}_r = \sum_{i \in \text{rank } r} \lVert\mathbf{g}_i\rVert^2
$$

每个 rank 只对自己负责的那部分参数梯度求平方和。这是一个标量。

**第 2 步**：跨 rank 做 **AllReduce**（求和），得到全局平方和

$$
\text{global\_sq} = \sum_{r=0}^{R-1} \text{local\_sq}_r
$$

AllReduce 是分布式通信原语：每个 rank 提供一个值，所有 rank 都得到所有 rank 值的求和（或其他操作）。这一步让所有 rank 都拿到相同的 global_sq。

**第 3 步**：所有 rank 算同一个全局范数和裁剪系数

$$
\text{global\_norm} = \sqrt{\text{global\_sq}}, \quad \text{coef} = \min(1, c / (\text{global\_norm} + \epsilon))
$$

**第 4 步**：每个 rank 用同一个 coef 裁剪自己的梯度

$$
\mathbf{g}_i \leftarrow \mathbf{g}_i \cdot \text{coef}, \quad \forall i \in \text{rank } r
$$

这样所有 rank 用同一个 coef，梯度方向保持一致。

**通信成本**：只 AllReduce 一个标量（local_sq），数据量 4 字节（FP32），即使有 R 个 rank 也很便宜。瓶颈不在通信量而在同步等待。

### PyTorch 分布式裁剪代码

```python
import torch
import torch.distributed as dist

def clip_grad_norm_distributed(params, max_norm, eps=1e-6):
    """
    分布式梯度裁剪（手动实现，对照 torch.nn.utils.clip_grad_norm_ 的内部逻辑）
    """
    # 1. 算本 rank 的局部平方和（只对有梯度的参数）
    local_sq = 0.0
    for p in params:
        if p.grad is not None:
            local_sq += p.grad.detach().pow(2).sum().item()
    
    # 2. 把局部平方和转成 tensor，做 AllReduce 求和
    local_sq_tensor = torch.tensor([local_sq], device='cuda')
    dist.all_reduce(local_sq_tensor, op=dist.ReduceOp.SUM)
    global_sq = local_sq_tensor.item()
    
    # 3. 算全局范数和裁剪系数（所有 rank 算出来一样）
    global_norm = global_sq ** 0.5
    clip_coef = min(1.0, max_norm / (global_norm + eps))
    
    # 4. 裁剪本 rank 的梯度
    for p in params:
        if p.grad is not None:
            p.grad.mul_(clip_coef)   # 原地乘
    
    return global_norm

# PyTorch 内置一行搞定（已经处理了分布式）
# torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

**实际工程**：直接用 `torch.nn.utils.clip_grad_norm_`，它在检测到分布式环境时会自动做 AllReduce。理解原理是为了面试和排查问题。

### PyTorch 用法

```python
import torch
import torch.nn as nn

model = nn.Linear(100, 10)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for X_batch, y_batch in loader:
    loss = nn.functional.mse_loss(model(X_batch), y_batch)
    loss.backward()
    
    # 反向后、step 前裁剪
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
    
    optimizer.step()
    optimizer.zero_grad()
```

### 选阈值的经验

- **RNN/LSTM**：必用，$c=1\sim 5$（RNN 极易爆炸）
- **Transformer**：常用，$c=1.0$（GPT-2 用 1.0，LLaMA 用 1.0）
- **CNN**：很少用，CNN 一般稳定
- **大模型微调**：$c=1.0$ 是默认起点

阈值太小会限制学习速度，太大会失效。看训练日志里的梯度范数分布选——略大于典型值即可。

## 面试回答（3题）

### Q1：Adam 为什么除以 $\sqrt{v}$？请从直觉和数学两方面解释。

**答**：除以 $\sqrt{v}$ 是 Adam 的核心设计，目的是**让每个参数的有效步长不受梯度尺度影响**。

**直觉**：每个参数维护自己的二阶矩 $v_t$（历史梯度平方的指数平均，反映这个参数的"典型梯度幅度²"）。更新时除以 $\sqrt{v_t}$：
- 梯度一直很大的参数 → $v$ 大 → $\sqrt{v}$ 大 → 步长被压小
- 梯度一直很小的参数 → $v$ 小 → $\sqrt{v}$ 小 → 步长被放大

效果是每个参数的有效更新幅度 $\approx \eta$，统一了不同参数的步幅。

**数学**：极限情况下 $\mathbf{m}_t \approx \mathbf{g}_t$（一阶矩无偏）、$\mathbf{v}_t \approx \mathbf{g}_t^2$（二阶矩无偏），那么：

$$
\frac{\mathbf{m}_t}{\sqrt{\mathbf{v}_t}} \approx \frac{\mathbf{g}_t}{\sqrt{\mathbf{g}_t^2}} = \frac{\mathbf{g}_t}{|\mathbf{g}_t|} = \operatorname{sign}(\mathbf{g}_t)
$$

更新方向退化为梯度的符号，幅度由 $\eta$ 统一控制。这就是 Adam 对尺度不敏感的数学根源——它把"梯度幅度"信息丢掉，只保留"梯度方向"。

**工程意义**：嵌入层和分类层梯度尺度差几个数量级，纯 SGD 会让嵌入层几乎不学。Adam 用 $\sqrt{v}$ 归一化后两者学习速度均衡，这是 Adam 成为 NLP 默认优化器的关键原因之一。

### Q2：梯度消失和梯度爆炸的数学根源是什么？残差连接为什么能缓解？

**答**：根源在链式法则的**连乘**。深网络 L 层，损失对第 0 层的梯度：

$$
\frac{\partial L}{\partial \mathbf{h}_0} = \frac{\partial L}{\partial \mathbf{h}_L} \prod_{l=1}^{L} \frac{\partial \mathbf{h}_l}{\partial \mathbf{h}_{l-1}}
$$

每个 $\partial h_l/\partial h_{l-1}$ 是该层的 Jacobian 矩阵。若每层 Jacobian 的特征值约为 $\lambda$：
- $\lambda < 1$：$\lambda^L \to 0$，梯度消失
- $\lambda > 1$：$\lambda^L \to \infty$，梯度爆炸
- $\lambda = 1$：稳定

举例：sigmoid 的导数最大 0.25，10 层连乘 $0.25^{10} \approx 10^{-6}$，必然消失。

**残差连接的缓解机制**：把 $h_{l+1} = F(h_l)$ 改成 $h_{l+1} = h_l + F(h_l)$，反向传播时：

$$
\frac{\partial h_{l+1}}{\partial h_l} = \mathbf{I} + \frac{\partial F}{\partial h_l}
$$

多出来的 **I**（单位矩阵）保证每层 Jacobian 至少有 +1 的保底特征值。即使 $\partial F/\partial h_l$ 的特征值在 $[-1, 1]$ 之间，$1 + \partial F/\partial h_l$ 的特征值也在 $[0, 2]$ 之间，几何平均 $\approx 1$，**长期连乘既不爆炸也不消失**。

数值例子：F 的 Jacobian 特征值 0.5、20 层。无残差时 $0.5^{20} \approx 10^{-6}$（消失）；有残差时 $1.5^{20} \approx 3325$（不消失，可被 LayerNorm + 学习率控制）。残差给梯度留了一条"紧急逃生通道"，不依赖 F 就能传到远处，这是 ResNet 能训到 152 层、Transformer 能训到 100+ 层的关键。

### Q3：LayerNorm 和 BatchNorm 的区别？为什么 Transformer 用 LayerNorm？分布式训练时梯度裁剪怎么做？

**答**：两者都是把激活归一化到均值 0、方差 1，再仿射 $\gamma$、$\beta$，但**归一化的维度不同**：

设输入形状 $(B, L, d)$（batch、序列长、特征维）：
- **BatchNorm**：在 $B$ 维（和 $L$ 维）上对每个特征维 $d$ 独立归一化，$\mu/\sigma$ 形状 $(d,)$。每个特征用全 batch 的统计量
- **LayerNorm**：在 $d$ 维上对每个样本独立归一化，$\mu/\sigma$ 形状 $(B, L, 1)$。每个样本用自己的统计量

**Transformer 用 LayerNorm 的原因**：
1. 序列长度可变，batch 内不同样本长度不同，BatchNorm 没法对齐统计量
2. 推理时 batch=1，BatchNorm 退化为用累积的 running mean/var，训练-推理行为不一致；LayerNorm 训练推理完全一致
3. NLP 任务里不同 token 的特征维语义差异大（不同词的 embedding），强行跨样本统计没意义

**分布式梯度裁剪**：阈值是全局梯度范数，但梯度分布在多个 rank 上，必须跨 rank 汇总。步骤：
1. 每个 rank 算自己参数梯度的局部平方和 $\text{local\_sq}_r = \sum_{i\in r} \lVert\mathbf{g}_i\rVert^2$
2. AllReduce 求和得 $\text{global\_sq} = \sum_r \text{local\_sq}_r$（所有 rank 都拿到同一个值）
3. 算 $\text{global\_norm} = \sqrt{\text{global\_sq}}$，裁剪系数 $\text{coef} = \min(1, c/(\text{global\_norm}+\epsilon))$
4. 每个 rank 用**同一个 coef** 缩放自己的梯度

关键点：必须用同一个 coef，否则不同 rank 缩放比不同，等于把梯度方向扭了。通信量极小（只 AllReduce 一个标量），但同步开销不可忽略。PyTorch 的 `torch.nn.utils.clip_grad_norm_` 已自动处理分布式场景。

## 深入追问（5题）

### Q4：Adam 的内存开销到底有多大？为什么大模型训练要省这个？

每个参数 Adam 维护 m 和 v 两个状态，各占一份参数大小的显存。加上参数本身（训练时通常 FP16 存参数 + FP32 存一份主权重用于更新），总显存结构（FP16 混合精度训练）：

- 参数 FP16：2 字节 $\times N$
- 参数主权重 FP32：4 字节 $\times N$（用于精确更新）
- 梯度 FP16：2 字节 $\times N$
- Adam m 状态 FP32：4 字节 $\times N$
- Adam v 状态 FP32：4 字节 $\times N$
- **合计 16 字节/参数**

7B 模型 $= 16 \times 7\text{G} =$ **112 GB**，单卡 80G 装不下。所以才需要 ZeRO 优化（把状态分片到多卡）、LoRA（只训少量参数，状态小）。换用 Adafactor 可省 v 状态（用低秩近似），但精度略损。

### Q5：为什么偏差修正 $1-\beta^t$ 不可省？早期不修会怎样？

$t=1$ 时 $m_1 = (1-\beta_1) g_1$。若 $\beta_1=0.9$，$m_1 = 0.1 g_1$，比真实梯度小 10 倍。更新量也小 10 倍——第一步几乎不学。$\beta_2=0.999$ 更极端，$v_1 = 0.001 g_1^2$，$\sqrt{v_1} \approx 0.032 |g_1|$，更新量被除以 0.032 放大约 30 倍——又会爆炸。

偏差修正后：$\hat{m}_1 = 0.1 g_1 / (1-0.9) = g_1$，$\hat{v}_1 = 0.001 g_1^2 / (1-0.999) = g_1^2$，完美还原。**前几百步必须修正**，$t\to\infty$ 后 $\beta^t \to 0$，修正系数 $\to 1$，影响消失。

### Q6：残差连接里 F 的 Jacobian 特征值若是 -0.9 会怎样？

$1 + (-0.9) = 0.1$，每层衰减 10 倍，100 层衰减 $10^{-100}$，**仍然消失**。残差不是万能药——它只保证"特征值在 $[-1, 1]$ 时连乘不爆炸"，但若 $\partial F/\partial \mathbf{x}$ 接近 $-1$，$1 + \partial F/\partial \mathbf{x}$ 接近 0，仍会消失。实际工程中 F 通常是若干层 Conv + ReLU，$\partial F/\partial \mathbf{x}$ 的特征值分布通常在 $[-0.5, 0.5]$，$1 + \partial F/\partial \mathbf{x}$ 在 $[0.5, 1.5]$，几何平均 $\approx 1$，所以稳定。如果 F 设计不当（如全 0 初始化的深层 F），残差也会失效。

### Q7：RMSNorm 比 LayerNorm 真的够用吗？什么场景下不够？

实验上在 Transformer 各任务（语言模型、翻译、分类）中 RMSNorm 和 LayerNorm 表现基本持平，部分任务 RMSNorm 略好（可能是减少了 $\beta$ 带来的正则化效果）。LLaMA、PaLM、Gemini 都用 RMSNorm。

不够用的场景：当激活均值明显偏离 0 时（如某些特殊激活函数后、或某些非 Transformer 架构），RMSNorm 不减均值的差异会放大。比如激活恒正（ReLU 后未接其他操作）时，均值非零，LayerNorm 减均值能去掉这个偏置，RMSNorm 不能。但深网络里通常多个操作叠加后均值接近 0，RMSNorm 的近似足够。

### Q8：mini-batch 的 B 怎么选？越大越好吗？

不是越大越好。$B$ 翻倍，方差只缩小 $\sqrt{2} \approx 1.4$ 倍，但显存翻倍。**$B$ 的选择权衡**：

- **B 太小（如 1-8）**：方差大，训练抖动；但每步快，单位时间更新多；GPU 利用率低
- **B 太大（如 4096+）**：方差小，方向准；但显存爆炸，每步慢；泛化能力反而可能下降（"泛化间隙"现象）
- **甜区**：CV 常用 256-512，NLP/Transformer 常用 32-256（受序列长影响）

大模型训练常用"梯度累积"——前向反向多次、梯度累加后再 step，等效 B 比单步 B 大。这能在显存不够时模拟大 batch。

学习率通常和 $B$ 同步调：$B$ 翻倍，$\eta$ 也大致翻倍（线性缩放规则），直到 $\eta$ 大到不稳定为止。

## 易混淆点表

| 易混淆概念 | 区别 | 记忆口诀 |
|-----------|------|---------|
| **SGD vs SGD+Momentum vs Adam** | SGD 只看当前梯度；Momentum 加历史梯度指数平均；Adam 再加每个参数的二阶矩自适应 | "SGD 走一步看一步，动量带惯性，Adam 自调步幅" |
| **全量梯度 vs mini-batch 梯度** | 全量用 N 个样本算精确梯度；mini-batch 用 B 个算无偏估计 | "全量准但慢，mini-batch 快且无偏" |
| **一阶矩 m vs 二阶矩 v** | m 是梯度的指数平均（方向）；v 是梯度平方的指数平均（尺度） | "m 看方向，v 看尺度" |
| **梯度消失 vs 梯度爆炸** | 连乘 <1 消失、>1 爆炸；前者参数不更新、后者 NaN | "小连乘消失，大连乘爆炸，1 才稳" |
| **残差连接 vs 普通连接** | 普通 $h=F(h)$；残差 $h=h+F(h)$，多恒等旁路 | "残差 = 主路 + 旁路" |
| **Xavier vs Kaiming 初始化** | Xavier 适合 tanh/sigmoid，$\operatorname{Var}(w)=2/(n_{in}+n_{out})$；Kaiming 适合 ReLU，$\operatorname{Var}(w)=2/n_{in}$ | "Xavier 配 tanh，Kaiming 配 ReLU" |
| **LayerNorm vs BatchNorm** | LayerNorm 在特征维归一化（每个样本独立）；BatchNorm 在 batch 维归一化（跨样本） | "LN 看自己，BN 看同事" |
| **LayerNorm vs RMSNorm** | LayerNorm 减均值除标准差 + γ、β；RMSNorm 只除 RMS + γ | "LN 全套，RMS 精简" |
| **Post-LN vs Pre-LN** | Post-LN 在残差后归一化 x=LN(x+Sub)；Pre-LN 在残差前归一化 x=x+Sub(LN(x)) | "Post 训练需 warmup，Pre 训练稳定" |
| **按范数裁剪 vs 按值裁剪** | 按范数缩放保方向；按值截断破坏方向 | "范数裁保方向，值裁断方向" |
| **局部范数 vs 全局范数（分布式）** | 局部范数只算本 rank；全局范数 AllReduce 所有 rank 平方和 | "分布式必须 AllReduce，否则各 rank 裁不一致" |
| **Adam 状态 vs 参数本身** | 状态 m、v 是优化器维护的辅助量；参数是要学的目标 | "参数是学生，状态是笔记" |

## 本章公式速查表

| 概念 | 公式 |
|------|------|
| mini-batch 梯度 | $\hat{g} = (1/B) \sum_{i\in\mathcal{B}} \nabla L_i(\theta)$ |
| 动量 | $\mathbf{v}_t = \beta\mathbf{v}_{t-1} + (1-\beta)\mathbf{g}_t$；$\theta \leftarrow \theta - \eta\mathbf{v}_t$ |
| Adam 一阶矩 | $\mathbf{m}_t = \beta_1\mathbf{m}_{t-1} + (1-\beta_1)\mathbf{g}_t$ |
| Adam 二阶矩 | $\mathbf{v}_t = \beta_2\mathbf{v}_{t-1} + (1-\beta_2)\mathbf{g}_t^2$ |
| Adam 偏差修正 | $\hat{\mathbf{m}}_t = \mathbf{m}_t/(1-\beta_1^t)$，$\hat{\mathbf{v}}_t = \mathbf{v}_t/(1-\beta_2^t)$ |
| Adam 更新 | $\theta \leftarrow \theta - \eta\hat{\mathbf{m}}_t/(\sqrt{\hat{\mathbf{v}}_t} + \epsilon)$ |
| 梯度连乘 | $\partial L/\partial \mathbf{h}_0 = (\partial L/\partial \mathbf{h}_L) \prod (\partial \mathbf{h}_l/\partial \mathbf{h}_{l-1})$ |
| 残差连接前向 | $\mathbf{h}_{l+1} = \mathbf{h}_l + F(\mathbf{h}_l)$ |
| 残差连接反向 | $\partial \mathbf{h}_{l+1}/\partial \mathbf{h}_l = \mathbf{I} + \partial F/\partial \mathbf{h}_l$ |
| 方差传播 | $\operatorname{Var}(y) = n \operatorname{Var}(w) \operatorname{Var}(x)$ |
| Xavier 方差 | $\operatorname{Var}(w) = 2/(n_{in}+n_{out})$ |
| Kaiming 方差 | $\operatorname{Var}(w) = 2/n_{in}$ |
| LayerNorm | $\operatorname{LayerNorm}(x_i) = \gamma_i (x_i-\mu)/\sqrt{\sigma^2+\epsilon} + \beta_i$ |
| RMSNorm | $\operatorname{RMSNorm}(x_i) = \gamma_i x_i/\operatorname{RMS}(\mathbf{x})$ |
| RMS | $\operatorname{RMS}(\mathbf{x}) = \sqrt{\frac{1}{H}\sum_i x_i^2 + \epsilon}$ |
| 梯度裁剪 | $\mathbf{g} \leftarrow \mathbf{g} \cdot \min(1, c/(\lVert\mathbf{g}\rVert_2+\epsilon))$ |
| 分布式全局范数 | $\text{global\_sq} = \sum_r \text{local\_sq}_r$（AllReduce 求和） |

## 概念关系图：把全章串起来

```
                ┌─────────────────────────────────────┐
                │   问题：深网络梯度怎么传得稳？       │
                └─────────────┬───────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
   更新策略层              结构层                  数值层
   (优化器)              (网络结构)              (归一化/裁剪)
        │                     │                     │
   ┌────┴────┐           ┌────┴────┐           ┌────┴────┐
   ▼         ▼           ▼         ▼           ▼         ▼
 SGD     Momentum      残差      初始化     LayerNorm  梯度裁剪
  │      (重球滚下坡)  (逃生通道) (Xavier/   (样本内部   (按范数缩放
  │         │            │       Kaiming)   归一化)     到阈值)
  │         │            │         │           │
  │         └──→ Adam ←──┘         │           │
  │           (动量 + 自适应)       │           │
  │              │                  │           │
  │         除以 √v 把每个           │           │
  │         参数梯度尺度归一         │           │
  │                                ▼           ▼
  │                            RMSNorm      分布式场景
  │                          (省掉减均值)    AllReduce
  │                                         算全局范数
  ▼
 mini-batch 是基础
 B 越大方差越小
 但收益递减
```

**主线**：mini-batch 是工程基础 → SGD 在噪声方向上抖动，加 Momentum 平滑 → 不同参数尺度差大，加 Adam 自适应 → 深网络连乘导致消失/爆炸，加残差保底、加良好初始化稳特征值 → 每层激活仍可能漂移，加 LayerNorm/RMSNorm 拉回标准范围 → 万一还爆炸，加梯度裁剪兜底。**所有这些技术都是为了一个目标：让梯度既更新得快、又传得远、还不爆炸**。
