# 练习 1：浅拷贝共享内层列表的问题与修复

> 关联笔记：`综合实战：跨层思维与自检练习.md` 5.1 练习 1
> 知识点：Python 对象引用模型、浅拷贝 vs 深拷贝、可变对象共享引用
> 难度：入门

---

## 一句话结论

`copy.copy()` 只复制外层容器，内层对象仍然是**共享引用**——改浅拷贝的内层，原始对象也会变。修复方式：数据重构（新建内层）或 `copy.deepcopy()`（递归复制所有层级）。

---

## 题目

复现浅拷贝共享内层列表的问题，并用数据重构和深拷贝两种方式修复。

---

## 知识点

| 概念 | 说明 |
|------|------|
| 变量是引用 | Python 变量不存值，存的是对象的地址（标签贴在对象上） |
| 浅拷贝 `copy.copy()` | 创建新外层容器，内层元素仍指向原对象（共享引用） |
| 深拷贝 `copy.deepcopy()` | 递归创建所有层级的新对象，完全独立 |
| `is` vs `==` | `is` 比较对象身份（id），`==` 比较值（`__eq__`） |

## 一步步实现

### 第 1 步：复现问题——浅拷贝后改内层，原始对象也被改了

```python
# shallow_copy_bug.py
import copy

# 原始数据：嵌套列表
original = [[1, 2], [3, 4]]
print("=== 初始状态 ===")
print(f"original: {original}")           # [[1, 2], [3, 4]]
print(f"original[0] id: {id(original[0])}")

# 浅拷贝
shallow = copy.copy(original)
print("\n=== 浅拷贝后 ===")
print(f"shallow:  {shallow}")            # [[1, 2], [3, 4]]
print(f"shallow[0] id: {id(shallow[0])}")
print(f"original[0] is shallow[0]: {original[0] is shallow[0]}")  # True ← 内层共享！

# 改浅拷贝的内层
shallow[0].append(99)
print("\n=== 改 shallow[0].append(99) 后 ===")
print(f"shallow:  {shallow}")            # [[1, 2, 99], [3, 4]]
print(f"original: {original}")           # [[1, 2, 99], [3, 4]]  ← original 也被改了！
```

#### 预期结果

```
=== 初始状态 ===
original: [[1, 2], [3, 4]]
original[0] id: 140234567890240

=== 浅拷贝后 ===
shallow:  [[1, 2], [3, 4]]
shallow[0] id: 140234567890240
original[0] is shallow[0]: True

=== 改 shallow[0].append(99) 后 ===
shallow:  [[1, 2, 99], [3, 4]]
original: [[1, 2, 99], [3, 4]]
```

> **id 值每次运行不同**，关键看 `original[0] is shallow[0]` 为 `True`——两个变量指向同一个内层列表对象。

### 第 2 步：修复一——数据重构（新建内层）

```python
# fix1_reconstruct.py
import copy

original = [[1, 2], [3, 4]]

# 数据重构：用列表推导式为每个内层创建新列表
fixed1 = [list(sub) for sub in original]
# 等价于：fixed1 = [sub[:] for sub in original]
# 等价于：fixed1 = [sub.copy() for sub in original]

print("=== 数据重构后 ===")
print(f"original:     {original}")
print(f"fixed1:       {fixed1}")
print(f"original[0] is fixed1[0]: {original[0] is fixed1[0]}")  # False ← 内层独立了

fixed1[0].append(99)
print("\n=== 改 fixed1[0].append(99) 后 ===")
print(f"fixed1:       {fixed1}")        # [[1, 2, 99], [3, 4]]
print(f"original:     {original}")       # [[1, 2], [3, 4]]     ← 不受影响
```

#### 预期结果

```
=== 数据重构后 ===
original:     [[1, 2], [3, 4]]
fixed1:       [[1, 2], [3, 4]]
original[0] is fixed1[0]: False

=== 改 fixed1[0].append(99) 后 ===
fixed1:       [[1, 2, 99], [3, 4]]
original:     [[1, 2], [3, 4]]
```

### 第 3 步：修复二——深拷贝

```python
# fix2_deepcopy.py
import copy

original = [[1, 2], [3, 4]]

# 深拷贝：递归复制所有层级
fixed2 = copy.deepcopy(original)

print("=== 深拷贝后 ===")
print(f"original:     {original}")
print(f"fixed2:       {fixed2}")
print(f"original[0] is fixed2[0]: {original[0] is fixed2[0]}")  # False
print(f"original is fixed2:       {original is fixed2}")        # False

fixed2[0].append(99)
fixed2.append([5, 6])
print("\n=== 改 fixed2 后 ===")
print(f"fixed2:       {fixed2}")        # [[1, 2, 99], [3, 4], [5, 6]]
print(f"original:     {original}")       # [[1, 2], [3, 4]]     ← 完全不受影响
```

#### 预期结果

```
=== 深拷贝后 ===
original:     [[1, 2], [3, 4]]
fixed2:       [[1, 2], [3, 4]]
original[0] is fixed2[0]: False
original is fixed2:       False

=== 改 fixed2 后 ===
fixed2:       [[1, 2, 99], [3, 4], [5, 6]]
original:     [[1, 2], [3, 4]]
```

## 逐行解释

### 为什么浅拷贝会共享内层？

```python
original = [[1, 2], [3, 4]]
```

内存状态：
```
original ──→ [ list_A, list_B ]
                │         │
                ↓         ↓
             [1, 2]    [3, 4]
             list_A    list_B
```

```python
shallow = copy.copy(original)
```

浅拷贝只复制外层列表，内层引用不变：
```
original ──→ [ list_A, list_B ]    ← 原外层
shallow  ──→ [ list_A, list_B ]    ← 新外层（但元素指向同一个 list_A、list_B）
                │         │
                ↓         ↓
             [1, 2]    [3, 4]
             list_A    list_B      ← 内层共享！
```

所以 `shallow[0].append(99)` 改的是 `list_A`，而 `original[0]` 也指向 `list_A`，所以 original 也变了。

### 数据重构为什么能修复？

```python
fixed1 = [list(sub) for sub in original]
```

`list(sub)` 对每个内层调用 `list()` 构造函数，创建新列表：
```
original ──→ [ list_A, list_B ]
                │         │
                ↓         ↓
             [1, 2]    [3, 4]
             list_A    list_B

fixed1   ──→ [ list_C, list_D ]    ← 新外层
                │         │
                ↓         ↓
             [1, 2]    [3, 4]
             list_C    list_D      ← 全新的内层！
```

### 深拷贝为什么能修复？

`copy.deepcopy()` 递归遍历整个对象树，每一层都创建新对象。对于嵌套列表、字典、自定义对象都能正确处理。它还能处理循环引用（通过 memo 字典记录已拷贝的对象）。

---

## 两种修复怎么选

| 方法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 数据重构 `[list(sub) for sub in ...]` | 只有一层嵌套，结构已知 | 显式、可控、快 | 多层嵌套要手写多层推导 |
| `copy.deepcopy()` | 任意深度嵌套、字典/对象混合 | 通用、一行搞定 | 慢（递归+类型检查）、可能拷贝不该拷贝的东西 |

---

## 常见错误

| 错误 | 原因 | 修复 |
|------|------|------|
| `fixed = original.copy()` 后改 fixed 内层，original 也变了 | `list.copy()` 也是浅拷贝 | 用 `copy.deepcopy()` |
| `fixed = original[:]` 后改 fixed 内层，original 也变了 | 切片 `[:]` 也是浅拷贝 | 同上 |
| `fixed = original` 后改 fixed，original 也变了 | 这不是拷贝，是赋值别名 | 这根本不是拷贝 |
| deepcopy 报 `TypeError: cannot pickle` | 对象含有不可序列化的属性（如文件句柄、线程锁） | 自定义 `__deepcopy__` 方法或用数据重构 |
