# 练习 2：实现带参数的 @retry 装饰器

> 关联笔记：`综合实战：跨层思维与自检练习.md` 5.1 练习 2
> 知识点：闭包、装饰器、`functools.wraps`、异常处理、可调用对象
> 难度：入门

---

## 一句话结论

带参数的装饰器是**三层嵌套函数**：外层接收参数（`max_attempts`、`exceptions`），中层接收被装饰函数，内层执行重试逻辑。用 `@functools.wraps` 保留原函数元信息，只捕获指定异常类型。

---

## 题目

实现一个 `@retry(max_attempts=3, exceptions=(ValueError, ConnectionError))` 装饰器，当被装饰函数抛出指定异常时自动重试，达到最大次数后重新抛出。

---

## 知识点

| 概念 | 说明 |
|------|------|
| 装饰器 | 接收函数返回函数的可调用对象，用于在不修改原函数的前提下增加功能 |
| 带参数的装饰器 | 三层嵌套：参数层 → 装饰器层 → 包装层 |
| 闭包 | 内层函数引用外层变量（`max_attempts`、`exceptions`、`func`） |
| `functools.wraps` | 把原函数的 `__name__`、`__doc__`、`__module__` 复制到包装函数 |
| 异常元组 | `except (A, B)` 同时捕获多种异常 |

## 一步步实现

### 第 1 步：理解普通装饰器（无参数）

先回顾不带参数的简单装饰器，理解"两层嵌套"：

```python
# simple_decorator.py
import functools

def log_call(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"调用 {func.__name__}({args}, {kwargs})")
        result = func(*args, **kwargs)
        print(f"{func.__name__} 返回 {result}")
        return result
    return wrapper

@log_call          # 等价于 add = log_call(add)
def add(a, b):
    """两数相加"""
    return a + b

print(add(1, 2))
print(f"函数名: {add.__name__}")     # add（因为用了 @functools.wraps）
print(f"文档: {add.__doc__}")        # 两数相加
```

#### 预期结果

```
调用 add((1, 2), {})
add 返回 3
3
函数名: add
文档: 两数相加
```

### 第 2 步：实现带参数的 retry 装饰器

```python
# retry_decorator.py
import functools
import random

def retry(max_attempts=3, exceptions=(Exception,)):
    """
    带参数的重试装饰器。

    参数：
        max_attempts: 最大尝试次数（含第一次）
        exceptions:   遇到哪些异常时重试（元组）
    """
    # 第一层：接收参数（max_attempts, exceptions）
    def decorator(func):
        # 第二层：接收被装饰的函数 func
        @functools.wraps(func)           # 保留 func 的 __name__/__doc__
        def wrapper(*args, **kwargs):
            # 第三层：实际执行的包装函数
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_attempts - 1:
                        raise             # 最后一次失败，原样抛出
                    print(f"  [retry] 第 {attempt+1} 次失败: {e}，重试中...")
        return wrapper
    return decorator


# ===== 测试：模拟不稳定的网络请求 =====

@retry(max_attempts=3, exceptions=(ConnectionError,))
def fetch_data(url):
    """模拟网络请求，有 70% 概率失败"""
    if random.random() < 0.7:
        raise ConnectionError(f"连接超时: {url}")
    return f"来自 {url} 的数据"

print("=== 测试 fetch_data（最多重试 3 次）===")
for i in range(3):
    try:
        data = fetch_data("https://api.example.com/data")
        print(f"  成功: {data}")
    except ConnectionError as e:
        print(f"  最终失败: {e}")

# ===== 测试：只捕获指定异常 =====

@retry(max_attempts=3, exceptions=(ValueError,))
def parse_int(s):
    """解析整数，只对 ValueError 重试"""
    return int(s)

print("\n=== 测试 parse_int（只重试 ValueError）===")
try:
    print(parse_int("abc"))    # ValueError 会被重试 3 次后抛出
except ValueError as e:
    print(f"  最终抛出: {type(e).__name__}: {e}")

# ===== 测试：不捕获的异常直接抛出 =====
print("\n=== 测试不捕获的异常直接抛出 ===")
try:
    print(parse_int(123))      # 正常返回
except Exception as e:
    print(f"  异常: {e}")
```

#### 预期结果（fetch_data 部分因随机性每次不同）

```
=== 测试 fetch_data（最多重试 3 次）===
  [retry] 第 1 次失败: 连接超时: https://api.example.com/data，重试中...
  成功: 来自 https://api.example.com/data 的数据
  [retry] 第 1 次失败: 连接超时: https://api.example.com/data，重试中...
  [retry] 第 2 次失败: 连接超时: https://api.example.com/data，重试中...
  最终失败: 连接超时: https://api.example.com/data
  成功: 来自 https://api.example.com/data 的数据

=== 测试 parse_int（只重试 ValueError）===
  [retry] 第 1 次失败: invalid literal for int() with base 10: 'abc'，重试中...
  [retry] 第 2 次失败: invalid literal for int() with base 10: 'abc'，重试中...
  最终抛出: ValueError: invalid literal for int() with base 10: 'abc'
```

### 第 3 步：验证 @functools.wraps 的作用

```python
# check_wraps.py
import functools

# 不用 @functools.wraps
def bad_retry(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

# 用 @functools.wraps
def good_retry(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@bad_retry
def func_a():
    """这是 func_a 的文档"""
    pass

@good_retry
def func_b():
    """这是 func_b 的文档"""
    pass

print("=== 不用 @functools.wraps ===")
print(f"  __name__: {func_a.__name__}")    # wrapper ← 名字丢了！
print(f"  __doc__:  {func_a.__doc__}")      # None    ← 文档丢了！

print("\n=== 用 @functools.wraps ===")
print(f"  __name__: {func_b.__name__}")    # func_b  ← 保留了
print(f"  __doc__:  {func_b.__doc__}")      # 这是 func_b 的文档 ← 保留了
```

#### 预期结果

```
=== 不用 @functools.wraps ===
  __name__: wrapper
  __doc__:  None

=== 用 @functools.wraps ===
  __name__: func_b
  __doc__:  这是 func_b 的文档
```

## 逐行解释

### 三层嵌套结构

```python
def retry(max_attempts=3, exceptions=(Exception,)):   # 第 1 层：参数层
    def decorator(func):                               # 第 2 层：装饰器层
        @functools.wraps(func)
        def wrapper(*args, **kwargs):                  # 第 3 层：包装层
            ...
        return wrapper
    return decorator
```

调用过程：

```python
@retry(max_attempts=3, exceptions=(ConnectionError,))
def fetch_data(url): ...

# Python 执行顺序：
# 1. retry(max_attempts=3, exceptions=(ConnectionError,)) → 返回 decorator 函数
# 2. decorator(fetch_data) → 返回 wrapper 函数
# 3. fetch_data 现在指向 wrapper
#
# 之后每次调用 fetch_data(url) 实际调用 wrapper(url)
# wrapper 通过闭包访问 max_attempts、exceptions、func
```

### 重试逻辑

```python
for attempt in range(max_attempts):     # 0, 1, 2（max_attempts=3）
    try:
        return func(*args, **kwargs)     # 成功就返回，不再重试
    except exceptions as e:              # 只捕获指定的异常
        if attempt == max_attempts - 1:  # 最后一次（attempt=2）
            raise                        # 不再重试，原样抛出
        print(f"...重试中...")            # 非最后一次，打印提示，继续循环
```

关键点：
- `return` 在 `try` 里——成功时直接返回，不会走到 `except`
- `except exceptions`——只捕获元组里的异常类型，其他异常（如 `TypeError`）直接向上抛出，不重试
- `if attempt == max_attempts - 1: raise`——最后一次不再 print，直接 raise 让调用者处理

## 常见错误

| 错误 | 原因 | 修复 |
|------|------|------|
| `TypeError: decorator() missing 1 required positional argument: 'func'` | 少写了一层嵌套，参数层直接接收了 func | 确保三层嵌套结构 |
| 装饰后 `func.__name__` 变成 `wrapper` | 没用 `@functools.wraps` | 加上 `@functools.wraps(func)` |
| 所有异常都被重试 | `exceptions` 默认值是 `(Exception,)`，太宽泛 | 显式指定 `exceptions=(ValueError, ConnectionError)` |
| 可变默认参数 `exceptions=[]` | 列表是可变对象，默认参数只创建一次 | 用元组 `(Exception,)` 做默认值 |

## 拓展：用类实现装饰器（等价写法）

```python
class Retry:
    """用类实现的 retry 装饰器，等价于函数版"""
    def __init__(self, max_attempts=3, exceptions=(Exception,)):
        self.max_attempts = max_attempts
        self.exceptions = exceptions

    def __call__(self, func):             # __call__ 让实例可调用，等价于 decorator(func)
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(self.max_attempts):
                try:
                    return func(*args, **kwargs)
                except self.exceptions as e:
                    if attempt == self.max_attempts - 1:
                        raise
                    print(f"  [retry] 第 {attempt+1} 次失败: {e}")
        return wrapper

@Retry(max_attempts=3, exceptions=(ValueError,))
def parse_int(s):
    return int(s)
```

类版和函数版完全等价，选哪个看个人偏好。类版的好处是状态管理更清晰（`self.max_attempts`），函数版更简洁。
