# 练习 5：RAII 文件句柄类（禁用复制，支持移动）

> 关联笔记：`综合实战：跨层思维与自检练习.md` 5.2 练习 5
> 知识点：RAII、Rule of Five、移动语义、`noexcept`、资源管理
> 难度：中等

---

## 一句话结论

RAII 类在构造时获取资源、析构时释放资源；禁用复制防止两个对象管理同一资源导致 double-free；支持移动转移所有权，源对象置空防止重复释放。

---

## 题目

写一个持有 `FILE*` 句柄的 C++ RAII 类，要求：
1. 构造时打开文件，析构时关闭文件
2. 禁用复制（防止 double close）
3. 支持移动（转移所有权）
4. 提供 `get()` 访问底层句柄

---

## 知识点

| 概念 | 说明 |
|------|------|
| RAII | Resource Acquisition Is Initialization——构造获取资源，析构释放资源 |
| Rule of Five | 如果自定义了析构/复制构造/复制赋值/移动构造/移动赋值中的一个，通常需要全部定义 |
| `= delete` | 禁用某个函数（C++11） |
| `noexcept` | 标记函数不抛异常——移动构造/赋值应该加 |
| 移动语义 | 转移资源所有权而非拷贝，源对象置为有效但未指定状态 |

## 一步步实现

### 第 1 步：创建项目文件

```
file_guard/
├── file_guard.hpp     # RAII 类声明+实现（头文件内联，方便教学）
└── main.cpp           # 测试程序
```

### 第 2 步：实现 RAII 类

```cpp
// file_guard.hpp
#pragma once

#include <cstdio>       // FILE, fopen, fclose
#include <stdexcept>    // std::runtime_error
#include <utility>      // std::swap (C++11 移动语义相关)

class FileGuard {
private:
    FILE* fp_;          // 底层文件句柄，nullptr 表示不持有

public:
    // ===== 构造函数：获取资源 =====
    explicit FileGuard(const char* path, const char* mode = "r")
        : fp_(std::fopen(path, mode)) {
        if (!fp_) {
            throw std::runtime_error(std::string("无法打开文件: ") + path);
        }
    }

    // ===== 析构函数：释放资源（RAII 核心）=====
    ~FileGuard() {
        if (fp_) {
            std::fclose(fp_);
            // 析构函数不能抛异常，fclose 的返回值在这里忽略
            // 生产代码可记日志
        }
    }

    // ===== 禁用复制（防止 double close）=====
    FileGuard(const FileGuard&) = delete;
    FileGuard& operator=(const FileGuard&) = delete;

    // ===== 移动构造函数：转移所有权 =====
    FileGuard(FileGuard&& other) noexcept
        : fp_(other.fp_) {
        other.fp_ = nullptr;    // 源置空，防止 other 析构时重复关闭
    }

    // ===== 移动赋值运算符 =====
    FileGuard& operator=(FileGuard&& other) noexcept {
        if (this != &other) {           // 自赋值检查
            if (fp_) std::fclose(fp_);  // 先释放自己持有的资源
            fp_ = other.fp_;            // 接管对方的资源
            other.fp_ = nullptr;        // 对方置空
        }
        return *this;
    }

    // ===== 访问底层句柄 =====
    FILE* get() const noexcept { return fp_; }

    // 释放所有权（交出句柄，不再负责关闭）
    FILE* release() noexcept {
        FILE* tmp = fp_;
        fp_ = nullptr;
        return tmp;
    }
};
```

### 第 3 步：写测试程序

```cpp
// main.cpp
#include "file_guard.hpp"
#include <iostream>
#include <string>

int main() {
    // ===== 1. 基本 RAII：构造打开，作用域结束自动关闭 =====
    std::cout << "=== 1. 基本 RAII ===" << std::endl;
    {
        FileGuard fg("test.txt", "w");
        std::fprintf(fg.get(), "Hello RAII!\n");
        std::cout << "  文件已打开并写入" << std::endl;
        // 离开作用域，~FileGuard() 自动调用 fclose
    }
    std::cout << "  文件已自动关闭（析构）" << std::endl;

    // ===== 2. 移动构造：转移所有权 =====
    std::cout << "\n=== 2. 移动构造 ===" << std::endl;
    {
        FileGuard fg1("test.txt", "r");
        std::cout << "  fg1.get() != nullptr: " << (fg1.get() != nullptr) << std::endl;

        FileGuard fg2(std::move(fg1));    // fg1 的资源转移给 fg2
        std::cout << "  移动后:" << std::endl;
        std::cout << "    fg1.get() == nullptr: " << (fg1.get() == nullptr) << std::endl;  // true
        std::cout << "    fg2.get() != nullptr: " << (fg2.get() != nullptr) << std::endl;  // true

        // fg1 现在是有效但空的状态，析构时不会 fclose（fp_ 是 nullptr）
        // fg2 析构时会 fclose
    }
    std::cout << "  移动构造测试完成" << std::endl;

    // ===== 3. 移动赋值 =====
    std::cout << "\n=== 3. 移动赋值 ===" << std::endl;
    {
        FileGuard fg1("test.txt", "r");
        FileGuard fg2("test2.txt", "w");
        std::cout << "  赋值前: fg1 持有 test.txt, fg2 持有 test2.txt" << std::endl;

        fg2 = std::move(fg1);    // fg2 先关闭 test2.txt，再接管 test.txt
        std::cout << "  赋值后:" << std::endl;
        std::cout << "    fg1.get() == nullptr: " << (fg1.get() == nullptr) << std::endl;
        std::cout << "    fg2.get() != nullptr: " << (fg2.get() != nullptr) << std::endl;
    }

    // ===== 4. 禁用复制 =====
    std::cout << "\n=== 4. 禁用复制 ===" << std::endl;
    {
        FileGuard fg1("test.txt", "r");
        // FileGuard fg2 = fg1;           // 编译错误！复制构造被 delete
        // FileGuard fg2(fg1);            // 编译错误！
        std::cout << "  复制构造已禁用（取消注释会编译报错）" << std::endl;
    }

    // ===== 5. 异常安全 =====
    std::cout << "\n=== 5. 异常安全 ===" << std::endl;
    try {
        FileGuard fg("不存在的文件.txt", "r");
    } catch (const std::exception& e) {
        std::cout << "  捕获异常: " << e.what() << std::endl;
    }

    // ===== 6. release() 放弃所有权 =====
    std::cout << "\n=== 6. release() ===" << std::endl;
    {
        FileGuard fg("test.txt", "r");
        FILE* raw = fg.release();         // fg 不再持有，不会 fclose
        std::cout << "  release 后 fg.get() == nullptr: " << (fg.get() == nullptr) << std::endl;
        std::fclose(raw);                 // 调用者负责关闭
    }

    return 0;
}
```

## 逐行解释

### 为什么禁用复制？

如果允许复制：
```cpp
FileGuard fg1("test.txt");
FileGuard fg2 = fg1;    // 如果允许：fg1.fp_ 和 fg2.fp_ 指向同一个 FILE*
// fg1 析构 → fclose(fp_)
// fg2 析构 → fclose(fp_)  ← double close！可能崩溃或关闭错误的文件
```

### 为什么移动构造要把源置空？

```cpp
FileGuard(FileGuard&& other) noexcept
    : fp_(other.fp_) {       // 接管 other 的资源
    other.fp_ = nullptr;     // ← 关键！other 析构时 if(fp_) 为 false，不会 fclose
}
```

不置空的话：other 析构时会 fclose 这个 FILE*，而新对象也会 fclose 同一个 FILE* → double close。

### 为什么移动操作要 `noexcept`？

`noexcept` 告诉编译器这个函数不会抛异常。好处：
1. **标准容器优化**：`std::vector` 扩容时，如果元素移动构造是 `noexcept`，用移动；否则用拷贝（保证异常安全）。不加 `noexcept` 会导致 vector 扩容时退化为拷贝，失去移动优势。
2. **性能**：编译器可以省略异常处理代码。

### 移动赋值为什么要先关自己的？

```cpp
FileGuard& operator=(FileGuard&& other) noexcept {
    if (this != &other) {
        if (fp_) std::fclose(fp_);  // 先释放自己当前持有的资源
        fp_ = other.fp_;             // 再接管 other 的
        other.fp_ = nullptr;
    }
    return *this;
}
```

不先关自己的话，原来持有的 FILE* 就泄漏了（没有任何对象再管理它）。

### 为什么用 `explicit`？

```cpp
explicit FileGuard(const char* path, const char* mode = "r");
```

防止隐式转换：
```cpp
FileGuard fg = "test.txt";    // 没有 explicit 时可能隐式转换，令人困惑
FileGuard fg("test.txt");     // 有 explicit 必须显式构造，意图清晰
```

## Rule of Five 对照

| 函数 | 本练习的实现 | 作用 |
|------|-------------|------|
| 析构函数 | `~FileGuard()` | 释放资源（fclose） |
| 复制构造 | `= delete` | 禁用（防止 double close） |
| 复制赋值 | `= delete` | 禁用 |
| 移动构造 | 实现，`noexcept` | 转移所有权 |
| 移动赋值 | 实现，`noexcept` | 转移所有权，先释放自身 |

**Rule of Zero**：如果用 `std::unique_ptr<FILE, decltype(&fclose)>` 管理资源，编译器自动生成正确的析构/移动/禁用复制，不需要手写任何特殊成员函数。这是现代 C++ 推荐的做法。

```cpp
// Rule of Zero 版本（等价但更简洁）
#include <memory>
#include <cstdio>

struct FileCloser {
    void operator()(FILE* fp) const { if (fp) std::fclose(fp); }
};

using UniqueFile = std::unique_ptr<FILE, FileCloser>;

// 使用：
UniqueFile fp(std::fopen("test.txt", "r"));
// 自动禁用复制，支持移动，析构自动 fclose——全靠 unique_ptr
```

## 常见错误

| 错误 | 原因 | 修复 |
|------|------|------|
| 移动后源对象崩溃 | 没把源对象的指针置 nullptr | 移动构造/赋值里必须 `other.fp_ = nullptr` |
| `std::vector<FileGuard>` 扩容时拷贝而不是移动 | 移动构造没加 `noexcept` | 加 `noexcept` |
| 析构里抛异常 | `fclose` 失败抛异常 | 析构函数里不抛异常，记日志或忽略 |
| 忘记自赋值检查 | `fg = std::move(fg)` 时先关自己再接管自己 | `if (this != &other)` |
