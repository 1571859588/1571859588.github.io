# 练习 9：在干净虚拟环境中验证 PyTorch GPU 环境

> 关联笔记：`综合实战：跨层思维与自检练习.md` 5.3 练习 9
> 知识点：venv、PyTorch 安装、CUDA 版本验证、GPU 软件栈四层
> 难度：中等

---

## 一句话结论

PyTorch 环境验证的核心是回答五个问题：用哪个 Python？装了哪个 PyTorch？构建时用的 CUDA 版本？GPU 是否可见？有几张什么型号的卡？——每一步都要看到实际输出，不能假设。

---

## 题目

在干净的虚拟环境中安装 PyTorch，记录 `sys.executable`、框架版本、构建 CUDA 版本和设备可见性。

---

## 知识点

| 概念 | 说明 |
|------|------|
| `venv` | Python 虚拟环境，隔离 Python 包 |
| `--index-url` | pip 的包索引地址，PyTorch 用专用索引区分 CUDA 版本 |
| `torch.__version__` | PyTorch 版本号（如 `2.3.0+cu121`，`+cu121` 表示 CUDA 12.1） |
| `torch.version.cuda` | PyTorch **构建时**链接的 CUDA 版本 |
| `nvidia-smi` 显示的 CUDA 版本 | 驱动**支持的最高** CUDA 版本，不等于实际安装的 |
| `torch.cuda.is_available()` | PyTorch 是否能真正使用 GPU |

---

## 一步步实现

### 第 1 步：创建干净虚拟环境

```bash
# 在项目目录下创建 venv
python -m venv .venv

# 激活
source .venv/bin/activate        # Linux/macOS
# .venv\Scripts\activate          # Windows

# 确认用的是 venv 的 Python
which python
# /home/user/project/.venv/bin/python   ← 应该是 venv 里的

python --version
# Python 3.11.x
```

### 第 2 步：安装 PyTorch（指定 CUDA 版本）

```bash
# 先看驱动支持的最高 CUDA 版本
nvidia-smi | head -5
# +-----------------------------------------------------------------------------+
# | NVIDIA-SMI 535.129.03   Driver Version: 535.129.03   CUDA Version: 12.2     |
# +-----------------------------------------------------------------------------+
#                                         ↑
#                   这是驱动支持的最高版本，不是已安装的版本
```

> **关键认知**：`nvidia-smi` 显示的 CUDA Version 是**驱动能支持的最高版本**。实际安装的 CUDA Toolkit 版本可以 ≤ 这个值。比如驱动显示 12.2，可以装 cu121（CUDA 12.1）的 PyTorch。

```bash
# 安装 PyTorch（CUDA 12.1 版本）
# --index-url 指定 PyTorch 的专用包索引
python -m pip install torch --index-url https://download.pytorch.org/whl/cu121
```

> 其他 CUDA 版本的索引 URL：
> - CUDA 11.8: `https://download.pytorch.org/whl/cu118`
> - CUDA 12.1: `https://download.pytorch.org/whl/cu121`
> - CPU only: `https://download.pytorch.org/whl/cpu`

### 第 3 步：验证脚本

```python
# verify_env.py
import sys
import torch

print("=" * 50)
print("PyTorch 环境验证")
print("=" * 50)

# 1. Python 信息
print(f"\n[Python]")
print(f"  sys.executable:  {sys.executable}")
print(f"  sys.version:     {sys.version}")

# 2. PyTorch 版本
print(f"\n[PyTorch]")
print(f"  torch.__version__:    {torch.__version__}")
print(f"  torch.version.cuda:   {torch.version.cuda}")
# torch.version.cuda = PyTorch 编译时链接的 CUDA 版本

# 3. GPU 可见性
print(f"\n[GPU]")
print(f"  torch.cuda.is_available(): {torch.cuda.is_available()}")
print(f"  torch.cuda.device_count(): {torch.cuda.device_count()}")

if torch.cuda.is_available():
    for i in range(torch.cuda.device_count()):
        print(f"  设备 {i}: {torch.cuda.get_device_name(i)}")
        props = torch.cuda.get_device_properties(i)
        print(f"    总显存:   {props.total_mem / 1024**3:.1f} GB")
        print(f"    计算能力: {props.major}.{props.minor}")
        print(f"    SM 数量:  {props.multi_processor_count}")

# 4. 驱动版本（从 PyTorch 内部查）
print(f"\n[Driver]")
try:
    print(f"  CUDA driver version (from PyTorch): "
          f"{torch.cuda.get_arch_list()}")
except:
    pass

# 5. 简单 GPU 运算测试
print(f"\n[功能测试]")
if torch.cuda.is_available():
    x = torch.randn(1000, 1000, device="cuda:0")
    y = torch.randn(1000, 1000, device="cuda:0")
    z = x @ y       # 矩阵乘法在 GPU 上执行
    torch.cuda.synchronize()
    print(f"  GPU 矩阵乘法: {z.shape}, dtype={z.dtype}, device={z.device}")
    print(f"  z[0][0] = {z[0][0].item():.6f}")
else:
    print("  GPU 不可用，跳过")

print("\n" + "=" * 50)
print("验证完成")
```

## 逐行解释

### 四层软件栈验证

```
┌──────────────────────────────────────────────┐
│  1. 驱动层（Driver）                          │
│     nvidia-smi → Driver Version: 535.129.03  │
│     nvidia-smi → CUDA Version: 12.2          │ ← 驱动支持的最高 CUDA
├──────────────────────────────────────────────┤
│  2. CUDA 运行时层（PyTorch 自带）              │
│     torch.version.cuda → 12.1                │ ← PyTorch 构建时的 CUDA
│     pip show torch → Requires nvidia-cuda...  │ ← pip 安装的 CUDA 库
├──────────────────────────────────────────────┤
│  3. 框架层（PyTorch）                          │
│     torch.__version__ → 2.3.0+cu121          │ ← 框架版本
├──────────────────────────────────────────────┤
│  4. 设备层（GPU 硬件）                         │
│     torch.cuda.is_available() → True/False   │ ← 框架能否看到设备
│     torch.cuda.get_device_name(0) → RTX 3090 │ ← 设备型号
└──────────────────────────────────────────────┘
```

### 为什么 nvidia-smi 和 torch.version.cuda 可能不同？

| 来源 | 版本 | 含义 |
|------|------|------|
| `nvidia-smi` CUDA Version | 12.2 | 驱动支持的最高 CUDA 版本 |
| `torch.version.cuda` | 12.1 | PyTorch 构建时链接的 CUDA 版本 |

**规则**：PyTorch 的 CUDA 版本 ≤ 驱动支持的最高版本即可。CUDA 12.1 的 PyTorch 可以在支持 12.2 的驱动上运行（向后兼容）。

### 为什么 `torch.cuda.is_available()` 可能为 False？

| 可能原因 | 排查方法 |
|----------|----------|
| 装了 CPU 版本（`+cpu`） | `torch.__version__` 是否有 `+cu` 后缀 |
| 驱动版本太低 | `nvidia-smi` 是否正常输出 |
| CUDA 版本不兼容 | `torch.version.cuda` vs `nvidia-smi` CUDA Version |
| 没有 NVIDIA GPU | `lspci | grep -i nvidia` |
| 容器内没挂载 GPU | `nvidia-smi` 在容器里是否能看到卡 |

---

## 常见错误

| 错误 | 原因 | 修复 |
|------|------|------|
| `torch.cuda.is_available()` 返回 False | 装了 CPU 版本 | 重装：`pip install torch --index-url https://download.pytorch.org/whl/cu121` |
| `CUDA driver version is insufficient` | 驱动版本太低 | 升级 NVIDIA 驱动 |
| `RuntimeError: CUDA error: no kernel image` | GPU 架构和 PyTorch 不匹配（如老卡 + 新 PyTorch） | 装支持老架构的 PyTorch 版本 |
| venv 里 pip 装到了全局 | 没激活 venv 或用了错误的 pip | 用 `python -m pip` 而非 `pip`，确认 `which python` 指向 venv |
