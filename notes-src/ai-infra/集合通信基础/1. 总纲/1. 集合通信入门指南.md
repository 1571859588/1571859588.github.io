## 一句话结论

集合通信是分布式训练的"血管系统"——从单机 NVLink 高速互联、跨机 InfiniBand 网络，到五大通信原语、Ring/Tree AllReduce 算法，再到 NCCL 通信库实战，这条链路决定了多卡训练能跑多快。本系列把这条链路拆成 10 篇笔记，从硬件到算法到工程实战层层递进。

## 背景与定位

单卡 GPU 再强也有天花板，大模型训练不可避免地需要多卡甚至多机协作。此时 GPU 之间"怎么说话、说话多快"就成了决定整体效率的关键。很多新学者学完 PyTorch 单卡训练后，面对 DDP、张量并行、流水线并行时一脸懵——根本原因是不理解底层通信。

本篇是整个系列的"地图"，帮你先看到全貌再逐篇深入。

## ⚠️ 环境与 sudo 权限说明（重要，先读）

本系列假设你的环境是 **WSL + 有 GPU 的 Linux 服务器，但无 sudo 权限**。不同篇章的命令可行性：

| 命令/操作 | 权限标注 | 说明 |
|-----------|:--------:|------|
| `nvidia-smi topo -m` | ✅ 无需 sudo | WSL 下可跑，但 NVLink 拓扑可能显示 SYS/NODE |
| `ibstat` / `ibv_devinfo` | ⚠️ WSL 不可用 | 需真实 IB 网卡，WSL 通常没有 |
| GPUDirect RDMA | ⚠️ 需 root | 无 sudo 不可用，NCCL 会自动回退 |
| `conda install -c nvidia nccl` | ✅ 无需 sudo | conda 用户态安装 |
| `torchrun --nproc_per_node=2` | ✅ 无需 sudo | 单机多进程可跑 |
| nccl-tests 编译（make） | ✅ 无需 sudo | conda 装 CUDA/NCCL/MPI，make 指向 conda 路径 |
| `mpirun` 多机 | ✅ 无需 sudo | 需配置 SSH 免密 |

**核心原则**：能跑的标注 ✅ 并给完整命令+预期输出；不能跑的标注 ⚠️ 并贴出标准输出供你对照阅读，理解概念即可。

## 各篇一句话定位

### 01 硬件篇（物理基础：路有多宽）
- **1.1 单机卡间通信**：PCIe 是普通马路，NVLink 是城际高速（900 GB/s），NVSwitch 是立交枢纽——TP 必须待在 NVLink 范围内
- **1.2 多机间通信**：传统以太网太慢，RDMA 绕过 CPU 直达显存，IB 是专用高速，RoCE 是以太网上的 RDMA

### 02 通信原语篇（软件抽象：数据怎么搬）
- **2.1 点对点通信**：Send/Recv 是两个人打电话，一对一定向传递，PP 靠它传激活值
- **2.2 集合通信五大原语**：Broadcast/Reduce/AllReduce/AllGather/ReduceScatter 各司其职，AllReduce = ReduceScatter + AllGather

### 03 算法篇（核心算法：怎么高效搬）
- **3.1 Ring AllReduce**：排成环传纸条，通信量 2(N-1)/N×M 与节点数无关，带宽最优
- **3.2 Tree AllReduce**：班长收作业，延迟 O(log N)，适合小数据量同步
- **3.3 通信与计算 Overlap**：把通信藏进计算时间里，DDP 的 bucket 机制自动实现

### 04 NCCL 实战篇（工程工具：谁来帮你搬）
- **4.1 NCCL 通信库基础**：NVIDIA 的"自动挡"通信库，自动探测拓扑选算法，PyTorch 默认后端
- **4.2 nccl-tests 带宽测量**：用 nccl-tests 量真实带宽，分清 algbw 和 busbw，判断集群是否达标

### 05 应用篇（融会贯通：为什么这么设计）
- **5.1 通信视角看并行策略**：从数据搬运角度解释 TP 为何不跨机、PP 可以跨机，附瓶颈排查清单

## 三条主线贯穿全程

### 主线 1：带宽分层（决定并行策略边界）

```
NVLink 4.0     900 GB/s   ┃  单机卡间，TP 必须在这里
PCIe 5.0        64 GB/s   ┃  单机卡间（无 NVLink 时）
IB NDR 4x       50 GB/s   ┃  跨机，DP/PP 可以走这里
以太网 100GbE   12.5 GB/s   ┃  回退方案，性能最差
```

**记住这个阶梯**：带宽每降一级，同样的数据搬运要花 14~70 倍的时间。这就是"TP 不跨机"的物理原因。

### 主线 2：通信量公式（量化分析工具）

| 公式 | 含义 | 出现篇 |
|------|------|--------|
| AllReduce 通信量 = 2(N-1)/N × M | Ring AllReduce 每 rank 通信量，N 大时趋近 2M | 3.1 |
| busbw = algbw × 2(N-1)/N | 总线带宽 = 算法带宽 × 算法倍数修正 | 4.2 |
| 气泡占比 = (P-1)/(P-1+M) | PP 流水线气泡占比，P=级数，M=micro-batch 数 | 5.1 |

### 主线 3：Overlap 思想（性能优化核心）

```
没有 Overlap：  [计算████] [通信████] [计算████] [通信████]  ← 串行，浪费时间
有 Overlap：    [计算████]                [计算████]          ← 计算连续
                    [通信████]    [通信████]              ← 通信藏在计算里
```

DDP 的 bucket 机制让"梯度一就绪就立即异步通信"，不需要等整个模型反向传播完。这是分布式训练性能优化的核心思想。
