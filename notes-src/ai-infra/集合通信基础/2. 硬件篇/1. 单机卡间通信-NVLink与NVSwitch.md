# 1.1 单机卡间通信：NVLink 与 NVSwitch

## 一句话结论

单机 8 卡的 GPU 之间用 NVLink（900 GB/s）直连，比 PCIe 5.0（64 GB/s）快约 14 倍；NVSwitch 让任意两张卡都能跑满带宽。✅ `nvidia-smi topo -m` 无需 sudo 可查看拓扑，但 WSL 下可能不显示 NV# 标记。**张量并行必须限制在 NVLink 互联的卡范围内**——这是本篇最重要的工程结论。

## 背景与定位

当一个模型需要多张 GPU 协同计算时，卡与卡之间搬运数据的速度直接决定了并行效率。张量并行（TP）在每一层前向和反向传播都要做 AllReduce 通信，如果卡间带宽不够，计算单元就会空等数据——这就是为什么理解单机互联如此重要。

本篇是硬件篇的第一篇，建立"带宽分层"的基本认知，为后续理解"TP 不跨机"打下基础。

## ⚠️ 环境与 sudo 权限说明

| 命令/操作 | 权限 | 说明 |
|-----------|:----:|------|
| `nvidia-smi topo -m` | ✅ 无需 sudo | WSL 下可跑，但 NVLink 拓扑可能显示 SYS/NODE 而非 NV# |
| `nvidia-smi -q` | ✅ 无需 sudo | 查看 GPU 详细信息 |

**WSL 特别说明**：WSL 是虚拟化环境，GPU 通过 WDDM/vGPU 暴露，NVLink 拓扑信息可能不完整。如果你的 `nvidia-smi topo -m` 输出里看不到 NV18 这类标记，不是你的卡没有 NVLink，而是 WSL 没有完全暴露底层拓扑。建议在真实 Linux 服务器（裸机）上验证。

## 核心概念（新人友好讲解）

### 1. PCIe：起步但不够用

PCIe（Peripheral Component Interconnect Express）**高速外设部件互连标准** 是 GPU 与 CPU、以及 GPU 之间最基本的互联通道。

> **生活类比**：PCIe 就像连接两栋大楼的**普通马路**——什么车都能跑，但车道窄、红绿灯多，高峰期容易堵。

**PCIe 带宽对比表**：

| PCIe 版本 | 单向带宽 (x16) | 双向带宽 (x16) | 典型用途 |
|-----------|:-------------:|:-------------:|---------|
| PCIe 4.0 | 32 GB/s | 64 GB/s | A100 部分配置 |
| PCIe 5.0 | 64 GB/s | 128 GB/s | 新一代服务器 |

对于张量并行来说，每一层前向和反向都要做 AllReduce。即便是最新的 PCIe 5.0，128 GB/s 的双向带宽也远远不够——好比上下班高峰时所有车都挤在同一条路上，堵成一锅粥。

### 2. NVLink：GPU 之间的专属高速公路

NVLink 是 NVIDIA 专为 GPU 间通信设计的高速互联协议，绕过了 PCIe 总线，在 GPU 之间搭起直通专线。

> **生活类比**：如果 PCIe 是限速 60 的城市道路，NVLink 就是不设红绿灯的**城际高速**——车道多、限速高、没有交叉路口。

#### NVLink 代际演进（新人必背）

| NVLink 版本 | 所属架构 | 单链路带宽 | 每 GPU 链路数 | 每 GPU 总带宽 | 与 PCIe 5.0 对比 |
|------------|---------|:---------:|:-----------:|:------------:|:---------------:|
| NVLink 3.0 | Ampere (A100) | 50 GB/s | 12 | 600 GB/s | ~9 倍 |
| NVLink 4.0 | Hopper (H100) | 50 GB/s | 18 | 900 GB/s | ~14 倍 |
| NVLink 5.0 | Blackwell (B200) | 100 GB/s | 18 | 1,800 GB/s | ~28 倍 |

**新人参数讲解**：
- **"单链路带宽"**：一条 NVLink 通道的速率。NVLink 3.0/4.0 每条 50 GB/s，5.0 翻倍到 100 GB/s。
- **"每 GPU 链路数"**：一张 GPU 上有几条 NVLink 通道。A100 有 12 条，H100 有 18 条，B200 也有 18 条但每条更快。
- **"总带宽 = 单链路 × 链路数"**：A100 = 50×12 = 600 GB/s，H100 = 50×18 = 900 GB/s。

> 💡 **面试记忆口诀**：A100 六百，H100 九百，B200 一千八。记住这三个数，面试被问 NVLink 带宽就不会卡壳。

这个 14~28 倍的差距直接决定了一条重要的工程准则：

> 📌 **关键结论**：需要高频通信的并行策略（如张量并行）必须限制在 NVLink 互联的范围内。跨 PCIe 或跨机的 TP 会让训练速度断崖式下降。

### 3. NVSwitch：全连接交换枢纽

一台 8 卡 GPU 服务器中，如果每两张卡之间都用 NVLink 直连，布线复杂度会爆炸（8 张卡两两直连需要 28 条链路）。

> **生活类比**：NVSwitch 就像一个**高速公路立交枢纽**——所有 GPU 通过它实现互联，任意两张卡之间都能跑满带宽，不需要逐一拉线。

NVSwitch 的核心价值在于**消除拓扑瓶颈**：无论哪两张卡之间通信，带宽都是满速的。这对张量并行至关重要——TP 在每一层都需要做 AllReduce，如果某些卡对之间的带宽比其他低，整体性能就会被最慢的链路拖累（木桶效应）。

### 4. nvidia-smi topo -m 拓扑标记详解

在 GPU 服务器上执行 `nvidia-smi topo -m` 可以查看卡间互联方式。这是排查通信问题最常用的命令。

#### 拓扑标记含义表（新人必懂）

| 标记 | 含义 | 通信性能 |
|------|------|---------|
| `NV#`（如 NV12, NV18） | 通过 # 条 NVLink 连接 | 🟢 最优，满速 NVLink |
| `NV#` 的数字含义 | NV18 = 18 条 NVLink（H100 满配） | 数字越大带宽越高 |
| `PHB` | 通过 PCIe Host Bridge 连接 | 🟡 较低，走 PCIe |
| `NODE` | 同一 NUMA 节点内，通过 PCIe 连接 | 🟡 中等，同 NUMA 内 PCIe |
| `SYS` | 跨 NUMA 节点，需经过系统总线 | 🔴 最差，跨 NUMA 走 QPI/UPI |
| `X` | 自身（GPU 和自己） | 无意义 |

> **新人参数讲解**：
> - **NUMA 节点**：多路 CPU 服务器中，每个 CPU 有自己的本地内存，访问其他 CPU 的内存要慢。`SYS` 表示数据要跨 NUMA 节点，走 CPU 之间的互联（QPI/UPI），性能最差。
> - **PHB（PCIe Host Bridge）**：GPU 通过 PCIe 连到同一个 CPU 的 Host Bridge 上，比跨 NUMA 快，但远不如 NVLink。
> - **NV18**：H100 的满配，18 条 NVLink 全部连通。A100 是 NV12（12 条）。

#### 规划并行策略的原则

在规划并行策略时，应该把 **TP 组内的 GPU 放在 NVLink 互联的卡上**（topo 显示 NV#），把通信量较小的 PP 或 DP 分配给跨 PCIe（PHB/NODE）或跨机（SYS）的链路。

## 动手实践

### 实验 1：查看 GPU 拓扑（✅ 无需 sudo）

```bash
nvidia-smi topo -m
```

**预期输出**（标准 8 卡 H100 SXM 服务器）：

```
        GPU0  GPU1  GPU2  GPU3  GPU4  GPU5  GPU6  GPU7  mlx5_0  mlx5_1  CPU Affinity    NUMA Affinity   GPU NUMA ID
GPU0     X    NV18  NV18  NV18  NV18  NV18  NV18  NV18  SYS     SYS     0-23,96-119     0               N/A
GPU1    NV18   X    NV18  NV18  NV18  NV18  NV18  NV18  SYS     SYS     0-23,96-119     0               N/A
GPU2    NV18  NV18   X    NV18  NV18  NV18  NV18  NV18  SYS     SYS     0-23,96-119     0               N/A
GPU3    NV18  NV18  NV18   X    NV18  NV18  NV18  NV18  PXB     PXB     0-23,96-119     0               N/A
GPU4    NV18  NV18  NV18  NV18   X    NV18  NV18  NV18  SYS     SYS     24-47,120-143   1               N/A
GPU5    NV18  NV18  NV18  NV18  NV18   X    NV18  NV18  SYS     SYS     24-47,120-143   1               N/A
GPU6    NV18  NV18  NV18  NV18  NV18  NV18   X    NV18  SYS     SYS     24-47,120-143   1               N/A
GPU7    NV18  NV18  NV18  NV18  NV18  NV18  NV18   X    PXB     PXB     24-47,120-143   1               N/A
mlx5_0  SYS   SYS   SYS   PXB   SYS   SYS   SYS   PXB    X      PIX
mlx5_1  SYS   SYS   SYS   PXB   SYS   SYS   SYS   PXB   PIX      X
```

**如何读懂这个输出**：
1. 看 GPU0 到 GPU1 的交叉格：`NV18` → 两张卡通过 18 条 NVLink 连接（H100 满配）
2. 看 GPU0 到 GPU4：`NV18` → 即使跨 NUMA（GPU0 在 NUMA 0，GPU4 在 NUMA 1），仍然有 NVLink 直连
3. 看 `mlx5_0`（IB 网卡）到 GPU3：`PXB` → 通过 PCIe 连接，这是 GPUDirect RDMA 的路径
4. 看 `NUMA Affinity` 列：GPU0-3 在 NUMA 0，GPU4-7 在 NUMA 1

**WSL 下的输出**（可能的样子）：

```
        GPU0  GPU1
GPU0     X    SYS
GPU1    SYS   X
```

> ⚠️ WSL 下只显示 SYS 是正常的，不代表你的卡没有 NVLink。WSL 虚拟化层没有完全暴露 NVLink 拓扑。如果你的 WSL 只看到 1 张卡，那连这个矩阵都没有。

### 实验 2：带宽对比计算练习

用公式估算不同互联方式下搬运 1GB 数据的耗时：

```
搬运 1GB 数据的耗时 = 数据量 / 带宽

NVLink 4.0 (H100):  1 GB / 900 GB/s ≈ 1.1 ms
PCIe 5.0:            1 GB / 64 GB/s  ≈ 15.6 ms
IB NDR 4x:           1 GB / 50 GB/s  ≈ 20 ms

差距：NVLink 比 PCIe 快 14 倍，比 IB 快 18 倍
```

> 💡 这就是 5.1 篇会详细推导的"TP 不跨机"的物理原因：一个 80 层模型每层 4 次 AllReduce 共 320 次，走 NVLink 约 0.35 秒，走 IB 约 6.4 秒，差 18 倍。

## 面试回答（30 秒口述版）

> 单机 8 卡 GPU 之间用 NVLink 高速互联。H100 的 NVLink 4.0 总带宽 900 GB/s，是 PCIe 5.0 的约 14 倍。NVSwitch 让任意两张卡之间都能跑满带宽，消除拓扑瓶颈。用 `nvidia-smi topo -m` 可以查看互联方式，NV18 表示 18 条 NVLink 满配，SYS 表示跨 NUMA 走系统总线性能最差。因为 NVLink 带宽远高于跨机网络，所以张量并行必须限制在单机 NVLink 范围内，而通信量较小的数据并行和流水线并行可以跨机。

## 深入追问

**Q1：为什么 NVLink 比 PCIe 快这么多？**
NVLink 是 GPU 间的专用点对点互联，绕过了 PCIe 总线的共享机制和协议开销。PCIe 是通用总线，要服务各种设备（网卡、SSD、GPU），有仲裁开销；NVLink 只为 GPU 间通信设计，多通道并行，每通道独立带宽。

**Q2：没有 NVSwitch 的服务器，8 卡怎么连？**
没有 NVSwitch 时，NVLink 只能在"邻居"卡之间直连（如 GPU0-GPU1, GPU2-GPU3 等），非邻居卡通信要走 PCIe 中转，性能不均匀。NVSwitch 的价值就是让 8 张卡全连接，任意两卡间都满速。

**Q3：NVLink 5.0 的 1800 GB/s 是双向还是单向？**
NVLink 官方标称的带宽通常是双向总和。例如 H100 的 900 GB/s 是双向，单向约 450 GB/s。实际通信中 AllReduce 等操作同时收发，所以用双向带宽估算更合理。

**Q4：WSL 下 `nvidia-smi topo -m` 显示 SYS，我该怎么确认有没有 NVLink？**
WSL 虚拟化层不完全暴露 NVLink 拓扑。确认方法：①在宿主机（Windows）的 PowerShell 跑 `nvidia-smi topo -m`；②查 GPU 型号规格表（H100 SXM 必有 NVLink 4.0）；③在真实 Linux 裸机服务器上验证。

**Q5：NUMA Affinity 对训练有什么影响？**
如果训练进程绑定在 NUMA 0 的 CPU 核上，但 GPU 在 NUMA 1，CPU 到 GPU 的数据拷贝要跨 NUMA，增加延迟。建议用 `numactl` 或 `taskset` 把训练进程绑定到 GPU 所在的 NUMA 节点。

## 易混淆点对比

| 易混概念 | 区别 | 记忆技巧 |
|---------|------|---------|
| NVLink vs NVSwitch | NVLink 是链路（路），NVSwitch 是交换芯片（立交桥） | 路 vs 立交桥 |
| NV12 vs NV18 | NV12=12条链路(A100)，NV18=18条链路(H100) | 数字=链路数 |
| PHB vs NODE vs SYS | PHB=同Host Bridge，NODE=同NUMA内PCIe，SYS=跨NUMA | 越远越慢：PHB > NODE > SYS |
| PCIe 5.0 vs NVLink 4.0 | 64 vs 900 GB/s，差14倍 | NVLink是GPU专属，PCIe是通用 |
| 单向 vs 双向带宽 | NVLink 4.0 双向 900，单向约 450 | 官方标称通常是双向 |

## 常见报错速查表

| ❌ 现象/报错 | 原因 | ✅ 解决 |
|-------------|------|---------|
| WSL 下 `topo -m` 只显示 SYS | WSL 虚拟化不暴露 NVLink 拓扑 | 在宿主机或裸机 Linux 验证；不影响理解概念 |
| `nvidia-smi` 命令找不到 | 驱动未装或 PATH 未配置 | 确认 NVIDIA 驱动已安装；`which nvidia-smi` 查路径 |
| topo 输出里没有 mlx5 网卡 | 机器没有 IB 网卡 | 正常现象，说明只有以太网或单机环境 |
| GPU 之间显示 PIX 而非 NV# | 通过 PCIe 交换机连接，无 NVLink | 该机型无 NVLink（如 PCIe 版 A100），TP 性能会受限 |
