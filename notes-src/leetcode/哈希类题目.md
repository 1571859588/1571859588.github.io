## 一句话总结

哈希类题目的核心思路：**用哈希表把"查找"从 O(n) 降到 O(1)**。常见模式：存访问历史（两数之和）、前缀和计数（子数组和为 K）、排序字符串作为分组 key（字母异位词）、判断元素是否存在（最长连续序列）。

## Q0: 1. 两数之和（哈希表一次遍历）

**题目**：给定数组 `nums` 和目标值 `target`，找出和为 target 的两个数的下标。字节/阿里/腾讯高频。

> [1. 两数之和 - 力扣](https://leetcode.cn/problems/two-sum/description/?envType=study-plan-v2&envId=top-100-liked)

### 解法

```python
def twoSum(nums: list[int], target: int) -> list[int]:
    """
    时间 O(n)，空间 O(n)
    核心：a + b = target → b = target - a
    遍历时，对每个 a，看 target-a 是否在哈希表中
    """
    seen = {}  # 数值 → 下标

    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i

    return []  # 题目保证有解
```

### 推导过程

```
nums = [2, 7, 11, 15], target = 9

i=0, num=2: complement=9-2=7, 不在seen, seen={2:0}
i=1, num=7: complement=9-7=2, 在seen! 返回 [0, 1]
```

### 为什么遍历时先查再存

```python
for i, num in enumerate(nums):
    complement = target - num
    if complement in seen:        # 先查：target-num 在不在已遍历的元素里
        return [seen[complement], i]
    seen[num] = i                 # 再存：当前元素存入哈希表
```

**如果先存再查会怎样**：当 `target = 2 * num` 时（如 `nums=[3], target=6`），先存 `seen[3]=0`，再查 `complement=3` 会在 seen 里找到自己，返回 `[0, 0]`，这是错的（不能用同一个元素两次）。**先查再存保证只用之前出现过的元素配对**。

### 面试追问

| 追问 | 答案 |
|------|------|
| 如果数组有序？ | 双指针，O(n) 时间 O(1) 空间（见 `双指针类题目.md` Q5） |
| 如果找三个数？ | 排序 + 固定一个 + 双指针 Two Sum，O(n²)（见 `双指针类题目.md` Q3 三数之和） |

## Q2: 560. 子数组和为 K（前缀和 + 哈希表）

**题目**：给定整数数组 `nums` 和整数 `k`，返回和为 `k` 的连续子数组的个数。

> [560. 和为 K 的子数组 - 力扣](https://leetcode.cn/problems/subarray-sum-equals-k/description/)

### 解法

```python
def subarraySum(nums: list[int], k: int) -> int:
    """
    时间 O(n)，空间 O(n)
    核心：pref[j] - pref[i-1] = k → pref[i-1] = pref[j] - k
    遍历到 j 时，看哈希表里有多少个 pref[j] - k
    """
    prefix_sum = 0
    count = 0
    prefix_count = {0: 1}  # 初始化 {0:1} 处理从开头开始的子数组

    for num in nums:
        prefix_sum += num
        if prefix_sum - k in prefix_count:
            count += prefix_count[prefix_sum - k]
        prefix_count[prefix_sum] = prefix_count.get(prefix_sum, 0) + 1

    return count
```

### 推导过程

```
pref[j] - pref[i-1] = k   →   pref[i-1] = pref[j] - k

举例：nums = [1, 2, 3], k = 3
i=0, num=1: pre=1, pre-k=-2(无), {0:1, 1:1}
i=1, num=2: pre=3, pre-k=0(有1个!), count=1, {0:1, 1:1, 3:1}  → [1,2]
i=2, num=3: pre=6, pre-k=3(有1个!), count=2, {0:1, 1:1, 3:1, 6:1}  → [3]
答案: 2
```

### `prefix_count = {0: 1}` 的含义

- **键**：前缀和的值（从数组开头累加到某位置的和）
- **值**：这个前缀和值出现过几次
- **`{0: 1}`**：表示"前缀和=0 出现过 1 次"——对应"从数组开头开始的子数组"的虚拟起点

没有它，从开头开始的子数组会被漏掉（如 `[3]` 本身和就是 k=3，需要 `pre - k = 0` 能匹配到）。

### 同类变体

| 变体 | 题目 | 差异 |
|------|------|------|
| 是否存在 | 子数组和是否等于 K | Set 替代 Map（不计数） |
| 最长子数组 | 和为 K 的最长子数组长度 | Map 只存首次出现位置 |
| 取模版 | 和可被 K 整除的子数组 | `(pre % K + K) % K` 处理负数 |

## Q3: 49. 字母异位词分组（排序字符串作 key）

**题目**：给定字符串数组，将字母异位词组合在一起。字母异位词 = 字母重新排列后相同的词。

> [49. 字母异位词分组 - 力扣](https://leetcode.cn/problems/group-anagrams/description/?envType=study-plan-v2&envId=top-100-liked)

**示例**：
```
输入: strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
输出: [["bat"],["nat","tan"],["ate","eat","tea"]]
```

### 解法

```python
from collections import defaultdict

def groupAnagrams(strs: list[str]) -> list[list[str]]:
    """
    时间 O(n * k log k)，n 是字符串数，k 是最长字符串长度
    空间 O(n * k)
    核心：字母异位词排序后相同，用排序后的字符串作为 key 分组
    """
    groups = defaultdict(list)

    for s in strs:
        # 排序后的字符串作为 key（"eat" → "aet"，"tea" → "aet"）
        key = "".join(sorted(s))
        groups[key].append(s)

    return list(groups.values())
```

### `defaultdict` vs `dict` 的区别

`defaultdict` 是 `dict` 的子类，区别在于**访问不存在的 key 时不会报 KeyError，而是自动创建一个默认值**。

```python
from collections import defaultdict

# === 普通 dict：访问不存在的 key 会报错 ===
d = {}
d["a"].append(1)    # ❌ KeyError: 'a'！必须先判断 key 在不在

# 所以用普通 dict 要这样写：
d = {}
for s in strs:
    key = "".join(sorted(s))
    if key not in d:        # 先判断
        d[key] = []         # 不在就先建空列表
    d[key].append(s)

# === defaultdict：访问不存在的 key 自动创建默认值 ===
d = defaultdict(list)       # list 表示默认值是空列表
d["a"].append(1)            # ✅ "a" 不存在 → 自动创建 d["a"]=[] → append(1)
# d 现在是 {"a": [1]}
```

**`defaultdict(list)` 的 `list` 是什么**：`list` 是一个**工厂函数**（调用 `list()` 返回空列表 `[]`）。`defaultdict(list)` 表示"访问不存在的 key 时，自动调用 `list()` 创建空列表"。

**常用工厂函数**：

| 工厂 | 默认值 | 用途 |
|------|--------|------|
| `list` | `[]` | 分组归类（本题） |
| `int` | `0` | 计数器 |
| `set` | `set()` | 去重分组 |
| `dict` | `{}` | 多级字典 |

**计数器示例**：

```python
# 用 defaultdict(int) 做计数
from collections import defaultdict

counter = defaultdict(int)
for word in ["apple", "banana", "apple"]:
    counter[word] += 1    # 不存在时默认 0，+1 后变 1
# counter = {"apple": 2, "banana": 1}
```

**在本题中的价值**：

```python
# 用 defaultdict：一行搞定
groups = defaultdict(list)
for s in strs:
    groups[key].append(s)    # key 不存在自动建空列表，直接 append

# 用普通 dict：要多两行判断
groups = {}
for s in strs:
    key = "".join(sorted(s))
    if key not in groups:    # 多了这两行
        groups[key] = []
    groups[key].append(s)
```

> **一句话记忆**：`defaultdict(list)` = 访问不存在的 key 时自动建空列表，不用手动判断 `if key not in dict`。刷题中"分组归类"场景必用。

### 推导过程

```
strs = ["eat", "tea", "tan", "ate", "nat", "bat"]

排序每个字符串：
"eat" → sorted → "aet"
"tea" → sorted → "aet"   ← 和 eat 同组
"tan" → sorted → "ant"
"ate" → sorted → "aet"   ← 和 eat 同组
"nat" → sorted → "ant"   ← 和 tan 同组
"bat" → sorted → "abt"

groups = {
    "aet": ["eat", "tea", "ate"],
    "ant": ["tan", "nat"],
    "abt": ["bat"]
}

返回: [["eat","tea","ate"], ["tan","nat"], ["bat"]]
```

### 为什么排序可以做 key

字母异位词的定义是"重新排列后相同"。排序就是把字母按固定顺序排列，所以两个异位词排序后必然得到相同的字符串。用这个相同的字符串做哈希表的 key，就能把所有异位词归到一组。

### 替代方案：用字符计数作 key

```python
def groupAnagrams_count(strs: list[str]) -> list[list[str]]:
    """用字符频率计数作 key，避免排序"""
    groups = defaultdict(list)
    for s in strs:
        # 26 个字母的计数元组作 key
        count = [0] * 26
        for ch in s:
            count[ord(ch) - ord('a')] += 1
        groups[tuple(count)].append(s)
    return list(groups.values())
```

| 方案 | 时间复杂度 | 适合场景 |
|------|------------|----------|
| 排序作 key | O(n * k log k) | 字符串较短时简洁 |
| 计数作 key | O(n * k) | 字符串较长时更快 |

## Q4: 128. 最长连续序列（哈希集合 + 只从起点开始）

**题目**：给定未排序整数数组，找出数字连续的最长序列的长度。要求 O(n) 时间。

> [128. 最长连续序列 - 力扣](https://leetcode.cn/problems/longest-consecutive-sequence/?envType=study-plan-v2&envId=top-100-liked)

**示例**：
```
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长连续序列是 [1, 2, 3, 4]，长度为 4
```

### 解法

```python
def longestConsecutive(nums: list[int]) -> int:
    """
    时间 O(n)，空间 O(n)
    核心：用集合 O(1) 判断元素是否存在
    只从"序列起点"开始数（num-1 不在集合中时才算起点）
    """
    num_set = set(nums)  # 去重 + O(1) 查找
    max_length = 0

    for num in num_set:
        # 只从序列起点开始数（num-1 不存在，说明 num 是起点）
        if num - 1 not in num_set:
            current = num
            length = 1

            # 向后数连续的数字
            while current + 1 in num_set:
                current += 1
                length += 1

            max_length = max(max_length, length)

    return max_length
```

### 推导过程

```
nums = [100, 4, 200, 1, 3, 2]
num_set = {1, 2, 3, 4, 100, 200}

遍历 num_set：
num=1: 1-1=0 不在集合 → 是起点！向后数：2✓ 3✓ 4✓ 5✗ → length=4
num=2: 2-1=1 在集合 → 跳过（不是起点，已经从 1 数过了）
num=3: 3-1=2 在集合 → 跳过
num=4: 4-1=3 在集合 → 跳过
num=100: 100-1=99 不在 → 是起点！向后数：101✗ → length=1
num=200: 200-1=199 不在 → 是起点！向后数：201✗ → length=1

max_length = 4
```

### 为什么是 O(n)

虽然有个 while 循环嵌套在 for 里，但**每个元素最多被 while 访问一次**。比如 [1,2,3,4]，从 1 开始数了 2、3、4，之后遍历到 2、3、4 时都因为 `num-1 in set` 而跳过，不会重复数。总访问次数 = 2n，是 O(n)。

### 为什么不用排序

排序是 O(n log n)，不满足题目 O(n) 要求。用哈希集合 + 只从起点开始数，巧妙实现 O(n)。

## 哈希类题目通用思路

| 模式 | 典型题目 | 核心操作 |
|------|----------|----------|
| 存访问历史 | 两数之和 | 遍历时存 `值→下标`，查 `target-当前值` |
| 前缀和计数 | 子数组和为 K | 遍历时存 `前缀和→次数`，查 `当前前缀和-k` |
| 分组归类 | 字母异位词分组 | 找共同 key（排序/计数），用 defaultdict 分组 |
| 判断存在 | 最长连续序列 | 存集合，O(1) 判断元素是否存在 |
