## 一句话总结

滑动窗口的核心：**维护一个满足条件的窗口 [left, right]，右指针探索扩大，左指针收缩调整**。把"枚举所有子串/子数组"的 O(n²) 优化到 O(n)。常见模式：定长窗口（窗口大小固定）、变长窗口（窗口大小随条件变化）。关键认知：**右指针始终向右移动，左指针只在条件被破坏时向右收缩**。

## Q1: 3. 无重复字符的最长子串（变长滑动窗口）

**题目**：给定一个字符串 `s`，请你找出其中不含有重复字符的 **最长子串** 的长度。**子串**是连续的字符序列。

> 🔗 题目链接：[3. 无重复字符的最长子串 - 力扣](https://leetcode.cn/problems/longest-substring-without-repeating-characters/?envType=study-plan-v2&envId=top-100-liked)
> 难度：中等 | 标签：滑动窗口、哈希表、字符串

**示例**：
```
输入: s = "abcabcbb"
输出: 3
解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。注意 "bca" 和 "cab" 也是正确答案。

输入: s = "bbbbb"
输出: 1
解释: 因为无重复字符的最长子串是 "b"，所以其长度为 1。

输入: s = "pwwkew"
输出: 3
解释: 因为无重复字符的最长子串是 "wke"，所以其长度为 3。
     请注意，答案必须是 子串 的长度，"pwke" 是一个子序列，不是子串。
```

**提示**：
- `0 <= s.length <= 5 * 10^4`
- `s` 由英文字母、数字、符号和空格组成

### 思路讲解（通俗版）

想象你在读一串字符，**用一个"窗口"框住一段没有重复字符的子串**。

- 右指针 `right` 不断向右走，把新字符放进窗口
- 如果新字符和窗口里已有的字符重复了，就把左指针 `left` 跳到"重复字符的下一个位置"，**直接跨过重复字符**，保证窗口内仍然无重复
- 每次更新最大长度

**关键优化**：用哈希表 `char_index` 记录每个字符最近出现的位置。发现 `s[right]` 已在哈希表中且位置 `≥ left` 时，直接把 `left` 跳到 `char_index[s[right]] + 1`，省去逐个移动。

### 解法

```python
def lengthOfLongestSubstring(s: str) -> int:
    """
    时间 O(n)，空间 O(min(m, n))，m 为字符集大小
    核心：变长滑动窗口 + 哈希表记录字符最近出现位置
    """
    char_index = {}  # 字符 → 最近一次出现的下标
    left = 0
    max_len = 0

    for right, ch in enumerate(s):
        # 如果字符已在窗口内（位置 >= left），左指针跳到它后面
        if ch in char_index and char_index[ch] >= left:
            left = char_index[ch] + 1

        # 更新字符的最新位置
        char_index[ch] = right

        # 更新最大长度
        max_len = max(max_len, right - left + 1)

    return max_len
```

### 为什么判断条件是 `char_index[ch] >= left`

```
情况1: char_index[ch] < left
  说明这个字符虽然在哈希表里，但它的位置已经在窗口左边界之外
  → 窗口里其实没有重复，不用移动 left

情况2: char_index[ch] >= left
  说明这个字符就在当前窗口内
  → 重复了，必须把 left 跳到 char_index[ch] + 1
```

如果不加 `>= left` 这个判断，会把窗口外的旧位置误判为重复，导致窗口错误收缩。

### 面试追问

| 追问 | 答案 |
|------|------|
| 如果只要返回最长子串本身？ | 用 `start, end` 记录最大长度对应的区间，最后返回 `s[start:end+1]` |
| 如果字符集很小（如只有小写字母）？ | 用长度 26 的数组代替哈希表，访问更快 |
| 能不能用 set 实现？ | 可以，但 left 只能逐个右移删除元素，最坏 O(2n)；哈希表版直接跳，O(n) |
| 时间复杂度真的是 O(n) 吗？ | 是。right 走 n 步，left 最多也走 n 步（均摊），整体 O(n) |

### 常见坑

1. **忘了 `char_index[ch] >= left` 判断**：会错误地把窗口外的旧位置当作重复，导致窗口异常收缩。
2. **先更新 max_len 还是先移动 left**：必须**先移动 left，再算窗口长度**，否则会把含重复字符的窗口算进去。
3. **初始化 `max_len = 0`**：空字符串 `s=""` 应返回 0，不能初始化为 1。

## Q2: 438. 找到字符串中所有字母异位词（定长滑动窗口）

**题目**：给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的 **异位词** 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

> **异位词**：由相同字母重新排列形成的字符串（包括原字符串本身）。例如 "abc" 的异位词有 "abc"、"bac"、"cab"、"bca"、"acb"、"cba"。

> 🔗 题目链接：[438. 找到字符串中所有字母异位词 - 力扣](https://leetcode.cn/problems/find-all-anagrams-in-a-string/?envType=study-plan-v2&envId=top-100-liked)
> 难度：中等 | 标签：滑动窗口、数组、哈希表、字符串

**示例**：
```
输入: s = "cbaebabacd", p = "abc"
输出: [0,6]
解释:
起始索引等于 0 的子串是 "cba", 它是 "abc" 的异位词。
起始索引等于 6 的子串是 "bac", 它是 "abc" 的异位词。

输入: s = "abab", p = "ab"
输出: [0,1,2]
解释:
起始索引等于 0 的子串是 "ab", 它是 "ab" 的异位词。
起始索引等于 1 的子串是 "ba", 它是 "ab" 的异位词。
起始索引等于 2 的子串是 "ab", 它是 "ab" 的异位词。
```

**提示**：
- `1 <= s.length, p.length <= 3 * 10^4`
- `s` 和 `p` 仅包含小写字母

### 思路讲解（通俗版）

**核心观察**：异位词的本质是"字母组成完全相同，只是顺序不同"。所以要判断 `s` 中长度为 `len(p)` 的子串是不是 `p` 的异位词，**只需要比较两个字符串的字母频次是否相同**。

**滑动窗口做法**：
1. 先统计 `p` 的字母频次 `p_count`（长度 26 的数组）
2. 在 `s` 上维护一个**定长窗口**，窗口大小 = `len(p)`
3. 窗口右移时：左边出一个字符（频次 -1），右边进一个字符（频次 +1）
4. 每次移动后比较窗口频次 `s_count` 和 `p_count`，相同就把窗口左边界加入结果

**优化**：不用每次都比较整个 26 长度的数组，而是维护一个"匹配字符数 `matched`"——当某个字符的窗口频次恰好等于 `p` 中频次时 `matched += 1`，等于 0 时 `matched -= 1`。`matched == 26`（或 `p` 中不同字符数）时就是异位词。但 26 长度数组比较是 O(26)=O(1)，直接比较也够快，这里给出更直观的版本。

### 解法

```python
from collections import Counter

def findAnagrams(s: str, p: str) -> list[int]:
    """
    时间 O(n)，空间 O(1)（字符集只有 26 个小写字母）
    核心：定长滑动窗口 + 字符频次计数
    """
    n, m = len(s), len(p)
    if n < m:
        return []

    p_count = Counter(p)        # p 的字符频次
    s_count = Counter(s[:m-1])  # 窗口内字符频次，初始包含前 m-1 个字符
    result = []

    # 窗口右边界从 m-1 开始（这样窗口刚好 m 个字符）
    for right in range(m - 1, n):
        # 右边进一个字符
        s_count[s[right]] += 1

        # 比较频次
        if s_count == p_count:
            result.append(right - m + 1)  # 左边界

        # 左边出一个字符（为下一次移动做准备）
        left_char = s[right - m + 1]
        s_count[left_char] -= 1
        if s_count[left_char] == 0:
            del s_count[left_char]  # 删掉 0 频次的 key，方便 == 比较

    return result
```

### 关键代码逐行解析

#### 1. `from collections import Counter` —— Counter 是什么

`Counter` 是 Python 标准库 `collections` 里的一个类，**专门用来统计可哈希对象的出现次数**，本质是 `dict` 的子类。

```python
from collections import Counter

# 基本用法：传入可迭代对象，返回 {元素: 出现次数}
Counter("aabbbc")   # Counter({'b': 3, 'a': 2, 'c': 1})
Counter([1, 1, 2])  # Counter({1: 2, 2: 1})

# 访问不存在的 key 返回 0（不会报 KeyError），这是 Counter 比 dict 方便的地方
c = Counter("abc")
c['x']              # 0（普通 dict 会报 KeyError）
c['a']              # 1

# 支持 == 比较：两个 Counter 的所有 key 和 value 都相等才返回 True
Counter("abc") == Counter("cba")   # True （字母频次相同 → 异位词）
Counter("abc") == Counter("abcd")  # False
```

**本题为什么用 Counter？** 因为异位词的本质就是"字母频次相同"，`Counter` 直接支持 `==` 比较，一行代码就能判断两个字符串是否互为异位词，比手写长度 26 的数组更直观、更 Pythonic。

#### 2. `p_count = Counter(p)` —— 统计目标串 p 的字符频次

```python
p_count = Counter(p)
# 例如 p = "abc" → p_count = Counter({'a': 1, 'b': 1, 'c': 1})
# 例如 p = "aab" → p_count = Counter({'a': 2, 'b': 1})
```

**含义**：把目标串 `p` 里每个字符出现了几次统计出来，作为后续比对的"标准答案"。这个表建好后**不再改动**，每次窗口滑动后拿窗口的频次表和它比。

#### 3. `s_count = Counter(s[:m-1])` —— 为什么初始化前 m-1 个字符（关键！）

这是本题最容易困惑的一行。先说结论：**故意少统计 1 个字符，留给循环里"右边进一个字符"那一步去补齐，这样循环体里的逻辑对所有窗口位置都统一**。

逐步拆解：

```python
m = len(p)            # 窗口大小 = p 的长度
# s[:m-1] 是 s 的前 m-1 个字符（切片，左闭右开，不含下标 m-1）

# 例：s = "cbaebabacd", p = "abc" → m = 3
# s[:m-1] = s[:2] = "cb"
# s_count 初始 = Counter("cb") = {'c': 1, 'b': 1}
```

**为什么不直接初始化 `Counter(s[:m])`（前 m 个）？**

因为循环是从 `right = m-1` 开始的，循环体里第一步是 `s_count[s[right]] += 1`（右边进一个字符）。如果初始就放满 m 个字符，第一次循环又进一个，窗口就变成 m+1 个字符了，逻辑错乱。

**正确的"错位配合"**：

```
初始:    s_count 只装前 m-1 个字符  →  窗口实际有 m-1 个字符（还差 1 个）
循环体:  先进一个字符 s[right]      →  窗口补满到 m 个字符  ← 此刻才是完整的窗口
         比较频次                    →  此时窗口大小正好是 m
         再出一个字符 s[right-m+1]   →  窗口又回到 m-1 个，为下一轮做准备
```

用一个具体例子看这个"进出节奏"（`s = "cbaebabacd"`, `p = "abc"`, `m = 3`）：

```
初始:        s_count = Counter("cb") = {'c':1, 'b':1}   窗口实际: "cb" (2个)

right=2:     进 s[2]='a'  → s_count={'c':1,'b':1,'a':1}  窗口="cba"(3个) ✓ 比较！
             出 s[0]='c'  → s_count={'b':1,'a':1}        窗口="ba" (2个)

right=3:     进 s[3]='e'  → s_count={'b':1,'a':1,'e':1}  窗口="bae"(3个) ✓ 比较！
             出 s[1]='b'  → s_count={'a':1,'e':1}        窗口="ae" (2个)

...以此类推
```

**核心心法**：循环体里"先进后出"——先把窗口补满到 m 个字符进行比较，再移除最左边的字符为下一轮腾位置。初始少装 1 个，是为了让"先进"这一步在第一轮循环就能成立，保证每一轮循环的逻辑完全一样。

> 💡 **记忆口诀**：**初始装 m-1，循环先进后出，窗口始终在 m 和 m-1 之间来回切换**。

### 为什么用 Counter 而不是排序

```
方法对比：
  排序法：对每个长度 m 的子串排序后和排序后的 p 比较
          时间 O(n * m log m)，超时
  频次法：滑动窗口维护频次，每次比较 O(26) = O(1)
          时间 O(n)，通过
```

**核心**：异位词的判定本质是"字母频次相同"，排序是 O(m log m) 的判定，频次计数是 O(m) 建表 + O(1) 判定。滑动窗口让建表也变成增量更新，每次 O(1)。

### 面试追问

| 追问 | 答案 |
|------|------|
| 如果字符集是 Unicode（不限小写字母）？ | Counter 仍然有效，空间 O(不同字符数)；或用 defaultdict(int) |
| 能不能优化到不用每次比较整个 Counter？ | 维护 `matched` 计数：某字符频次恰好匹配时 matched+1，不匹配时 matched-1，matched == 需要匹配的字符数即为答案 |
| 如果 p 比 s 长？ | 直接返回空列表（题目保证 `1 <= s.length, p.length`，但要处理边界） |
| 和题 76（最小覆盖子串）的区别？ | 438 是**定长窗口**且要求频次**完全相等**；76 是**变长窗口**且要求频次**至少覆盖** |

### 常见坑

1. **窗口初始包含前 m-1 个字符**：循环从 `right = m-1` 开始，这样第一次进字符后窗口正好是 m 个字符。如果初始为空，循环里要先出后进会乱。
2. **删除 0 频次的 key**：`Counter` 里 `s_count['x'] = 0` 和 `p_count` 里没有 'x' 不相等，必须 `del` 掉 0 频次的 key，`==` 比较才正确。
3. **结果存的是左边界**：`right - m + 1`，不是 `right`。

## 滑动窗口通用模板

### 变长窗口（求最长 / 最短满足条件的子串）

```python
def sliding_window_variable(s):
    left = 0
    # 维护窗口状态的变量（如 char_index、counter、sum 等）
    result = 0

    for right in range(len(s)):
        # 1. 右指针进入窗口，更新状态
        add(s[right])

        # 2. 当窗口不满足条件时，收缩左指针
        while not is_valid():
            remove(s[left])
            left += 1

        # 3. 此时窗口满足条件，更新结果
        result = max(result, right - left + 1)

    return result
```

### 定长窗口（求固定大小的窗口满足条件的个数）

```python
def sliding_window_fixed(s, k):
    # 1. 初始化前 k-1 个元素的状态
    init_state(s[:k-1])
    result = []

    # 2. 右边界从 k-1 开始，窗口刚好 k 个元素
    for right in range(k - 1, len(s)):
        add(s[right])              # 右边进
        if is_valid():             # 判断
            result.append(right - k + 1)
        remove(s[right - k + 1])   # 左边出

    return result
```
