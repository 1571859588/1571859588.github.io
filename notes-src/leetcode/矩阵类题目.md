## 一句话总结

矩阵题的核心套路就三种：**① 用首行首列当标记位实现 O(1) 原地操作（73 置零）；② 按层/按方向模拟（54 螺旋、48 旋转）；③ 利用矩阵的单调性把 O(mn) 搜索降到 O(m+n)（240 搜索）**。关键认知：矩阵不是"二维数组暴力遍历"，而是要找到其中的**单调性**或**对称性**来降复杂度。

## Q1: 73. 矩阵置零（O(1) 原地标记法）

**题目**：给定 m×n 矩阵，如果一个元素为 0，则将其所在行和列的所有元素都设为 0。**必须使用原地算法**。

> [73. 矩阵置零 - 力扣](https://leetcode.cn/problems/set-matrix-zeroes/description/?envType=study-plan-v2&envId=top-100-liked)
> 标签：数组 | 哈希 | 矩阵

**示例**：
```
输入：matrix = [[1,1,1],[1,0,1],[1,1,1]]
输出：[[1,0,1],[0,0,0],[1,0,1]]
```

### 一句话结论

**用第一行和第一列本身作为"标记数组"**，额外只开 2 个变量记录第一行/第一列自己是否要置零，就能做到 O(1) 空间。

### 解法（最优：O(1) 空间标记法）

```python
def setZeroes(matrix: list[list[int]]) -> None:
    """
    时间 O(m*n)，空间 O(1)
    核心：把第一行、第一列当作标记数组，额外用两个变量记录首行首列自身
    """
    m, n = len(matrix), len(matrix[0])
    first_row_has_zero = any(matrix[0][j] == 0 for j in range(n))
    first_col_has_zero = any(matrix[i][0] == 0 for i in range(m))

    # 1. 用第一行/第一列记录其余位置是否要置零
    for i in range(1, m):
        for j in range(1, n):
            if matrix[i][j] == 0:
                matrix[i][0] = 0   # 标记第 i 行要置零
                matrix[0][j] = 0   # 标记第 j 列要置零

    # 2. 根据标记位，把内部（除首行首列）置零
    for i in range(1, m):
        for j in range(1, n):
            if matrix[i][0] == 0 or matrix[0][j] == 0:
                matrix[i][j] = 0

    # 3. 最后处理首行首列自己（必须放最后，否则会污染标记）
    if first_row_has_zero:
        for j in range(n):
            matrix[0][j] = 0
    if first_col_has_zero:
        for i in range(m):
            matrix[i][0] = 0
```

### 推导过程

```
matrix = [[1,1,1],
          [1,0,1],
          [1,1,1]]

第 0 步：首行无 0，首列无 0
  first_row_has_zero = False
  first_col_has_zero = False

第 1 步：扫描内部，matrix[1][1]=0 → 标记 matrix[1][0]=0, matrix[0][1]=0
  matrix = [[1,0,1],
            [0,0,1],
            [1,1,1]]

第 2 步：根据标记把内部置零
  matrix[1][1]: 行标记 matrix[1][0]==0 → 置 0
  matrix[1][2]: 行标记 matrix[1][0]==0 → 置 0
  matrix[2][1]: 列标记 matrix[0][1]==0 → 置 0
  matrix = [[1,0,1],
            [0,0,0],
            [1,0,1]]

第 3 步：首行首列不置零（标记为 False）

结果: [[1,0,1],[0,0,0],[1,0,1]] ✓
```

### 为什么首行首列的处理必须放最后

```
假设先处理首行，把 matrix[0][1] 置 0：
  此时 matrix[0][1]=0，但它是"列标记位"！
  它原本是 1（表示第 1 列不需要置零），
  被你置 0 后，会误判"第 1 列要置零"，导致错误。
```

所以必须**先处理内部，再处理首行首列**，避免污染标记。

### 其他解法对比

| 解法 | 时间 | 空间 | 思路 |
|------|------|------|------|
| 两个标记数组 | O(mn) | O(m+n) | 开 row[n]、col[m] 记录哪些行列要置零 |
| **首行首列标记（本解）** | O(mn) | **O(1)** | 把标记位塞回矩阵第一行第一列 |
| 单标记变量进阶 | O(mn) | O(1) | 只用一个变量记录首列，更省 |

**面试要点**：题目明确进阶要求 O(1) 空间，所以必须用首行首列标记法。解释清楚"为什么首行首列最后处理"是关键加分项。

### 易错点

1. 首行首列的处理顺序错了 → 标记位被污染。
2. 忘记用 `any()` 提前记录首行首列自身状态 → 后面无法判断。
3. 扫描内部时从 `i=0, j=0` 开始 → 会把首行首列本身当标记位改掉。

## Q2: 54. 螺旋矩阵（按层模拟）

**题目**：给你一个 m×n 矩阵，按**顺时针螺旋顺序**返回所有元素。

> [54. 螺旋矩阵 - 力扣](https://leetcode.cn/problems/spiral-matrix/?envType=study-plan-v2&envId=top-100-liked)
> 标签：数组 | 矩阵 | 模拟

**示例**：
```
输入：matrix = [[1,2,3],[4,5,6],[7,8,9]]
输出：[1,2,3,6,9,8,7,4,5]
```

### 一句话结论

**维护四个边界 top/bottom/left/right，按"右→下→左→上"循环走，每走完一边收缩对应边界**，直到边界交叉。

### 解法（最优：边界模拟法）

```python
def spiralOrder(matrix: list[list[int]]) -> list[int]:
    """
    时间 O(m*n)，空间 O(1)（不计输出数组）
    核心：四个边界，按右→下→左→上走，走完一边收缩
    """
    if not matrix:
        return []

    top, bottom = 0, len(matrix) - 1
    left, right = 0, len(matrix[0]) - 1
    result = []

    while top <= bottom and left <= right:
        # 1. 从左到右走上边
        for j in range(left, right + 1):
            result.append(matrix[top][j])
        top += 1  # 上边收缩

        # 2. 从上到下走右边
        for i in range(top, bottom + 1):
            result.append(matrix[i][right])
        right -= 1  # 右边收缩

        # 3. 从右到左走下边（注意要判断 top <= bottom，避免重复）
        if top <= bottom:
            for j in range(right, left - 1, -1):
                result.append(matrix[bottom][j])
            bottom -= 1  # 下边收缩

        # 4. 从下到上走左边（注意要判断 left <= right，避免重复）
        if left <= right:
            for i in range(bottom, top - 1, -1):
                result.append(matrix[i][left])
            left += 1  # 左边收缩

    return result
```

### 推导过程

```
matrix = [[1,2,3],
          [4,5,6],
          [7,8,9]]
top=0, bottom=2, left=0, right=2

第 1 轮：
  右：走第 0 行 [1,2,3] → result=[1,2,3], top=1
  下：走第 2 列 [6,9]    → result=[1,2,3,6,9], right=1
  左：top(1)<=bottom(2)，走第 2 行反向 [8,7] → result=[1,2,3,6,9,8,7], bottom=1
  上：left(0)<=right(1)，走第 0 列反向 [4]   → result=[1,2,3,6,9,8,7,4], left=1

第 2 轮：top=1, bottom=1, left=1, right=1
  右：走第 1 行 [5] → result=[1,2,3,6,9,8,7,4,5], top=2
  下：top(2)>bottom(1)，跳过
  左：top>bottom 跳过
  上：left<=right 跳过

结束，结果 [1,2,3,6,9,8,7,4,5] ✓
```

### 为什么第 3、4 步要加边界判断

```
考虑单行矩阵 [[1,2,3]]：
  第 1 步右走后 top=1，但 bottom=0，top>bottom
  如果不加 if 直接左走，会反向再走一遍第 0 行 → 重复！

考虑单列矩阵 [[1],[2],[3]]：
  第 2 步下走后 right=-1，left=0，right<left
  如果不加 if 直接上走，会反向再走一遍第 0 列 → 重复！
```

所以**左走和上走前必须判断边界是否还合法**，否则会重复收集元素。

### 其他解法对比

| 解法 | 时间 | 空间 | 思路 |
|------|------|------|------|
| **边界模拟（本解）** | O(mn) | O(1) | 四边界收缩 |
| 方向数组法 | O(mn) | O(mn) | visited 数组 + 方向转向 |
| 层层剥离 | O(mn) | O(1) | 递归剥最外层 |

边界模拟法最直观，且不需要额外 visited 数组，是最优解。

### 易错点

1. 左走、上走前忘记判断 `top <= bottom` / `left <= right` → 重复收集。
2. 收缩边界时方向搞反（top 应 +1，bottom 应 -1）。
3. `range` 的反向写法 `range(right, left-1, -1)` 容易漏掉 -1。

## Q3: 48. 旋转图像（转置 + 翻转）

**题目**：将 n×n 矩阵**顺时针旋转 90 度**，必须**原地**操作。

> [48. 旋转图像 - 力扣](https://leetcode.cn/problems/rotate-image/?envType=study-plan-v2&envId=top-100-liked)
> 标签：数组 | 数学 | 矩阵

**示例**：
```
输入：matrix = [[1,2,3],[4,5,6],[7,8,9]]
输出：[[7,4,1],[8,5,2],[9,6,3]]
```

### 一句话结论

**顺时针旋转 90° = 沿主对角线转置 + 每行左右翻转**。两步都是原地操作，空间 O(1)。

### 解法（最优：转置 + 翻转）

```python
def rotate(matrix: list[list[int]]) -> None:
    """
    时间 O(n²)，空间 O(1)
    核心：顺时针 90° = 主对角线转置 + 每行翻转
        逆时针 90° = 副对角线转置 + 每行翻转（或主对角线转置 + 每列翻转）
    """
    n = len(matrix)

    # 第 1 步：沿主对角线转置（matrix[i][j] <-> matrix[j][i]）
    for i in range(n):
        for j in range(i + 1, n):  # 只处理上三角，避免交换两次
            matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]

    # 第 2 步：每行左右翻转
    for i in range(n):
        matrix[i].reverse()
```

### 推导过程

```
matrix = [[1,2,3],
          [4,5,6],
          [7,8,9]]

第 1 步：主对角线转置（沿左上到右下的对角线对称）
  matrix[0][1] <-> matrix[1][0]:  2 <-> 4
  matrix[0][2] <-> matrix[2][0]:  3 <-> 7
  matrix[1][2] <-> matrix[2][1]:  6 <-> 8
  转置后:
  [[1,4,7],
   [2,5,8],
   [3,6,9]]

第 2 步：每行翻转
  [1,4,7] → [7,4,1]
  [2,5,8] → [8,5,2]
  [3,6,9] → [9,6,3]

结果: [[7,4,1],[8,5,2],[9,6,3]] ✓
```

### 为什么"转置 + 翻转"等于旋转 90°

```
设原矩阵元素位置为 (i, j)，值为 a[i][j]。

转置后：a[i][j] 移到了 (j, i)
行翻转后：行内的列下标 j 变成 (n-1-j)
所以转置+行翻转后：a[i][j] 最终到了 (j, n-1-i)

而顺时针旋转 90° 的坐标变换正是：(i, j) → (j, n-1-i)
两者一致，所以等价。
```

记忆口诀：
- **顺时针 90°**：主对角线转置 + 行翻转
- **逆时针 90°**：主对角线转置 + 列翻转
- **旋转 180°**：每行翻转 + 每列翻转（或整体翻转两次）

### 其他解法对比

| 解法 | 时间 | 空间 | 思路 |
|------|------|------|------|
| **转置+翻转（本解）** | O(n²) | O(1) | 两次原地操作 |
| 四元素原地旋转 | O(n²) | O(1) | 一次旋转 4 个点 |
| 辅助矩阵 | O(n²) | O(n²) | 开新矩阵直接算下标 |

转置+翻转最易记、不易错，面试推荐。

### 四元素旋转法（拓展）

```python
def rotate_four(matrix: list[list[int]]) -> None:
    """一次旋转 4 个对称位置的点"""
    n = len(matrix)
    for i in range(n // 2):
        for j in range(i, n - i - 1):
            # (i,j) -> (j,n-1-i) -> (n-1-i,n-1-j) -> (n-1-j,i) -> (i,j)
            temp = matrix[i][j]
            matrix[i][j] = matrix[n - 1 - j][i]
            matrix[n - 1 - j][i] = matrix[n - 1 - i][n - 1 - j]
            matrix[n - 1 - i][n - 1 - j] = matrix[j][n - 1 - i]
            matrix[j][n - 1 - i] = temp
```

这种一次转 4 个点，少一次遍历，但下标容易写错，面试不如转置+翻转稳妥。

### 易错点

1. 转置时内层循环从 `j=0` 开始 → 会把每个元素交换两次（等于没转）。必须从 `j=i+1` 开始，只处理上三角。
2. 用了 `matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]` 时，Python 的多元赋值是安全的（先算右边再赋值），但其他语言要注意临时变量。

## Q4: 240. 搜索二维矩阵 II（从右上角起搜索）

**题目**：在 m×n 矩阵中搜索 target。矩阵满足：**每行从左到右升序，每列从上到下升序**。

> [240. 搜索二维矩阵 II - 力扣](https://leetcode.cn/problems/search-a-2d-matrix-ii/?envType=study-plan-v2&envId=top-100-liked)
> 标签：数组 | 二分查找 | 分治 | 矩阵

**示例**：
```
输入：matrix = [[1,4,7,11,15],[2,5,8,12,19],[3,6,9,16,22],[10,13,14,17,24],[18,21,23,26,30]], target = 5
输出：true
```

### 一句话结论

**从右上角（或左下角）出发**：当前值 > target 往左走（变小），< target 往下走（变大）。每一步都能排除一整行或一整列，像在二叉搜索树上走。

### 解法（最优：BST 搜索法）

```python
def searchMatrix(matrix: list[list[int]], target: int) -> bool:
    """
    时间 O(m+n)，空间 O(1)
    核心：从右上角出发，大了左移，小了下移
    右上角是"行内最大、列内最小"的特殊点
    """
    if not matrix or not matrix[0]:
        return False

    m, n = len(matrix), len(matrix[0])
    # 从右上角出发
    row, col = 0, n - 1

    while row < m and col >= 0:
        cur = matrix[row][col]
        if cur == target:
            return True
        elif cur > target:
            col -= 1  # 当前太大，往左走（列变小）
        else:
            row += 1  # 当前太小，往下走（行变大）

    return False
```

### 推导过程

```
matrix = [[1, 4, 7, 11,15],
          [2, 5, 8, 12,19],
          [3, 6, 9, 16,22],
          [10,13,14,17,24],
          [18,21,23,26,30]]
target = 5

从右上角 (0,4)=15 出发：
  (0,4)=15 > 5 → 左走 → (0,3)=11
  (0,3)=11 > 5 → 左走 → (0,2)=7
  (0,2)=7  > 5 → 左走 → (0,1)=4
  (0,1)=4  < 5 → 下走 → (1,1)=5
  (1,1)=5 == 5 → 找到，返回 true ✓
```

### 为什么从右上角出发能搜全

```
右上角这个点有个特殊性质：
  - 它是所在行的最大值（行升序，它在最右）
  - 它是所在列的最小值（列升序，它在最上）

所以：
  cur > target → target 不可能在当前列（列里更大的都在下面），排除当前列，左移
  cur < target → target 不可能在当前行（行里更小的都在左边），排除当前行，下移

每一步排除一行或一列，最多走 m+n 步，O(m+n)。

类比：把这个矩阵看成一棵"右上角是根"的搜索树，往左是左子树（更小），往下是右子树（更大）。
```

**注意**：从左上角或右下角出发不行——左上角是"行最小列最小"，无论大于还是小于 target，都可能往两个方向走，无法排除。

### 其他解法对比

| 解法 | 时间 | 空间 | 思路 |
|------|------|------|------|
| **右上角搜索（本解）** | **O(m+n)** | O(1) | 利用单调性逐行/列排除 |
| 每行二分 | O(m log n) | O(1) | 对每行做二分 |
| 分治（四象限） | O(n log n)（n=max(m,n)） | O(log n) | 每次切四块递归 |

右上角搜索法是最优且最直观。当 m 很大、n 很小时，每行二分可能更快（O(m log n)），但通用场景下 O(m+n) 已是最优渐进复杂度。

### 易错点

1. 从左上角出发 → 无法排除方向，是常见误区。
2. 起点写成 `(0,0)` 或 `(m-1, n-1)` → 错，必须是右上角 `(0, n-1)` 或左下角 `(m-1, 0)`。
3. 边界条件 `row < m and col >= 0` 用 `or` → 会越界。

## 矩阵类题目通用思路

| 模式 | 典型题目 | 关键技巧 |
|------|----------|----------|
| 原地标记 | 73 置零 | 借用首行首列当标记数组，省空间 |
| 边界模拟 | 54 螺旋 | 四个变量维护边界，走完一边收缩 |
| 转置+翻转 | 48 旋转 | 旋转 = 转置 + 翻转的组合 |
| 单调性搜索 | 240 搜索 | 找到"行内极值"的特殊角点出发 |

**核心认知**：
1. 看到"原地算法"要求 → 想办法把额外信息塞回矩阵本身（首行首列、首尾元素）。
2. 看到矩阵有"单调性"（行升序、列升序）→ 找角点（右上/左下）做 BST 式搜索。
3. 看到旋转/对称问题 → 先想转置 + 翻转的组合拳。
