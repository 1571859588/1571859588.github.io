## 一句话总结

进阶链表题的核心套路：**① 双指针的"间距控制"（19 删倒数第 N、142 找环入口）；② dummy 哨兵节点处理头节点变动（21 合并、24 交换）；③ 反转链表作为子组件的复用（25 K 个一组、148 排序）；④ 空间压缩技巧（138 原地复制、148 自底向上归并、146 双向链表+哈希）**。关键认知：**链表题不靠"聪明算法"，靠"指针操作细节 + 哨兵节点"**——遇到头节点可能变动的题，第一反应就是加 dummy。

## Q1: 2. 两数相加（模拟加法 + 进位）

**题目**：两个非空链表表示两个非负整数，数字**逆序**存储，每个节点存一位。相加后以同样形式返回链表。

> [2. 两数相加 - 力扣](https://leetcode.cn/problems/add-two-numbers/?envType=study-plan-v2&envId=top-100-liked)
> 标签：链表 | 数学 | 模拟

**示例**：
```
输入：l1 = [2,4,3], l2 = [5,6,4]
输出：[7,0,8]
解释：342 + 465 = 807，逆序存储即 [7,0,8]
```

### 一句话结论

**用 dummy 哨兵 + tail 指针逐位相加，维护一个进位 carry**。循环条件是 `l1 or l2 or carry`，最后一个 `or carry` 是为了处理最高位进位（如 999+1=1000 多出一位）。

### 解法（最优：迭代模拟）

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def addTwoNumbers(l1: ListNode, l2: ListNode) -> ListNode:
    """
    时间 O(max(m,n))，空间 O(max(m,n))（结果链表，不计入算法空间）
    核心：逐位相加，carry = sum // 10，当前位 = sum % 10
    """
    dummy = ListNode()   # 哨兵，避免单独处理头节点
    tail = dummy
    carry = 0

    # 三个条件缺一不可：l1、l2、carry 任何一个还有就要继续
    while l1 or l2 or carry:
        total = carry
        if l1:
            total += l1.val
            l1 = l1.next
        if l2:
            total += l2.val
            l2 = l2.next

        carry = total // 10          # 进位
        tail.next = ListNode(total % 10)  # 当前位
        tail = tail.next

    return dummy.next
```

### 推导过程

```
l1 = 2→4→3   (表示 342)
l2 = 5→6→4   (表示 465)

第 1 位：carry=0, total=2+5+0=7, carry=0, 结果 7, tail 前进
  l1=4, l2=6
第 2 位：carry=0, total=4+6+0=10, carry=1, 结果 0, tail 前进
  l1=3, l2=4
第 3 位：carry=1, total=3+4+1=8, carry=0, 结果 8, tail 前进
  l1=None, l2=None, carry=0 → 循环退出

结果：7→0→8（表示 807）✓
```

### 为什么循环条件必须带 `or carry`

```
极端例子：l1=[9,9,9,9] (9999), l2=[1] (1)
  逐位加完三个 9 后，l1=l2=None，但 carry=1（最后一位 9+1=10）
  如果循环条件只有 "l1 or l2"，此时会退出 → 漏掉最高位的 1！

带 or carry 后：
  最后一次循环：total=0+0+1=1, carry=0, 结果 1
  结果：0→0→0→0→1（表示 10000）✓
```

### 其他解法

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| **迭代模拟（本解）** | O(max(m,n)) | O(max(m,n)) | 推荐，不易错 |
| 递归 | O(max(m,n)) | O(max(m,n)) | 递归栈额外空间，思路类似 |

递归版本质一样，把"进位"作为递归返回值传递。迭代版更直观，面试推荐。

### 易错点

1. 忘记循环条件带 `or carry` → 最高位进位丢失。
2. 用 `total += l1.val` 前没判 `if l1` → 空指针。
3. 忘记移动 `l1 = l1.next` → 死循环。
4. 直接修改原链表节点（如复用 l1 节点）→ 题目允许但容易出错，建议新建节点。

## Q2: 19. 删除链表的倒数第 N 个结点（快慢指针间距）

**题目**：删除链表倒数第 n 个节点，返回头节点。要求**一趟扫描**。

> [19. 删除链表的倒数第 N 个结点 - 力扣](https://leetcode.cn/problems/remove-nth-node-from-end-of-list/?envType=study-plan-v2&envId=top-100-liked)
> 标签：链表 | 双指针

**示例**：
```
输入：head = [1,2,3,4,5], n = 2
输出：[1,2,3,5]   （删除倒数第 2 个，即节点 4）
```

### 一句话结论

**dummy 哨兵 + 快慢指针，快指针先走 n+1 步**（不是 n 步！），然后快慢同速走，快到尾时慢正好停在要删节点的前驱。

### 解法（最优：双指针一趟扫描）

```python
def removeNthFromEnd(head: ListNode, n: int) -> ListNode:
    """
    时间 O(n)，空间 O(1)
    核心：快指针先走 n+1 步，慢指针停在被删节点的前驱
    """
    # dummy 处理"删头节点"的边界（n = 链表长度时）
    dummy = ListNode(0, head)
    fast = slow = dummy

    # 快指针先走 n+1 步，制造 n+1 的间距
    for _ in range(n + 1):
        fast = fast.next

    # 快慢同速，快到尾时慢在被删节点的前驱
    while fast:
        slow = slow.next
        fast = fast.next

    # 删除 slow 的下一个节点
    slow.next = slow.next.next

    return dummy.next
```

### 推导过程

```
head = 1→2→3→4→5, n=2
加 dummy：dummy→1→2→3→4→5→None
fast=slow=dummy

第 1 步：fast 先走 n+1=3 步
  fast=dummy→1→2→3，fast 停在 3
  此时 fast 在 3，slow 在 dummy，间距 3（即 slow 后面数 3 个是 fast）

第 2 步：快慢同速
  fast=3→4, slow=dummy→1
  fast=4→5, slow=1→2
  fast=5→None, slow=2→3
  fast=None 退出

此时 slow 停在节点 2，它是要删节点 4 的前驱（2→3→4→5）。
等一下，slow=3，slow.next=4，删 4 ✓
结果：1→2→3→5 ✓
```

### 为什么快指针走 n+1 步而不是 n 步

```
要删倒数第 n 个，需要它的"前驱"（倒数第 n+1 个）。
如果快指针只走 n 步，制造 n 的间距：
  快到尾时，慢停在倒数第 n 个（要删的节点本身），但删节点要知道前驱！
  此时 slow.next 是要删的，slow 本身可能就是头节点（n=长度时），slow.next 操作 OK，但返回 head 时若删的是头节点就出错。

走 n+1 步：
  快到尾时，慢停在倒数第 n+1 个（前驱）。
  slow.next.next 操作安全，且 dummy 保证了即使删头节点也能正确返回 dummy.next。

口诀：要删第 n 个，找第 n+1 个，所以快走 n+1 步。
```

### 为什么必须用 dummy

```
考虑删除头节点的情况：head=[1], n=1
  不加 dummy：fast 先走 2 步会越界（链表只有 1 个节点 + dummy 才够）
  加 dummy：dummy→1→None
    fast 走 2 步：dummy→1→None，fast=None
    while fast 不执行（fast 已 None）
    slow=dummy，slow.next=1，删除：slow.next=None
    返回 dummy.next=None ✓

不加 dummy，删除头节点后返回的 head 还是原来的头（已被删），逻辑出错。
```

### 其他解法

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 两趟扫描 | O(n) | O(1) | 第一趟算长度 L，第二趟走 L-n 步 |
| **双指针一趟（本解）** | O(n) | O(1) | 满足"一趟扫描"进阶要求 |

两趟扫描更易懂，但题目进阶要求一趟，所以双指针是最优。

### 易错点

1. 快指针走 n 步而非 n+1 步 → slow 停错位置。
2. 不加 dummy → 删头节点时出错。
3. 循环条件写成 `while fast.next` → 快指针少走一步。

## Q3: 21. 合并两个有序链表（dummy + 双指针）

**题目**：合并两个升序链表为一个新的升序链表，拼接所有节点。

> [21. 合并两个有序链表 - 力扣](https://leetcode.cn/problems/merge-two-sorted-lists/?envType=study-plan-v2&envId=top-100-liked)
> 标签：链表 | 递归

**示例**：
```
输入：l1 = [1,2,4], l2 = [1,3,4]
输出：[1,1,2,3,4,4]
```

### 一句话结论

**dummy 哨兵 + tail 指针，每次比较两链表头，小的接到 tail 后面，移动对应链表**。最后把剩余链表直接接上。

### 解法（最优：迭代）

```python
def mergeTwoLists(l1: ListNode, l2: ListNode) -> ListNode:
    """
    时间 O(m+n)，空间 O(1)（复用原节点）
    核心：比较两链表头，小的接到 tail，移动该链表
    """
    dummy = ListNode()
    tail = dummy

    while l1 and l2:
        if l1.val <= l2.val:
            tail.next = l1
            l1 = l1.next
        else:
            tail.next = l2
            l2 = l2.next
        tail = tail.next

    # 其中一个链表已空，把另一个剩余部分直接接上
    tail.next = l1 if l1 else l2

    return dummy.next
```

### 推导过程

```
l1 = 1→2→4, l2 = 1→3→4

第 1 次：1 vs 1, 1<=1 取 l1 → dummy→1, l1=2, tail=1
第 2 次：2 vs 1, 1<2 取 l2 → ...→1, l2=3, tail=1
第 3 次：2 vs 3, 2<3 取 l1 → ...→2, l1=4, tail=2
第 4 次：4 vs 3, 3<4 取 l2 → ...→3, l2=4, tail=3
第 5 次：4 vs 4, 4<=4 取 l1 → ...→4, l1=None, tail=4
循环退出，l1=None, l2=4→None
tail.next = l2 → ...→4→4→None

结果：1→1→2→3→4→4 ✓
```

### 递归解法

```python
def mergeTwoLists(l1: ListNode, l2: ListNode) -> ListNode:
    """时间 O(m+n)，空间 O(m+n)（递归栈）"""
    if not l1:
        return l2
    if not l2:
        return l1
    if l1.val <= l2.val:
        l1.next = mergeTwoLists(l1.next, l2)
        return l1
    else:
        l2.next = mergeTwoLists(l1, l2.next)
        return l2
```

### 两种解法对比

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| **迭代（本解）** | O(m+n) | **O(1)** | 推荐，省栈 |
| 递归 | O(m+n) | O(m+n) | 代码短，但栈空间 |

面试推荐迭代。这个 merge 函数是 23、148 的核心组件，务必熟练。

### 易错点

1. 忘记 `tail = tail.next` → 所有节点都接到同一个位置。
2. 忘记最后接剩余链表 `tail.next = l1 if l1 else l2` → 丢失剩余节点。
3. 用 `<` 而非 `<=` → 也能过，但相等时优先取 l1 更稳定。

## Q4: 23. 合并 K 个升序链表（最小堆）

**题目**：合并 K 个已排序的链表为一个升序链表。

> [23. 合并 K 个升序链表 - 力扣](https://leetcode.cn/problems/merge-k-sorted-lists/?envType=study-plan-v2&envId=top-100-liked)
> 标签：链表 | 堆（优先队列）| 分治

**示例**：
```
输入：lists = [[1,4,5],[1,3,4],[2,6]]
输出：[1,1,2,3,4,4,5,6]
```

### 一句话结论

**用最小堆维护 K 个链表的当前头**：每次弹出最小值接到结果，再把它所在链表的下一个节点压入堆。时间 O(N log k)。

### 解法（最优：最小堆）

```python
import heapq

def mergeKLists(lists: list[ListNode]) -> ListNode:
    """
    时间 O(N log k)，空间 O(k)
    N = 所有节点总数，k = 链表个数
    核心：堆里维护 k 个链表当前最小头，每次取最小，补一个
    """
    dummy = ListNode()
    tail = dummy
    heap = []

    # 入堆：用 (val, 链表序号, node) 避免节点不可比较
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))

    while heap:
        val, i, node = heapq.heappop(heap)
        tail.next = node
        tail = tail.next
        # 把该链表的下一个节点压入堆
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next
```

### 推导过程

```
lists = [[1,4,5],[1,3,4],[2,6]]
初始堆：[(1,0,n0), (1,1,n1), (2,2,n2)]  （三个链表头）

第 1 次：弹 (1,0,n0)，接 n0(值1)
  补 n0.next=4 → 堆 [(1,1,n1),(2,2,n2),(4,0,n0_2)]
第 2 次：弹 (1,1,n1)，接 n1(值1)
  补 n1.next=3 → 堆 [(2,2,n2),(3,1,n1_2),(4,0,n0_2)]
第 3 次：弹 (2,2,n2)，接 n2(值2)
  补 n2.next=6 → 堆 [(3,1,n1_2),(4,0,n0_2),(6,2,n2_2)]
...依此类推
最终结果：1→1→2→3→4→4→5→6 ✓
```

### 为什么堆元素用 (val, i, node) 三元组

```
Python 的 heapq 在 val 相同时会比较下一个元素。
如果用 (val, node)，两个 val 相同的节点会比较 node，而 ListNode 不可比较 → TypeError。
加 i（链表序号）后，val 相同时比较 i，永远比较不到 node，安全。
```

### 分治解法（O(N log k)，O(log k) 空间）

```python
def mergeKLists(lists: list[ListNode]) -> ListNode:
    """分治：两两合并，log k 轮"""
    if not lists:
        return None

    while len(lists) > 1:
        merged = []
        for i in range(0, len(lists), 2):
            l1 = lists[i]
            l2 = lists[i + 1] if i + 1 < len(lists) else None
            merged.append(mergeTwo(l1, l2))  # 复用 21 题的 merge
        lists = merged
    return lists[0]
```

### 解法对比

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 依次合并 | O(kN) | O(1) | 顺序合并，每轮从头扫，最慢 |
| **最小堆（本解）** | **O(N log k)** | O(k) | 通用最优 |
| 分治 | O(N log k) | O(log k) | 递归栈，k 很大时省 |

堆法最通用。分治法空间略优，但要复用 mergeTwo 函数。

### 易错点

1. 堆元素直接用 `(val, node)` → val 相同时 TypeError。必须加索引。
2. 忘记判 `if node` 就入堆 → 空节点入堆报错。
3. 补节点时用 `if node.next` 判断 → 正确，避免空节点入堆。

## Q5: 24. 两两交换链表中的节点（dummy + 三指针）

**题目**：两两交换相邻节点，只能节点交换不能改值。

> [24. 两两交换链表中的节点 - 力扣](https://leetcode.cn/problems/swap-nodes-in-pairs/?envType=study-plan-v2&envId=top-100-liked)
> 标签：链表 | 递归

**示例**：
```
输入：head = [1,2,3,4]
输出：[2,1,4,3]
```

### 一句话结论

**dummy 哨兵 + prev/a/b 三指针**，每次把 `prev→a→b→b.next` 改成 `prev→b→a→b.next`，然后 prev 移到 a（下一对的前驱）。

### 解法（最优：迭代）

```python
def swapPairs(head: ListNode) -> ListNode:
    """
    时间 O(n)，空间 O(1)
    核心：每对用三指针交换，prev 是当前对的前驱
    """
    dummy = ListNode(0, head)
    prev = dummy

    while prev.next and prev.next.next:
        a = prev.next          # 当前对第一个
        b = a.next             # 当前对第二个

        # 交换：prev→a→b→b.next  →  prev→b→a→b.next
        a.next = b.next
        b.next = a
        prev.next = b

        # a 现在是这对的第二个，是下一对的前驱
        prev = a

    return dummy.next
```

### 推导过程

```
head = 1→2→3→4→None
dummy→1→2→3→4→None, prev=dummy

第 1 轮：a=1, b=2
  a.next = 4? 不，b.next=3
  a.next = 3   (1 指向 3)
  b.next = a   (2 指向 1)
  prev.next = b (dummy 指向 2)
  链表：dummy→2→1→3→4→None
  prev = a = 1（下一对的前驱）

第 2 轮：prev=1, prev.next=3, prev.next.next=4
  a=3, b=4
  a.next = None (b.next=None)
  b.next = a (4 指向 3)
  prev.next = b (1 指向 4)
  链表：dummy→2→1→4→3→None
  prev = a = 3

下一轮：prev=3, prev.next=None → 退出

结果：2→1→4→3 ✓
```

### 三条指针赋值的顺序很重要

```
错误顺序：先改 prev.next 再改 a.next
  prev.next = b   (dummy 指向 2)
  a.next = b.next (此时 b.next 是 1 还是 3？b.next 没变还是 1，所以 a.next=1 错误循环)

正确顺序：
  a.next = b.next  (先存 b 的下一个，a 指向它)
  b.next = a       (b 指向 a)
  prev.next = b    (prev 指向 b)

核心：先让 a 接管 b 的后续，再让 b 指向 a，最后让 prev 接 b。
```

### 递归解法

```python
def swapPairs(head: ListNode) -> ListNode:
    """时间 O(n)，空间 O(n)（递归栈）"""
    if not head or not head.next:
        return head
    new_head = head.next           # 第二个变新头
    head.next = swapPairs(head.next.next)  # 第一个指向后面交换后的头
    new_head.next = head           # 新头指向原第一个
    return new_head
```

### 解法对比

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| **迭代（本解）** | O(n) | **O(1)** | 推荐 |
| 递归 | O(n) | O(n) | 代码短，思路清晰 |

### 易错点

1. 三条赋值顺序错 → 形成环或丢失节点。必须先 `a.next=b.next`。
2. 循环条件只用 `prev.next` → 奇数个节点时最后一个漏判，应同时判 `prev.next and prev.next.next`。
3. 忘记 `prev = a` 后 prev 没前进 → 死循环。

## Q6: 25. K 个一组翻转链表（分组反转）

**题目**：每 k 个节点一组翻转，不足 k 个保持原序。要求实际节点交换。

> [25. K 个一组翻转链表 - 力扣](https://leetcode.cn/problems/reverse-nodes-in-k-group/?envType=study-plan-v2&envId=top-100-liked)
> 标签：链表 | 递归

**示例**：
```
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]   （最后 5 不足 k=2，保持）
```

### 一句话结论

**dummy + 分组循环**：每组先检查够不够 k 个，够则用反转链表函数翻转整组，再连接前后；不够则结束。反转链表是 206 题的复用。

### 解法（最优：迭代分组反转）

```python
def reverseKGroup(head: ListNode, k: int) -> ListNode:
    """
    时间 O(n)，空间 O(1)
    核心：每组先检查够 k 个，够则反转整组并连接前后
    """
    def reverse(head: ListNode) -> ListNode:
        """反转链表，返回新头"""
        prev = None
        curr = head
        while curr:
            nxt = curr.next
            curr.next = prev
            prev = curr
            curr = nxt
        return prev

    dummy = ListNode(0, head)
    prev = dummy  # 上一组的尾

    while True:
        # 检查从 prev.next 起是否够 k 个
        tail = prev
        for _ in range(k):
            tail = tail.next
            if not tail:          # 不足 k 个
                return dummy.next

        # 记录下一组起点
        nxt_group = tail.next
        # 断开当前组（tail 后面切断）
        tail.next = None

        # 翻转当前组，prev.next 是组头，翻转后变组尾
        group_head = prev.next
        prev.next = reverse(group_head)  # reverse 返回新头（原 tail）

        # 连接：group_head 现在是组尾，接下一组
        group_head.next = nxt_group
        # prev 移到组尾，作为下一组的前驱
        prev = group_head
```

### 推导过程

```
head = 1→2→3→4→5, k=2
dummy→1→2→3→4→5, prev=dummy

第 1 组：
  检查：tail=dummy→1→2，走 k=2 步，tail 停在 2（够）
  nxt_group = 3 (tail.next)
  tail.next = None（断开：1→2→None）
  group_head = 1
  prev.next = reverse(1→2) = 2→1   (反转后 2→1→None)
  group_head(=1).next = nxt_group(=3)  → 2→1→3→4→5
  prev = group_head = 1

第 2 组：
  检查：tail=1→3→4，走 2 步，tail 停在 4（够）
  nxt_group = 5
  tail.next = None（断开：3→4→None）
  group_head = 3
  prev.next = reverse(3→4) = 4→3
  group_head(=3).next = 5  → 4→3→5
  prev = 3

第 3 组：
  检查：tail=3→5→None，走 2 步，tail 到 None（不足）
  返回 dummy.next

链表：dummy→2→1→4→3→5，即 2→1→4→3→5 ✓
```

### 关键认知：反转后原组头变组尾

```
原组：group_head → ... → tail → None
reverse 后：tail → ... → group_head → None
group_head 变成了组尾，所以 group_head.next = nxt_group 能正确连接下一组。
prev 也应该移到 group_head（新的组尾，下一组的前驱）。
```

### 递归解法

```python
def reverseKGroup(head: ListNode, k: int) -> ListNode:
    """时间 O(n)，空间 O(n/k)（递归栈）"""
    # 检查够不够 k 个
    tail = head
    for _ in range(k):
        if not tail:
            return head  # 不足 k 个，不翻
        tail = tail.next
    # tail 现在是下一组起点（走 k 步后）

    # 翻转前 k 个
    prev = None
    curr = head
    for _ in range(k):
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt

    # head 现在是组尾，接上后面递归翻转的结果
    head.next = reverseKGroup(tail, k)
    return prev  # prev 是组头
```

### 解法对比

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| **迭代（本解）** | O(n) | **O(1)** | 满足进阶 O(1) |
| 递归 | O(n) | O(n/k) | 代码短，思路清晰 |

### 易错点

1. 反转前没断开 `tail.next = None` → reverse 会翻到链表末尾。
2. 反转后用 `prev.next = reverse(...)` 但忘了 `group_head.next = nxt_group` → 链表断开。
3. 检查够 k 个时循环条件错 → 误翻不足的组。
4. 这是 Hard 题，但本质是 206 反转链表的组合应用，拆解后不难。

## Q7: 138. 随机链表的复制（原地拼接三次遍历）

**题目**：深拷贝带 random 指针的链表。每个节点有 val、next、random（可指向任意节点或空）。

> [138. 随机链表的复制 - 力扣](https://leetcode.cn/problems/copy-list-with-random-pointer/?envType=study-plan-v2&envId=top-100-liked)
> 标签：哈希 | 链表

**示例**：
```
输入：head = [[7,null],[13,0],[11,4],[10,2],[1,0]]
输出：[[7,null],[13,0],[11,4],[10,2],[1,0]]
```

### 一句话结论

**原地拼接法三步走**：① 在每个原节点后插入复制节点；② 复制节点的 random = 原节点 random 的 next；③ 分离两个链表。O(n) 时间 O(1) 额外空间。

### 解法（最优：原地拼接）

```python
class Node:
    def __init__(self, x: int, next: 'Node' = None, random: 'Node' = None):
        self.val = int(x)
        self.next = next
        self.random = random

def copyRandomList(head: 'Node') -> 'Node':
    """
    时间 O(n)，空间 O(1)（不计结果链表）
    核心：在原节点后插入复制节点，借用原链表建立 random 映射
    """
    if not head:
        return None

    # 第 1 遍：每个原节点后插入复制节点
    # A→B→C  变成  A→A'→B→B'→C→C'
    cur = head
    while cur:
        copy = Node(cur.val)
        copy.next = cur.next
        cur.next = copy
        cur = copy.next  # 跳到下一个原节点

    # 第 2 遍：设置复制节点的 random
    # copy.random = cur.random.next（原节点 random 的复制版本）
    cur = head
    while cur:
        if cur.random:
            cur.next.random = cur.random.next  # cur.random 是原节点，它的 next 就是复制节点
        cur = cur.next.next  # 跳到下一个原节点

    # 第 3 遍：分离两个链表
    cur = head
    new_head = head.next
    while cur:
        copy = cur.next
        cur.next = copy.next        # 恢复原链表
        cur = copy.next             # 下一个原节点
        copy.next = cur.next if cur else None  # 复制链表连接
    return new_head
```

### 推导过程（关键看第 2 步）

```
原链表：A(random=C)→B(random=A)→C(random=None)
（A 的 random 指向 C，B 的 random 指向 A）

第 1 遍后（每个原节点后插复制）：
  A→A'→B→B'→C→C'→None
  此时 A'、B'、C' 的 random 都是 None（还没设）

第 2 遍设置 random：
  A.random = C（原节点），C.next = C'（复制节点）
  → A'.random = C.next = C' ✓
  B.random = A，A.next = A'
  → B'.random = A.next = A' ✓
  C.random = None
  → C'.random = None ✓

第 3 遍分离：
  原链表恢复：A→B→C
  复制链表：A'→B'→C'（random 已正确）
```

### 为什么 `copy.random = cur.random.next`

```
cur.random 指向某个原节点（比如 C），
而我们在第 1 步已经在 C 后面插入了 C'（C.next = C'）。
所以 cur.random.next 就是 C 对应的复制节点 C'。
这正是 cur 的复制节点 cur.next 的 random 应该指向的目标。
```

### 哈希表解法（O(n) 空间，更易理解）

```python
def copyRandomList_hash(head: 'Node') -> 'Node':
    """时间 O(n)，空间 O(n)"""
    if not head:
        return None
    # 第一遍：建原节点→复制节点的映射
    memo = {}
    cur = head
    while cur:
        memo[cur] = Node(cur.val)
        cur = cur.next
    # 第二遍：设置 next 和 random
    cur = head
    while cur:
        memo[cur].next = memo.get(cur.next)
        memo[cur].random = memo.get(cur.random)
        cur = cur.next
    return memo[head]
```

### 解法对比

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 哈希表 | O(n) | O(n) | 易理解，先用这个 AC |
| **原地拼接（本解）** | O(n) | **O(1)** | 满足进阶 O(1) |

面试可以先讲哈希表法建立思路，再优化到原地拼接展示空间优化能力。

### 易错点

1. 第 3 遍分离时 `copy.next = cur.next` → 要在 `cur.next = copy.next` 之前取，或用临时变量。
2. 第 2 遍跳步用 `cur = cur.next` → 应该 `cur.next.next`，否则会卡在复制节点。
3. `cur.random` 为 None 时没判空 → 空指针。必须 `if cur.random`。

## Q8: 142. 环形链表 II（Floyd 找环入口）

**题目**：返回链表入环的第一个节点，无环返回 null。不允许修改链表。

> [142. 环形链表 II - 力扣](https://leetcode.cn/problems/linked-list-cycle-ii/description/?envType=study-plan-v2&envId=top-100-liked)
> 标签：哈希 | 链表 | 双指针

**示例**：
```
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的节点（tail 连接到节点 2）
```

### 一句话结论

**Floyd 算法两阶段**：① 快慢指针找相遇点；② 一指针从头、一指针从相遇点，同速走，相遇即环入口。数学保证：从头走 a 步 = 从相遇点走 c + (n-1) 圈，两者必在入口相遇。

### 解法（最优：Floyd 快慢指针）

```python
def detectCycle(head: ListNode) -> ListNode:
    """
    时间 O(n)，空间 O(1)
    核心：阶段一找相遇点，阶段二一头一相遇点同速走
    """
    # 阶段一：快慢指针找相遇点
    slow = fast = head
    meet = None
    while fast and fast.next:
        slow = slow.next         # 慢走 1
        fast = fast.next.next    # 快走 2
        if slow is fast:
            meet = slow
            break

    # 无环
    if meet is None:
        return None

    # 阶段二：一头一相遇点，同速走
    p = head
    while p is not meet:
        p = p.next
        meet = meet.next
    return p  # 相遇即入口
```

### 推导过程

```
head = 3→2→0→4→(回到 2)，环入口是节点 2
      a (头到入口)
      3 → 2 ←入口
          ↑   ↓
      b   0   4
          ↑___↓ (c)
  a = 1（3→2），b = 0（入口 2 到相遇点），c = 3（相遇点到入口）

阶段一：快慢指针
  slow=3, fast=3
  第1步: slow=2, fast=0
  第2步: slow=0, fast=4
  第3步: slow=4, fast=2（相遇，meet=2？）
  实际相遇点取决于环结构，假设相遇在某个点

阶段二：p 从 head(3) 走，meet 从相遇点走
  p=3→2, meet=...→2
  两者在某点相遇，相遇点即入口 2 ✓
```

### 数学证明（为什么阶段二必在入口相遇）

```
设：
  a = 头到环入口的距离
  b = 环入口到相遇点的距离（顺时针）
  c = 相遇点到环入口的距离（顺时针）
  环长 L = b + c

阶段一相遇时：
  慢指针走：a + b 步（进环后走 b 到相遇，b < L）
  快指针走：a + b + nL 步（在环里绕了 n 圈，n≥1）
  快 = 2 × 慢：
    a + b + nL = 2(a + b)
    → nL = a + b
    → a = nL - b = (n-1)L + (L - b) = (n-1)L + c

所以从头走 a 步 = 从相遇点走 c 步 + 绕 (n-1) 圈。
两者同时走、同速，步数相同（都是 a），必在环入口相遇。
```

通俗理解：从头到入口的距离 a，等于从相遇点绕回入口的距离（c + 若干整圈）。所以两头同时出发同速走，走的步数一样，会在入口碰头。

### 其他解法

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 哈希集合 | O(n) | O(n) | 访问过的节点入集合，第一个重复即入口 |
| **Floyd（本解）** | O(n) | **O(1)** | 满足进阶 O(1) |

### 与 141 的关系

141 只判断"有没有环"（到相遇点即可），142 要"找到入口"（多走阶段二）。本解在 141 基础上加了阶段二。见 `链表类题目.md` 的 141 题。

### 易错点

1. 阶段一慢指针进环前快指针已绕圈 → 不影响，数学证明对任意情况成立。
2. 用 `while fast and fast.next` 但忘记判 `meet is None` → 无环时阶段二报错。
3. 阶段二判断用 `==` → 应用 `is`，节点对象比较。
4. 无环时慢指针走 n 步就到尾，不会相遇，meet 保持 None。

## Q9: 146. LRU 缓存（哈希 + 双向链表）

**题目**：设计 LRU 缓存，get/put 均 O(1)。容量满时淘汰最久未使用的。

> [146. LRU 缓存 - 力扣](https://leetcode.cn/problems/lru-cache/?envType=study-plan-v2&envId=top-100-liked)
> 标签：设计 | 哈希 | 链表 | 双向链表

**示例**：
```
LRUCache lRUCache = new LRUCache(2);
lRUCache.put(1, 1); // {1=1}
lRUCache.put(2, 2); // {1=1, 2=2}
lRUCache.get(1);    // 返回 1
lRUCache.put(3, 3); // 淘汰 2，{1=1, 3=3}
lRUCache.get(2);    // 返回 -1
lRUCache.put(4, 4); // 淘汰 1，{4=4, 3=3}
lRUCache.get(1);    // 返回 -1
lRUCache.get(3);    // 返回 3
lRUCache.get(4);    // 返回 4
```

### 一句话结论

**哈希表 O(1) 查找 + 双向链表 O(1) 调整顺序**：节点放在双向链表，最近访问的放头部，淘汰尾部。dummy 头尾节点省边界处理。

### 解法（最优：哈希 + 双向链表）

```python
class DLinkedNode:
    """双向链表节点，存 key 是为了淘汰时能从哈希表删 key"""
    def __init__(self, key=0, value=0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.size = 0
        self.cache = {}  # key -> 节点

        # dummy 头尾节点，省去 None 判断
        self.head = DLinkedNode()  # dummy head（最近使用）
        self.tail = DLinkedNode()  # dummy tail（最久未使用）
        self.head.next = self.tail
        self.tail.prev = self.head

    def get(self, key: int) -> int:
        """O(1)：查到则移到头部并返回值，否则 -1"""
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self.moveToHead(node)   # 访问过，移到头部
        return node.value

    def put(self, key: int, value: int) -> None:
        """O(1)：存在则更新+移头部；不存在则新建+加头部，超容则删尾"""
        if key in self.cache:
            node = self.cache[key]
            node.value = value
            self.moveToHead(node)
        else:
            node = DLinkedNode(key, value)
            self.cache[key] = node
            self.addToHead(node)
            self.size += 1
            if self.size > self.capacity:
                removed = self.removeTail()  # 淘汰尾部
                self.cache.pop(removed.key)   # 同步删哈希表
                self.size -= 1

    # ===== 双向链表操作，全是 O(1) =====
    def addToHead(self, node: DLinkedNode) -> None:
        """插入到 dummy head 之后（最近使用位）"""
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node

    def removeNode(self, node: DLinkedNode) -> None:
        """删除任意节点"""
        node.prev.next = node.next
        node.next.prev = node.prev

    def moveToHead(self, node: DLinkedNode) -> None:
        """移到头部 = 删除 + 加头"""
        self.removeNode(node)
        self.addToHead(node)

    def removeTail(self) -> DLinkedNode:
        """删除尾部（dummy tail 前一个），返回被删节点"""
        node = self.tail.prev
        self.removeNode(node)
        return node
```

### 结构示意图

```
哈希表：{key1: node1, key2: node2, ...}

双向链表（head 侧是最近使用，tail 侧是最久未使用）：
  head(d) ↔ node1 ↔ node2 ↔ ... ↔ nodeN ↔ tail(d)

get(key1)：哈希表查到 node1 → 移到 head 侧 → 返回值
put(key_new)：新建节点加 head 侧 → 若超容删 tail 侧节点（同步删哈希表）
```

### 为什么必须用双向链表

```
要 O(1) 删除任意节点，必须知道前驱。
单向链表删节点要从头找前驱，O(n)。
双向链表每个节点存 prev，删节点直接 node.prev.next = node.next，O(1)。
```

### 为什么节点要存 key

```
淘汰尾部节点时，要从哈希表删对应 key。
如果节点不存 key，删尾后无法知道它对应的 key，哈希表无法同步。
所以 DLinkedNode 同时存 key 和 value。
```

### 为什么用 dummy 头尾

```
不用 dummy 的话：
  加头节点时要判链表是否空
  删尾节点时要判是不是最后一个
  移动节点时要判是不是头

用 dummy 后，head 和 tail 永远存在，所有操作统一，省去边界判断。
```

### Python 偷懒解法（用 OrderedDict）

```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.cap = capacity
        self.od = OrderedDict()

    def get(self, key):
        if key not in self.od:
            return -1
        self.od.move_to_end(key)  # 移到末尾（最近使用）
        return self.od[key]

    def put(self, key, value):
        if key in self.od:
            self.od.move_to_end(key)
        self.od[key] = value
        if len(self.od) > self.cap:
            self.od.popitem(last=False)  # 删头部（最久未使用）
```

注意：`move_to_end` 默认移到末尾，`popitem(last=False)` 删头部（最老的）。这与手写版的"头部是最近"相反，但效果一样。

### 解法对比

| 解法 | get/put 时间 | 空间 | 说明 |
|------|------|------|------|
| **哈希+双向链表（本解）** | **O(1)** | O(n) | 面试标准答案 |
| OrderedDict | O(1) | O(n) | Python 偷懒，面试要能手写 |

面试必须手写双向链表版。OrderedDict 只能作为"知道 Python 有这工具"的加分项。

### 易错点

1. 节点不存 key → 淘汰时无法同步删哈希表。
2. `pop(removed.key)` 忘记写 → 哈希表残留，下次 get 返回已删节点。
3. `removeTail` 返回的是节点不是值 → 要拿 node.key。
4. dummy 头尾没初始化相连 → 第一节点加入时 prev 为空。
5. `put` 已存在的 key 时忘记更新 value → 数据不一致。

## Q10: 148. 排序链表（归并排序自底向上）

**题目**：链表升序排序，要求 O(n log n) 时间、O(1) 空间（进阶）。

> [148. 排序链表 - 力扣](https://leetcode.cn/problems/sort-list/?envType=study-plan-v2&envId=top-100-liked)
> 标签：链表 | 双指针 | 分治 | 排序

**示例**：
```
输入：head = [4,2,1,3]
输出：[1,2,3,4]
```

### 一句话结论

**归并排序自底向上**：size 从 1 倍增，每轮把相邻两个 size 长的子链表归并。用迭代代替递归，省掉 O(log n) 栈空间，实现 O(1) 额外空间。

### 解法（最优：自底向上归并）

```python
def sortList(head: ListNode) -> ListNode:
    """
    时间 O(n log n)，空间 O(1)（自底向上迭代，无递归栈）
    核心：size 从 1 倍增，每轮归并相邻两段
    """
    if not head or not head.next:
        return head

    # 求长度
    length = 0
    node = head
    while node:
        length += 1
        node = node.next

    dummy = ListNode(0, head)
    size = 1
    while size < length:
        tail = dummy       # 已归并部分的尾
        cur = dummy.next   # 当前待归并起点
        while cur:
            # 取出第一段（长度 size）
            left = cur
            right = split(left, size)
            # 取出第二段后的下一段
            cur = split(right, size) if right else None
            # 归并两段，接回主链
            merged = merge(left, right)
            tail.next = merged
            while tail.next:   # tail 走到归并后末尾
                tail = tail.next
        size *= 2
    return dummy.next


def split(head: ListNode, n: int) -> ListNode:
    """从 head 走 n 步，断开，返回后半段头"""
    if not head:
        return None
    cur = head
    for _ in range(n - 1):     # 走 n-1 步，停在长度 n 的最后节点
        if not cur.next:
            break
        cur = cur.next
    nxt = cur.next
    cur.next = None            # 断开
    return nxt


def merge(l1: ListNode, l2: ListNode) -> ListNode:
    """合并两个有序链表，复用 21 题"""
    dummy = ListNode()
    tail = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            tail.next = l1
            l1 = l1.next
        else:
            tail.next = l2
            l2 = l2.next
        tail = tail.next
    tail.next = l1 if l1 else l2
    return dummy.next
```

### 推导过程

```
head = 4→2→1→3, length=4

size=1（每段 1 个节点）：
  第1轮：left=4, right=2, 归并得 2→4, cur=1→3
  第2轮：left=1, right=3, 归并得 1→3, cur=None
  链表：2→4→1→3

size=2（每段 2 个节点）：
  第1轮：left=2→4, right=1→3, 归并得 1→2→3→4, cur=None
  链表：1→2→3→4

size=4 >= length=4，结束。
结果：1→2→3→4 ✓
```

### 自顶向下递归版（O(log n) 栈空间）

```python
def sortList_recursive(head: ListNode) -> ListNode:
    """时间 O(n log n)，空间 O(log n)（递归栈）"""
    if not head or not head.next:
        return head
    # 快慢指针找中点
    slow, fast = head, head.next
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    mid = slow.next
    slow.next = None  # 断开
    # 递归排序两半，再归并
    return merge(sortList_recursive(head), sortList_recursive(mid))
```

递归版更易懂，但不满足 O(1) 空间进阶要求。

### 为什么用归并而不是快排

```
快排在数组上很快是因为随机访问分区点。
链表无法 O(1) 随机访问，快排分区效率低。
归并排序天然适合链表：
  - 找中点：快慢指针 O(n)
  - 归并：双指针 O(n)
  - 不需要随机访问
且归并是稳定排序，链表归并能 O(1) 额外空间（数组归并要 O(n) 辅助空间，但链表靠改指针即可）。
```

### 解法对比

| 解法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| **自底向上归并（本解）** | O(n log n) | **O(1)** | 满足进阶 |
| 递归归并 | O(n log n) | O(log n) | 易理解 |
| 插入排序 | O(n²) | O(1) | 不满足时间要求 |

### 易错点

1. `split` 走 n 步写成 `range(n)` → 多走一步，段长不对。
2. 每轮归并后忘记把 `tail` 移到末尾 → 下一段接错位置。
3. `merge` 函数和 21 题一样，直接复用，但要确保 `split` 正确断开。
4. `size` 倍增用 `size *= 2`，不是 `size += 1`。
5. 递归版快慢指针找中点时，fast 从 head.next 出发（保证偶数时 slow 停在前半尾）。

## 链表进阶题通用思路

| 模式 | 典型题目 | 关键技巧 |
|------|----------|----------|
| dummy 哨兵 | 19/21/24/25 | 头节点可能变动时，加 dummy 省边界 |
| 双指针间距 | 19 删倒数N / 142 找环入口 | 快指针先走制造间距，或快慢追及 |
| 反转链表复用 | 25 K个一组 / 148 排序 | reverse 是组件，组合使用 |
| 空间压缩 | 138 原地复制 / 148 自底向上 | 用原链表结构存储映射，省哈希表 |
| 堆/优先队列 | 23 合并K个 | 维护多路数据的最值 |
| 哈希+双向链表 | 146 LRU | 查找 O(1) + 顺序调整 O(1) |

### 核心认知

1. **dummy 哨兵是链表题第一工具**：凡是涉及头节点可能变动（删除、插入、合并、交换），都加 dummy，省去大量边界判断。
2. **快慢指针间距可控**：快指针先走 n 步 → 制造 n 的间距；快指针走 2 倍速 → 找中点或判环。间距和速度是两种用法。
3. **反转链表是万能组件**：25、148、206 都用到，务必写熟。
4. **链表排序用归并**：快排不适合链表（无随机访问），归并天然适配。
5. **进阶 O(1) 空间的套路**：借用原链表节点存映射关系（138）、用迭代代替递归（148）、用指针代替哈希（142）。

### 常见陷阱速查

| 陷阱 | 出现题目 | 规避方法 |
|------|----------|----------|
| 改 next 前没存下一个节点 | 206/24/25 | 先 `nxt = curr.next` |
| 删节点没找前驱 | 19 | 快指针多走 1 步，或用 dummy |
| 循环条件漏判 | 24/141 | 同时判 `fast and fast.next` |
| 哈希表不同步 | 146 LRU | 删尾后 `cache.pop(key)` |
| 比较节点用 `==` | 141/142 | 用 `is` 比较对象身份 |
