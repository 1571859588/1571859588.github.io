    SSH 本地端口转发

    SSH 本地端口转发：访问远程服务器上的 3001 服务

    一句话结论

    ssh -L 3001:127.0.0.1:3001 empyrean@172.21.20.244 的作用是：登录远程服务器 172.21.20.244，同时在本机打开一个 3001 端口，把本机访问 127.0.0.1:3001 的流量转发到远程服务器自己的 127.0.0.1:3001。

    简单说：远程服务器上如果有一个只监听本机的服务，例如 Web 应用、API、Jupyter、前端 dev server，运行在远程的 127.0.0.1:3001，本机可以通过 SSH 隧道访问它。

    命令

    ssh -L 3001:127.0.0.1:3001 empyrean@172.21.20.244

    参数拆解

    ssh

    使用 SSH 连接远程服务器。

    -L 3001:127.0.0.1:3001

    开启本地端口转发，格式是：

    -L 本机端口:远程目标地址:远程目标端口

    对应这条命令就是：

    本机端口：3001
远程目标地址：127.0.0.1
远程目标端口：3001

    含义是：

    本机 127.0.0.1:3001  -->  SSH 服务器 172.21.20.244  -->  远程服务器上的 127.0.0.1:3001

    注意这里的第二个 127.0.0.1 是站在远程服务器视角看的，不是本机自己的 127.0.0.1。

    empyrean@172.21.20.244

    表示用用户名 empyrean 登录 IP 为 172.21.20.244 的远程服务器。

    什么时候需要用

    典型场景：

    • 远程服务器上启动了一个服务，只能在远程服务器内部访问，比如 http://127.0.0.1:3001。
    • 这个服务没有直接暴露公网端口，或者服务器防火墙没有开放 3001。
    • 想在自己电脑浏览器里访问远程服务。
    • 想安全地访问远程开发服务，而不是直接开放端口。

    怎么使用

    1. 先确认远程服务已经启动

    在远程服务器上，目标服务需要已经监听 3001 端口。例如远程服务器里可能有：

    npm run dev -- --host 127.0.0.1 --port 3001

    或者某个服务已经能在远程服务器上通过下面地址访问：

    http://127.0.0.1:3001

    2. 在本机终端运行 SSH 转发命令

    在自己的电脑上打开终端，运行：

    ssh -L 3001:127.0.0.1:3001 empyrean@172.21.20.244

    如果需要密码，就输入远程服务器用户 empyrean 的 SSH 密码。

    如果使用密钥登录，一般会自动使用默认密钥；也可以显式指定：

    ssh -i /path/to/private_key -L 3001:127.0.0.1:3001 empyrean@172.21.20.244

    Windows PowerShell 也可以直接运行这条命令，前提是系统已经有 OpenSSH 客户端。

    3. 保持这个 SSH 窗口不要关闭

    只要这个 SSH 连接还开着，端口转发就有效。

    如果关闭终端、断开 SSH、网络中断，本地 3001 到远程 3001 的转发也会断开。

    4. 在本机浏览器访问

    打开本机浏览器访问：

    http://127.0.0.1:3001

    或：

    http://localhost:3001

    这时看到的应该是远程服务器上运行在 127.0.0.1:3001 的服务。

    验证方法

    验证本机端口是否能访问

    在本机执行：

    curl http://127.0.0.1:3001

    如果返回页面 HTML、JSON、接口响应，说明转发成功。

    验证远程服务是否真的在运行

    如果本机访问失败，可以先 SSH 到远程服务器后执行：

    curl http://127.0.0.1:3001

    如果远程服务器自己都访问不了，说明问题不是 SSH 转发，而是远程服务没有启动、端口不对，或者服务绑定地址不对。

    常见问题

    1. 本机提示端口被占用

    可能报错类似：

    bind: Address already in use

    说明本机 3001 已经被其他程序占用了。可以换一个本机端口，例如：

    ssh -L 13001:127.0.0.1:3001 empyrean@172.21.20.244

    然后本机浏览器访问：

    http://127.0.0.1:13001

    注意：这里本机端口变成了 13001，但远程目标端口仍然是 3001。

    2. SSH 能登录，但浏览器打不开

    优先检查：

    • 远程服务是否真的运行在 127.0.0.1:3001。
    • 远程服务端口是不是 3001，有没有实际跑在 3000、7860、8000 等其他端口。
    • SSH 转发窗口是否还开着。
    • 本机访问的是不是 http://127.0.0.1:3001，不要访问远程 IP 的 3001。

    3. 远程服务绑定在 0.0.0.0 怎么办

    通常也可以用这条命令访问，因为远程服务监听 0.0.0.0:3001 时，一般也能通过远程服务器自己的 127.0.0.1:3001 访问。

    如果不行，需要确认服务实际监听地址和端口。

    4. 想让 SSH 只做转发，不进入交互终端

    可以加 -N：

    ssh -N -L 3001:127.0.0.1:3001 empyrean@172.21.20.244

    含义是只建立端口转发，不打开远程 shell。

    记忆方式

    -L 可以理解为 Local，本地转发：

    把“本机端口”接到“远程服务器能访问到的地址和端口”。

    这条命令最容易混淆的是：

    3001:127.0.0.1:3001

    左边的 3001 是自己电脑上的端口；中间的 127.0.0.1 和右边的 3001 是远程服务器视角下的目标服务地址和端口。

    关联命令变体

    换本机端口，避免本机端口冲突：

    ssh -L 13001:127.0.0.1:3001 empyrean@172.21.20.244

    只转发不进入远程终端：

    ssh -N -L 3001:127.0.0.1:3001 empyrean@172.21.20.244

    指定 SSH 私钥：

    ssh -i /path/to/private_key -L 3001:127.0.0.1:3001 empyrean@172.21.20.244
