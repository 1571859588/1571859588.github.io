    普通数组类题目

    LeetCode Hot 100 普通数组类题目

    更新时间：2026-07-02 来源：LeetCode Hot 100 普通数组专题 状态：刷题笔记，含完整代码 + 推导 + 面试追问

    一句话总结

    普通数组类题目的核心：把”看起来要 O(n²) 甚至 O(n³)”的暴力做法，利用”前缀/后缀/原地标记/排序”等技巧优化到 O(n)。常见模式：Kadane 算法（最大子数组和）、排序后线性扫描（合并区间）、三次翻转（轮转数组）、前后缀乘积（除自身以外的乘积）、原地哈希（缺失的第一个正数）。关键认知：数组本身就可以当作哈希表来用——下标 i 天然对应一个值，这是 O(1) 空间解题的钥匙。

    Q1: 53. 最大子数组和（Kadane 算法）

    题目：给你一个整数数组 nums，找出一个具有最大和的连续子数组（最少包含一个元素），返回其最大和。

    53. 最大子数组和 - 力扣

    示例：

    输入: nums = [-2,1,-3,4,-1,2,1,-5,4]
输出: 6
解释: 连续子数组 [4,-1,2,1] 的和最大，为 6。

输入: nums = [1]
输出: 1

输入: nums = [5,4,-1,7,8]
输出: 23

    解法（Kadane 算法 / 动态规划）

    def maxSubArray(nums: list[int]) -> int:
    """
    时间 O(n)，空间 O(1)
    核心：维护"以 nums[i] 结尾的最大子数组和"
    状态转移：dp[i] = max(nums[i], dp[i-1] + nums[i])
    """
    # cur 表示"以当前元素结尾"的最大子数组和
    # ans 表示全局最大子数组和
    cur = ans = nums[0]

    for i in range(1, len(nums)):
        # 关键决策：是把 nums[i] 接到前面的子数组后面，还是从 nums[i] 重新开始？
        # 如果 cur < 0，说明前面的累加是负收益，不如扔掉重新开始
        cur = max(nums[i], cur + nums[i])
        ans = max(ans, cur)

    return ans

    推导过程

    以 nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4] 为例：

    i=0: nums[0]=-2
     cur = -2, ans = -2

i=1: nums[1]=1
     cur = max(1, -2+1) = max(1, -1) = 1   ← 前面是负的，扔掉重新开始
     ans = max(-2, 1) = 1

i=2: nums[2]=-3
     cur = max(-3, 1+(-3)) = max(-3, -2) = -2  ← 虽然 -2 比 -3 大，但接着走
     ans = max(1, -2) = 1

i=3: nums[3]=4
     cur = max(4, -2+4) = max(4, 2) = 4    ← 前面是负的，扔掉重新开始
     ans = max(1, 4) = 4

i=4: nums[4]=-1
     cur = max(-1, 4+(-1)) = max(-1, 3) = 3  ← 4 是正收益，接着走
     ans = max(4, 3) = 4

i=5: nums[5]=2
     cur = max(2, 3+2) = max(2, 5) = 5
     ans = max(4, 5) = 5

i=6: nums[6]=1
     cur = max(1, 5+1) = max(1, 6) = 6
     ans = max(5, 6) = 6   ← 全局最大值出现在这里

i=7: nums[7]=-5
     cur = max(-5, 6+(-5)) = max(-5, 1) = 1
     ans = max(6, 1) = 6

i=8: nums[8]=4
     cur = max(4, 1+4) = max(4, 5) = 5
     ans = max(6, 5) = 6

最终答案: 6 ✓
对应子数组: [4, -1, 2, 1]（i=3 到 i=6）

    直觉理解

    想象你在爬一座山，山有起伏（有正有负）。你只能往右走，问：哪一段连续的路程累计海拔最高？

    • 如果前面累计的海拔已经是负数了，那继续走只会拖累后面的路程——不如从当前位置重新开始累计。
    • 如果前面累计是正数，那继续接着走，越走越高。

    cur 就是”以当前位置结尾的累计海拔”，ans 是历史最高记录。

    复杂度

     复杂度
    时间O(n)，一次遍历
    空间O(1)，只用两个变量

    面试追问

    Q：题目说”进阶用分治法”，怎么做？

    分治法：把数组从中点切成两半，最大子数组和有三种情况：

    1. 完全在左半边 → 递归左半
    2. 完全在右半边 → 递归右半
    3. 跨中点 → 从中点向左扫最大后缀和 + 向右扫最大前缀和
    def maxSubArray_divide(nums: list[int]) -> int:
    def helper(l, r):
        if l == r:
            return nums[l]
        mid = (l + r) // 2
        # 情况3：跨中点
        left_max = nums[mid]
        s = 0
        for i in range(mid, l - 1, -1):
            s += nums[i]
            left_max = max(left_max, s)
        right_max = nums[mid + 1]
        s = 0
        for i in range(mid + 1, r + 1):
            s += nums[i]
            right_max = max(right_max, s)
        cross = left_max + right_max
        # 三者取最大
        return max(helper(l, mid), helper(mid + 1, r), cross)
    return helper(0, len(nums) - 1)

    时间 O(n log n)，空间 O(log n)。比 Kadane 慢，但能练习分治思想，且能扩展到”支持区间修改/查询最大子数组和”的线段树场景。

    Q：如果数组全是负数，结果是什么？

    返回最大的那个负数（不是 0）。Kadane 算法的初始值 cur = ans = nums[0] 保证了这一点。如果初始值设成 0 会错——会返回 0，但题目要求”子数组最少包含一个元素”。

    Q：怎么求最大子数组和的”具体子数组”（返回下标）？

    记录 cur 更新时的起始位置，以及 ans 更新时的结束位置：

    def maxSubArray_with_index(nums: list[int]) -> tuple[int, int, int]:
    cur = ans = nums[0]
    cur_start = ans_start = ans_end = 0
    for i in range(1, len(nums)):
        if nums[i] > cur + nums[i]:
            cur = nums[i]
            cur_start = i
        else:
            cur = cur + nums[i]
        if cur > ans:
            ans = cur
            ans_start, ans_end = cur_start, i
    return ans, ans_start, ans_end

    Q2: 56. 合并区间（排序 + 线性扫描）

    题目：以数组 intervals 表示若干个区间的集合，合并所有重叠的区间，返回一个不重叠的区间数组。

    56. 合并区间 - 力扣

    示例：

    输入: intervals = [[1,3],[2,6],[8,10],[15,18]]
输出: [[1,6],[8,10],[15,18]]
解释: 区间 [1,3] 和 [2,6] 重叠，合并为 [1,6]。

输入: intervals = [[1,4],[4,5]]
输出: [[1,5]]
解释: [1,4] 和 [4,5] 可被视为重叠（端点接触也算重叠）。

输入: intervals = [[4,7],[1,4]]
输出: [[1,7]]
解释: 无序输入也要合并。

    解法

    def merge(intervals: list[list[int]]) -> list[list[int]]:
    """
    时间 O(n log n)（排序），空间 O(n)（结果数组）
    核心：按区间左端点排序，然后线性扫描合并
    """
    # 第一步：按左端点排序
    intervals.sort(key=lambda x: x[0])

    merged = []
    for interval in intervals:
        # 如果结果为空，或当前区间与上一个不重叠（左端 > 上一个的右端）
        if not merged or merged[-1][1] < interval[0]:
            merged.append(interval)
        else:
            # 重叠：合并，取右端较大值
            merged[-1][1] = max(merged[-1][1], interval[1])

    return merged

    推导过程

    以 intervals = [[1,3],[2,6],[8,10],[15,18]] 为例（已排序）：

    初始: merged = []

处理 [1,3]:
  merged 为空 → 直接加入
  merged = [[1,3]]

处理 [2,6]:
  merged[-1] = [1,3], 右端 = 3
  当前左端 = 2，2 <= 3 → 重叠！
  合并：右端 = max(3, 6) = 6
  merged = [[1,6]]

处理 [8,10]:
  merged[-1] = [1,6], 右端 = 6
  当前左端 = 8，8 > 6 → 不重叠
  直接加入
  merged = [[1,6],[8,10]]

处理 [15,18]:
  merged[-1] = [8,10], 右端 = 10
  当前左端 = 15，15 > 10 → 不重叠
  直接加入
  merged = [[1,6],[8,10],[15,18]]

最终结果: [[1,6],[8,10],[15,18]] ✓

    为什么排序后就能线性扫描？

    排序后，所有可能重叠的区间一定相邻。因为：

    • 如果区间 A 和区间 B 重叠（A.left ≤ B.left），且排序后它们不相邻，中间隔着 C
    • 那么 A.left ≤ C.left ≤ B.left（排序保证）
    • 又 A.right ≥ B.left（A 和 B 重叠）
    • 所以 C.left ≤ B.left ≤ A.right，即 C 也和 A 重叠
    • 矛盾：C 应该被合并到 A 里，不可能”隔着”

    所以排序后只要看相邻就行。

    重叠的判定

    核心判定式：当前区间左端 <= 上一个区间右端，即 interval[0] <= merged[-1][1]

    情况图示是否重叠
    不重叠[1,3] [5,7] → 3 < 5❌
    端点接触[1,4] [4,5] → 4 <= 4✅（题目说算重叠）
    部分重叠[1,3] [2,6] → 2 <= 3✅
    包含[1,10] [3,5] → 3 <= 10✅

    复杂度

     复杂度
    时间O(n log n)，瓶颈在排序
    空间O(n)，存结果（不算结果数组则是 O(log n) 的排序栈）

    面试追问

    Q：如果端点接触不算重叠（[1,4] 和 [4,5] 不合并），怎么改？

    把判定条件改成严格小于：merged[-1][1] < interval[0] → 不重叠，merged[-1][1] >= interval[0] → 重叠。等价于把 if 条件改成 merged[-1][1] < interval[0] 改为 merged[-1][1] < interval[0]（不变），但 else 分支不变。其实只有”等于”的边界要改：把判定改成 merged[-1][1] < interval[0] 表示不重叠时直接加入，那么 merged[-1][1] >= interval[0]（含等号）就是重叠。如果要让等号不算重叠，把条件改成 merged[-1][1] < interval[0] 改为 merged[-1][1] < interval[0] 不变，但你需要思考：接触不合并意味着 [1,4] 和 [4,5] 分开，那么”不重叠”的判定应该是 merged[-1][1] < interval[0]（严格小于，等于就算重叠的反面）。

    简单记：判定条件 merged[-1][1] < interval[0] 是”不重叠”，把 < 改成 <= 就变成”接触也不重叠”。本题要”接触算重叠”，所以用 <。

    Q：如果要返回哪些区间被合并到一起（保留原始下标），怎么做？

    排序前给每个区间打上原始下标，排序后合并时记录每组包含的原始下标：

    # 给每个区间加原始下标
indexed = [(interval[0], interval[1], i) for i, interval in enumerate(intervals)]
indexed.sort(key=lambda x: x[0])
# 合并时记录原始下标
groups = []
for start, end, idx in indexed:
    if not groups or groups[-1][1] < start:
        groups.append([start, end, [idx]])
    else:
        groups[-1][1] = max(groups[-1][1], end)
        groups[-1][2].append(idx)

    Q：插入一个新区间到已合并的区间集合里（LeetCode 57 题），怎么做？

    思路类似：遍历原区间，分三段处理——

    1. 右端 < 新区间左端：直接加入结果
    2. 与新区间重叠：合并（左端取小、右端取大）
    3. 左端 > 新区间右端：先把合并后的新区间加入结果，再把剩余区间加入

    Q3: 189. 轮转数组（三次翻转）

    题目：给定一个整数数组 nums，将数组中的元素向右轮转 k 个位置。

    189. 轮转数组 - 力扣

    示例：

    输入: nums = [1,2,3,4,5,6,7], k = 3
输出: [5,6,7,1,2,3,4]
解释:
  向右轮转 1 步: [7,1,2,3,4,5,6]
  向右轮转 2 步: [6,7,1,2,3,4,5]
  向右轮转 3 步: [5,6,7,1,2,3,4]

输入: nums = [-1,-100,3,99], k = 2
输出: [3,99,-1,-100]

    解法一：三次翻转（原地 O(1) 空间，推荐）

    def rotate(nums: list[int], k: int) -> None:
    """
    时间 O(n)，空间 O(1)（原地修改）
    核心：三次翻转
      1. 整体翻转
      2. 翻转前 k 个
      3. 翻转后 n-k 个
    """
    n = len(nums)
    k = k % n  # 关键：k 可能大于 n，取模
    if k == 0:
        return

    def reverse(l: int, r: int) -> None:
        """翻转 nums[l..r]（双指针相向交换）"""
        while l < r:
            nums[l], nums[r] = nums[r], nums[l]
            l += 1
            r -= 1

    # 第一步：整体翻转
    reverse(0, n - 1)
    # 第二步：翻转前 k 个
    reverse(0, k - 1)
    # 第三步：翻转后 n-k 个
    reverse(k, n - 1)

    推导过程

    以 nums = [1,2,3,4,5,6,7], k = 3 为例：

    原始:       [1, 2, 3, 4, 5, 6, 7]
                  │           │
              前 n-k=4 个  后 k=3 个

第1步 整体翻转:
            [7, 6, 5, 4, 3, 2, 1]
             ←后k个→  ←前n-k个→
             (倒着了)  (倒着了)

第2步 翻转前 k=3 个:
            [5, 6, 7, 4, 3, 2, 1]
             ←正过来→  (还是倒的)

第3步 翻转后 n-k=4 个:
            [5, 6, 7, 1, 2, 3, 4]
             ←正过来→  ←正过来→

最终结果: [5, 6, 7, 1, 2, 3, 4] ✓

    直觉理解：右轮转 k 位，等价于把”最后 k 个元素”挪到前面，”前面 n-k 个元素”挪到后面。整体翻转后，”最后 k 个”跑到了最前面（但顺序倒了），”前 n-k 个”跑到了后面（也倒了）。再分别翻转这两段，顺序就正过来了。

    解法二：额外数组（直观但 O(n) 空间）

    def rotate_extra(nums: list[int], k: int) -> None:
    """时间 O(n)，空间 O(n)"""
    n = len(nums)
    k = k % n
    tmp = [0] * n
    for i in range(n):
        # 第 i 个元素右移 k 位后到 (i + k) % n
        tmp[(i + k) % n] = nums[i]
    for i in range(n):
        nums[i] = tmp[i]

    关键公式：新位置 = (原位置 + k) % n

    解法三：环状替换（O(1) 空间，但难理解）

    def rotate_cyclic(nums: list[int], k: int) -> None:
    """时间 O(n)，空间 O(1)"""
    n = len(nums)
    k = k % n
    count = 0  # 已替换的元素个数
    start = 0
    while count < n:
        cur = start
        prev = nums[start]
        while True:
            nxt = (cur + k) % n
            nums[nxt], prev = prev, nums[nxt]
            cur = nxt
            count += 1
            if cur == start:
                break
        start += 1

    核心思想：从位置 0 出发，把元素放到它该去的位置，被挤出来的元素再放到它的目的地，直到回到起点。如果一圈下来没替换完所有元素（当 gcd(n, k) > 1 时会这样），从下一个起点继续。

    为什么会有多圈：当 n 和 k 有公约数时，元素会分成多个独立的环。比如 n=6, k=2，gcd=2，会有 2 个环各 3 个元素。所以要遍历 gcd(n, k) 个起点。

    复杂度对比

    解法时间空间推荐度
    三次翻转O(n)O(1)⭐⭐⭐ 面试首选
    额外数组O(n)O(n)⭐⭐ 容易理解
    环状替换O(n)O(1)⭐ 难理解，面试不推荐

    面试追问

    Q：为什么必须 k = k % n？

    如果 k > n，右轮转 k 次等价于右轮转 k % n 次（每 n 次回到原位）。比如 n=7, k=10，等价于 k=3。如果不取模，k=10 时三次翻转的边界就错了。

    Q：如果题目改成”左轮转 k 位”呢？

    左轮转 k 位 = 右轮转 n-k 位。所以 k = k % n 后改成 rotate(nums, n - k) 即可。或者用三次翻转，但顺序反过来：

    1. 翻转前 k 个
    2. 翻转后 n-k 个
    3. 整体翻转

    Q：能不用 reverse 辅助函数，用 Python 切片吗？

    可以，但切片会创建新列表（不是真正原地）：

    def rotate_slice(nums: list[int], k: int) -> None:
    n = len(nums)
    k = k % n
    nums[:] = nums[-k:] + nums[:-k]  # nums[:] 是原地修改

    nums[:] = ... 是把右边的结果逐个赋值回 nums，看起来像原地，但右边 nums[-k:] + nums[:-k] 创建了新列表，空间是 O(n)。面试时如果要求严格 O(1) 空间，不要用切片。

    Q4: 238. 除自身以外数组的乘积（前后缀乘积）

    题目：返回数组 answer，其中 answer[i] 等于 nums 中除了 nums[i] 之外其余各元素的乘积。不要使用除法，且在 O(n) 时间内完成。

    238. 除自身以外数组的乘积 - 力扣

    示例：

    输入: nums = [1,2,3,4]
输出: [24,12,8,6]
解释:
  answer[0] = 2*3*4 = 24
  answer[1] = 1*3*4 = 12
  answer[2] = 1*2*4 = 8
  answer[3] = 1*2*3 = 6

输入: nums = [-1,1,0,-3,3]
输出: [0,0,9,0,0]

    解法（前后缀乘积，O(1) 额外空间）

    def productExceptSelf(nums: list[int]) -> list[int]:
    """
    时间 O(n)，空间 O(1)（输出数组不算额外空间）
    核心：answer[i] = 左边所有元素的乘积 × 右边所有元素的乘积
    分两趟扫描：先从左到右算左前缀乘积，再从右到左乘上右后缀乘积
    """
    n = len(nums)
    answer = [1] * n

    # 第一趟：从左到右，answer[i] 存"nums[i] 左边所有元素的乘积"
    left_product = 1
    for i in range(n):
        answer[i] = left_product
        left_product *= nums[i]

    # 第二趟：从右到左，把"右边所有元素的乘积"乘上去
    right_product = 1
    for i in range(n - 1, -1, -1):
        answer[i] *= right_product
        right_product *= nums[i]

    return answer

    推导过程

    以 nums = [1, 2, 3, 4] 为例：

    第一趟（从左到右，存左前缀乘积）：

    初始: answer = [1, 1, 1, 1], left_product = 1

i=0: answer[0] = left_product = 1
     left_product *= nums[0] = 1*1 = 1
     answer = [1, 1, 1, 1]

i=1: answer[1] = left_product = 1    ← nums[0]
     left_product *= nums[1] = 1*2 = 2
     answer = [1, 1, 1, 1]

i=2: answer[2] = left_product = 2    ← nums[0]*nums[1]
     left_product *= nums[2] = 2*3 = 6
     answer = [1, 1, 2, 1]

i=3: answer[3] = left_product = 6    ← nums[0]*nums[1]*nums[2]
     left_product *= nums[3] = 6*4 = 24
     answer = [1, 1, 2, 6]

第一趟结束: answer = [1, 1, 2, 6]
           含义: answer[i] = nums[0] * ... * nums[i-1]

    第二趟（从右到左，乘上右后缀乘积）：

    初始: answer = [1, 1, 2, 6], right_product = 1

i=3: answer[3] *= right_product = 6 * 1 = 6   ← 左乘积 × 右乘积(空)
     right_product *= nums[3] = 1*4 = 4
     answer = [1, 1, 2, 6]

i=2: answer[2] *= right_product = 2 * 4 = 8   ← 左乘积(1*2) × 右乘积(4)
     right_product *= nums[2] = 4*3 = 12
     answer = [1, 1, 8, 6]

i=1: answer[1] *= right_product = 1 * 12 = 12 ← 左乘积(1) × 右乘积(3*4)
     right_product *= nums[1] = 12*2 = 24
     answer = [1, 12, 8, 6]

i=0: answer[0] *= right_product = 1 * 24 = 24 ← 左乘积(空) × 右乘积(2*3*4)
     right_product *= nums[0] = 24*1 = 24
     answer = [24, 12, 8, 6]

最终结果: [24, 12, 8, 6] ✓

    直觉理解

    对每个位置 i，answer[i] 就是”它左边所有人的乘积” × “它右边所有人的乘积”。

    nums:      [1,  2,  3,  4]
              ↓ i=2
左边: [1, 2]      右边: [4]
answer[2] = (1*2) * (4) = 8 ✓

    用两个变量 left_product 和 right_product，分别从左右两端扫描，边走边累积乘积，避免开两个额外数组。

    复杂度

     复杂度
    时间O(n)，两趟线性扫描
    空间O(1)（输出数组不算额外空间，题目明确允许）

    面试追问

    Q：为什么不用除法？

    最直观的解法是：先算总乘积 total = nums[0] * nums[1] * ... * nums[n-1]，然后 answer[i] = total / nums[i]。但有两个问题：

    1. 题目明确禁止用除法（考察前后缀技巧）
    2. 如果有 0：nums = [-1,1,0,-3,3]，total = 0，除法全部得 0，但 answer[2] 应该是 (-1)*1*(-3)*3 = 9，不是 0。要处理 0 的话需要单独统计 0 的个数和位置，变复杂了。

    前后缀乘积法天然处理 0，不需要特殊逻辑。

    Q：如果允许用除法，怎么处理 0？

    def product_except_self_div(nums: list[int]) -> list[int]:
    n = len(nums)
    zero_count = nums.count(0)
    if zero_count >= 2:
        # 有两个或以上 0，所有位置都是 0
        return [0] * n
    if zero_count == 1:
        # 只有一个 0，只有那个 0 位置的非零乘积不为 0
        product = 1
        zero_idx = -1
        for i, x in enumerate(nums):
            if x == 0:
                zero_idx = i
            else:
                product *= x
        ans = [0] * n
        ans[zero_idx] = product
        return ans
    # 没有 0，直接除
    total = 1
    for x in nums:
        total *= x
    return [total // x for x in nums]

    可以看出，处理 0 的逻辑很啰嗦，这也是前后缀乘积法更优雅的原因。

    Q：如果数组里只有一个元素，怎么办？

    题目保证 2 <= nums.length，所以不用考虑。如果真要考虑，answer[0] 应该是”空乘积”，按数学约定等于 1。

    Q5: 41. 缺失的第一个正数（原地哈希）

    题目：给你一个未排序的整数数组 nums，找出其中没有出现的最小的正整数。要求时间 O(n) 且只使用常数级额外空间。

    41. 缺失的第一个正数 - 力扣

    示例：

    输入: nums = [1,2,0]
输出: 3
解释: 范围 [1,2] 中的数字都在数组中。

输入: nums = [3,4,-1,1]
输出: 2
解释: 1 在数组中，但 2 没有。

输入: nums = [7,8,9,11,12]
输出: 1
解释: 最小的正数 1 没有出现。

    核心观察（解题钥匙）

    关键认知：长度为 n 的数组，答案一定在 [1, n+1] 之间。

    为什么？

    • 最好情况：数组里恰好是 [1, 2, 3, ..., n]，那答案是 n+1。
    • 其他情况：数组里至少缺一个 [1, n] 范围内的数，答案就是缺失的最小那个。
    • 数组里出现 > n 或 <= 0 的数，对答案没影响——它们不可能是答案（答案 ≤ n+1）。

    所以可以把数组本身当哈希表：让 nums[i] 存值 i+1（即把值 v 放到下标 v-1 的位置）。最后扫一遍，第一个 nums[i] != i+1 的位置，答案就是 i+1。

    解法（原地哈希 / 标记法）

    def firstMissingPositive(nums: list[int]) -> int:
    """
    时间 O(n)，空间 O(1)
    核心：把值 v 放到下标 v-1 的位置（原地哈希）
    """
    n = len(nums)

    # 第一阶段：原地归位
    # 每个值 v（如果 1 <= v <= n）应该放到下标 v-1 的位置
    for i in range(n):
        # 注意是 while 不是 if，因为交换过来的新值可能也要归位
        while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
            # 把 nums[i] 放到它该去的位置 nums[i]-1
            nums[nums[i] - 1], nums[i] = nums[i], nums[nums[i] - 1]

    # 第二阶段：扫描找第一个不在位的
    for i in range(n):
        if nums[i] != i + 1:
            return i + 1

    # 所有 1~n 都在位，答案是 n+1
    return n + 1

    推导过程

    示例 1：nums = [1, 2, 0]，n = 3

    第一阶段（归位）:
i=0: nums[0]=1, 1<=1<=3, 目标位置 nums[0]-1=0, nums[0]==nums[0] → 已在位，跳过
i=1: nums[1]=2, 1<=2<=3, 目标位置 nums[1]-1=1, nums[1]==nums[1] → 已在位，跳过
i=2: nums[2]=0, 不在 [1,3] 范围 → 跳过

归位后: nums = [1, 2, 0]

第二阶段（扫描）:
i=0: nums[0]=1 == 1 ✓
i=1: nums[1]=2 == 2 ✓
i=2: nums[2]=0 != 3 ✗ → 返回 3

答案: 3 ✓

    示例 2：nums = [3, 4, -1, 1]，n = 4

    第一阶段（归位）:
i=0: nums[0]=3, 1<=3<=4, 目标位置 3-1=2, nums[2]=-1 != 3 → 交换
     交换 nums[0] 和 nums[2]: nums = [-1, 4, 3, 1]
     现在 nums[0]=-1, 不在 [1,4] → 跳过

i=1: nums[1]=4, 1<=4<=4, 目标位置 4-1=3, nums[3]=1 != 4 → 交换
     交换 nums[1] 和 nums[3]: nums = [-1, 1, 3, 4]
     现在 nums[1]=1, 1<=1<=4, 目标位置 1-1=0, nums[0]=-1 != 1 → 交换
     交换 nums[1] 和 nums[0]: nums = [1, -1, 3, 4]
     现在 nums[1]=-1, 不在 [1,4] → 跳过

i=2: nums[2]=3, 1<=3<=4, 目标位置 3-1=2, nums[2]==nums[2] → 已在位，跳过

i=3: nums[3]=4, 1<=4<=4, 目标位置 4-1=3, nums[3]==nums[3] → 已在位，跳过

归位后: nums = [1, -1, 3, 4]

第二阶段（扫描）:
i=0: nums[0]=1 == 1 ✓
i=1: nums[1]=-1 != 2 ✗ → 返回 2

答案: 2 ✓

    示例 3：nums = [7, 8, 9, 11, 12]，n = 5

    第一阶段（归位）:
所有元素都 > 5（不在 [1,5] 范围），全部跳过
归位后: nums = [7, 8, 9, 11, 12]（没变）

第二阶段（扫描）:
i=0: nums[0]=7 != 1 ✗ → 返回 1

答案: 1 ✓

    为什么用 while 而不是 if？

    交换过来的新值可能也需要归位。比如 nums = [3, 4, -1, 1]，i=1 时：

    nums[1]=4 → 交换到位置 3 → nums = [-1, 1, 3, 4]
                              ↑ 新值是 1，也要归位！

    如果用 if，只交换一次就走了，新值 1 不会被归位。用 while 持续交换，直到当前位置的值”不需要归位”（不在 [1,n] 范围，或已经在正确位置）。

    为什么是 O(n) 虽然有两层循环？

    均摊分析：每次交换都会让一个元素”归位”（放到它该去的地方）。一旦归位，下次再遇到它就不会再交换（因为 nums[nums[i]-1] != nums[i] 条件不满足）。所以每个元素最多被交换一次，总共最多 n 次交换。外层 for 走 n 次，内层 while 累计执行 ≤ n 次，总体 O(n)。

    复杂度

     复杂度
    时间O(n)（均摊，每元素最多交换一次）
    空间O(1)（原地修改）

    面试追问

    Q：为什么答案一定在 [1, n+1] 之间？

    反证：如果 [1, n] 都在数组里，那数组至少要有 n 个位置存这 n 个数，正好填满。此时答案是 n+1。如果 [1, n] 中有缺失（说明数组里有 <= 0 或 > n 的数占了位置），答案就是缺失的最小正数，一定 ≤ n。所以答案 ∈ [1, n+1]。

    Q：如果不能用”修改原数组”的方式，怎么做？

    那就只能用额外空间。最直接的：把 [1, n] 范围内的数存进哈希集合，然后从 1 开始查：

    def firstMissingPositive_set(nums: list[int]) -> int:
    s = set(nums)
    for i in range(1, len(nums) + 2):
        if i not in s:
            return i

    时间 O(n)，空间 O(n)。不满足题目”常数空间”要求，但能过。题目要求 O(1) 空间就只能用原地哈希。

    Q：能不能用”标记法”（不交换，只改符号）？

    可以，思路类似：

    1. 第一趟：把所有 <= 0 的数改成 n+1（这样它们不影响后续标记）
    2. 第二趟：对每个 |nums[i]|（如果 <= n），把下标 |nums[i]|-1 位置的数改成负数（标记”这个值出现过”）
    3. 第三趟：第一个 nums[i] > 0 的位置 i，答案就是 i+1
    def firstMissingPositive_mark(nums: list[int]) -> int:
    n = len(nums)
    # 第一趟：<= 0 的改成 n+1
    for i in range(n):
        if nums[i] <= 0:
            nums[i] = n + 1
    # 第二趟：标记
    for i in range(n):
        v = abs(nums[i])
        if v <= n:
            nums[v - 1] = -abs(nums[v - 1])
    # 第三趟：找第一个正数
    for i in range(n):
        if nums[i] > 0:
            return i + 1
    return n + 1

    两种方法思路不同但等价，交换法更直观（直接归位），标记法更巧妙（用正负号当标记位）。

    Q：这题为什么是”困难”难度？

    难在两个约束同时满足：

    1. O(n) 时间：排个序就 O(n log n) 了，不行。
    2. O(1) 空间：开哈希表就 O(n) 空间，不行。

    两个约束逼着你想到”把数组本身当哈希表”这个技巧。一旦想到，代码不难写；想不到，就卡死。这是面试中区分”会背模板”和”真正理解”的经典题。

    普通数组类题目通用思路

    模式典型题目核心技巧
    动态规划（Kadane）最大子数组和dp[i] = max(nums[i], dp[i-1]+nums[i])
    排序 + 线性扫描合并区间排序后只看相邻
    三次翻转轮转数组整体翻 + 前段翻 + 后段翻
    前后缀乘积除自身以外数组的乘积左趟累积 × 右趟累积
    原地哈希缺失的第一个正数把值 v 放到下标 v-1

    核心认知：

    1. 数组本身可以当哈希表——下标 i 天然对应值 i+1，这是 O(1) 空间解题的钥匙（41 题、287 寻找重复数都用这个思想）。
    2. 前后缀分解是通用技巧——任何”位置 i 的结果 = 左边某种累积 × 右边某种累积”的问题都能用（238 题、除自身以外的乘积、接雨水的左右最大值）。
    3. 排序能消除”无序”带来的复杂性——很多数组题排序后变成线性扫描（56 合并区间、15 三数之和、18 四数之和）。
    4. 翻转/旋转是原地操作的好朋友——三次翻转实现轮转、反转链表、反转字符串，都是 O(1) 空间。
    5. 动态规划降维——子数组/子序列问题，状态定义成”以 i 结尾”就能线性递推（53 最大子数组和、300 最长递增子序列）。

    遇到数组题的思考顺序：

    1. 能排序吗？排序后问题变简单吗？
   ├─ 能 → 排序 + 线性扫描 / 双指针
   └─ 不能（要保留下标/顺序）→ 转下一步

2. 能用"以 i 结尾"定义状态吗？
   ├─ 能 → 动态规划
   └─ 不能 → 转下一步

3. 位置 i 的结果能拆成"左边 × 右边"吗？
   ├─ 能 → 前后缀分解
   └─ 不能 → 转下一步

4. 答案范围有限吗（比如 [1, n]）？
   ├─ 有限 → 原地哈希
   └─ 无限 → 老实开哈希表 / 其他方法
