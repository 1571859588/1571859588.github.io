    第1章_学习目标与读公式方法

    第1章 学习目标与读公式方法

    来源：AIInfraGuide 第2章 数学基础 第1节 目标读者：高中数学基础，没学过线性代数/微积分/概率论 整理原则：生活直觉 → 数学定义 → 代码验证，公式不跳步

    一句话结论

    AI Infra 的数学不是”算难题”，而是”读公式 → 推 shape → 估代价 → 找风险”这四个动作的肌肉记忆。 你不需要会做高难度的证明题，但必须一眼看出 $XW$ 这个乘法要花多少 FLOPs、输出什么形状、在显存里怎么摆。

    一、核心概念（通俗解释）

    1. 为什么 AI Infra 工程师要会读公式？

    打个比方：你是一个工厂的车间主任（GPU 调度员），工程师（算法研究员）给你一张图纸（公式 $Y = XW$），你要回答四个问题：

    1. 这台机器要加工多大的料？（输入 shape）
    2. 出来多大的成品？（输出 shape）
    3. 要花多少电费、多少工时？（FLOPs、访存量）
    4. 哪一步容易出废品？（数值稳定性、溢出）

    你不需要会设计图纸，但必须会看图纸报预算。 这就是 AI Infra 工程师和算法工程师的分界线。

    2. 这本笔记要教会你的 10 件事

    下面 10 条学习目标，每条后面都加了一句”高中能懂”的解释，让你知道这条到底在学什么。

    #学习目标高中能懂的解释
    1区分标量、向量、矩阵和高阶张量，推导运算后 shape数（一个数字）、一排数（一行数字）、一张表（数字方阵）、一摞表（多维数组），运算后要能说出”出来的是几张表、几行几列”
    2解释点积、范数、矩阵乘法、秩、特征分解与 SVD 的直觉点积=两组数”同向程度”打分；范数=向量的”长度”；矩阵乘法=批量算点积；秩=这张表有多少”独立信息”；SVD=把一张表拆成”三个简单动作”的组合
    3理解 GEMM tiling 的数学等价性GEMM=通用矩阵乘；tiling=把大表切成小块一块块算，结果和一次性算完全一样，只是为了塞进 GPU 的”小胃”（SRAM）
    4计算期望、方差、协方差，理解概率分布与条件概率期望=长期平均值（掷骰子 3.5）；方差=波动大小；协方差=两个量”同涨同跌”还是”你涨我跌”；条件概率=”已知 A 发生时 B 的概率”（雨天带伞的概率）
    5从 logits 得到稳定 Softmax，理解温度、交叉熵、熵和 KL 散度logits=模型给每个词打的”裸分”；Softmax=把裸分变成概率（加起来=1）；温度=调”软硬”；交叉熵=预测概率和真实答案差多远；熵=分布有多”不确定”；KL=两个分布差多远
    6用链式法则读懂计算图，手推梯度链式法则=”复合函数一层层求导”（y=f(g(x))，先对 g 求再对 x 求）；计算图=把公式画成流程图；梯度=每个参数该往哪调、调多少
    7解释梯度消失/爆炸、残差连接和归一化的影响梯度消失=信号传着传着没了（像传话游戏越传越听不见）；梯度爆炸=信号越传越大（像雪崩）；残差连接=给信号开一条”高速旁路”绕过衰减；归一化=把每一层的输出”拉回正常范围”防失控
    8区分 FP32、TF32、FP16、BF16 与 FP8不同精度=小数点后留几位的问题。FP32=标准款（32位）；TF32=FP32 的”截短版”专门给矩阵乘用；FP16=半精度（容易溢出）；BF16=半精度但范围大（不容易溢出，但精度低）；FP8=极致省显存（新一代卡才支持）
    9识别溢出、下溢、灾难性消减等数值风险溢出=数太大装不下（爆了）；下溢=数太小被当成 0（没了）；灾难性消减=两个接近的大数相减，有效数字丢光（10.0001 - 10.0000 = 0.0001，但本来可能更精确）
    10估算参数量、激活量、FLOPs、访存量与算术强度参数量=模型存了多少”知识”；激活量=中间结果多大；FLOPs=算多少次浮点运算；访存量=从显存读了多少字节；算术强度=FLOPs÷访存量，决定你是”算力瓶颈”还是”带宽瓶颈”

    学习建议：第 1、4、5、8、10 条是”看完就能用”的，优先掌握；第 2、3、6、7、9 条需要配合后面章节的代码理解，别急。

    三、读公式的三个问题（含完整例子）

    这是全章最核心的部分。任何公式拿来，问这三个问题，你就知道它在工程上要做什么。

    3.1 三个问题是什么？

    以语言模型输出投影为例：

    正式数学公式（LaTeX）：

    \[\mathbf{Y} = \mathbf{X}\mathbf{W}, \qquad \mathbf{X} \in \mathbb{R}^{(BS) \times H},\quad \mathbf{W} \in \mathbb{R}^{H \times V}\]

    纯文本对照：

    Y = XW
其中 X ∈ R^((B·S) × H),  W ∈ R^(H × V),  Y ∈ R^((B·S) × V)

    问题 1：形状是否匹配？

    • 看乘号两边的”内维”是否相同：$X$ 的列数 $H$ 必须 == $W$ 的行数 $H$。
    • 输出形状 = 两边的”外维”拼起来：$(B \cdot S) \times V$。

    问题 2：代价是多少？

    • 矩阵乘 $(m \times k) @ (k \times n)$ 的 FLOPs ≈ $2 \cdot m \cdot k \cdot n$（乘法和加法各算一次，所以乘 2）。
    • 这里 $m = B \cdot S$, $k = H$, $n = V$，所以 $\text{FLOPs} \approx 2 \cdot B \cdot S \cdot H \cdot V$。

    正式公式（对应网页）：

    \[2BSHV \quad \text{FLOPs}\]

    这里把一次乘法和一次加法各计一个浮点操作。

    问题 3：实现风险是什么？

    • 数据类型：FP16 算会不会溢出？要不要 BF16？
    • 内存布局：X 是 row-major 还是 column-major？影响访存效率。
    • 分块：矩阵太大塞不进 SRAM，要不要 tiling？
    • 并行策略：在 B 维切还是 S 维切？（数据并行 vs 序列并行）
    • 数值稳定性：要不要先减最大值再做 Softmax？

    3.2 端到端例子：把一个公式从头算到尾

    设定：一个 LLM 的最后一层（输出投影层 / lm_head），输入是一批 token 的隐藏向量，输出是每个 token 在词表上的得分。

    实际数字：B=8（batch size），S=2048（序列长度），H=4096（隐藏维度），V=32000（词表大小）。

    第一步：读公式

    Y = XW
X ∈ R^((B·S) × H)   = R^(16384 × 4096)     # B·S = 8·2048 = 16384
W ∈ R^(H × V)       = R^(4096 × 32000)
Y ∈ R^((B·S) × V)   = R^(16384 × 32000)

    为什么 X 是 (B·S) × H 而不是 B × S × H？ 因为矩阵乘法只认二维。我们把 B 个序列、每个序列 S 个 token”摊平”成 B·S 行，每行是一个 token 的 H 维向量。这是工程上的常见操作（view/reshape）。

    第二步：推 shape（逐个对齐）

            X         @        W         =        Y
形状: (16384 × 4096)   (4096 × 32000)   (16384 × 32000)
            └─内维 4096 == 4096 ─┘
       └── 外维 ──┘                └── 外维 ──┘
       16384                          16384 和 32000

    判断：内维都是 4096，✅ 匹配。输出是 (16384, 32000)。

    如果内维不等会怎样？ 直接报错 RuntimeError: mat1 and mat2 shapes cannot be multiplied。这是新手最常见的 bug。

    第三步：估 FLOPs

    FLOPs ≈ 2 × m × k × n
      = 2 × (B·S) × H × V
      = 2 × 16384 × 4096 × 32000
      = 2 × 16384 × 4096 × 32000

    逐项算：

    16384 × 4096 = 67,108,864            ≈ 6.7 × 10^7
67,108,864 × 32000 = 2,147,483,648,000  ≈ 2.15 × 10^12
再 × 2 = 4,294,967,296,000            ≈ 4.29 × 10^12 FLOPs
即约 4.29 TFLOPs（一次前向）

    换算：1 TFLOPs = 10^12 次浮点运算。A100 显卡 FP16 算力约 312 TFLOPS，所以这一层理论上要花 4.29 / 312 ≈ 0.014 秒（理论值，实际受带宽和其他因素影响）。

    第四步：估显存（参数 + 激活）

    参数量（W 的大小）：

    W 有 H × V = 4096 × 32000 = 131,072,000 ≈ 1.31 亿参数
FP16 下显存 = 131,072,000 × 2 字节 ≈ 262 MB

    激活量（X 和 Y 的大小）：

    X: B·S × H = 16384 × 4096 = 67,108,864 个数
   FP16 显存 ≈ 67,108,864 × 2 = 134 MB
Y: B·S × V = 16384 × 32000 = 524,288,000 个数
   FP16 显存 ≈ 524,288,000 × 2 ≈ 1 GB  ← 注意 Y 巨大！

    工程洞察：输出 Y 占 1GB，比参数 W 还大！这就是为什么大词表场景要做 vocab parallel（把 V 切到多卡）或 lazy softmax（不全算）。

    第五步：找实现风险

    风险点具体表现缓解方法
    数据类型FP16 下 4096×32000 累加容易溢出用 BF16 或 FP32 累加
    显存Y 占 1GB，多卡每卡都存一份太浪费在 V 维切（张量并行）
    带宽W 262MB 每次都要从 HBM 读kernel fusion，把 Y 直接喂给 Softmax
    数值稳定Y 直接做 Softmax 会溢出先减行最大值再 exp

    第六步：代码验证（NumPy 从零 + PyTorch 工程用法）

    # ============ NumPy 从零验证 ============
import numpy as np

B, S, H, V = 8, 2048, 4096, 32000

# 造数据
X = np.random.randn(B * S, H).astype(np.float16)   # (16384, 4096)
W = np.random.randn(H, V).astype(np.float16)       # (4096, 32000)

# 推 shape
print(X.shape, W.shape)        # (16384, 4096) (4096, 32000)
assert X.shape[1] == W.shape[0], "内维不匹配！"

# 算
Y = X @ W                       # (16384, 32000)
print(Y.shape)                  # (16384, 32000) ✅

# 估 FLOPs
flops = 2 * (B * S) * H * V
print(f"FLOPs = {flops:,}")     # 4,294,967,296,000 ≈ 4.29 TFLOPs

# 估显存（FP16，2 字节）
param_bytes = H * V * 2
act_x_bytes = (B * S) * H * 2
act_y_bytes = (B * S) * V * 2
print(f"参数 W: {param_bytes / 1e6:.1f} MB")    # 262.1 MB
print(f"激活 X: {act_x_bytes / 1e6:.1f} MB")    # 134.2 MB
print(f"激活 Y: {act_y_bytes / 1e6:.1f} MB")    # 1048.6 MB ≈ 1 GB

    # ============ PyTorch 工程用法 ============
import torch
import torch.nn as nn

B, S, H, V = 8, 2048, 4096, 32000

# 工程上一般这么写
class LMHead(nn.Module):
    def __init__(self, H, V):
        super().__init__()
        self.proj = nn.Linear(H, V, bias=False)   # 内部就是 Y = XW^T
    def forward(self, x):                          # x: (B, S, H)
        return self.proj(x)                        # -> (B, S, V)

# 实际跑一次
x = torch.randn(B, S, H, device='cuda', dtype=torch.bfloat16)
model = LMHead(H, V).cuda().bfloat16()

# 推 shape（PyTorch 自动处理 batch 维，不用摊平）
y = model(x)
print(y.shape)   # torch.Size([8, 2048, 32000])

# 估参数量
n_params = sum(p.numel() for p in model.parameters())
print(f"参数量 = {n_params:,}")   # 131,072,000

# 估 FLOPs（用公式对一下）
flops = 2 * B * S * H * V
print(f"FLOPs = {flops:,}")       # 4,294,967,296,000

    新手常见困惑：为什么 PyTorch 里 nn.Linear 的权重是 (V, H) 而不是 (H, V)？ 答：因为 nn.Linear 内部做的是 x @ W^T，权重存的是转置后的版本。数学公式和代码实现差一个转置，这是面试常考点。

    3.3 读公式习惯清单

    养成下面 4 步肌肉记忆：

    1. 一眼找形状：扫一眼公式里所有张量，写出它们的 shape，看内维是否相等。
    2. 二眼估代价：乘法用 $2 \cdot m \cdot k \cdot n$，加法用元素个数，访存用字节数 ×数据类型大小。
    3. 三眼找风险：数据类型、布局、分块、并行、数值稳定性——按这五条过一遍。
    4. 四眼写代码：用 NumPy 验证 shape 和数值，用 PyTorch 验证工程可行性。

    四、深度学习常用记法

    这套字母是深度学习论文/代码的”通用缩写”，记住它们才能读懂论文里的 shape 推导。

    4.1 核心字母表

    字母全称含义高中类比代码示例
    BBatch size一次喂给模型多少个样本一次烤 8 个蛋糕x.shape = (8, ...)
    SSequence length序列长度（多少个 token）一句话有多少个字x.shape = (..., 2048, ...)
    HHidden size隐藏维度（每个 token 的向量有多长）每个字用多少个数字描述x.shape = (..., 4096)
    VVocabulary size词表大小（能输出的词有多少个）字典有多少个字logits.shape = (..., 32000)
    NhNumber of heads注意力头数（把 H 拆几路并行看）8 个翻译员同时看一段话n_head = 32
    DhHead dim每个 head 的维度 = H / Nh每个翻译员分到多少信息Dh = 4096 / 32 = 128

    4.2 常见组合

    组合含义出现场景
    B·Sbatch 摊平后的 token 总数矩阵乘法前要 reshape
    B·S·H一层的激活总元素数估激活显存
    Nh·Dh= $H$（必须相等，否则拆不开）多头注意力的形状约束
    B·S·V输出 logits 总元素数（巨大！）大词表是显存大户

    4.3 典型 shape 速查（以 LLaMA-7B 配置为例）

    B = 8          # batch size
S = 2048       # 序列长度
H = 4096       # 隐藏维度
V = 32000      # 词表
Nh = 32        # 注意力头数
Dh = H / Nh = 128   # 每个 head 维度
    张量shape元素数FP16 显存
    输入 embedding(B, S, H)8·2048·4096 = 67M134 MB
    Q/K/V 投影权重(H, H) ×33·4096² = 50M100 MB
    注意力分数（单 head）(B, Nh, S, S)8·32·2048·2048 = 1.07B2.1 GB ⚠️
    输出投影权重(H, V)4096·32000 = 131M262 MB
    输出 logits(B, S, V)8·2048·32000 = 524M1 GB ⚠️

    两个显存大户：注意力分数（S² 太大）和 logits（V 太大）。所有推理优化都在盯这两个。

    4.4 代码：把字母表跑一遍

    import torch

B, S, H, V, Nh = 8, 2048, 4096, 32000, 32
Dh = H // Nh
assert Nh * Dh == H, "head 必须能整除 hidden"

# 输入：(B, S, H)
x = torch.randn(B, S, H)
print(f"输入 x: {x.shape}")              # (8, 2048, 4096)

# QKV 投影后：(B, S, H) -> 拆成 (B, Nh, S, Dh)
q = x.view(B, S, Nh, Dh).transpose(1, 2)
print(f"Q: {q.shape}")                   # (8, 32, 2048, 128)

# 注意力分数：(B, Nh, S, S)  ← 这里 S² 很大
attn = q @ q.transpose(-2, -1)
print(f"attn scores: {attn.shape}")      # (8, 32, 2048, 2048)
print(f"attn 元素数: {attn.numel():,}")  # 1,073,741,824 ≈ 10 亿

# 输出投影：logits (B, S, V)
W_out = torch.randn(H, V)
logits = x @ W_out
print(f"logits: {logits.shape}")         # (8, 2048, 32000)
print(f"logits 元素数: {logits.numel():,}")  # 524,288,000 ≈ 5 亿

    五、面试回答

    Q1：你读一个新公式时，会怎么分析？

    参考回答： “我会按三个问题来：第一看形状，把公式里每个张量的 shape 写出来，确认内维匹配、输出 shape 正确；第二估代价，用 $2 \cdot m \cdot k \cdot n$ 估 FLOPs，用元素数 ×数据类型字节数估显存和访存；第三找风险，按数据类型、内存布局、分块策略、并行方式、数值稳定性这五条过一遍。比如 LLM 的输出投影 $Y=XW$，B=8、S=2048、H=4096、V=32000 时，FLOPs 约 4.29T，参数 262MB，但输出 logits 就要 1GB——所以工程上要考虑 vocab 并行。”

    Q2：B、S、H、V、Nh、Dh 之间什么关系？

    参考回答： “B 是 batch size，S 是序列长度，H 是隐藏维度，V 是词表大小。多头注意力里 $H = N_h \cdot D_h$，即总维度 = 头数 × 每头维度。比如 H=4096、Nh=32，则 Dh=128。这个等式是注意力 reshape 的硬约束，不整除会直接报错。”

    Q3：为什么矩阵乘法的 FLOPs 是 $2 \cdot m \cdot k \cdot n$？

    参考回答： “矩阵乘 $(m \times k) @ (k \times n)$ 的结果有 $m \cdot n$ 个元素，每个元素是 k 次乘加。一次乘 + 一次加算 2 FLOPs，所以总共 $m \cdot n \cdot k \cdot 2$。工程上有时把乘加合并叫 1 FMA（fused multiply-add），那 FLOPs 就写成 $m \cdot n \cdot k$ FMA。面试时说清楚自己用的是哪种口径即可。”

    Q4：标量、向量、矩阵、张量在代码里怎么区分？

    参考回答： “看 ndim（维度数）：标量 ndim=0（Python 的 float 或 torch.tensor(3.14)），向量 ndim=1、shape (n,)，矩阵 ndim=2、shape (m,n)，3 阶及以上张量 ndim≥3。数学上写法也不同：标量小写斜体 $x$，向量粗体小写 $\mathbf{x}$，矩阵/张量粗体大写 $\mathbf{X}$。”

    六、易混淆点

    6.1 @ vs *（矩阵乘 vs 逐元素乘）

    A = np.array([[1, 2], [3, 4]])
B = np.array([[5, 6], [7, 8]])

A @ B   # 矩阵乘 = [[19,22],[43,50]]
A * B   # 逐元素乘 = [[5,12],[21,32]]

    口诀：@ 是”行乘列”（矩阵乘，有 shape 约束），* 是”格子对格子”（逐元素，要求同形）。数学符号 $\odot$ 对应 *，不是 @。

    6.2 (n,) 和 (1,n) 不一样

    a = np.array([1, 2, 3])       # shape (3,)  ← 一维向量
b = np.array([[1, 2, 3]])     # shape (1,3) ← 二维矩阵（1 行 3 列）

    向量没有”行/列”之分，只有”长度”。要当行向量或列向量，得显式升一维。新手最容易在这里报 shape 错误。

    6.3 nn.Linear 的权重是转置存的

    import torch.nn as nn
lin = nn.Linear(4096, 32000, bias=False)
print(lin.weight.shape)   # (32000, 4096) ← 注意是 (out, in)，不是 (in, out)！

    数学公式写 $Y = XW$，$W$ 是 $(H, V)$；但 PyTorch 内部存的是 $(V, H)$，forward 时算 X @ W.T。差一个转置，面试常考。

    6.4 FLOPs 和 FLOPS 不是一回事

    写法含义单位
    FLOPs（小写 s）总运算次数（一个量）次
    FLOPS（大写 S）每秒运算次数（一个速率）次/秒

    “这次前向花了 4.29 TFLOPs”（总运算量）vs “A100 算力 312 TFLOPS”（每秒能算多少）。差一个大小写，含义完全不同。

    6.5 “形状匹配” ≠ “语义正确”

    # 数学上能乘，但语义可能错
X = np.random.randn(8, 4096)      # 想表示 (B, H)，B=8
W = np.random.randn(4096, 32000)  # (H, V)
Y = X @ W                          # 能算，shape (8, 32000)
# 但如果其实 X 是 (B·S, H) 被错误 reshape 成 (B, H) 了，
# 结果虽然能算出来，但完全没意义！

    教训：shape 匹配只是必要条件，不是充分条件。还要确认”语义”对——这个张量在业务上到底代表什么。

    6.6 期望/方差/梯度的”维度”

    概念输入输出
    期望 $\mathbb{E}[X]$一组数一个数（标量）
    方差 $\operatorname{Var}(X)$一组数一个数（标量）
    梯度 $\nabla_x f$向量 x和 x 同形的向量

    梯度不是”一个数”，而是”和参数同形的向量”——每个参数都有自己的梯度。新手常把”梯度”想成一个数，导致理解反向传播时卡住。

    七、本章自测题

    1. X shape (8, 2048, 4096)，W shape (4096, 32000)，要怎么乘？输出什么 shape？
    2. 上题如果用 FP16，参数和输出各占多少显存？
    3. nn.Linear(4096, 32000) 的 weight.shape 是什么？为什么？
    4. H=4096、Nh=32，Dh 是多少？注意力分数张量（B=8, S=2048）有多少元素？
    5. @ 和 * 的区别？数学符号 $\odot$ 对应哪个？
    参考答案 1. 先把 X reshape 成 `(8·2048, 4096) = (16384, 4096)`，再 `@ W`，输出 `(16384, 32000)`，再 reshape 回 `(8, 2048, 32000)`。 2. 参数 `4096·32000·2 = 262 MB`；输出 `16384·32000·2 ≈ 1 GB`。 3. `(32000, 4096)`。因为 `nn.Linear` 内部做 `x @ W.T`，权重存转置版本。 4. `Dh = 128`。注意力分数 `(8, 32, 2048, 2048)`，元素数 `8·32·2048·2048 = 1,073,741,824 ≈ 10.7 亿`。 5. `@` 是矩阵乘（行乘列），`*` 是逐元素乘（格子对格子）。$\odot$ 对应 `*`。
