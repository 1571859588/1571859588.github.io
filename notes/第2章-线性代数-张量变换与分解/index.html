    第2章_线性代数_张量变换与分解

    第2章 线性代数：张量变换与分解

    来源：AIInfraGuide 第2章 数学基础 第2节 目标读者：高中数学基础，没学过线性代数 整理原则：生活直觉 → 数学定义 → 公式不跳步 → 代码验证（NumPy + PyTorch 双轨） 阅读建议：每节先读”直觉”，再读”定义”，最后跑代码。代码可以复制直接运行。

    一句话结论

    线性代数研究的就是”一摞数”怎么摆、怎么搬、怎么乘、怎么拆。 在 AI Infra 里，它具体化为四件事：

    1. 摆（shape/stride）：数据在内存里怎么排，索引怎么算
    2. 搬（广播/转置/reshape）：不改数据，只改”看法”
    3. 乘（矩阵乘/BMM）：所有大模型算力的 90% 都花在这里
    4. 拆（SVD/低秩/LoRA）：把一个大矩阵拆成两个小的，省显存、省训练参数

    你不需要会证任何定理，但必须会：推 shape、估 FLOPs、看出广播 bug、算 LoRA 参数量。 这四件事是 AI Infra 工程师面试的核心考点。

    2.1 标量、向量、矩阵和张量

    直觉

    打比方，你要记录气温：

    概念生活类比例子
    标量 scalar一个温度值“今天 25℃” → 一个数
    向量 vector一周的温度表[周一 22, 周二 23, …] → 一排数
    矩阵 matrix一年 12 月 × 31 日的温度表一张二维表
    张量 tensor多个年份的这种表一摞表，可以无限摞下去

    关键认知：张量不是什么神秘东西，它就是”多维数组”。标量是 0 阶张量，向量是 1 阶，矩阵是 2 阶，再往上都叫张量。NumPy 的 ndarray 和 PyTorch 的 Tensor 本质上都是张量。

    定义

    数学对象记号维度（阶）shape 例子含义
    标量$x \in \mathbb{R}$0 阶()一个实数
    向量$\mathbf{x} \in \mathbb{R}^H$1 阶(H,)H 个实数排成一排
    矩阵$\mathbf{W} \in \mathbb{R}^{H \times 4H}$2 阶(H, 4H)H 行 4H 列的表
    张量$\mathbf{X} \in \mathbb{R}^{B \times S \times H}$3 阶(B, S, H)B 个 S×H 的表摞起来

    AI 里的实际例子：

    • 标量：学习率 $\eta = 10^{-4}$
    • 向量：一个 token 的隐藏状态 $\mathbf{x} \in \mathbb{R}^H$（H=4096）
    • 矩阵：线性层权重 $\mathbf{W} \in \mathbb{R}^{H \times 4H}$（FFN 第一层）
    • 张量：一个 batch 的隐藏状态 $\mathbf{X} \in \mathbb{R}^{B \times S \times H}$（B=8, S=2048, H=4096）

    工程中张量的完整信息（重点！）

    这是原稿缺的，必须补：一个张量在内存里不是”凭空存在”的，它由 6 个属性 完全决定：

    张量 = 数据地址(data) + 形状(shape) + 步幅(stride) + 数据类型(dtype) + 设备(device) + 布局(layout)
    属性含义例子
    data内存起始地址（指针）0x7f3a2c000000
    shape每个维度有多大(2, 3)
    stride每个维度走一步要跳几个元素(3, 1)
    dtype每个元素占多少字节、什么格式float32（4 字节）
    device数据躺在哪个硬件上cpu / cuda:0
    layout内存排列方式row-major（行主序，默认）

    为什么这 6 个都重要？

    • data + stride + shape 决定”怎么从内存取数”（下一节详讲）
    • dtype 决定”占多少显存”：FP32 是 FP16 的两倍
    • device 决定”能不能直接运算”：CPU 张量和 GPU 张量相加会报错
    • layout 决定”访存效率”：row-major 对行切片快，col-major 对列切片快

    NumPy 代码：把 6 个属性打印出来

    import numpy as np

# 造一个 (2, 3) 的 float32 矩阵
arr = np.array([[1, 2, 3],
                [4, 5, 6]], dtype=np.float32)

print("值:\n", arr)
print("shape    :", arr.shape)        # (2, 3)      —— 2 行 3 列
print("strides  :", arr.strides)      # (12, 4)     —— 行步幅 12 字节，列步幅 4 字节
print("dtype    :", arr.dtype)        # float32
print("ndim     :", arr.ndim)         # 2           —— 阶数
print("size     :", arr.size)         # 6           —— 总元素数
print("itemsize :", arr.itemsize)     # 4           —— 每个元素 4 字节
print("nbytes   :", arr.nbytes)       # 24          —— 总字节数 = 6 × 4

    预期输出：

    值:
 [[1. 2. 3.]
 [4. 5. 6.]]
shape    : (2, 3)
strides  : (12, 4)
dtype    : float32
ndim     : 2
size     : 6
itemsize : 4
nbytes   : 24

    注意 strides 是字节数，不是元素数：float32 一个元素 4 字节，所以行步幅 = 3 个元素 × 4 字节 = 12 字节；列步幅 = 1 个元素 × 4 字节 = 4 字节。这是新手最容易搞混的点。

    PyTorch 用法

    import torch

t = torch.tensor([[1, 2, 3],
                  [4, 5, 6]], dtype=torch.float32, device='cpu')

print("shape    :", t.shape)          # torch.Size([2, 3])
print("stride   :", t.stride())       # (3, 1)      —— 注意：PyTorch 返回的是"元素数"不是字节
print("dtype    :", t.dtype)          # torch.float32
print("device   :", t.device)         # cpu
print("layout   :", t.layout)         # torch.strided
print("element_size:", t.element_size())  # 4 字节
print("numel    :", t.numel())        # 6

    NumPy vs PyTorch 的 stride 区别：NumPy 的 arr.strides 单位是字节，PyTorch 的 t.stride() 单位是元素数。这是两边 API 最容易踩的坑。

    易混淆点

    1. 向量 shape 是 (H,) 不是 (H, 1)：前者是 1 维数组，后者是 2 维矩阵（只有 1 列）。在矩阵乘法里两者行为完全不同。
    2. 标量不是 0 维向量的”特殊情况”：x = np.array(3.14) 的 shape 是 ()，x.ndim == 0，和 np.array([3.14]) 完全不同。
    3. “张量”在物理/数学里和工程里含义不同：物理里的张量要满足坐标变换规则，工程里就是”多维数组”。AI 里都是后者。

    2.2 Shape、索引和 stride（重点中的重点）

    直觉

    stride 回答一个问题：”要从内存里取第 i 行第 j 列的元素，要跳过多少个元素？”

    打个比方：一栋楼有 3 层，每层 4 个房间。房间号从 0 开始连续编：0,1,2,3（1楼），4,5,6,7（2楼），8,9,10,11（3楼）。

    • 要找”第 2 层第 3 个房间”（0-indexed：i=2, j=3），房间号 = 2×4 + 3 = 11。
    • 这里”每层 4 个房间”就是行步幅 = 4，”同一层内隔壁房间”就是列步幅 = 1。

    stride 就是”每个维度走一步要跳多少个元素”。

    定义

    对一个 shape 为 (d0, d1, ..., dn) 的行主序张量：

    • stride[n-1] = 1（最后一维连续存放）
    • stride[i] = stride[i+1] × d[i+1]（前一维的步幅 = 后一维步幅 × 后一维大小）

    偏移公式（核心！）：要取元素 X[i0, i1, ..., in-1]，它在内存里的位置是：

    offset = i0 × stride[0] + i1 × stride[1] + ... + in-1 × stride[n-1]

    公式不跳步：2×3 矩阵的完整 stride 推导

    设 X = [[1, 2, 3], [4, 5, 6]]，shape = (2, 3)，行主序存储。

    第一步：底层存储（内存里就是一排）

    内存索引:  0   1   2   3   4   5
值:        1   2   3   4   5   6
对应位置: (0,0)(0,1)(0,2)(1,0)(1,1)(1,2)

    第二步：算 stride

    • stride[1]（列方向，最后一维）= 1（同一行里相邻元素挨着）
    • stride[0]（行方向）= stride[1] × d1 = 1 × 3 = 3（跳到下一行要跨 3 个元素）

    所以 stride = (3, 1)。

    第三步：用偏移公式验证

    取 X[1, 2]（第 1 行第 2 列，即数字 6）：

    offset = 1 × stride[0] + 2 × stride[1]
       = 1 × 3 + 2 × 1
       = 3 + 2
       = 5

    内存索引 5 的值是 6 ✅。

    第四步：所有 6 个元素都验证一遍

    索引 (i,j)偏移 = 3i + j内存值
    (0,0)0+0=01 ✅
    (0,1)0+1=12 ✅
    (0,2)0+2=23 ✅
    (1,0)3+0=34 ✅
    (1,1)3+1=45 ✅
    (1,2)3+2=56 ✅

    NumPy 代码：亲手验证

    import numpy as np

X = np.array([[1, 2, 3],
              [4, 5, 6]], dtype=np.float32)

print("shape  :", X.shape)        # (2, 3)
print("strides:", X.strides)      # (12, 4)  —— 字节！12 = 3元素×4字节, 4 = 1元素×4字节
print("flags  :\n", X.flags)      # 看 C_CONTIGUOUS 为 True（行主序连续）

# 手动用偏移公式取 X[1, 2]
i, j = 1, 2
stride_elements = (X.strides[0] // X.itemsize, X.strides[1] // X.itemsize)  # 转成元素数
offset = i * stride_elements[0] + j * stride_elements[1]
print("手动取 X[1,2] =", X.ravel()[offset])  # 6.0
print("直接取 X[1,2] =", X[1, 2])            # 6.0

    转置：stride 的魔法（不复制数据！）

    这是 stride 最强大的地方：转置一个矩阵，根本不用搬数据，只要把 stride 和 shape 调一下。

    X = np.array([[1, 2, 3],
              [4, 5, 6]], dtype=np.float32)
# 原来：shape=(2,3), strides=(12, 4)  即元素 stride (3, 1)

Xt = X.T
print("Xt shape  :", Xt.shape)     # (3, 2)
print("Xt strides:", Xt.strides)   # (4, 12)  —— 元素 stride 变成 (1, 3)！
print("Xt:\n", Xt)
# [[1. 4.]
#  [2. 5.]
#  [3. 6.]]

# 关键：底层数据没动！
print("X.data 地址 和 Xt.data 地址相同？", X.ctypes.data == Xt.ctypes.data)  # True

    原理：

    • 原来 X[i, j] 用 stride (3, 1) 取
    • 转置后 Xt[i, j] = X[j, i]，相当于用 stride (1, 3) 取（两个维度对调）
    • 数据还是 [1,2,3,4,5,6]，只是”看法”变了

    这就是为什么 .T 是 O(1) 操作——不复制数据，只改 metadata。同理 reshape、expand_dims、slice 很多时候也是 O(1) 的（叫 “view”）。

    PyTorch 用法

    import torch

X = torch.tensor([[1, 2, 3],
                  [4, 5, 6]], dtype=torch.float32)

print("shape  :", X.shape)        # torch.Size([2, 3])
print("stride :", X.stride())     # (3, 1)  —— 元素数！
print("X[1,2] :", X[1, 2])        # tensor(6.)

Xt = X.T
print("Xt shape:", Xt.shape)      # torch.Size([3, 2])
print("Xt stride:", Xt.stride())  # (1, 3)
print("Xt.data_ptr() == X.data_ptr():", Xt.data_ptr() == X.data_ptr())  # True

# 一个常见操作：判断张量是否连续
print("X.is_contiguous()  :", X.is_contiguous())   # True
print("Xt.is_contiguous() :", Xt.is_contiguous())  # False（转置后不连续了）

    is_contiguous() 是什么意思？ 行主序下，”连续”意味着沿最后一维走一步正好跳 1 个元素。转置后 Xt 的最后一维是原来的行方向，stride=3，所以不连续了。不连续的张量在某些操作（如 view）前需要先 .contiguous() 复制成连续的。

    易混淆点

    问题答案
    NumPy 的 strides 和 PyTorch 的 stride() 单位一样吗？不一样。NumPy 是字节，PyTorch 是元素数。
    转置会复制数据吗？不会，只改 shape 和 stride（view 操作）。
    reshape 一定不复制数据吗？不一定。能”凑出”新 shape 的 stride 就不复制（view），否则复制（copy）。
    为什么转置后张量”不连续”了？因为行主序要求最后一维 stride=1，转置后最后一维 stride≠1。

    2.3 逐元素运算与广播

    直觉

    逐元素运算：两张表”对应格子”做运算，像 Excel 里拖公式。

    广播（broadcasting）：当两张表 shape 不一样时，”小的那张”自动复制成”大的那张”的形状，再逐元素算。像 Excel 里”每个格子都乘一个常数”——常数被”广播”到每个格子。

    定义

    逐元素加法：C[i,j] = A[i,j] + B[i,j]（要求 A 和 B 同 shape，或者能广播到同 shape）

    广播规则（从尾部对齐，逐维判断）：

    对每一对维度，必须满足以下三个条件之一：

    1. 两个维度大小相等，或
    2. 其中一个维度大小为 1，或
    3. 其中一个张量根本不存在这个维度（即维度数少）

    满足后，大小为 1 的那个维度会被”复制”成另一边的大小。

    公式不跳步：广播怎么扩

    例：A.shape = (3, 1), B.shape = (1, 4)，求 A + B 的 shape。

    A: (3, 1)
B: (1, 4)
对齐:
  维度 -1（最后）: A=1, B=4 → 取 max(1,4)=4
  维度 -2:        A=3, B=1 → 取 max(3,1)=3
结果 shape: (3, 4)

    广播后等价于：

    A 被复制成 (3, 4):  每列都是原来的 (3,1)
B 被复制成 (3, 4):  每行都是原来的 (1,4)
然后逐元素相加

    4 个广播例子（含正确和错误，重点！）

    这是原稿缺的，必须补：

    import numpy as np

# ===== 例 1: 正确广播——标量加矩阵 =====
A = np.array([[1, 2, 3],
              [4, 5, 6]])       # shape (2, 3)
b = 10                          # 标量 shape ()
print("例1:", A + b)
# [[11 12 13]
#  [14 15 16]]
# 解释：标量广播到 (2,3)，每个元素 +10

# ===== 例 2: 正确广播——列向量加行向量 → 矩阵 =====
col = np.array([[1],            # shape (3, 1)
                [2],
                [3]])
row = np.array([10, 20, 30])    # shape (3,)  → 视为 (1, 3)
print("例2:\n", col + row)
# [[11 21 31]
#  [12 22 32]
#  [13 23 33]]
# 解释：(3,1) + (1,3) → (3,3)。这就是"外积"的效果

# ===== 例 3: 错误广播——形状完全不匹配 =====
A = np.zeros((2, 3))
B = np.ones((4, 5))
try:
    A + B
except Exception as e:
    print("例3 报错:", e)
# operands could not be broadcast together with shapes (2,3) (4,5)
# 解释：最后维 3 vs 5 不等、都不是 1，直接报错

# ===== 例 4: 危险！能广播但语义错了 =====
# 场景：有 3 个样本，每个样本 4 个特征
X = np.random.randn(3, 4)       # (3 个样本, 4 个特征)
mean_per_feature = np.random.randn(3)  # 想减去每个特征的均值，应该是 (4,)

# 程序员写错了，算成了 (3,) 的"每样本均值"
# 现在尝试 X - mean_per_feature：
try:
    result = X - mean_per_feature  # (3,4) - (3,) → 广播！
    print("例4 shape:", result.shape)  # (3, 4) —— 不报错！
except Exception as e:
    print("例4 报错:", e)
# 解释：(3,4) 和 (3,) 对齐：
#   最后维: 4 vs 3 → 不等且都不是 1 → 应该报错！

    等等，例 4 这里 (3,4) - (3,) 实际上会报错（最后维 4 vs 3 不匹配）。让我重新构造一个真正”能广播但语义错”的例子：

    # ===== 例 4 修正: 能广播但语义错了（最隐蔽的 bug）=====
# 场景：4 个样本，每个 3 个特征
X = np.random.randn(4, 3)            # (4 样本, 3 特征)
# 想算：每个样本减去"每个特征的全局均值"
# 正确：mean 应该是 (3,) 即每个特征一个均值
correct_mean = X.mean(axis=0)        # shape (3,)  ✓
correct = X - correct_mean           # (4,3) - (3,) → 广播成 (4,3)，每行减同一个均值

# 程序员手滑算成了 axis=1，得到 (4,) 即每个样本一个均值
wrong_mean = X.mean(axis=1)          # shape (4,)
# X - wrong_mean： (4,3) - (4,) 对齐：
#   最后维 3 vs 4 → 不等，会报错！
# 程序员又手滑加了 keepdims：
wrong_mean_keep = X.mean(axis=1, keepdims=True)  # shape (4, 1)
wrong = X - wrong_mean_keep          # (4,3) - (4,1) → 广播成 (4,3)
print("例4 wrong shape:", wrong.shape)   # (4, 3) —— 不报错！
# 但语义完全错了：本该减"每列均值"，却减了"每行均值"
# 结果：每个样本减去了自己的均值，相当于把每个样本中心化
# 这是真实工程里最常见的"广播 bug"：shape 凑巧能对上，但语义完全不同

    教训：“能广播” ≠ “语义对”。永远在代码里写 assert X.shape == expected 或者用 keepdims=True 让 shape 显式化。

    广播规则速查表

    A.shapeB.shape能广播？结果 shape说明
    (2,3)(2,3)✅(2,3)完全相同
    (2,3)()✅(2,3)标量广播
    (2,3)(3,)✅(2,3)行向量广播到每行
    (2,3)(2,1)✅(2,3)列向量广播到每列
    (2,3)(1,3)✅(2,3)单行广播到多行
    (3,1)(1,4)✅(3,4)双向广播
    (2,3)(2,4)❌—最后维 3≠4
    (2,3)(3,2)❌—最后维 3≠2，倒数第二维 2≠3

    PyTorch 用法

    import torch

A = torch.tensor([[1., 2., 3.],
                  [4., 5., 6.]])   # (2, 3)
b = torch.tensor([10., 20., 30.])  # (3,)

print(A + b)   # 广播，shape (2, 3)
# tensor([[11., 22., 33.],
#         [14., 25., 36.]])

# PyTorch 广播规则和 NumPy 完全一致
# 但 PyTorch 有个额外的广播函数 expand（不复制数据）
b_expanded = b.unsqueeze(0).expand(2, 3)  # (1,3) → (2,3)，view 操作
print("expand 是否共享数据:", b_expanded.data_ptr() == b.unsqueeze(0).data_ptr())  # True

    易混淆点

    1. 广播是”逻辑复制”，不真的占内存：底层用 stride=0 实现，所以 (1000, 1000) + (1,) 不会真的复制 100 万个标量。
    2. keepdims=True 是防 bug 利器：X.mean(axis=1, keepdims=True) 保留 shape (4,1)，比 (4,) 更不容易和别的 shape 误广播。
    3. 广播对齐从尾部开始：(2,3) + (3,) 能广播，但 (2,3) + (2,) 不能。新手常以为”前面维度相同就行”，错了。

    2.4 点积

    直觉

    点积有两种等价的直觉：

    直觉 1：加权求和。你有 H 个特征值 x = [x1, x2, ..., xH]，和 H 个权重 w = [w1, w2, ..., wH]，点积就是”加权总分”： \(\mathbf{x} \cdot \mathbf{w} = x_1 \cdot w_1 + x_2 \cdot w_2 + \cdots + x_H \cdot w_H\) 比如你的成绩 [语文 90, 数学 80, 英语 70]，权重 [0.3, 0.5, 0.2]，点积 = $90 \times 0.3 + 80 \times 0.5 + 70 \times 0.2 = 27 + 40 + 14 = 81$，加权平均分。

    直觉 2：方向相似性。点积衡量两个向量”方向有多一致”： \(\mathbf{x} \cdot \mathbf{y} = \|\mathbf{x}\| \times \|\mathbf{y}\| \times \cos\theta\)

    • θ=0°（同向）：cosθ=1，点积最大 = x · y 
    • θ=90°（垂直）：cosθ=0，点积=0（”正交”=毫无关联）
    • θ=180°（反向）：cosθ=-1，点积最小 = - x · y 

    这就是为什么 Attention 里用 $\mathbf{Q} \cdot \mathbf{K}$ 衡量”查询向量和键向量有多匹配”——点积大=方向一致=相关。

    定义

    两个同维向量 $\mathbf{x}, \mathbf{y} \in \mathbb{R}^n$ 的点积：

    \[\mathbf{x} \cdot \mathbf{y} = \mathbf{x}^\top \mathbf{y} = \sum_{i=1}^{n} x_i y_i = x_1 y_1 + x_2 y_2 + \cdots + x_n y_n\]

    几何定义：

    \[\mathbf{x} \cdot \mathbf{y} = \|\mathbf{x}\| \cdot \|\mathbf{y}\| \cdot \cos\theta\]

    余弦相似度（消除长度影响，只看方向）：

    \[\cos\theta = \frac{\mathbf{x} \cdot \mathbf{y}}{\|\mathbf{x}\| \cdot \|\mathbf{y}\|}\]

    值域 $[-1, 1]$：1=完全同向，0=正交，-1=完全反向。

    公式不跳步：从加权求和推到几何定义

    设 x = [3, 4], y = [4, 3]。

    第一步：算点积

    \[\mathbf{x} \cdot \mathbf{y} = 3 \times 4 + 4 \times 3 = 12 + 12 = 24\]

    第二步：算两个向量的 L2 范数

    \[\|\mathbf{x}\| = \sqrt{3^2 + 4^2} = \sqrt{9+16} = \sqrt{25} = 5\] \[\|\mathbf{y}\| = \sqrt{4^2 + 3^2} = \sqrt{16+9} = \sqrt{25} = 5\]

    第三步：算 cosθ

    \[\cos\theta = \frac{\mathbf{x} \cdot \mathbf{y}}{\|\mathbf{x}\| \cdot \|\mathbf{y}\|} = \frac{24}{5 \times 5} = \frac{24}{25} = 0.96\]

    θ ≈ arccos(0.96) ≈ 16.3°，两个向量方向很接近。

    def cosine_sim(x, y):
    """手写余弦相似度，每一步都看清"""
    dot = np.dot(x, y)                    # 分子：点积
    norm_x = np.linalg.norm(x)            # ||x||
    norm_y = np.linalg.norm(y)            # ||y||
    return dot / (norm_x * norm_y + 1e-8) # 加 epsilon 防止除零

print("cosine_sim:", cosine_sim(x, y))   # 0.96

# 等价于

print("cosine_sim:", np.dot(x, y) / (norm(x) * norm(y)))  # 0.96

    3. 余弦相似度 ≠ 欧氏距离：余弦只看方向，欧氏距离受长度影响。归一化后两者等价：$|\mathbf{x}-\mathbf{y}|^2 = 2(1 - \cos\theta)$（当 $|\mathbf{x}|=|\mathbf{y}|=1$ 时）。

    2.5 范数

    直觉

    范数就是”长度”或”大小”的推广。

    • 一根棍子有多长？→ 向量的 L2 范数（勾股定理）
    • 一堆零件总误差多大？→ 取绝对值求和（L1 范数）
    • 最坏那个零件误差多大？→ 取最大绝对值（L∞ 范数）

    不同范数衡量”大小”的角度不同，就像问”这堆人有多高”：

    • L2：先平方和再开根号（像算”等效能量”）
    • L1：直接求和（像算”总身高”）
    • L∞：只看最高的（”瓶颈”）

    定义

    向量范数（x ∈ R^n）：

    正式数学公式（LaTeX，对应网页 2.5 节）：

    \[\lVert \mathbf{x} \rVert_1 = \sum_i |x_i|\] \[\lVert \mathbf{x} \rVert_2 = \sqrt{\sum_i x_i^2}\] \[\lVert \mathbf{x} \rVert_\infty = \max_i |x_i|\]

    矩阵的 Frobenius 范数（把所有元素视为一个长向量）：

    \[\lVert \mathbf{A} \rVert_F = \sqrt{\sum_i\sum_j A_{ij}^2}\]

    纯文本对照表：

    名称公式直觉
    L1 范数\|x\|_1 = Σ\|x_i\|绝对值求和，”曼哈顿距离”
    L2 范数\|x\|_2 = √(Σ x_i²)勾股定理，”直线距离”
    L∞ 范数\|x\|_∞ = max\|x_i\|最大绝对值，”切比雪夫距离”
    Lp 范数（一般）\|x\|_p = (Σ\|x_i\|^p)^(1/p)p 越大越看重最大值

    矩阵范数：

    名称公式用途
    Frobenius 范数\|A\|_F = √(ΣΣ A_ij²)矩阵的”总大小”，类似 L2

    浮点误差比较（对应网页，工程常用）：

    \[e_{abs} = \|\hat{x}-x\|,\qquad e_{rel} = \frac{\|\hat{x}-x\|}{\|x\|}\]

    当参考值 $x$ 接近 0 时，相对误差会被放大，所以测试库通常组合绝对容差 atol 和相对容差 rtol：

    \[\|\hat{x}-x\| \le \text{atol} + \text{rtol}\cdot \|x\|\]

    公式不跳步：算一个具体向量的三种范数

    设 x = [3, -4, 0]。

    L1： \(\|\mathbf{x}\|_1 = |3| + |-4| + |0| = 3 + 4 + 0 = 7\)

    L2： \(\|\mathbf{x}\|_2 = \sqrt{3^2 + (-4)^2 + 0^2} = \sqrt{9 + 16 + 0} = \sqrt{25} = 5\)

    L∞： \(\|\mathbf{x}\|_\infty = \max(|3|, |-4|, |0|) = \max(3, 4, 0) = 4\)

    注意：L2 ≤ L1 ≤ √n · L2（这是常用不等式），这里 $5 \le 7 \le \sqrt{3} \cdot 5 \approx 8.66$ ✅。

    Frobenius 范数例子：

    设 A = [[1, 2], [3, 4]]。

    \[\|\mathbf{A}\|_F = \sqrt{1^2 + 2^2 + 3^2 + 4^2} = \sqrt{1+4+9+16} = \sqrt{30} \approx 5.477\]

    2.6 矩阵乘法

    直觉

    矩阵乘法 = 批量算点积。

    C = A @ B 中：

    • A 的每一行是一个”查询向量”
    • B 的每一列是一个”被查询的键向量”
    • C 的每个元素 C[i,j] = A 的第 i 行 · B 的第 j 列

    打比方：相亲大会，A 是 10 个男嘉宾（每人填 5 项资料），B 是 8 个女嘉宾（每人 5 项资料）。C[i,j] 就是男 i 和女 j 的”匹配度”——一次算出 10×8=80 个匹配度。

    定义

    设 $\mathbf{A} \in \mathbb{R}^{M \times K}$, $\mathbf{B} \in \mathbb{R}^{K \times N}$, 则 $\mathbf{C} = \mathbf{A}\mathbf{B} \in \mathbb{R}^{M \times N}$：

    正式数学公式（LaTeX，对应网页 2.6 节）：

    \[\mathbf{A} \in \mathbb{R}^{M \times K}, \qquad \mathbf{B} \in \mathbb{R}^{K \times N}, \qquad \mathbf{C} = \mathbf{A}\mathbf{B} \in \mathbb{R}^{M \times N}\]

    每个元素为：

    \[C_{ij} = \sum_{k=1}^{K} A_{ik}B_{kj}\]

    标准 GEMM 常写成（带缩放和累加）：

    \[\mathbf{C} \leftarrow \alpha\mathbf{A}\mathbf{B} + \beta\mathbf{C}\]

    结合律（数学上成立，但浮点结果不一定逐位相同）：

    \[(\mathbf{A}\mathbf{B})\mathbf{C} = \mathbf{A}(\mathbf{B}\mathbf{C})\]

    矩阵乘法一般不满足交换律：$\mathbf{A}\mathbf{B}$ 与 $\mathbf{B}\mathbf{A}$ 可能 shape 不同，或数值不同。数学上相等不代表浮点结果逐位相同，也不代表计算代价相同。选择不同结合顺序可以显著改变中间张量大小和 FLOPs。

    纯文本对照：

    C[i,j] = Σ(k=0 to K-1) A[i,k] · B[k,j]

    形状规则（核心）：

    (M, K) @ (K, N) → (M, N)
  ↑  ↑     ↑  ↑     ↑  ↑
  外 内   内  外   外  外
    • 内维必须相等（K == K），否则报错
    • 输出形状取外维（M, N）

    公式不跳步：手算 2×3 @ 3×2 的完整例子

    这是原稿缺的，必须补：

    设：

    A = [[1, 2, 3],     shape (2, 3)
     [4, 5, 6]]

B = [[7,  8],       shape (3, 2)
     [9,  10],
     [11, 12]]

    内维都是 3 ✅，输出 shape = (2, 2)。

    算 C[0,0]（A 第 0 行 · B 第 0 列）：

    A 的第 0 行 = [1, 2, 3]
B 的第 0 列 = [7, 9, 11]   （B[0,0], B[1,0], B[2,0]）

C[0,0] = 1×7 + 2×9 + 3×11
       = 7 + 18 + 33
       = 58

    算 C[0,1]（A 第 0 行 · B 第 1 列）：

    B 的第 1 列 = [8, 10, 12]

C[0,1] = 1×8 + 2×10 + 3×12
       = 8 + 20 + 36
       = 64

    算 C[1,0]（A 第 1 行 · B 第 0 列）：

    A 的第 1 行 = [4, 5, 6]

C[1,0] = 4×7 + 5×9 + 6×11
       = 28 + 45 + 66
       = 139

    算 C[1,1]（A 第 1 行 · B 第 1 列）：

    C[1,1] = 4×8 + 5×10 + 6×12
       = 32 + 50 + 72
       = 154

    最终结果：

    C = [[ 58,  64],
     [139, 154]]

A = np.array([[1, 2, 3],

              [11, 12]], dtype=np.float64)

# ===== 手写三重 for 循环（最直观，最慢）=====
def matmul_loop(A, B):
    M, K = A.shape
    K2, N = B.shape
    assert K == K2, f"内维不等: {K} vs {K2}"
    C = np.zeros((M, N), dtype=A.dtype)
    for i in range(M):
        for j in range(N):
            for k in range(K):
                C[i, j] += A[i, k] * B[k, j]
    return C

print("手写循环:\n", matmul_loop(A, B))   # 和上面一致

    标准 GEMM：C ← αAB + βC

    工程上矩阵乘法叫 GEMM（General Matrix Multiply），完整接口是：

    \[\mathbf{C} \leftarrow \alpha \cdot (\mathbf{A}\mathbf{B}) + \beta \cdot \mathbf{C}\]
    • $\alpha$, $\beta$ 是标量
    • 这一步同时完成”矩阵乘 + 加偏置 + 缩放”
    # PyTorch 的 torch.addmm 就是 GEMM
import torch

A = torch.randn(2, 3)
B = torch.randn(3, 2)
C = torch.randn(2, 2)   # 初始 C
alpha, beta = 1.0, 0.5

# C = alpha * (A @ B) + beta * C
result = torch.addmm(C, A, B, alpha=alpha, beta=beta)
print(result)

# 等价于
manual = alpha * (A @ B) + beta * C
print(torch.allclose(result, manual))   # True

    为什么要有 α 和 β？ 一次 kernel 调用同时完成乘加，比”先乘再加”少一次显存读写。在 GPU 上这是重要的性能优化。

    三个重要性质

    1. 不满足交换律：$\mathbf{A}\mathbf{B} \ne \mathbf{B}\mathbf{A}$（一般情况）
    2. 满足结合律：$(\mathbf{A}\mathbf{B})\mathbf{C} = \mathbf{A}(\mathbf{B}\mathbf{C})$ —— 这就是 tiling 的数学基础
    3. 数学相等 ≠ 浮点结果相同：
    # 演示：结合律在数学上成立，但浮点结果可能不同
import numpy as np
np.random.seed(0)
A = np.random.randn(100, 100).astype(np.float32)
B = np.random.randn(100, 100).astype(np.float32)
C = np.random.randn(100, 100).astype(np.float32)

# 数学上 (AB)C == A(BC)
left  = (A @ B) @ C
right = A @ (B @ C)
print("完全相等:", np.array_equal(left, right))      # False
print("近似相等:", np.allclose(left, right, rtol=1e-4))  # True
print("最大差:", np.max(np.abs(left - right)))       # ~1e-4

    工程含义：调换乘法顺序会改变浮点结果（差 ~1e-5 级别），训练 checkpoint 复现要对齐乘法顺序。

    PyTorch 用法

    import torch

# 2D 矩阵乘
A = torch.randn(2, 3)
B = torch.randn(3, 2)
print(torch.matmul(A, B).shape)   # torch.Size([2, 2])
print((A @ B).shape)              # torch.Size([2, 2])

# 带偏置的线性层：Y = XW^T + b
X = torch.randn(8, 4096)          # 8 个样本，每个 4096 维
W = torch.randn(2048, 4096)       # 2048 个输出神经元，每个 4096 维权重
b = torch.randn(2048)             # 偏置
Y = X @ W.T + b                   # (8, 4096) @ (4096, 2048) + (2048,) → (8, 2048)
print("Y shape:", Y.shape)        # torch.Size([8, 2048])

# 等价于 F.linear
import torch.nn.functional as F
Y2 = F.linear(X, W, b)            # 内部就是 X @ W.T + b
print(torch.allclose(Y, Y2))      # True

    FLOPs 估算（必考）

    矩阵乘 (M, K) @ (K, N) → (M, N)
FLOPs = 2 · M · K · N    （每个 C[i,j] 算 K 次乘 + K-1 次加 ≈ 2K，共 M·N 个元素）

    例：XW，X 是 (B·S, H) = (16384, 4096)，W 是 (H, V) = (4096, 32000)：

    \[\text{FLOPs} = 2 \times 16384 \times 4096 \times 32000 \approx 4.29 \times 10^{12} = 4.29 \text{ TFLOPs}\]

    易混淆点

    1. * vs @：A * B 是逐元素乘（Hadamard 积，要求同 shape）；A @ B 是矩阵乘。这是新手最常写错的。
    2. 矩阵乘不交换：$\mathbf{A}\mathbf{B}$ 和 $\mathbf{B}\mathbf{A}$ 形状可能不同，就算形状相同结果也不同。
    3. 向量点积是矩阵乘的特例：$\mathbf{x} \cdot \mathbf{y} = \mathbf{x}^\top \mathbf{y}$（行向量 × 列向量 = 标量）。
    4. 结合律浮点不严格成立：$(\mathbf{A}\mathbf{B})\mathbf{C}$ 和 $\mathbf{A}(\mathbf{B}\mathbf{C})$ 数值上有微小差异，做精确复现时要注意。

    2.7 Batched Matrix Multiplication (BMM)

    直觉

    普通矩阵乘是”一次算一组”。BMM 是”一次算多组独立的矩阵乘”，像工厂流水线同时加工多批料。

    典型场景：多头 Attention，B 个样本 × Nh 个头，每个头独立做 $\mathbf{Q}\mathbf{K}^\top$，互相不干扰。

    定义

    \[\mathbf{A} \in \mathbb{R}^{B \times M \times K}, \quad \mathbf{B} \in \mathbb{R}^{B \times K \times N}\] \[\mathbf{A}\mathbf{B} \in \mathbb{R}^{B \times M \times N}\]

    每个 batch b 独立做一次 (M, K) @ (K, N) → (M, N)：

    \[C[b, i, j] = \sum_k A[b, i, k] \cdot B[b, k, j]\]

    多维广播规则：PyTorch 的 @ / torch.matmul 对高维张量自动按”最后两维做矩阵乘，前面维度做广播”处理。

    多头 Attention 的 shape 推导

    正式数学公式（LaTeX，对应网页 2.7 节）：

    \[\mathbf{Q} \in \mathbb{R}^{B \times N_h \times S_q \times D_h}\] \[\mathbf{K} \in \mathbb{R}^{B \times N_h \times S_k \times D_h}\]

    转置最后两维后：

    \[\mathbf{K}^\top \in \mathbb{R}^{B \times N_h \times D_h \times S_k}\]

    批量矩阵乘法：

    \[\mathbf{Q}\mathbf{K}^\top \in \mathbb{R}^{B \times N_h \times S_q \times S_k}\]

    前两维 $(B,N_h)$ 是 batch 维，每个 (batch, head) 独立执行一个矩阵乘法。不要把”高阶张量乘法”想得过于神秘，它通常只是对最后两维做矩阵乘法，前面的维度负责批处理和广播。

    纯文本对照：

    Q ∈ R^(B, Nh, Sq, Dk)   —— B 个样本，Nh 个头，每个头 Sq 个 query，每个 query 是 Dk 维
K ∈ R^(B, Nh, Sk, Dk)
Q @ K^T ∈ R^(B, Nh, Sq, Sk)

    推导：

    • 把 Q 看作 (B·Nh, Sq, Dk)，K^T 看作 (B·Nh, Dk, SK)
    • 每个 batch 独立做 (Sq, Dk) @ (Dk, Sk) → (Sq, Sk)
    • 结果拼回 (B, Nh, Sq, Sk)

    FLOPs：

    每个 batch: 2 · Sq · Dk · Sk
总共: B · Nh · 2 · Sq · Dk · Sk

    NumPy 代码

    import numpy as np

# 模拟多头 Attention 的 QK^T
B, Nh, Sq, Sk, Dk = 2, 4, 10, 12, 64

Q = np.random.randn(B, Nh, Sq, Dk)
K = np.random.randn(B, Nh, Sk, Dk)

# 方法 1: np.matmul 自动批量
scores = np.matmul(Q, K.transpose(0, 1, 3, 2))  # (B, Nh, Sq, Sk)
print("scores shape:", scores.shape)   # (2, 4, 10, 12)

# 方法 2: 用 @ 操作符（NumPy 和 PyTorch 都支持）
scores2 = Q @ K.swapaxes(-1, -2)       # 更简洁
print("scores2 shape:", scores2.shape) # (2, 4, 10, 12)
print("相等:", np.allclose(scores, scores2))  # True

    PyTorch 用法

    import torch

B, Nh, Sq, Sk, Dk = 2, 4, 10, 12, 64
Q = torch.randn(B, Nh, Sq, Dk)
K = torch.randn(B, Nh, Sk, Dk)

# torch.matmul 会自动 batch
scores = torch.matmul(Q, K.transpose(-2, -1))   # (B, Nh, Sq, Sk)
print("scores shape:", scores.shape)   # torch.Size([2, 4, 10, 12])

# 完整 Attention（简化版）
import math
V = torch.randn(B, Nh, Sq, Dk)
attn = torch.softmax(scores / math.sqrt(Dk), dim=-1)   # (B, Nh, Sq, Sk)
out = attn @ V                                          # (B, Nh, Sq, Dk)
print("out shape:", out.shape)                          # torch.Size([2, 4, 10, 64])

# 专门的 bmm：只支持 3D（不含 Nh 那维，要把 B 和 Nh 合并）
Q_3d = Q.view(B*Nh, Sq, Dk)
K_3d = K.view(B*Nh, Sk, Dk)
out_bmm = torch.bmm(Q_3d, K_3d.transpose(1, 2))   # (B*Nh, Sq, Sk)
print("bmm shape:", out_bmm.shape)   # torch.Size([8, 10, 12])

    torch.bmm vs torch.matmul：

    • bmm：只接受 3D (B, M, N)，不支持广播，更严格
    • matmul/@：支持任意维，前面维度自动广播，更灵活

    易混淆点

    1. 前两维是 batch 维，最后两维才是”做矩阵乘”的维：PyTorch 里 matmul 只在最后两维做乘法，前面全当 batch。
    2. bmm 不支持广播：(B, M, N) @ (B, N, P) 可以，但 (B, M, N) @ (1, N, P) 不行（要用 matmul）。
    3. 转置用 transpose(-2, -1)：只转最后两维，前面 batch 维不动。写 K.T 会把所有维度反转，错！

    2.8 线性变换与仿射变换

    直觉

    线性变换：把空间”拉扯/旋转/压缩”，但保持”直线还是直线，原点还在原点”。像橡皮筋上画格子，拉一拉，格子变了但格子线还是直的。

    仿射变换：线性变换 + 平移。多了一个”整体搬家”的自由度。神经网络的全连接层就是仿射变换。

    定义

    线性变换 f 满足两条性质（对任意向量 x, y 和标量 a, b）：

    1. $f(\mathbf{x} + \mathbf{y}) = f(\mathbf{x}) + f(\mathbf{y})$（可加性）
    2. $f(a \cdot \mathbf{x}) = a \cdot f(\mathbf{x})$（齐次性）

    合并写：$f(a\mathbf{x} + b\mathbf{y}) = af(\mathbf{x}) + bf(\mathbf{y})$

    正式数学公式（LaTeX，对应网页 2.8 节）：

    \[f(a\mathbf{x}+b\mathbf{y}) = af(\mathbf{x})+bf(\mathbf{y})\]

    矩阵乘法 $f(\mathbf{x})=\mathbf{W}\mathbf{x}$ 是线性变换。神经网络中所谓”Linear 层”通常还包含 bias：

    \[\mathbf{y}=\mathbf{W}\mathbf{x}+\mathbf{b}\]

    这在数学上是仿射变换。多个没有激活函数的线性/仿射层可以合并成一个，因此非线性激活是深层网络表达复杂函数的关键。

    重要事实：任何线性变换都可以写成矩阵乘法 $f(\mathbf{x}) = \mathbf{W}\mathbf{x}$，其中 W 是矩阵。反之亦然。

    仿射变换：在线性变换基础上加一个平移 b：

    \(\mathbf{y} = \mathbf{W}\mathbf{x} + \mathbf{b}\)

    • $\mathbf{W}\mathbf{x}$ 是线性部分（拉扯/旋转）
    • b 是平移部分（整体搬家）

    公式不跳步：验证 $f(\mathbf{x}) = \mathbf{W}\mathbf{x}$ 是线性变换

    设 W = [[2, 0], [0, 3]], x = [1, 1], y = [2, 0], a = 2, b = 3。

    第一步：算 f(a·x + b·y)

    a·x + b·y = 2·[1,1] + 3·[2,0] = [2,2] + [6,0] = [8, 2]
f([8,2]) = W @ [8,2] = [2·8 + 0·2, 0·8 + 3·2] = [16, 6]

    第二步：算 a·f(x) + b·f(y)

    f(x) = W @ [1,1] = [2·1, 3·1] = [2, 3]
f(y) = W @ [2,0] = [2·2, 3·0] = [4, 0]
a·f(x) + b·f(y) = 2·[2,3] + 3·[4,0] = [4,6] + [12,0] = [16, 6]

    两边相等 ✅，验证了线性。

    仿射变换不满足线性：$g(\mathbf{x}) = \mathbf{W}\mathbf{x} + \mathbf{b}$，则 $g(\mathbf{x}+\mathbf{y}) = \mathbf{W}(\mathbf{x}+\mathbf{y}) + \mathbf{b} = \mathbf{W}\mathbf{x} + \mathbf{W}\mathbf{y} + \mathbf{b}$，但 $g(\mathbf{x}) + g(\mathbf{y}) = \mathbf{W}\mathbf{x} + \mathbf{b} + \mathbf{W}\mathbf{y} + \mathbf{b} = \mathbf{W}\mathbf{x} + \mathbf{W}\mathbf{y} + 2\mathbf{b}$。差了一个 b。所以”仿射”不是”线性”的子概念，是”线性 + 平移”。

    NumPy 代码

    import numpy as np

W = np.array([[2, 0],
              [0, 3]], dtype=np.float32)
b = np.array([1, -1], dtype=np.float32)
x = np.array([1, 1], dtype=np.float32)

# 线性变换
linear = W @ x
print("线性 Wx:", linear)   # [2, 3]

# 仿射变换
affine = W @ x + b
print("仿射 Wx+b:", affine)  # [3, 2]

# 批量：X 的每一行是一个样本
X = np.array([[1, 1],
              [2, 0],
              [0, 2]], dtype=np.float32)   # (3, 2)
Y = X @ W.T + b    # (3, 2) @ (2, 2) + (2,) → (3, 2)
print("批量仿射:\n", Y)
# [[3. 2.]
#  [5. -1.]
#  [1. 5.]]

    PyTorch 用法

    import torch
import torch.nn as nn

# nn.Linear 就是仿射变换 y = xW^T + b
linear = nn.Linear(in_features=4096, out_features=2048, bias=True)
# 内部参数：
print("W shape:", linear.weight.shape)   # (2048, 4096)  —— 注意是 (out, in)
print("b shape:", linear.bias.shape)     # (2048,)

Y = linear(X)              # (8, 2048)
print("Y shape:", Y.shape) # torch.Size([8, 2048])

# 等价于
Y_manual = X @ linear.weight.T + linear.bias
print(torch.allclose(Y, Y_manual))   # True

# 关掉 bias（纯线性变换）
linear_no_bias = nn.Linear(4096, 2048, bias=False)
print("bias is None:", linear_no_bias.bias is None)   # True

    易混淆点

    1. nn.Linear 存的权重是 (out, in) 不是 (in, out)：因为它内部算的是 x @ W.T + b，W 的形状是 (out_features, in_features)。这是新手常错。
    2. bias 让变换”非线性”：严格说仿射变换不是线性变换（除非 b=0）。但工程上大家混着叫。
    3. 激活函数才真正引入非线性：线性层堆 100 层还是线性（$\mathbf{W}_2(\mathbf{W}_1 \mathbf{x}) = (\mathbf{W}_2 \mathbf{W}_1) \mathbf{x}$），所以中间必须加 ReLU/GeLU 等非线性。

    2.9 转置、单位矩阵与逆矩阵

    直觉

    • 转置：把表格”翻一下”，行列互换。像 Excel 里”复制粘贴特殊-转置”。
    • 单位矩阵：乘法里的”1”。任何矩阵乘它都等于自己。像数字里的 $\mathbf{A} \times 1 = \mathbf{A}$。
    • 逆矩阵：乘法里的”倒数”。$\mathbf{A} \mathbf{A}^{-1} = \mathbf{I}$。像数字里 $5 \times (1/5) = 1$。

    定义

    转置 $\mathbf{A}^\top$：$\mathbf{A}^\top_{ij} = A_{ji}$。shape (M, N) → (N, M)。

    正式数学公式（LaTeX，对应网页 2.9 节）：

    \[(\mathbf{A}^\top)_{ij} = A_{ji}\]

    转置的乘积反转：

    \[(\mathbf{A}\mathbf{B})^\top = \mathbf{B}^\top\mathbf{A}^\top\]

    单位矩阵 $\mathbf{I}$：对角线为 1，其余为 0 的方阵。

    \[\mathbf{I}\mathbf{A}=\mathbf{A}\mathbf{I}=\mathbf{A}\]

    逆矩阵 $\mathbf{A}^{-1}$（仅对方阵且行列式 ≠ 0）：

    \[\mathbf{A}^{-1}\mathbf{A}=\mathbf{A}\mathbf{A}^{-1}=\mathbf{I}\]

    求解线性方程 $\mathbf{A}\mathbf{x}=\mathbf{b}$ 时，数学上可写 $\mathbf{x}=\mathbf{A}^{-1}\mathbf{b}$，但数值实现通常不显式求逆，而使用 LU、QR、Cholesky 等分解直接求解。显式求逆往往计算更多、误差更大。

    三个重要公式

    1. 转置的乘积反转： \((\mathbf{A}\mathbf{B})^\top = \mathbf{B}^\top \mathbf{A}^\top\)

    证明（不跳步）：

    (AB)^T[i, j] = (AB)[j, i]                    # 转置定义
             = Σ_k A[j, k] · B[k, i]         # 矩阵乘法定义
             = Σ_k B[k, i] · A[j, k]         # 乘法交换
             = Σ_k B^T[i, k] · A^T[k, j]     # 转置定义（B[k,i]=B^T[i,k]）
             = (B^T A^T)[i, j]               # 矩阵乘法定义

    2. 单位矩阵： \(\mathbf{I}\mathbf{A} = \mathbf{A}\mathbf{I} = \mathbf{A}\)

    3. 逆矩阵： \(\mathbf{A}^{-1} \mathbf{A} = \mathbf{A} \mathbf{A}^{-1} = \mathbf{I}\)

    NumPy 代码

    import numpy as np

A = np.array([[1, 2],
              [3, 4]], dtype=np.float64)

# 转置
print("A^T:\n", A.T)
# [[1. 3.]
#  [2. 4.]]

# 单位矩阵
I = np.eye(2)
print("I:\n", I)
# [[1. 0.]
#  [0. 1.]]

# 验证 IA = A
print("IA == A:", np.array_equal(I @ A, A))   # True

# 逆矩阵
A_inv = np.linalg.inv(A)
print("A^(-1):\n", A_inv)
# [[-2.   1. ]
#  [ 1.5 -0.5]]

# 验证 A @ A^(-1) = I
print("A @ A^(-1):\n", A @ A_inv)
# [[1. 0.]
#  [0. 1.]]   （可能有 1e-16 级误差）

# 验证 (AB)^T = B^T A^T
B = np.array([[5, 6],
              [7, 8]], dtype=np.float64)
left  = (A @ B).T
right = B.T @ A.T
print("(AB)^T == B^T A^T:", np.allclose(left, right))   # True

    工程实践：不要直接求逆！

    为什么？ 直接求逆数值不稳定（O(n³) 且误差大），工程上用 分解 替代：

    想做的事不要用用这个
    解 $\mathbf{A}\mathbf{x} = \mathbf{b}$x = inv(A) @ bnp.linalg.solve(A, b)
    求 $\mathbf{A}^{-1} \mathbf{B}$inv(A) @ Bscipy.linalg.solve(A, B)
    稳定分解—LU / QR / Cholesky 分解
    import numpy as np
from scipy.linalg import lu, qr, cholesky

# 解 Ax = b
A = np.array([[4., 3.],
              [6., 3.]])
b = np.array([10., 12.])

# 错误做法：直接求逆
x_wrong = np.linalg.inv(A) @ b
print("inv 法:", x_wrong)   # [2. 0.]

# 正确做法：solve
x_right = np.linalg.solve(A, b)
print("solve法:", x_right)  # [2. 0.]

# LU 分解
P, L, U = lu(A)
print("L:\n", L)
print("U:\n", U)
# PA = LU，分解后可快速解多个 b

    PyTorch 用法

    import torch

A = torch.tensor([[1., 2.],
                  [3., 4.]])
print("A.T:\n", A.T)
print("inv(A):\n", torch.inverse(A))
print("solve(A, b):\n", torch.linalg.solve(A, torch.tensor([10., 12.])))

# 矩阵的迹（trace，对角线和）

# 行列式

    易混淆点

    1. 不是所有矩阵都有逆：只有方阵且行列式 ≠ 0 才有逆。长方矩阵没有逆，但有”伪逆” pinv。
    2. $(\mathbf{A}\mathbf{B})^{-1} = \mathbf{B}^{-1} \mathbf{A}^{-1}$：和转置一样会”反转顺序”。
    3. 工程上几乎不需要显式求逆：训练时反向传播算的是 $\mathbf{W}^\top \mathbf{g}$（转置乘梯度），不是 $\mathbf{W}^{-1}$。

    2.10 线性相关、张成空间与秩

    直觉

    线性相关：一组向量里，有”多余”的——某个向量能被其他向量凑出来。

    打比方：

    • 你说”我身高 175cm”和”我身高 1.75m”——这两个信息线性相关（一个能算出另一个，只携带 1 份信息）
    • “我身高 175cm”和”我体重 70kg”——线性无关（两个独立信息）

    秩（rank）：一组向量里”真正独立的维度数”。两个相关的向量秩还是 1，两个无关的秩是 2。

    低秩近似：一个矩阵虽然写着 1000×1000（100 万个数），但真正独立的信息可能只有 16 维——可以用 1000×16 和 16×1000 两个小矩阵（3.2 万个数）近似，省 30 倍存储。

    定义

    线性组合：$c_1\mathbf{v}_1 + c_2\mathbf{v}_2 + \cdots + c_n\mathbf{v}_n$

    线性相关：存在不全为 0 的系数 $c_1, \ldots, c_n$ 使得 $\sum c_i\mathbf{v}_i = \mathbf{0}$。等价于”某个 vi 能被其他向量凑出来”。

    张成空间：所有线性组合构成的集合 $\operatorname{span}(\mathbf{v}_1, \ldots, \mathbf{v}_n)$。

    秩 $\operatorname{rank}(\mathbf{A})$：A 的列向量中”线性无关”的最大个数 = 行向量中”线性无关”的最大个数。两者相等。

    重要性质：$\operatorname{rank}(\mathbf{A}) \le \min(M, N)$（A 是 M×N 矩阵）

    正式数学公式（LaTeX，对应网页 2.10 节）：

    \[\operatorname{rank}(\mathbf{A}) \le \min(M,N)\]

    秩低意味着矩阵中的信息存在冗余，可以由更小的两个矩阵近似：

    \[\mathbf{A} \approx \mathbf{U}\mathbf{V},\qquad \mathbf{U}\in\mathbb{R}^{M\times r}, \mathbf{V}\in\mathbb{R}^{r\times N}, \quad r \ll \min(M,N)\]

    这会把参数量从 $MN$ 降到 $r(M+N)$，也是低秩适配和模型压缩的数学基础。

    低秩近似：如果 $\operatorname{rank}(\mathbf{A}) = r$，A 可以分解为 $\mathbf{A} = \mathbf{U}\mathbf{V}$，其中 $\mathbf{U} \in \mathbb{R}^{M \times r}$, $\mathbf{V} \in \mathbb{R}^{r \times N}$，参数从 $M \cdot N$ 降到 $r(M+N)$。

    公式不跳步：算一个矩阵的秩

    A = [[1, 2, 3],
     [2, 4, 6],
     [1, 0, 1]]

    观察：第 2 行 = 2 × 第 1 行（$[2,4,6] = 2 \cdot [1,2,3]$），所以第 2 行是”多余的”。 第 1 行和第 3 行 $[1,0,1]$ 显然不成比例，独立。

    所以秩 = 2（前两行相关，第 1、3 行独立）。

    参数量对比：

    • 原矩阵 A 是 3×3，9 个参数
    • 低秩近似 A ≈ U·V，U 是 3×2，V 是 2×3，共 6+6=12 个参数

    嗯，这里 r=2，参数量没减少反而增加了？因为矩阵本来就小。低秩近似只在原矩阵远大于 r 时才省：

    原矩阵 M×N = 1000×1000 = 1,000,000 参数
低秩 r=16: U(1000×16) + V(16×1000) = 16000 + 16000 = 32,000 参数
节省: 1,000,000 / 32,000 ≈ 31 倍

    NumPy 代码

    import numpy as np

              [2, 4, 6],

    2.11 特征值与特征向量

    直觉

    特征向量：矩阵作用在它身上，只伸缩不改变方向。

    打比方：你拉一根橡皮筋，大多数方向上的”花纹”会被拉歪；但顺着橡皮筋方向的花纹只会被拉长/缩短，方向不变。这些方向就是”特征向量”，拉长/缩短的比例就是”特征值”。

    物理意义：特征向量是系统的”主轴方向”，特征值是这些方向上的”放大率”。

    定义

    对方阵 A，如果存在标量 λ 和非零向量 v 使得：

    正式数学公式（LaTeX，对应网页 2.11 节）：

    \[\mathbf{A}\mathbf{v}=\lambda\mathbf{v}\]

    则称 v 是 A 的特征向量，λ 是对应的特征值。直觉上，矩阵变换在这个方向上只缩放，不改变方向。

    特征值可帮助理解：线性动态系统是否放大/衰减；Hessian 在不同方向的曲率；梯度经过多层 Jacobian 连乘时为什么爆炸或消失；协方差矩阵的主要变化方向（PCA）。不是所有矩阵都能在实数域下良好特征分解；工程中应根据矩阵结构选择合适算法，而不是把 eig 当万能工具。

    纯文本对照：

    A v = λ v

    直觉解读：$\mathbf{A}\mathbf{v}$ 本来应该把 v “拉扯”成另一个方向的向量；但如果 v 是特征向量，$\mathbf{A}\mathbf{v}$ 还在 v 的方向上，只是大小变了 λ 倍。

    公式不跳步：手算一个 2×2 矩阵的特征值

    设 A = [[4, 2], [1, 3]]，求特征值和特征向量。

    第一步：列方程 $\mathbf{A}\mathbf{v} = \lambda\mathbf{v}$，即 $(\mathbf{A} - \lambda\mathbf{I})\mathbf{v} = \mathbf{0}$

    \[\mathbf{A} - \lambda\mathbf{I} = \begin{bmatrix} 4-\lambda & 2 \\ 1 & 3-\lambda \end{bmatrix}\]

    第二步：非零解条件是行列式 = 0

    \[\det(\mathbf{A} - \lambda\mathbf{I}) = (4-\lambda)(3-\lambda) - 2 \cdot 1 = 0\] \[= 12 - 4\lambda - 3\lambda + \lambda^2 - 2 = \lambda^2 - 7\lambda + 10 = (\lambda-2)(\lambda-5) = 0\]

    第三步：解出 λ

    \[\lambda_1 = 2, \quad \lambda_2 = 5\]

    第四步：求特征向量

    对 $\lambda_1 = 2$：

    (A - 2I) v = 0
[[2, 2],     [v1]     [0]
 [1, 1]]  @  [v2]  =  [0]

    等价于 $2v_1 + 2v_2 = 0$，即 $v_1 = -v_2$。取 $\mathbf{v} = [1, -1]$。

    对 $\lambda_2 = 5$：

    (A - 5I) v = 0
[[-1, 2],     [v1]     [0]
 [1, -2]]  @  [v2]  =  [0]

    等价于 $-v_1 + 2v_2 = 0$，即 $v_1 = 2v_2$。取 $\mathbf{v} = [2, 1]$。

    验证：

    A @ [1, -1] = [4·1 + 2·(-1), 1·1 + 3·(-1)] = [2, -2] = 2 · [1, -1]  ✅ λ1=2
A @ [2, 1]  = [4·2 + 2·1, 1·2 + 3·1] = [10, 5] = 5 · [2, 1]  ✅ λ2=5

    NumPy 代码

              [1, 3]], dtype=np.float64)

print("特征向量（每列一个）:\n", eigenvectors)
# [[ 0.894 -0.707]
#  [ 0.447  0.707]]

    v = eigenvectors[:, i]
    print(f"λ={lam:.2f}, v={v}, A@v={A @ v}, λv={lam * v}")
    print(f"  相等: {np.allclose(A @ v, lam * v)}")

    易混淆点

    1. 只有方阵才有特征值：长方矩阵没有，但有”奇异值”（下一节 SVD）。
    2. 特征值可以是负数、零、复数：实矩阵也可能有复数特征值（比如旋转矩阵）。
    3. 特征向量不唯一：任何 $c\mathbf{v}$（c ≠ 0）也是特征向量，所以代码里返回的特征向量通常被归一化成单位长度。
    4. 工程上很少直接算特征值：太慢且不稳定。算”最大的几个”用幂迭代，算全部用 SVD 替代。

    2.12 SVD（奇异值分解）

    直觉

    SVD：把任何矩阵拆成”旋转 → 沿轴缩放 → 再旋转”三个简单动作的组合。

    打比方：任何线性变换（不管多复杂）都可以分解为：

    1. 先旋转一下（V^T）
    2. 沿坐标轴拉伸/压缩不同倍数（Σ）
    3. 再旋转一下（U）

    这就是 SVD 的物理意义。它适用于任何矩阵（不必方阵），所以比特征分解更通用。

    最重要的应用——降维/压缩：Σ 矩阵的对角线（奇异值）按从大到小排列，扔掉小的那些，就能用很少的参数近似原矩阵。这就是图像压缩、推荐系统、LoRA 的数学基础。

    定义

    对任意矩阵 $\mathbf{A} \in \mathbb{R}^{M \times N}$，SVD 分解为：

    正式数学公式（LaTeX，对应网页 2.12 节）：

    \[\mathbf{A} = \mathbf{U}\mathbf{\Sigma}\mathbf{V}^\top\]

    其中：

    • $\mathbf{U}$ 的列是左奇异向量；
    • $\mathbf{V}$ 的列是右奇异向量；
    • $\mathbf{\Sigma}$ 对角线上的 $\sigma_1\ge\sigma_2\ge\cdots\ge0$ 是奇异值。

    保留最大的前 $r$ 个奇异值（截断 SVD，最优 rank-$r$ 近似）：

    \[\mathbf{A}_r = \mathbf{U}_{:,1:r} \mathbf{\Sigma}_{1:r,1:r} \mathbf{V}_{:,1:r}^\top\]

    $\mathbf{A}_r$ 是一种最优的 rank-$r$ 近似（在常见的 2-范数和 Frobenius 范数意义下）。直觉上，小奇异值对应影响较弱的方向，可以舍弃以换取压缩。SVD 本身也有计算和内存成本。大型模型权重的实际压缩通常使用截断、随机化算法或直接训练低秩因子，而不是对所有巨大矩阵做完整 SVD。

    纯文本对照：

    A = U Σ V^T
    • $\mathbf{U} \in \mathbb{R}^{M \times M}$：左奇异矩阵，列向量两两正交（$\mathbf{U}^\top \mathbf{U} = \mathbf{I}$）
    • $\mathbf{\Sigma} \in \mathbb{R}^{M \times N}$：只有对角线非零，对角元素 $\sigma_1 \ge \sigma_2 \ge \cdots \ge 0$ 叫奇异值
    • $\mathbf{V} \in \mathbb{R}^{N \times N}$：右奇异矩阵，列向量两两正交（$\mathbf{V}^\top \mathbf{V} = \mathbf{I}$）

    形状：

    (M, N) = (M, M) @ (M, N) @ (N, N)

    截断 SVD（rank-r 近似）：只保留前 r 个最大的奇异值：

    \[\mathbf{A}_r = \mathbf{U}_r \mathbf{\Sigma}_r \mathbf{V}_r^\top\]

    其中 $\mathbf{U}_r \in \mathbb{R}^{M \times r}$, $\mathbf{\Sigma}_r \in \mathbb{R}^{r \times r}$, $\mathbf{V}_r \in \mathbb{R}^{N \times r}$。

    Eckart-Young 定理：$\mathbf{A}_r$ 是所有秩为 r 的矩阵中，与 A 的 Frobenius 距离最小的那个。即 SVD 给出的低秩近似是最优的。

    公式不跳步：SVD 怎么和”低秩”挂钩

    原矩阵 A 的”信息量”可以用奇异值总和衡量：$\sum \sigma_i$。前 r 个奇异值占比越大，截断 SVD 越准确：

    \[\text{能量保留率} = \frac{\sum_{i=1}^{r} \sigma_i^2}{\sum_{\text{all}} \sigma_i^2}\]

    如果前 10 个奇异值占 99% 能量，那用 rank-10 近似就能保留 99% 信息，参数量从 $M \cdot N$ 降到 $r(M+N)$。

    易混淆点

    1. SVD vs 特征分解：SVD 适用于任何矩阵；特征分解只对方阵。SVD 的奇异值 = $\mathbf{A}^\top \mathbf{A}$ 特征值的平方根。
    2. np.linalg.svd 返回 s 是向量，不是对角矩阵：要自己 np.diag(s) 才能乘。
    3. full_matrices=True vs False：True 返回完整 U(M,M) 和 Vt(N,N)；False 返回”瘦”SVD U(M,K) 和 Vt(K,N)，K=min(M,N)。工程上几乎都用 False。
    4. 截断 SVD 是最优低秩近似（Eckart-Young 定理），任何其他 rank-r 近似都不会比它更接近原矩阵（在 Frobenius 范数下）。

    2.13 LoRA：低秩更新

    直觉

    LoRA（Low-Rank Adaptation）：大模型已经训练好（参数 W0 固定不动），现在要做微调。直接更新 W0 太贵（参数太多），于是用一个小补丁 ΔW 来近似更新。

    核心思想：微调产生的权重变化 ΔW 是”低秩”的——虽然 ΔW 形状和 W0 一样大，但它真正需要的信息维度很少。所以把 ΔW 拆成两个小矩阵相乘：

    \[\Delta\mathbf{W} = \frac{\alpha}{r} \mathbf{B}\mathbf{A}\]
    • A 形状 (r, d_in)：把输入从 d_in 维压到 r 维
    • B 形状 (d_out, r)：把 r 维还原回 d_out 维
    • r 远小于 d_in 和 d_out（比如 r=16，d=4096）
    • α/r 是缩放系数

    打比方：W0 是一本厚厚的”通用百科全书”（4096×4096），微调时只想加一点”医学专业知识”。直接改整本书太贵，于是加一个”医学附录”——附录只是一个 r=16 维的”小索引”，但它和原书配合就能查医学内容。

    定义

    正式数学公式（LaTeX，对应网页 2.13 节）：

    对预训练权重 $\mathbf{W}0\in\mathbb{R}^{d{out}\times d_{in}}$，LoRA 冻结 $\mathbf{W}_0$，只训练低秩更新：

    \[\mathbf{W} = \mathbf{W}_0 + \Delta\mathbf{W}\] \[\Delta\mathbf{W} = \frac{\alpha}{r}\mathbf{B}\mathbf{A}\]

    其中：

    \[\mathbf{A}\in\mathbb{R}^{r\times d_{in}},\qquad \mathbf{B}\in\mathbb{R}^{d_{out}\times r}\]

    新增参数量从完整更新的 $d_{out}d_{in}$ 变为：

    \[r(d_{in}+d_{out})\]

    例如 $d_{in}=d_{out}=4096$、$r=16$：

    \[\frac{16(4096+4096)}{4096^2} = 0.0078125 \approx 0.78\%\]

    低秩只减少可训练参数与相应优化器状态，不自动消除基座权重、激活和前向计算成本。部署时可选择合并权重，或保留 adapter 动态切换，两者有不同的显存、延迟和灵活性权衡。

    纯文本对照：

    原始前向：

    h = W0 x

    LoRA 微调时：

    h = W0 x + ΔW x = W0 x + (α/r) B A x
    • W0 冻结（不更新）
    • A, B 是新增的可训练参数
    • A 用随机高斯初始化，B 初始化为 0（所以训练开始时 ΔW=0，模型和原来完全一样）

    参数量计算（重点！面试常考）

    这是原稿缺的，必须补：

    配置全量微调LoRA比例
    W0: 4096×4096, r=1616,777,216A(16×4096) + B(4096×16) = 131,0720.78%

    手算过程：

    全量参数量 = d_in × d_out = 4096 × 4096 = 16,777,216

LoRA 参数量 = r × d_in + d_out × r
            = r × (d_in + d_out)
            = 16 × (4096 + 4096)
            = 16 × 8192
            = 131,072

占比 = 131,072 / 16,777,216
     = 131,072 / 16,777,216
     = 1 / 128
     = 0.0078125
     = 0.78%

    通用公式：

    LoRA 参数占比 = r × (d_in + d_out) / (d_in × d_out)
            = r × (1/d_out + 1/d_in)
            ≈ 2r / d      （当 d_in = d_out = d 时）

    当 d=4096, r=16：$2 \times 16/4096 = 32/4096 = 1/128 \approx 0.78\%$ ✅

    NumPy 代码：参数量计算 + 前向对比

    import numpy as np

d_in, d_out, r = 4096, 4096, 16
alpha = 32   # LoRA 缩放系数

# 原始权重（冻结）
W0 = np.random.randn(d_out, d_in) * 0.02

# LoRA 参数（可训练）
A = np.random.randn(r, d_in) * 0.01    # (r, d_in)
B = np.zeros((d_out, r))               # (d_out, r)，初始为 0

# 参数量对比
full_params = d_in * d_out
lora_params = r * d_in + d_out * r
print(f"全量参数: {full_params:,}")        # 16,777,216
print(f"LoRA参数: {lora_params:,}")        # 131,072
print(f"占比: {lora_params/full_params*100:.2f}%")   # 0.78%

# 前向对比
x = np.random.randn(d_in)   # 输入

# 原始前向
h_full = W0 @ x

# LoRA 前向
delta_W = (alpha / r) * (B @ A)   # (d_out, r) @ (r, d_in) = (d_out, d_in)
h_lora = W0 @ x + delta_W @ x

# 等价形式（更省内存）
h_lora_efficient = W0 @ x + (alpha / r) * (B @ (A @ x))
# A @ x: (r, d_in) @ (d_in,) = (r,)
# B @ (...): (d_out, r) @ (r,) = (d_out,)
print("两种 LoRA 前向相等:", np.allclose(h_lora, h_lora_efficient))   # True

# 训练开始时（B=0），LoRA 前向 = 原始前向
print("初始时 ΔW=0:", np.allclose(B @ A, 0))   # True
print("初始时 h_lora == h_full:", np.allclose(h_lora, h_full))   # True

# FLOPs 对比
flops_full = 2 * d_out * d_in                          # 全量前向
flops_lora = 2 * d_out * d_in + 2 * (r * d_in + d_out * r)   # 原始 + LoRA 增量
print(f"全量 FLOPs: {flops_full:,}")        # 33,554,432
print(f"LoRA 增量 FLOPs: {2*(r*d_in + d_out*r):,}")   # 262,144
print(f"FLOPs 增加: {2*(r*d_in + d_out*r)/flops_full*100:.2f}%")   # 0.78%

    PyTorch 实现：一个完整 LoRA 层

    import torch
import torch.nn as nn

class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, r=16, alpha=32):
        super().__init__()
        self.r = r
        self.alpha = alpha
        self.scaling = alpha / r
        
        # 原始权重（冻结）
        self.base = nn.Linear(in_features, out_features, bias=False)
        for p in self.base.parameters():
            p.requires_grad = False   # 冻结
        
        # LoRA 增量（可训练）
        self.A = nn.Parameter(torch.randn(r, in_features) * 0.01)   # (r, d_in)
        self.B = nn.Parameter(torch.zeros(out_features, r))         # (d_out, r)
    
    def forward(self, x):
        # x: (batch, in_features)
        h = self.base(x)                              # 原始前向
        delta = self.scaling * (x @ self.A.T) @ self.B.T   # LoRA 增量
        return h + delta

# 测试
d_in, d_out, r = 4096, 4096, 16
lora_layer = LoRALinear(d_in, d_out, r=r, alpha=32)

# 统计可训练参数
trainable = sum(p.numel() for p in lora_layer.parameters() if p.requires_grad)
total = sum(p.numel() for p in lora_layer.parameters())
print(f"可训练参数: {trainable:,}")    # 131,072
print(f"总参数: {total:,}")            # 16,908,288
print(f"可训练占比: {trainable/total*100:.2f}%")   # 0.78%

# 前向
x = torch.randn(8, d_in)
out = lora_layer(x)
print("输出 shape:", out.shape)        # torch.Size([8, 4096])

    易混淆点

    1. LoRA 不是”减少模型大小”，是”减少可训练参数”：W0 还在显存里，只是不更新。所以推理时如果合并 W0+ΔW，参数量和原来一样。
    2. B 初始化为 0 是关键：保证训练开始时 ΔW=0，模型行为和原始完全一致，不会因为加了 LoRA 就跑飞。
    3. α/r 缩放系数的作用：解耦学习率和 LoRA rank。调 r 时不影响学习率，调学习率时不影响 r 的语义。
    4. 合并推理：训练完后可以把 $\mathbf{W} = \mathbf{W}_0 + \frac{\alpha}{r}\mathbf{B}\mathbf{A}$ 合并成一个矩阵，推理时和原模型完全一样快（不增加延迟）。

    面试回答（3 题）

    Q1: 请解释 stride 是什么，为什么转置不复制数据？

    答题框架（30 秒版本）：

    stride 是”每个维度走一步要跳过的元素数”，它和 shape 一起决定怎么从一维内存里取数。 偏移公式是 $\text{offset} = \sum i_k \times \text{stride}[k]$。 转置一个矩阵只是把 shape 的两个维度对调、stride 的两个值对调，底层数据完全不动。 比如 (2,3) stride (3,1) 的矩阵，转置后变成 (3,2) stride (1,3)——内存里还是 [1,2,3,4,5,6]，只是取 Xt[i,j] 时用新的 stride 算偏移。 这就是为什么 .T、reshape、expand 这些操作是 O(1) 的 view 操作。 代价：转置后张量 is_contiguous()=False，某些操作（如 view）会要求先 .contiguous() 复制成连续内存。

    Q2: 矩阵乘法 A @ B 的 FLOPs 怎么算？为什么是 $2MKN$？

    答题框架：

    对 $\mathbf{A} \in \mathbb{R}^{M \times K}$, $\mathbf{B} \in \mathbb{R}^{K \times N}$, 输出 $\mathbf{C} \in \mathbb{R}^{M \times N}$。 每个 C[i,j] 要算 $\sum_k A[i,k] \cdot B[k,j]$，这是 K 次乘法 + (K-1) 次加法 ≈ 2K 次浮点运算。 一共 M×N 个元素，所以总 FLOPs = $2 \cdot M \cdot K \cdot N$。 举例：(16384, 4096) @ (4096, 32000) 的 FLOPs = $2 \times 16384 \times 4096 \times 32000 \approx 4.29 \text{ TFLOPs}$。 在 A100（312 TFLOPS FP16）上理论耗时约 $4.29 / 312 \approx 14 \text{ ms}$。 工程含义：估算模型训练算力时，几乎全是矩阵乘的 FLOPs，所以记住 $2MKN$ 这个公式就够用 80% 场景。

    Q3: LoRA 为什么省参数？算一下 4096×4096, r=16 的具体数字。

    答题框架：

    LoRA 把权重更新 ΔW 拆成两个小矩阵相乘：$\Delta\mathbf{W} = \frac{\alpha}{r}\mathbf{B}\mathbf{A}$，其中 A 是 (r, d_in)，B 是 (d_out, r)。 全量微调要训练整个 W0，参数量 = $d_{in} \times d_{out}$。 LoRA 只训练 A 和 B，参数量 = $r \times d_{in} + d_{out} \times r = r(d_{in} + d_{out})$。 对 4096×4096, r=16：

    • 全量：$4096 \times 4096 = 16,777,216$ 个参数
    • LoRA：$16 \times (4096 + 4096) = 131,072$ 个参数
    • 占比：$131072 / 16777216 = 1/128 \approx 0.78\%$

    通用公式：当 d_in = d_out = d 时，占比 = $2r/d$。 为什么 work：研究表明微调产生的 ΔW 本质是低秩的，前几个奇异值就携带了大部分信息，所以用 r=8 或 16 就能很好近似。 推理时：可以把 $\mathbf{W}_0 + \Delta\mathbf{W}$ 合并成一个矩阵，不增加推理延迟。

    深入追问（5 题）

    Q1: 为什么广播”能对上”不等于”语义对”？给个真实 bug 例子。

    答：广播只检查 shape 能否对齐，不懂你的语义。经典 bug：

    X = np.random.randn(4, 3)   # 4 个样本，3 个特征
# 想减去"每列均值"（3 个特征各自的均值）
mean = X.mean(axis=1, keepdims=True)   # 程序员写错 axis，得到 (4, 1)
wrong = X - mean            # (4,3) - (4,1) → 广播成 (4,3)，不报错
# 但实际减的是"每行均值"，把每个样本中心化了，完全不是想要的

    防范：

    • 用 keepdims=True 让 shape 显式
    • 写 assert mean.shape == (3,) 这种断言
    • Code review 时重点关注广播操作的 shape

    Q2: (AB)C 和 A(BC) 数值结果一样吗？为什么？

    答：数学上相等（结合律），但浮点结果有微小差异（~1e-5）。

    原因：浮点运算不满足结合律。比如 (a+b)+c 和 a+(b+c) 在浮点下可能不同，因为舍入误差累积方式不同。矩阵乘法涉及大量加减，顺序不同误差不同。

    工程影响：

    • 训练复现要对齐乘法顺序（比如分布式训练里 A(B1+B2) vs AB1+AB2）
    • 不同 BLAS 库（MKL vs OpenBLAS）结果可能不同
    • 单元测试要用 allclose(rtol=1e-4) 而不是 ==

    Q3: SVD 截断后是”最优 rank-r 近似”，这个最优怎么理解？

    答：Eckart-Young 定理——在所有秩 ≤ r 的矩阵中，截断 SVD 得到的 $\mathbf{A}_r = \mathbf{U}_r \mathbf{\Sigma}_r \mathbf{V}_r^\top$ 使 Frobenius 范数 $|\mathbf{A} - \mathbf{A}_r|_F$ 最小。

    直觉：SVD 把矩阵的”信息”按奇异值从大到小排列，扔掉小的那些相当于扔掉最不重要的信息。任何其他 rank-r 近似都无法比 SVD 保留更多”能量”（奇异值平方和）。

    为什么对 LoRA 有启发：如果权重更新的 ΔW 是低秩的（前几个奇异值占大头），那 LoRA 用 r 维子空间就能很好近似 ΔW。但 LoRA 不是直接做 SVD——SVD 需要先有 ΔW，而 LoRA 是”边训练边学”低秩结构。

    Q4: 为什么工程上不直接求逆矩阵？

    答：三个原因：

    1. 数值不稳定：求逆的误差可能被放大，特别是矩阵接近奇异时（条件数大）。
    2. 效率低：求逆 O(n³)，而且求完逆还要再做一次矩阵乘法 O(n²)；直接解方程 $\mathbf{A}\mathbf{x}=\mathbf{b}$ 也是 O(n³) 但常数更小。
    3. 没必要：实际需求通常是”解方程”或”算 $\mathbf{A}^{-1}\mathbf{b}$”，这些用 LU/QR/Cholesky 分解更稳定更快。

    正确做法：

    • 解 $\mathbf{A}\mathbf{x} = \mathbf{b}$ → np.linalg.solve(A, b)（内部 LU 分解）
    • 算 $\mathbf{A}^{-1}\mathbf{B}$ → np.linalg.solve(A, B)
    • 最小二乘 $\mathbf{A}\mathbf{x} \approx \mathbf{b}$ → np.linalg.lstsq（内部 SVD）

    Q5: LoRA 的 r 怎么选？r 太大或太小有什么问题？

    答：

    • r 太小（如 r=1, 2）：表达能力不够，微调效果差。对于简单任务（如风格迁移）可能够用，复杂任务（如学习新领域知识）会欠拟合。
    • r 太大（如 r=256）：参数量上升，失去 LoRA 的省参数优势；且可能过拟合微调数据。
    • 经验值：
      • 通用微调：r=8 ~ 32
      • 困难任务（领域差异大）：r=64 ~ 128
      • 简单任务（指令跟随）：r=4 ~ 8
    • 实践经验：先从 r=8 或 16 试起，看验证集 loss 调整。LoRA 论文里 r=4 在多数任务上已经接近全量微调。

    为什么不是越大越好？ 因为大模型微调的 ΔW 本质是低秩的（研究表明前 1-10 个奇异值就占大部分能量），r 超过”真实秩”后只是浪费参数，反而容易过拟合。

    本章总结

    你学到了什么：

    1. 张量的 6 个属性（data/shape/stride/dtype/device/layout）决定它在工程上的一切
    2. stride 是 O(1) view 操作的核心：转置/reshape/expand 不复制数据的秘密
    3. 广播是双刃剑：方便但容易掩盖 shape bug，要写 assert
    4. 点积 = 加权求和 = 方向相似性，是 Attention 的数学基础
    5. 范数衡量大小，工程上常用 L2 和 Frobenius，比较浮点要 allclose
    6. 矩阵乘 FLOPs = 2MKN，是大模型算力估算的 90%
    7. BMM 是批量矩阵乘，多头 Attention 就是 BMM
    8. 线性变换 = 矩阵乘，仿射 = 矩阵乘 + 偏置，nn.Linear 就是仿射
    9. 工程上不求逆，用 LU/QR/Cholesky 分解
    10. 秩 = 独立信息维度，低秩近似能省存储
    11. SVD 是最优低秩近似，适用于任何矩阵
    12. LoRA 利用低秩性，用 r=16 把可训练参数压到 0.78%
