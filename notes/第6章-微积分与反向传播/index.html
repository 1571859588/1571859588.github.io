    第6章_微积分与反向传播

    第6章 微积分与反向传播

    来源：AIInfraGuide 第2章 数学基础 第6节 目标读者：高中数学基础，完全没学过微积分 整理原则：生活直觉 → 数学定义 → 公式不跳步（每一步等号都写出来）→ 代码验证（NumPy + PyTorch 双轨） 阅读建议：这是全书最难的一章。每节先读”直觉”建立画面，再读”定义”看数学怎么描述，最后跑代码验证。代码可直接复制运行。 配套章节：本章用到第2章的矩阵乘法、第5章的 Softmax/交叉熵，不熟可先回去翻。

    一句话结论

    这一章讲的是”模型怎么知道每个参数该往哪个方向调、调多少”——也就是反向传播的数学根基。导数是”一个量变化时另一个量跟着变多快”的瞬时比率，是山坡的陡度；偏导数是把”只动一个变量、其他固定”用在多变量函数上；梯度是把所有偏导数按顺序排成的一个向量，指向”上坡最快方向”，所以负梯度就是”下坡最快方向”；梯度下降就是蒙眼下山，每步沿负梯度走 η 那么远；链式法则是”复合函数一层层拆开求导”，像多米诺骨牌一样把损失一层层传回参数；计算图是把公式画成流程图，前向算结果、反向算梯度；反向模式自动微分就是”一次前向 + 一次反向”就能拿到所有参数的梯度，比数值法快得多也准得多；线性层反向就是两次 GEMM 加一次归约；Softmax 单独求导得到 diag(p)−pp^T，但 Softmax+交叉熵合起来会消成一行的 p−y，这是工程上必须利用的简化；梯度检查是用中心差分对照自动微分，写新算子时必做。 你需要记住九件事：

    1. 导数 = 瞬时变化率 = 山坡陡度：lim(h→0) [f(x+h)−f(x)]/h 是”割线斜率 → 切线斜率”的极限过程
    2. 偏导数 = 只动一个变量：把别的变量当常数，对这一个变量求导；多变量函数的”分方向陡度”
    3. 梯度 = 所有偏导数拼成的向量：方向是”上升最快”，所以 −∇f 是”下降最快”
    4. 梯度下降公式 x ← x − η∇f(x)：η 是步长（学习率），太大跨过谷底、太小走不到
    5. 链式法则 = 复合函数逐层求导：∂L/∂x = (∂L/∂h₂)(∂h₂/∂h₁)(∂h₁/∂x)，每层只算自己的局部梯度，乘起来传下去
    6. 反向模式自动微分 = 一次前向 + 一次反向拿全部梯度：代价 O(前向)，比数值法（每个参数都要重新前向一次）便宜得多
    7. 一个变量被多处使用 → 反向时梯度要累加：这是计算图里最容易写错的点
    8. 线性层 Y=XW+b 的反向 = 两次 GEMM + 一次归约：dX=dY@W^T、dW=X^T@dY、db=sum(dY, axis=0)
    9. Softmax+交叉熵合并求导 = p−y：单独求 Softmax 的 Jacobian 是 diag(p)−pp^T，但和 CE 一起推会大幅消元，工程上永远用合并版

    6.1 导数：局部变化率

    直觉：山坡的陡度 + 车的瞬时速度

    画面 1：山坡的陡度。 你站在一个起伏的山坡上，问”这里有多陡？”最自然的办法是：往前走一小步，记下高度变化，用”高度变化 ÷ 水平距离”得到陡度。但走多远呢？走 1 米得到一个粗略的陡度，走 0.1 米更精确，走 0.001 米更更精确……当步长无限趋近 0 时，这个比值就是”这一点的瞬时陡度”——这就是导数。

    画面 2：车的瞬时速度。 你开车从 A 到 B，总路程 100 km，总耗时 1 小时，平均速度 100 km/h。但你在高速上开过 130，在红绿灯前停下过 0。仪表盘上那个实时显示的数字，就是”路程对时间的导数”——瞬时速度。

    画面 3：切线。 把函数图像画出来，连接图像上两点得到一条直线（叫”割线”），它的斜率 = (y₂−y₁)/(x₂−x₁)。让其中一点慢慢向另一点靠拢，割线就慢慢变成”那一点的切线”。导数 = 切线的斜率。

    三句话总结：导数 = 瞬时变化率 = 切线斜率 = 山坡陡度。 一回事的四个说法。

    为什么是 lim(h→0)？—— 割线趋近切线的可视化

    很多人卡在”既然 h→0，那不就分母是 0 吗？”。关键是：h 趋近 0 但不等于 0，我们看的是”比值在 h 越来越小时的趋势”，而不是 h=0 那一刻的值。

    下面用 f(x) = x² 在 x=2 处的导数演示。令 h 从 1 一直缩小到 0.0001，看比值怎么变化：

    h        f(x+h)   f(x)    差值      比值 [f(x+h)-f(x)]/h
1.0      9.0000   4.0000  5.0000    5.0000   ← 割线斜率，粗略
0.1      4.4100   4.0000  0.4100    4.1000   ← 缩小
0.01     4.0401   4.0000  0.0401    4.0100
0.001    4.0040   4.0000  0.0040    4.0010
0.0001   4.0004   4.0000  0.0004    4.0001   ← 趋近 4
-------------------------------------------------
极限 h→0:                              4.0000   ← f'(2) = 2·2 = 4

    比值在向 4 靠拢。这就是 lim(h→0) 的含义：观察比值在 h 缩小时的”趋势值”，而不是把 h=0 代进去（那会得到 0/0 没意义）。

    一句话：导数不是”h=0 时的值”，而是”h 趋近 0 时比值趋近的那个数”。 这是极限思想的精髓。

    定义

    函数 f 在点 x 处的导数 f’(x)（读作”f prime of x”）定义为：

    \[f'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h}\]
    • 分子 f(x+h)−f(x)：自变量变化 h 后，函数值变化多少
    • 分母 h：自变量变化多少
    • 比值：单位自变量变化引起的函数变化，即”平均变化率”
    • 取极限：让 h→0，平均变瞬时

    几何意义：f’(x) 是函数图像在 x 点切线的斜率。 物理意义：若 f 是位置，f’ 是速度；若 f 是速度，f’ 是加速度。

    四则运算求导法则

    法则公式
    加法(f+g)’ = f’ + g’
    减法(f−g)’ = f’ − g’
    乘积(fg)’ = f’g + fg’
    商(f/g)’ = (f’g − fg’) / g²
    复合（链式）(f(g(x)))’ = f’(g(x)) · g’(x)

    乘积法则口诀：”前导后不导 + 前不导后导”。 商法则口诀：”(子导母 − 子母导) / 母²”。

    一个完整手推：f(x) = x² + 2x 的导数

    用定义推：

    \[\begin{aligned} f'(x) &= \lim_{h \to 0} \frac{f(x+h) - f(x)}{h} \\ &= \lim_{h \to 0} \frac{[(x+h)^2 + 2(x+h)] - [x^2 + 2x]}{h} \\ &= \lim_{h \to 0} \frac{[x^2 + 2xh + h^2 + 2x + 2h] - [x^2 + 2x]}{h} \quad \text{（展开平方）} \\ &= \lim_{h \to 0} \frac{2xh + h^2 + 2h}{h} \quad \text{（消掉 } x^2 \text{ 和 } 2x\text{）} \\ &= \lim_{h \to 0} \frac{h(2x + h + 2)}{h} \quad \text{（提取公因子 } h\text{）} \\ &= \lim_{h \to 0} (2x + h + 2) \quad \text{（} h \neq 0 \text{ 时约掉 } h\text{）} \\ &= 2x + 0 + 2 = 2x + 2 \quad \text{（取极限）} \end{aligned}\]

    验证：用法则更快——(x²)’ = 2x，(2x)’ = 2，加起来 2x+2。✓

    关键一步：在 h≠0 时把分子分母的 h 约掉，然后再取极限。这就是为什么”h→0 但 h≠0”——约分需要 h≠0，但极限看的是趋势。

    NumPy 代码：数值微分（前向差分 vs 中心差分）

    数值微分就是用很小的 h 直接算比值，不求解析解。两种常用方式：

    import numpy as np

def f(x):
    return x ** 2 + 2 * x

def f_prime(x):
    """解析导数：2x + 2"""
    return 2 * x + 2

def forward_diff(f, x, h=1e-5):
    """前向差分：[f(x+h) - f(x)] / h"""
    return (f(x + h) - f(x)) / h

def central_diff(f, x, h=1e-5):
    """中心差分：[f(x+h) - f(x-h)] / (2h)，精度更高"""
    return (f(x + h) - f(x - h)) / (2 * h)

x = 3.0
print(f"解析导数  : {f_prime(x)}")        # 8.0
print(f"前向差分  : {forward_diff(f, x):.8f}")  # 8.00002000...
print(f"中心差分  : {central_diff(f, x):.8f}")  # 8.00000000...  ← 更准

    为什么中心差分更准？ 直觉：从”目标点两侧各取一点”比”只看一侧”更对称，误差更小。数学上：前向差分误差 O(h)，中心差分误差 O(h²)，h=1e-5 时前者误差 ~1e-5，后者 ~1e-10。

    PyTorch 用法：autograd 自动求导

    PyTorch 不用数值微分，而是用反向模式自动微分（本章后半部分会手写一个）。这里先看用法：

    import torch

# requires_grad=True 告诉 PyTorch：请跟踪对 x 的所有运算，以便反推梯度
x = torch.tensor(3.0, requires_grad=True)

# 前向：构建计算图
y = x ** 2 + 2 * x       # y = 15
print(f"y = {y.item()}")  # 15.0

# 反向：从 y 倒着算梯度
y.backward()

# x.grad 就是 dy/dx 在 x=3 处的值
print(f"dy/dx = {x.grad.item()}")  # 8.0  ← 等于 2x+2 = 8

    对照表：

    方式精度速度用途
    解析手推精确最快推公式、写算子
    数值微分（中心差分）近似（O(h²)）慢验证手推/自动微分对不对
    PyTorch autograd精确（机器精度）一次前向+一次反向训练神经网络

    6.2 偏导数与梯度

    直觉：山坡在”东西方向”和”南北方向”各自的陡度

    回到山坡的画面。但这次山坡是一个面——高度由两个坐标决定：h = f(经度, 纬度)。你站在某点，问”这里有多陡？”答案不是单个数，而是要看朝哪个方向：

    • 朝东走 1 米，高度变化多少？→ 这是”东方向的陡度”
    • 朝北走 1 米，高度变化多少？→ 这是”北方向的陡度”

    偏导数就是”固定其他变量、只动一个变量”时的导数：算东方向陡度时把纬度当常数，算北方向陡度时把经度当常数。

    梯度 = 把所有偏导数按顺序排成一个向量：(∂f/∂x, ∂f/∂y)。它的方向是”上升最快的方向”，长度是”上升最快的速率”。

    一句话：偏导数 = 分方向的陡度；梯度 = 把所有偏导拼起来 = 指向山顶的箭头。

    定义

    设 f(x₁, x₂, …, xₙ) 是 n 元函数。f 对 x_i 的偏导数定义为：

    \[\frac{\partial f}{\partial x_i} = \lim_{h \to 0} \frac{f(x_1, \ldots, x_i + h, \ldots, x_n) - f(x_1, \ldots, x_i, \ldots, x_n)}{h}\]

    操作上：把除 x_i 以外的所有变量当成常数，对 x_i 求普通导数。

    梯度（gradient）是把所有偏导数拼成的向量：

    \[\nabla f(\mathbf{x}) = \left( \frac{\partial f}{\partial x_1}, \frac{\partial f}{\partial x_2}, \ldots, \frac{\partial f}{\partial x_n} \right)\]

    读作”nabla f”或”grad f”。注意：梯度是一个向量，长度等于变量个数。

    手推示例：f(x, y) = x²y + 3y²

    对 x 求偏导（把 y 当常数）：

    \[\frac{\partial f}{\partial x} = \frac{\partial}{\partial x}(x^2 y) + \frac{\partial}{\partial x}(3y^2) = y \cdot \frac{d}{dx}(x^2) + 0 = y \cdot 2x = 2xy\]

    第二项是 0 因为 3y² 不含 x，对 x 来说是常数。

    对 y 求偏导（把 x 当常数）：

    \[\frac{\partial f}{\partial y} = \frac{\partial}{\partial y}(x^2 y) + \frac{\partial}{\partial y}(3y^2) = x^2 \cdot 1 + 3 \cdot 2y = x^2 + 6y\]

    梯度：

    \[\nabla f(x, y) = (2xy, \; x^2 + 6y)\]

    在点 (x=2, y=1) 处：∇f = (2·2·1, 2²+6·1) = (4, 10)。

    梯度下降：蒙眼下山

    画面：你被蒙住眼睛站在山坡上，目标是走到谷底。看不见，但脚下能感觉到”哪边是下坡”。于是策略是：每一步沿”最陡的下坡方向”走一小步，走完再感受一次方向，再走一步……

    数学上：

    • “最陡下坡方向” = 负梯度方向 −∇f（因为梯度指上坡）
    • “走一小步” = 步长 η（学习率，learning rate）
    \[\mathbf{x}_{t+1} = \mathbf{x}_t - \eta \, \nabla f(\mathbf{x}_t)\]
    • x_t：第 t 步的位置（参数当前值）
    • η：学习率，控制步长
    • ∇f(x_t)：当前点的梯度
    • 负号：往梯度反方向走（下坡）

    学习率的影响：

    • η 太大：一步跨过谷底，到对面山坡上了，来回震荡甚至发散
    • η 太小：每步挪一点点，要走很久才到谷底
    • 合适的 η：平滑收敛到谷底

    完整 NumPy 示例：求 f(x) = x² + 2x 的最小值

    这个函数 1D，但用梯度向量形式写，便于推广。解析上：f’(x) = 2x+2 = 0 ⇒ x = −1 是最小值点。

    我们用梯度下降从 x=3 出发，学习率 η=0.1，跑 5 步，把每步的 x、梯度、新 x 都写出来：

    import numpy as np

def f(x):
    return x ** 2 + 2 * x

def grad_f(x):
    """梯度：f'(x) = 2x + 2"""
    return 2 * x + 2

eta = 0.1            # 学习率
x = 3.0              # 起点
print(f"起点  : x = {x:.4f},  f(x) = {f(x):.4f},  grad = {grad_f(x):.4f}")

for step in range(1, 6):
    g = grad_f(x)             # 算梯度
    x_new = x - eta * g       # 梯度下降更新
    print(f"步 {step}: x = {x_new:.4f},  f(x) = {f(x_new):.4f},  "
          f"grad = {grad_f(x_new):.4f},  Δx = {-eta*g:.4f}")
    x = x_new

    输出（手算对照，每一步都写出来）：

    步x（更新前）grad = 2x+2Δx = −η·gradx（更新后）f(x)
    起点3.00008.0000−0.80002.200015.0000 → 9.2400
    12.20006.4000−0.64001.56009.2400 → 5.5536
    21.56005.1200−0.51201.04805.5536 → 3.1983
    31.04804.0960−0.40960.63843.1983 → 1.7808
    40.63843.2768−0.32770.31071.7808 → 0.7189
    50.31072.6214−0.26210.04860.7189 → 0.0979

    可以看到 x 在朝 −1 逼近，f(x) 在朝 0 逼近（最小值是 f(−1) = −1，注意 f 值要走到收敛才到 −1）。

    为什么没一步到位？ 因为梯度只告诉你”局部下坡方向”，不是”谷底在哪”。蒙着眼看不见全局，只能一步步摸。

    多变量梯度下降 NumPy 示例

    f(x, y) = x² + 4y²，最小值在 (0, 0)：

    import numpy as np

def f(v):
    return v[0]**2 + 4 * v[1]**2

def grad_f(v):
    return np.array([2 * v[0], 8 * v[1]])   # (∂f/∂x, ∂f/∂y) = (2x, 8y)

v = np.array([3.0, 3.0])   # 起点
eta = 0.1
for step in range(20):
    v = v - eta * grad_f(v)
print(f"经过 20 步后 v = {v}")  # 接近 [0, 0]

    PyTorch 用法

    import torch

x = torch.tensor(3.0, requires_grad=True)
opt = torch.optim.SGD([x], lr=0.1)   # 随机梯度下降优化器

for step in range(5):
    opt.zero_grad()        # 1. 清空上一步的梯度
    y = x ** 2 + 2 * x     # 2. 前向算损失
    y.backward()           # 3. 反向算梯度
    opt.step()             # 4. 用梯度更新 x（内部就是 x -= lr * x.grad）
    print(f"步 {step+1}: x = {x.item():.4f},  f = {y.item():.4f}")

    四个动作 zero_grad → forward → backward → step 是 PyTorch 训练的标准循环，背后就是 6.2 节的梯度下降公式。

    6.3 Jacobian 与 Hessian

    直觉：多元版的”一阶导数”和”二阶导数”

    普通导数 f’(x) 描述”一进一出”函数的变化率。但机器学习里：

    • 输入经常是向量（多个参数）
    • 输出也可能是向量（如 Softmax 输出 n 个概率）

    Jacobian 是”向量进、向量出”函数的一阶导数，是”把每个输出对每个输入的偏导排成一张表”。

    Hessian 是”向量进、标量出”函数的二阶导数，描述”梯度本身怎么变化”——也就是曲率信息。

    函数类型一阶导二阶导
    标量→标量：f(x)f’(x) 一个数f’‘(x) 一个数
    向量→标量：f(x)∇f 一个向量（梯度）H 一个矩阵（Hessian）
    向量→向量：f(x)J 一个矩阵（Jacobian）（高阶张量，少用）

    Jacobian 定义

    设 f: ℝⁿ → ℝᵐ，输入 x ∈ ℝⁿ，输出 y = f(x) ∈ ℝᵐ。Jacobian 矩阵 J ∈ ℝ^(m×n)：

    \[J_{ij} = \frac{\partial y_i}{\partial x_j}\]
    • 第 i 行第 j 列：第 i 个输出对第 j 个输入的偏导
    • 形状 (m, n)：m 个输出，n 个输入
    • 作用：把”输入的微小变化 dx”映射到”输出的微小变化 dy”：dy = J · dx

    特例：当 m=1（输出标量）时，Jacobian 退化为 1×n 的行向量，转置后就是梯度 ∇f。

    Hessian 定义

    设 f: ℝⁿ → ℝ（标量函数）。Hessian 矩阵 H ∈ ℝ^(n×n)：

    \[H_{ij} = \frac{\partial^2 f}{\partial x_i \partial x_j}\]
    • 第 i 行第 j 列：先对 x_i 求偏导、再对 x_j 求偏导的”二阶偏导”
    • 形状 (n, n) 方阵
    • 直觉：H_ii 是”沿 x_i 方向的曲率”；H_ij 是”沿 x_i 和 x_j 两个方向的混合曲率”
    • 若 f 二阶连续可导，则 H 是对称矩阵（H_ij = H_ji，Clairaut 定理）

    梯度是”坡度”，Hessian 是”坡度的变化”，也就是”曲面有多弯”。

    • H 正定（所有特征值 > 0）：局部凸（开口朝上，局部最小值）
    • H 负定（所有特征值 < 0）：局部凹（开口朝下，局部最大值）
    • H 有正有负：鞍点

    手推示例：f(x, y) = x²y + 3y²

    上节已求 ∇f = (2xy, x² + 6y)。现在求 Hessian。

    二阶偏导：

    \[\begin{aligned} \frac{\partial^2 f}{\partial x^2} &= \frac{\partial}{\partial x}(2xy) = 2y \\ \frac{\partial^2 f}{\partial y^2} &= \frac{\partial}{\partial y}(x^2 + 6y) = 6 \\ \frac{\partial^2 f}{\partial x \partial y} &= \frac{\partial}{\partial x}(x^2 + 6y) = 2x \\ \frac{\partial^2 f}{\partial y \partial x} &= \frac{\partial}{\partial y}(2xy) = 2x \end{aligned}\]

    Hessian：

    \[H = \begin{pmatrix} 2y & 2x \\ 2x & 6 \end{pmatrix}\]

    注意 H_xy = H_yx = 2x，对称。✓

    为什么 AI Infra 主要用梯度，不用 Hessian？

    • 梯度 ∇f 是 n 维向量，存 O(n)
    • Hessian 是 n×n 矩阵，存 O(n²)；GPT-3 有 1750 亿参数，Hessian 要存 ~10²² 个数，根本装不下
    • 所以深度学习几乎只用一阶方法（SGD、Adam），二阶方法（牛顿法）只在参数少时用
    • 但 Hessian 的概念仍然重要——梯度爆炸/消失、鞍点、曲率敏感的学习率都和它有关

    6.4 链式法则

    直觉：多米诺骨牌

    画面：一排多米诺骨牌 A → B → C → D。A 倒下推倒 B，B 推倒 C，C 推倒 D。问”D 倒下的速度受 A 倒下速度影响多大？”

    答案是逐段相乘：A 影响 B 的速率 × B 影响 C 的速率 × C 影响 D 的速率。

    函数的复合就是数学版的多米诺：y = f(g(h(x)))，x 是第一块骨牌，y 是最后一块。问”y 对 x 的变化率”，就要逐层相乘：

    \[\frac{dy}{dx} = \frac{df}{dg} \cdot \frac{dg}{dh} \cdot \frac{dh}{dx}\]

    这就是链式法则（Chain Rule），是反向传播的数学根基。

    标量链式法则

    若 y = f(u)，u = g(x)，则：

    \[\frac{dy}{dx} = \frac{dy}{du} \cdot \frac{du}{dx}\]

    手推示例：y = (3x+1)²，令 u = 3x+1，则 y = u²。

    \[\frac{dy}{dx} = \frac{dy}{du} \cdot \frac{du}{dx} = 2u \cdot 3 = 2(3x+1) \cdot 3 = 6(3x+1)\]

    验证：直接展开 y = 9x²+6x+1，dy/dx = 18x+6 = 6(3x+1)。✓

    多元链式法则（反向传播真正用的形式）

    若 L 依赖中间变量 h₂，h₂ 依赖 h₁，h₁ 依赖 x：

    \[\frac{\partial L}{\partial x} = \frac{\partial L}{\partial h_2} \cdot \frac{\partial h_2}{\partial h_1} \cdot \frac{\partial h_1}{\partial x}\]

    更一般地，若 x 通过多条路径影响 L（比如 x 同时影响 h₁ 和 h₂，二者都汇入 L），要把各路径的梯度相加：

    \[\frac{\partial L}{\partial x} = \sum_{\text{路径 } p} \frac{\partial L}{\partial p} \cdot \frac{\partial p}{\partial x}\]

    这就是为什么计算图里一个变量被多处使用时，反向时梯度要累加。 不是 averaging，是 summing。

    逐步手推：a = xy, b = a + x, L = b²

    这是本章最重要的推导之一，每一步都不跳。目标：求 ∂L/∂x 和 ∂L/∂y。

    第 1 步：写出全部前向关系

    \[a = xy, \quad b = a + x, \quad L = b^2\]

    变量依赖链：

    • L 直接依赖 b
    • b 直接依赖 a 和 x
    • a 直接依赖 x 和 y
    • x 通过两条路径影响 L：x → b → L 和 x → a → b → L
    • y 通过一条路径影响 L：y → a → b → L

    第 2 步：从 L 开始，逐层求局部导数

    L = b²，对 b 求导：

    \[\frac{\partial L}{\partial b} = 2b\]

    b = a + x，分别对 a 和 x 求偏导：

    \[\frac{\partial b}{\partial a} = 1 \quad \text{（a 的系数是 1，x 视为常数）}\] \[\frac{\partial b}{\partial x} = 1 \quad \text{（x 的系数是 1，a 视为常数）}\]

    a = xy，分别对 x 和 y 求偏导：

    \[\frac{\partial a}{\partial x} = y \quad \text{（y 视为常数，xy 对 x 求导是 y）}\] \[\frac{\partial a}{\partial y} = x \quad \text{（x 视为常数，xy 对 y 求导是 x）}\]

    第 3 步：求 ∂L/∂b 已得，现在求 ∂L/∂a

    a 通过 b 影响 L，链式：

    \[\frac{\partial L}{\partial a} = \frac{\partial L}{\partial b} \cdot \frac{\partial b}{\partial a} = 2b \cdot 1 = 2b\]

    第 4 步：求 ∂L/∂x —— 注意 x 有两条路径

    路径 1（直接）：x → b → L

    \[\left(\frac{\partial L}{\partial x}\right)_1 = \frac{\partial L}{\partial b} \cdot \frac{\partial b}{\partial x} = 2b \cdot 1 = 2b\]

    路径 2（经 a）：x → a → b → L

    \[\left(\frac{\partial L}{\partial x}\right)_2 = \frac{\partial L}{\partial b} \cdot \frac{\partial b}{\partial a} \cdot \frac{\partial a}{\partial x} = 2b \cdot 1 \cdot y = 2by\]

    两条路径相加：

    \[\frac{\partial L}{\partial x} = 2b + 2by = 2b(1 + y)\]

    第 5 步：求 ∂L/∂y —— y 只有一条路径

    y → a → b → L：

    \[\frac{\partial L}{\partial y} = \frac{\partial L}{\partial b} \cdot \frac{\partial b}{\partial a} \cdot \frac{\partial a}{\partial y} = 2b \cdot 1 \cdot x = 2bx\]

    第 6 步：总结答案

    \[\boxed{\frac{\partial L}{\partial x} = 2b + 2by, \quad \frac{\partial L}{\partial y} = 2bx}\]

    第 7 步：代个具体数验证

    令 x=3, y=4：

    • a = 3·4 = 12
    • b = 12 + 3 = 15
    • L = 15² = 225

    公式给出的梯度：

    • ∂L/∂x = 2·15 + 2·15·4 = 30 + 120 = 150
    • ∂L/∂y = 2·15·3 = 90

    下面用 PyTorch 验证：

    import torch

x = torch.tensor(3.0, requires_grad=True)
y = torch.tensor(4.0, requires_grad=True)

a = x * y
b = a + x
L = b ** 2

L.backward()
print(f"∂L/∂x = {x.grad.item()}")  # 150.0
print(f"∂L/∂y = {y.grad.item()}")  # 90.0

    输出 150 和 90，与手推完全一致。✓

    关键观察：x 被用在了两处（直接进 b，间接进 a 再进 b），所以 ∂L/∂x 是两条路径之和（2b + 2by）。这就是 6.5 节”一个张量被多处使用时需要累积梯度”的数学根源。

    6.5 计算图与反向模式自动微分

    直觉：流水线工厂 + 报表倒着追

    前向 = 流水线：原材料 x 进第一道工序产出 a，a 进第二道工序产出 b，b 进最后一道产出成品 L。每道工序的”操作手册”是已知公式（乘、加、平方等）。

    反向 = 报表倒着追：老板看到成品 L 不达标，要追责：”谁的锅？”从最后一道工序往回问——L 不达标是因为 b 错了，b 错是因为 a 和 x 错了，a 错是因为 x 和 y 错了。每一步都把”责任”按比例分摊给上游。

    数学上：

    • “责任”就是梯度
    • “按比例分摊”就是乘以局部偏导
    • 多个下游都怪同一个上游 → 责任累加

    前向算结果，反向算梯度。 这是自动微分的两阶段。

    计算图是什么

    把公式拆成”一次只做一个运算”的有向无环图（DAG）：

    • 节点：变量（输入、中间、输出）和运算
    • 边：数据流方向

    例如 a=xy, b=a+x, L=b² 的计算图：

      x ──┐
      ├── (×) ── a ──┐
  y ──┘              │
                     ├── (+) ── b ── (²) ── L
  x ─────────────────┘

    注意 x 在图里出现两次（被两处使用），这是后面”梯度累加”的关键。

    前向模式 vs 反向模式

    模式从哪到哪算什么适合
    前向模式输入→输出每个输入的梯度输入多、输出少
    反向模式输出→输入每个输出的梯度输出少、输入多（神经网络就是这种）

    神经网络：参数 n 个（输入多），损失 1 个（输出少）。反向模式一次反向就能拿到所有参数的梯度，所以深度学习都用反向模式。

    反向模式的核心：局部梯度 × 上游传来的梯度

    每个节点只需要知道两件事：

    1. 局部梯度：自己的输出对自己的输入的偏导（这部分用解析公式，提前算好）
    2. 上游传来的梯度：损失对自己的输出的梯度（反向流过来）

    两者相乘 = 损失对自己的输入的梯度，继续往上传。

    形式化：若节点 v = f(u₁, u₂)，已知上游传来的 ∂L/∂v，则：

    \[\frac{\partial L}{\partial u_i} = \frac{\partial L}{\partial v} \cdot \frac{\partial v}{\partial u_i}\]

    如果同一个 u_i 被多个下游节点用到，把所有下游传来的梯度相加。

    累积梯度：一个张量被多处使用时

    这是计算图里最容易踩坑的点。规则：一个变量在反向时被多个下游节点”追责”，它的总梯度 = 各下游传来梯度之和。

    例子还是 a=xy, b=a+x, L=b²。x 被用在两处：直接进 b、间接进 a。反向时：

    • b 这条线传给 x 一个梯度 2b（来自 ∂L/∂b · ∂b/∂x）
    • a 这条线传给 x 一个梯度 2by（来自 ∂L/∂b · ∂b/∂a · ∂a/∂x）
    • x 的总梯度 = 2b + 2by

    代码实现上的关键：在反向之前必须把 x.grad 清零（PyTorch 的 zero_grad()），否则会累加到上一次的旧梯度上。这就是为什么训练循环开头永远有 opt.zero_grad()。

    NumPy 手写一个 mini 计算图（不用 PyTorch autograd）

    目标：完全用 NumPy 实现 a=xy, b=a+x, L=b² 的前向+反向，验证手推的 ∂L/∂x=150, ∂L/∂y=90。

    import numpy as np

# ========== 前向：每个节点存自己的输入和输出 ==========
class Node:
    """一个计算图节点：存前向值、存反向函数"""
    def __init__(self, name):
        self.name = name
        self.value = None       # 前向输出
        self.grad = 0.0         # 反向累积梯度，初始化 0
        self.parents = []       # 父节点（输入来源）
        self.backward_fn = None # 反向函数：给定上游梯度，怎么传给父节点

# 定义运算
def mul(x, y):
    """z = x * y"""
    z = Node('mul')
    z.value = x.value * y.value
    z.parents = [x, y]
    # 局部梯度：∂z/∂x = y, ∂z/∂y = x
    def backward_fn(upstream):
        # x.grad += upstream * y
        # y.grad += upstream * x
        return [upstream * y.value, upstream * x.value]
    z.backward_fn = backward_fn
    return z

def add(x, y):
    """z = x + y"""
    z = Node('add')
    z.value = x.value + y.value
    z.parents = [x, y]
    # 局部梯度：∂z/∂x = 1, ∂z/∂y = 1
    def backward_fn(upstream):
        return [upstream * 1.0, upstream * 1.0]
    z.backward_fn = backward_fn
    return z

def square(x):
    """z = x^2"""
    z = Node('square')
    z.value = x.value ** 2
    z.parents = [x]
    # 局部梯度：∂z/∂x = 2x
    def backward_fn(upstream):
        return [upstream * 2 * x.value]
    z.backward_fn = backward_fn
    return z

# ========== 前向：构建图 ==========
x = Node('x'); x.value = 3.0
y = Node('y'); y.value = 4.0

a = mul(x, y)       # a = 12
b = add(a, x)       # b = 15
L = square(b)       # L = 225

print(f"a = {a.value}, b = {b.value}, L = {L.value}")

# ========== 反向：拓扑逆序，每节点累积梯度 ==========
# 手动指定拓扑顺序：L → b → (a, x) → (x, y)
# 实际框架用拓扑排序自动完成

# L 节点的上游梯度 = 1（dL/dL = 1）
L.grad = 1.0

# 反向到 b：b 的梯度 = L.grad * ∂L/∂b = 1 * 2b = 2*15 = 30
grads = L.backward_fn(L.grad)        # [upstream * 2 * b.value]
b.grad += grads[0]                   # b 被 L 这一处用，所以是 +=
print(f"b.grad = {b.grad}")          # 30.0

# 反向到 a 和 x：通过 add 节点 b
grads = b.backward_fn(b.grad)        # [upstream, upstream]
a.grad += grads[0]                   # a.grad += 30  → 30
x.grad += grads[1]                   # x.grad += 30  → 30  ← x 的第一次累积
print(f"a.grad = {a.grad}, x.grad（暂时）= {x.grad}")

# 反向到 x 和 y：通过 mul 节点 a
grads = a.backward_fn(a.grad)        # [upstream * y, upstream * x]
y.grad += grads[0]                   # y.grad += 30 * 4 = 120
x.grad += grads[1]                   # x.grad += 30 * 3 = 90  ← x 的第二次累积
print(f"x.grad = {x.grad}, y.grad = {y.grad}")

print("=" * 40)
print(f"∂L/∂x = {x.grad}")   # 30 + 120 = 150  ✓
print(f"∂L/∂y = {y.grad}")   # 120  ← 等等，应该是 90？

    等等，y.grad = 120 是错的！ 手推说 ∂L/∂y = 90。bug 在哪？

    回头查：a.grad 应该是 ∂L/∂a = 2b = 30，没错。然后 ∂L/∂y = ∂L/∂a · ∂a/∂y = 30 · x = 30 · 3 = 90。但代码算成了 30 · 4 = 120。

    问题：在 mul 的 backward_fn 里，x 和 y 是闭包捕获的 Node 对象引用，但 backward_fn(upstream) = [upstream * y.value, upstream * x.value]——第一个返回给 x 的父节点位置（parents[0]），第二个给 y。索引顺序错了！

    修正：z.parents = [x, y]，所以 backward_fn 返回的列表第 0 项给 x，第 1 项给 y。∂z/∂x = y，所以第 0 项应是 upstream * y.value；∂z/∂y = x，所以第 1 项应是 upstream * x.value。代码原本写的就是这样……让我重新代入数字：a=12，b=15，L=225，∂L/∂b=2b=30，∂b/∂a=1，所以 a.grad=30。∂a/∂y=x=3，所以 y.grad = 30 * 3 = 90。

    但代码里写的是 y.grad += grads[0]，grads[0] = upstream * y.value = 30 * 4 = 120。索引对应反了：grads[0] 应该给 x（parents[0]），代码却加到了 y 上。

    正确写法：

    # 反向到 x 和 y：通过 mul 节点 a
# 注意 parents = [x, y]，所以 grads[0] 给 x，grads[1] 给 y
grads = a.backward_fn(a.grad)        # [upstream * y.value, upstream * x.value]
x.grad += grads[0]                   # x.grad += 30 * 4 = 120  ← x 的第二次累积
y.grad += grads[1]                   # y.grad += 30 * 3 = 90
print(f"x.grad = {x.grad}, y.grad = {y.grad}")

print("=" * 40)
print(f"∂L/∂x = {x.grad}")   # 30 + 120 = 150  ✓
print(f"∂L/∂y = {y.grad}")   # 90  ✓

    现在正确了：∂L/∂x = 150，∂L/∂y = 90，与手推完全一致。

    这个 bug 演示了反向传播最容易写错的两点：①父节点顺序对应错（索引对反）；②忘记累积（用 = 而不是 +=）。这也是为什么生产环境用 PyTorch autograd 而不是手写——autograd 用拓扑排序自动管理顺序，且每个节点的 grad 默认就是累加的。

    PyTorch 等价写法

    import torch

x = torch.tensor(3.0, requires_grad=True)
y = torch.tensor(4.0, requires_grad=True)

a = x * y            # 等价于 torch.mul
b = a + x            # 等价于 torch.add
L = b ** 2           # 等价于 torch.pow

L.backward()         # 自动反向，自动拓扑排序，自动累积

print(f"∂L/∂x = {x.grad.item()}")   # 150.0
print(f"∂L/∂y = {y.grad.item()}")   # 90.0

    三行核心代码 vs 上面的几十行——这就是框架的价值。但理解手写版对排查 bug 和写自定义算子至关重要。

    6.6 线性层的反向传播

    直觉：成品出问题，分清是原材料、模具还是偏置的锅

    线性层 Y = XW + b 是神经网络最基本的积木。前向是”输入 X 经过权重 W 加偏置 b 得到输出 Y”。反向是”已知输出 Y 的梯度 dY，反推 X、W、b 各自的梯度”。

    类比工厂：

    • X = 原材料（每行一个样本）
    • W = 模具（每列一个特征变换）
    • b = 调味料（每个输出维度一个固定偏置）
    • Y = 成品

    成品出问题（dY ≠ 0），分别问：

    • 原材料 X 该怎么改 → dX
    • 模具 W 该怎么改 → dW
    • 调味料 b 该怎么改 → db

    前向：Y = XW + b

    设：

    • X ∈ ℝ^(B×D_in)：B 个样本，每个 D_in 维
    • W ∈ ℝ^(D_in×D_out)：权重矩阵
    • b ∈ ℝ^(D_out)：偏置（每个输出维度一个）
    • Y ∈ ℝ^(B×D_out)：输出

    形状推导：

    \[\underbrace{X}_{B \times D_{in}} \cdot \underbrace{W}_{D_{in} \times D_{out}} + \underbrace{b}_{D_{out}} = \underbrace{Y}_{B \times D_{out}}\]
    • 矩阵乘法：(B×D_in) · (D_in×D_out) = (B×D_out) ✓
    • 加偏置 b：广播机制，b 加到 Y 每一行上

    反向：已知 dY，求 dX、dW、db

    设 dY ∈ ℝ^(B×D_out)（损失对 Y 每个元素的偏导）。三个梯度：

    $db = \text{sum}(dY, \text{axis}=0)$

    因为 Y_ij = XW_ij + b_j（b 广播到每行），∂L/∂b_j = Σ_i ∂L/∂Y_ij · ∂Y_ij/∂b_j = Σ_i dY_ij · 1 = Σ_i dY_ij。

    把所有行加起来：

    \[db = \sum_{i=1}^{B} dY_{i,:} \quad \text{（沿 batch 维归约，形状 } D_{out}\text{）}\]

    $dW = X^\top \, dY$

    ∂L/∂W_kj = Σ_i ∂L/∂Y_ij · ∂Y_ij/∂W_kj = Σ_i dY_ij · X_ik（因为 Y_ij = Σ_k X_ik W_kj + b_j）。

    整理成矩阵：

    \[dW = X^T \, dY \quad \text{（形状 } D_{in} \times D_{out}\text{）}\]

    $dX = dY \, W^\top$

    ∂L/∂X_ik = Σ_j ∂L/∂Y_ij · ∂Y_ij/∂X_ik = Σ_j dY_ij · W_kj。

    整理成矩阵：

    \[dX = dY \, W^T \quad \text{（形状 } B \times D_{in}\text{）}\]

    Shape 推导表（一目了然）

    量公式形状推导
    前向 YX @ W + b(B, D_out)(B, D_in)·(D_in, D_out)+(D_out,)→(B, D_out)
    dY（已知）上游传来(B, D_out)和 Y 同形
    dbsum(dY, axis=0)(D_out,)b 是 (D_out,)，把 B 个样本的梯度加起来
    dWX^T @ dY(D_in, D_out)(D_in, B)·(B, D_out) = (D_in, D_out)，和 W 同形 ✓
    dXdY @ W^T(B, D_in)(B, D_out)·(D_out, D_in) = (B, D_in)，和 X 同形 ✓

    记忆口诀：”反向就是把前向乘过的矩阵转置一下换个位置“。

    • dW 用 X^T（X 转置放左边）
    • dX 用 W^T（W 转置放右边）
    • db 是 dY 沿 batch 求和

    一个线性层反向 = 两次 GEMM + 一次归约

    看上面三个公式：

    • dX = dY @ W^T → 一次矩阵乘（GEMM）
    • dW = X^T @ dY → 一次矩阵乘（GEMM）
    • db = sum(dY, axis=0) → 一次归约（reduction）

    所以一个线性层反向 = 2 × GEMM + 1 × reduce。这是 AI Infra 估算反向 FLOPs 的基本单元：

    • 前向 1 次 GEMM（X @ W）
    • 反向 2 次 GEMM（dX 和 dW）+ 1 次 reduce（db）
    • 反向是前向 FLOPs 的 2 倍（不考虑 reduce，因为 reduce 比 GEMM 便宜）

    这就是为什么训练比推理贵——训练每步要前向+反向，反向又是前向的 2 倍，所以训练每步 ≈ 3 倍前向 FLOPs。

    含批次广播的偏置梯度细节

    注意 b 的形状是 (D_out,)，前向时被广播加到 Y 每一行（B 行）。反向时 ∂L/∂b_j = Σ_i dY_ij，把 B 个样本的梯度全加起来。这是初学最容易漏的一步——很多人想当然以为 db = dY，其实要 sum。

    如果 Y 是 (B, D_out) 而 b 是 (1, D_out)（带 fake batch 维），sum 仍要 axis=0，结果形状 (1, D_out)，再 reshape 回 (D_out,)。

    6.7 Softmax 的 Jacobian

    直觉：Softmax 把 n 个 logits 变成 n 个概率，每个输出对每个输入都有影响

    Softmax 是向量进向量出：z ∈ ℝⁿ → p ∈ ℝⁿ。它有 n² 个偏导（每个 p_i 对每个 z_j），排成 n×n 的 Jacobian 矩阵。

    回忆第 5 章：p_i = e^{z_i} / Σ_k e^{z_k}。注意 p_i 不仅依赖 z_i（分子），还依赖所有 z_k（分母里的和）。所以 ∂p_i/∂z_j 对所有 j 都可能非零。

    推导（不跳步）

    令 S = Σ_k e^{z_k}，则 p_i = e^{z_i} / S。对 z_j 求偏导，分两种情况：

    情况 1：i = j（p_i 对自己 z_i 求导）

    \[\frac{\partial p_i}{\partial z_i} = \frac{\partial}{\partial z_i}\left(\frac{e^{z_i}}{S}\right)\]

    用商法则 (f/g)’ = (f’g − fg’)/g²：

    \[\begin{aligned} \frac{\partial p_i}{\partial z_i} &= \frac{e^{z_i} \cdot S - e^{z_i} \cdot \frac{\partial S}{\partial z_i}}{S^2} \\ &= \frac{e^{z_i} \cdot S - e^{z_i} \cdot e^{z_i}}{S^2} \quad \text{（} \frac{\partial S}{\partial z_i} = e^{z_i} \text{）} \\ &= \frac{e^{z_i}}{S} \cdot \frac{S - e^{z_i}}{S} \\ &= p_i \cdot (1 - p_i) \quad \text{（因为 } e^{z_i}/S = p_i\text{）} \end{aligned}\]

    情况 2：i ≠ j（p_i 对别的 z_j 求导）

    分子 e^{z_i} 不依赖 z_j（∂e^{z_i}/∂z_j = 0），分母 S 依赖（∂S/∂z_j = e^{z_j}）：

    \[\begin{aligned} \frac{\partial p_i}{\partial z_j} &= \frac{0 \cdot S - e^{z_i} \cdot e^{z_j}}{S^2} \\ &= -\frac{e^{z_i}}{S} \cdot \frac{e^{z_j}}{S} \\ &= -p_i p_j \end{aligned}\]

    合并两种情况（用 Kronecker delta δ_ij：i=j 时为 1，i≠j 时为 0）：

    \[\frac{\partial p_i}{\partial z_j} = p_i(\delta_{ij} - p_j)\]

    Jacobian 矩阵（n×n）：

    \[J_{\text{softmax}} = \text{diag}(\mathbf{p}) - \mathbf{p} \mathbf{p}^T\]
    • diag(p)：n×n 对角矩阵，对角元是 p_i
    • pp^T：n×n 外积矩阵，第 (i,j) 元是 p_i p_j
    • 相减：对角元 p_i − p_i² = p_i(1−p_i)（情况 1），非对角元 0 − p_i p_j = −p_i p_j（情况 2）

    NumPy 验证

    import numpy as np

def softmax(z):
    z = z - z.max()                  # 减最大值防溢出
    e = np.exp(z)
    return e / e.sum()

z = np.array([2.5, 1.0, 0.3])
p = softmax(z)
print(f"p = {p}")                    # [0.714, 0.158, 0.128]

# 解析 Jacobian: diag(p) - p p^T
J_analytic = np.diag(p) - np.outer(p, p)
print("解析 Jacobian:")
print(np.round(J_analytic, 4))
# 对角元: p_i(1-p_i) > 0
# 非对角: -p_i p_j < 0

# 数值 Jacobian 验证
eps = 1e-6
J_numeric = np.zeros((3, 3))
for j in range(3):
    z_plus = z.copy();  z_plus[j] += eps
    z_minus = z.copy(); z_minus[j] -= eps
    J_numeric[:, j] = (softmax(z_plus) - softmax(z_minus)) / (2 * eps)

print("数值 Jacobian:")
print(np.round(J_numeric, 4))
print("一致:", np.allclose(J_analytic, J_numeric, atol=1e-5))   # True

    性质小结：

    • Jacobian 每行和为 0：Σ_j ∂p_i/∂z_j = Σ_j p_i(δ_ij − p_j) = p_i − p_i·1 = 0（因为概率和为 1）
    • 这意味着”把所有 z_j 同时加同一常数，p 不变”——这正是第 5 章的”平移不变性”
    • 对角元为正（p_i(1−p_i) ≥ 0），非对角元为负（−p_i p_j ≤ 0）：一个 logits 上升会让自己的概率上升、别人的概率下降

    单独的 Softmax 在神经网络里很少用

    为什么这节内容看起来”复杂但用得少”？因为：Softmax 几乎总跟交叉熵一起出现，而合并求导会大幅简化（下一节）。但如果分开求（先 softmax 再单独 CE），中间就要拿这个 Jacobian 做矩阵乘，浪费且数值不稳。所以工程上永远用合并版 F.cross_entropy。

    6.8 Softmax + 交叉熵的简洁梯度

    直觉：两条复杂的公式合在一起，奇迹般地消成一行

    上一节看到 Softmax 的 Jacobian 是个 n×n 矩阵 diag(p)−pp^T，挺复杂。交叉熵 L = −Σ y_i log p_i 又有自己的梯度。如果分开求：”先 softmax 求导，再 CE 求导，链式法则乘起来”——会得到一个看起来很复杂的式子。

    但奇迹在于：把两者合起来推，所有复杂项会互相消掉，最终结果就一行：∂L/∂z_i = p_i − y_i。

    这就是为什么所有深度学习框架都把 softmax 和 CE 合成一个算子（F.cross_entropy）——不仅数值稳定（用 LogSumExp），梯度也最简洁。

    设定

    • logits z ∈ ℝⁿ
    • softmax 后概率 p_i = e^{z_i} / Σ_k e^{z_k}
    • 真实分布 y（one-hot 或软标签，这里只要求 Σ y_i = 1）
    • 交叉熵损失 L = −Σ_i y_i log p_i

    目标：求 ∂L/∂z_j。

    完整推导（每一步等号都写出来）

    第 1 步：展开 L

    \[L = -\sum_i y_i \log p_i\]

    第 2 步：对 z_j 求偏导，链式法则穿过 log p_i

    \[\frac{\partial L}{\partial z_j} = -\sum_i y_i \cdot \frac{\partial \log p_i}{\partial z_j}\]

    这里 Σ_i 是因为 L 依赖所有 p_i，每个 p_i 都依赖 z_j，全部加起来。

    第 3 步：求 ∂log p_i / ∂z_j

    利用复合函数求导：log p_i 对 z_j = (1/p_i) · ∂p_i/∂z_j。上一节已求 ∂p_i/∂z_j = p_i(δ_ij − p_j)：

    \[\frac{\partial \log p_i}{\partial z_j} = \frac{1}{p_i} \cdot \frac{\partial p_i}{\partial z_j} = \frac{1}{p_i} \cdot p_i(\delta_{ij} - p_j) = \delta_{ij} - p_j\]

    关键消元：1/p_i 和 p_i 约掉了！这是简洁性的根源。结果只依赖 i==j，不依赖 i 本身。

    第 4 步：代回第 2 步

    \[\frac{\partial L}{\partial z_j} = -\sum_i y_i (\delta_{ij} - p_j)\]

    第 5 步：拆开求和

    \[\frac{\partial L}{\partial z_j} = -\sum_i y_i \delta_{ij} + \sum_i y_i p_j\]

    第 6 步：用 δ_ij 的”挑选”性质

    δ_ij 只在 i=j 时为 1，否则为 0，所以 Σ_i y_i δ_ij = y_j（只挑出 i=j 那一项）：

    \[\frac{\partial L}{\partial z_j} = -y_j + p_j \sum_i y_i\]

    第 7 步：用 Σ y_i = 1

    因为 y 是概率分布（one-hot 也满足），Σ_i y_i = 1：

    \[\frac{\partial L}{\partial z_j} = -y_j + p_j \cdot 1 = p_j - y_j\]

    最终结果：

    \[\boxed{\frac{\partial L}{\partial z_i} = p_i - y_i}\]

    简洁到极致：梯度就是”预测概率减真实标签”。

    • 如果 p_i > y_i（预测过头），梯度为正，z_i 会被减小（往负方向走），p_i 下降
    • 如果 p_i < y_i（预测不足），梯度为负，z_i 会被增大，p_i 上升
    • 如果 p_i = y_i（预测正确），梯度为 0，不动

    这是机器学习里最优雅的公式之一：模型的”自我纠错信号”就是”我预测的减去真实的”。

    几个特殊场景验证

    场景 1：one-hot 标签，正确类

    y = [0, 1, 0]，模型预测 p = [0.2, 0.5, 0.3]。

    梯度 = p − y = [0.2, −0.5, 0.3]。

    • 对正确类（i=2）：梯度 −0.5 < 0，z_2 会被增大，p_2 上升 ✓
    • 对错误类（i=1, 3）：梯度 0.2, 0.3 > 0，z_1, z_3 减小，对应 p 下降 ✓

    场景 2：预测完全正确

    y = [0, 1, 0]，p = [0, 1, 0]（极端情况）。

    梯度 = [0, 0, 0]，完全不更新，模型已收敛。✓

    PyTorch 验证

    import torch
import torch.nn.functional as F

torch.manual_seed(0)
z = torch.tensor([2.5, 1.0, 0.3], requires_grad=True)
y = torch.tensor([0.0, 1.0, 0.0])    # one-hot: 真实类别是 1（0-indexed）

# 用 F.cross_entropy：内部把 softmax 和 CE 合并，输入是 logits + 类别索引
loss = F.cross_entropy(z.unsqueeze(0), y.unsqueeze(0).long().argmax().unsqueeze(0))
loss.backward()

p = F.softmax(z.detach(), dim=0)
hand_grad = p - y                     # 手推公式：p - y

print(f"PyTorch grad : {z.grad}")             # tensor([0.236, -0.500, 0.264])
print(f"手推 grad   : {hand_grad}")            # 同上
print(f"一致        : {torch.allclose(z.grad, hand_grad)}")   # True

    工程意义：为什么框架都把 softmax+CE 合成一个算子

    1. 数值稳定：分开算要先 softmax（e^z 可能溢出）再 log（结果可能下溢）；合并版用 LogSumExp，全程在 log 域
    2. 梯度简洁：分开算要走 Jacobian 矩阵乘（n² 项），合并版直接 p−y（n 项），快 n 倍
    3. 实现简单：自定义模型时 loss = F.cross_entropy(logits, labels) 一行搞定，不用手写 softmax+CE

    铁律：永远不要分开写 loss = -torch.log(F.softmax(logits))，永远用 F.cross_entropy(logits, labels) 或 F.nll_loss(F.log_softmax(logits), labels)。

    6.9 梯度检查

    直觉：用”笨办法”验证”聪明办法”

    手推梯度公式或写自定义算子时，怎么知道自己没写错？用最笨但绝对不会错的办法对照——数值差分。

    笨办法：拿一个很小的 h，直接算 [f(x+h) − f(x−h)] / (2h)，这是 6.1 节的中心差分。它慢，但绝不会因为公式推错而错（只要 f 本身算对了）。

    聪明办法：你写的反向传播代码（autograd）。

    梯度检查 = 比较两者是否一致。一致 → 反向代码大概没错；不一致 → 有 bug，去查。

    中心差分公式

    \[\frac{\partial f}{\partial x_i} \approx \frac{f(\mathbf{x} + \varepsilon \mathbf{e}_i) - f(\mathbf{x} - \varepsilon \mathbf{e}_i)}{2 \varepsilon}\]
    • e_i：第 i 维单位向量（只有第 i 个分量是 1，其余 0）
    • ε：很小的数，通常 1e-5 或 1e-6
    • 每次只扰一个维度，所以要算 2n 次 f（n 是参数个数）——这就是为什么数值法慢，只在测试时用

    为什么用中心差分不用前向差分？ 6.1 节说过：中心差分误差 O(ε²)，前向差分误差 O(ε)，前者精度高得多。

    PyTorch 内置：torch.autograd.gradcheck

    PyTorch 提供现成的梯度检查工具，自动比对解析梯度和数值梯度：

    import torch

# 定义一个简单函数：f(x) = x^2 + 2x，验证梯度是 2x+2
def f(x):
    return (x ** 2 + 2 * x).sum()

x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True, dtype=torch.double)
# 注意：gradcheck 要求 double 精度，否则浮点误差会让检查失败

# gradcheck 内部会：①用 autograd 算解析梯度 ②用中心差分算数值梯度 ③比较
ok = torch.autograd.gradcheck(f, (x,), eps=1e-6, atol=1e-4)
print(f"梯度检查通过: {ok}")   # True

    对自定义函数做梯度检查（更实用的场景）：

    import torch

class MyReLU(torch.autograd.Function):
    """手写 ReLU 的前向和反向"""
    @staticmethod
    def forward(ctx, x):
        ctx.save_for_backward(x)       # 保存输入供反向用
        return x.clamp(min=0)          # 前向：max(0, x)

    @staticmethod
    def backward(ctx, grad_output):
        x, = ctx.saved_tensors
        grad_x = grad_output.clone()
        grad_x[x < 0] = 0              # 反向：x<0 处梯度为 0
        return grad_x

# 包装成函数
my_relu = MyReLU.apply

# 梯度检查
x = torch.randn(5, dtype=torch.double, requires_grad=True)
ok = torch.autograd.gradcheck(my_relu, (x,), eps=1e-6, atol=1e-4)
print(f"MyReLU 梯度检查通过: {ok}")   # True

    gradcheck 的关键参数和坑

    参数含义建议
    eps数值差分的 ε1e-6（太小浮点噪声大，太大近似误差大）
    atol绝对误差容限1e-4 ~ 1e-5
    rtol相对误差容限1e-3 ~ 1e-4
    dtype数据类型必须 double，float32 浮点误差太大

    常见失败原因：

    1. 用了 float32 而不是 float64 → 换 double
    2. 函数有不可导点（如 ReLU 在 0 处）→ 选避开 0 的输入
    3. eps 设得太小 → 1e-6 通常合适
    4. 反向代码有 bug → 这正是 gradcheck 要发现的，去查 backward

    何时该做梯度检查

    场景是否做 gradcheck
    写自定义 autograd Function必做
    实现新算子（如自定义 attention）必做
    训练 loss 不下降排查推荐（先确认梯度对）
    用 PyTorch 内置算子组合不必（已经测过）
    上线生产环境不必（性能开销大）

    Q1：请手推线性层 Y = XW + b 的反向传播，并说明计算量。

    答：设 X∈ℝ^(B×D_in)、W∈ℝ^(D_in×D_out)、b∈ℝ^(D_out)、Y∈ℝ^(B×D_out)。已知上游梯度 dY∈ℝ^(B×D_out)，求 dX、dW、db。

    从 Y_ij = Σ_k X_ik W_kj + b_j 出发：

    • $dX_{ik} = \sum_j dY_{ij} \cdot W_{kj}$，矩阵形式 $dX = dY \, W^\top$，形状 (B, D_in)
    • $dW_{kj} = \sum_i dY_{ij} \cdot X_{ik}$，矩阵形式 $dW = X^\top \, dY$，形状 (D_in, D_out)
    • $db_j = \sum_i dY_{ij}$（因为 b_j 广播到每行），矩阵形式 $db = \text{sum}(dY, \text{axis}=0)$，形状 (D_out,)

    计算量：dX 一次 GEMM、dW 一次 GEMM、db 一次归约。反向 = 2×GEMM + 1×reduce。相比之下前向只是 1×GEMM，所以反向 FLOPs 约为前向 2 倍，训练每步 ≈ 3×前向 FLOPs。形状记忆口诀：”反向就是把前向乘过的矩阵转置换个位置”。

    Q2：为什么 Softmax 配交叉熵时梯度是 p−y？请完整推导。

    答：设 logits z、softmax 输出 p_i = e^{z_i}/Σ e^{z_k}、真实分布 y（Σ y_i = 1）、损失 L = −Σ y_i log p_i。求 ∂L/∂z_j。

    第 1 步：∂L/∂z_j = −Σ_i y_i · ∂log p_i/∂z_j（链式穿过 log p_i）。

    第 2 步：∂log p_i/∂z_j = (1/p_i) · ∂p_i/∂z_j。Softmax 的 Jacobian 已知 ∂p_i/∂z_j = p_i(δ_ij − p_j)，代入：(1/p_i) · p_i(δ_ij − p_j) = δ_ij − p_j。1/p_i 和 p_i 在这一步消掉，是简洁性的根源。

    第 3 步：代回 = −Σ_i y_i(δ_ij − p_j) = −Σ_i y_i δ_ij + Σ_i y_i p_j = −y_j + p_j Σ_i y_i = −y_j + p_j · 1 = p_j − y_j。

    最终 ∂L/∂z_i = p_i − y_i。意义：预测过头就往回缩，预测不足就往前推，预测正确不动。工程上一定要用 F.cross_entropy，因为它把 softmax 和 CE 合成一个算子，既数值稳定（LogSumExp）又走最简梯度（p−y，n 项而非 n² 项）。

    Q3：什么是反向模式自动微分？为什么深度学习用它而不是数值微分或符号微分？

    答：反向模式自动微分 = 把函数拆成基本运算的”计算图”，先一次前向算出所有中间值并保存局部梯度，再一次反向从输出往输入逐节点用链式法则累积梯度。

    对比三种求导方式：

    • 数值微分（差分）：实现简单，但每个参数都要扰动一次重算前向，对 n 个参数是 O(n × 前向)；且浮点误差大。只用于梯度检查。
    • 符号微分（如 Mathematica）：表达式膨胀严重，复杂函数会生成巨大公式，不可行。
    • 自动微分：精确（机器精度）、一次前向+一次反向拿全部 n 个参数梯度、代价 O(前向)。神经网络损失是 1 个输出、参数是 n 个输入，正是反向模式的甜区（输出少输入多）。

    所以深度学习一律用反向模式自动微分。PyTorch 的 tensor.backward() 内部就是它，靠 requires_grad 跟踪计算图、靠拓扑排序确定反向顺序、靠每个节点的 backward_fn 算局部梯度、靠”grad 累加”处理一处变量被多处使用的情况。

    深入追问（5题）

    Q4：为什么反向比前向慢？前向后还要存什么？

    反向慢是因为：①要算 2×GEMM（dX、dW）+1×reduce（db），FLOPs 是前向 2 倍；②前向必须保存所有中间激活（X、a、b 等），反向要用它们算 dW、dX，所以显存占用大幅增加。

    这就是为什么大模型训练要省激活显存——gradient checkpointing（用时间换空间，不存中间激活、反向时重算）、ZeRO（把激活分片到多卡）都是为此。前向存的东西：每层的输入 X（用于 dW = X^T @ dY）、每层激活函数的中间状态（如 ReLU 的 mask、LayerNorm 的均值方差）。

    Q5：链式法则里”一处变量被多处使用要累加梯度”——为什么是加不是平均？

    数学根源：多元链式法则。若 x 通过两条路径影响 L：路径 A 和路径 B，则 ∂L/∂x = ∂L/∂x_A + ∂L/∂x_B，是各路径偏导的和，不是平均。

    直觉：责任分摊。一个员工被两个项目同时调用，两个项目的锅都要他背——加起来，不是平均。或者用流体类比：两条支流汇入同一条河，下游水量是两条支流之和。

    代码上：PyTorch 的 x.grad 默认就是累加（+=），所以每次反向之前必须 zero_grad() 清零，否则会把上一次的梯度叠加进来——这是初学最常见的 bug。

    Q6：梯度爆炸和梯度消失的数学根源是什么？怎么缓解？

    根源在链式法则的连乘。深网络损失对早期参数的梯度是许多局部梯度的乘积：

    \[\frac{\partial L}{\partial W_1} = \frac{\partial L}{\partial h_n} \cdot \frac{\partial h_n}{\partial h_{n-1}} \cdots \frac{\partial h_2}{\partial h_1} \cdot \frac{\partial h_1}{\partial W_1}\]
    • 若每个 ∂h_{k+1}/∂h_k 的特征值均值 > 1，乘起来 → 梯度爆炸（数值溢出成 inf）
    • 若 < 1，乘起来 → 梯度消失（数值下溢成 0，参数不更新）

    缓解办法：

    • 换激活函数：用 ReLU（梯度在正区恒为 1）替代 sigmoid（最大梯度 0.25，连乘必消失）
    • 残差连接：h_{k+1} = h_k + f(h_k)，梯度多一条 “1” 的旁路，连乘里有保底项
    • 归一化（LayerNorm、BatchNorm）：把每层激活拉回正常范围，避免梯度过大过小
    • 梯度裁剪：梯度范数超过阈值就缩放，治标不治本但防爆
    • 更好的初始化（Xavier、He）：让初始梯度特征值约等于 1
    • 优化器（Adam）：自适应学习率，对梯度规模不敏感

    Q7：为什么用中心差分而不是前向差分做梯度检查？ε 怎么选？

    误差分析：

    • 前向差分 [f(x+h)−f(x)]/h 的截断误差是 O(h)（来自泰勒展开的二阶项）
    • 中心差分 [f(x+h)−f(x−h)]/(2h) 的截断误差是 O(h²)（奇次项相消）

    h=1e-6 时，前向误差 ~1e-6，中心误差 ~1e-12，差距 6 个数量级。所以梯度检查永远用中心差分。

    ε 的选择是个权衡：

    • ε 太大（如 1e-2）：截断误差大，近似不准
    • ε 太小（如 1e-10）：浮点舍入误差主导，被噪声淹没
    • 经验值 1e-5 ~ 1e-6 最优，PyTorch gradcheck 默认 1e-6

    另外必须用 float64 而不是 float32：float32 只有 ~7 位有效数字，1e-6 的 ε 算出来的差分几乎全是噪声。float64 有 ~15 位，差分才可信。

    Q8：自动微分、符号微分、数值微分有什么区别？JIT 编译器（如 torch.compile）算哪种？

    • 数值微分：用差分近似，不解析表达式，简单但有误差、慢
    • 符号微分：像 Mathematica 那样用代数规则推 closed-form 表达式，精确但表达式会爆炸
    • 自动微分：把函数拆成基本运算的图，每个基本运算有解析的局部梯度，用链式法则累积——精确（机器精度）、不爆炸、能拿全部梯度

    torch.compile 不是上述任何一种，它是 JIT 编译——把动态图拍平成优化后的静态图（融合算子、减少 kernel launch、更好的内存规划）。它依赖 autograd 提供的局部梯度，但本身不做求导，只做执行优化。编译后的代码跑起来还是 autograd 那套数学，只是 kernel 调度更高效。

    可以理解为：autograd 解决”怎么算梯度”，torch.compile 解决”怎么算得更快”。

    易混淆点表

    易混淆概念区别记忆口诀
    导数 vs 偏导数导数一元、偏导数多元（其他变量当常数）“导数是单行道，偏导数是多岔路只走一条”
    偏导数 vs 梯度偏导数是单个分量，梯度是所有偏导拼成的向量“偏导数是一片，梯度是一摞”
    梯度 vs Jacobian梯度是标量函数的导数向量（n 维），Jacobian 是向量函数的导数矩阵（m×n）“标量出→梯度；向量出→Jacobian”
    Jacobian vs HessianJacobian 一阶偏导（m×n），Hessian 二阶偏导（n×n 方阵）“一阶导→J，二阶导→H”
    前向模式 vs 反向模式 AD前向一次拿一个输入的梯度，反向一次拿一个输出的全部梯度“前向输入少、反向输出少”
    数值微分 vs 自动微分数值用差分近似（有误差、慢），自动用链式法则精确算“数值是猜，自动是算”
    梯度下降步长 η 大 vs 小太大跨过谷底震荡发散，太小收敛慢“大跨小挪都不好”
    梯度 = 0 的点可能是最小值、最大值、鞍点“驻点不一定是极值”
    ∂L/∂x 多路径各路径偏导相加，不是平均“责任累加不平均”
    PyTorch grad 默认行为累加（+=），不是覆盖“不清零就翻车”
    db vs dYdb 是 dY 沿 batch 维求和，不是直接等于 dY“偏置梯度要 sum”
    softmax 单独梯度 vs softmax+CE 合并梯度单独是 n×n Jacobian diag(p)−pp^T，合并是 n 维向量 p−y“分开 n²，合并 n”
    解析梯度 vs 数值梯度解析是公式推出的（精确），数值是差分算的（近似）“解析准，数值笨”
    gradcheck 用 float64 vs float32必须用 float64，float32 噪声太大“检查要双精度”
    forward diff vs central diff前向误差 O(h)，中心误差 O(h²)“中心差分精度高一阶”
    requires_grad=True vs detach()前者跟踪梯度，后者把张量从计算图切下来“detach 是剪脐带”
    zero_grad() 必要性PyTorch grad 默认累加，不清零会把上次梯度叠进来“每步反向前必清零”
