    链表类题目

    LeetCode Hot 100 链表类题目

    更新时间：2026-07-04 来源：LeetCode Hot 100 链表专题 状态：刷题笔记，含完整代码 + 推导 + 面试追问

    一句话总结

    链表题的核心套路就三板斧：① 快慢指针（找中点、判环、找交点）；② 反转链表（迭代 / 递归，是很多题的子步骤）；③ 指针换向（改 next 指向）。关键认知：链表题不靠”聪明算法”，靠”指针操作细节”——画图、理清谁指向谁、注意空指针，比想高级解法更重要。

    Q1: 160. 相交链表（双指针走对方链表）

    题目：给你两个单链表的头节点 headA 和 headB，找出并返回两个单链表相交的起始节点。若不相交返回 null。要求链表保持原始结构。

    160. 相交链表 - 力扣 标签：哈希 | 链表 | 双指针

    示例：

    输入：intersectVal = 8, listA = [4,1,8,4,5], listB = [5,6,1,8,4,5], skipA = 2, skipB = 3
输出：Intersected at '8'

    先解释”自定义评测系统”是什么意思

    题目里那段”自定义评测”看起来很绕，其实就是力扣后台构造测试用例的方式，不是给你的函数的输入。逐个解释：

    intersectVal：相交节点的值。不相交时为 0。
listA、listB：两条链表的所有节点值。
skipA：链表 A 从头走到相交节点，要跳过多少个节点。
skipB：链表 B 从头走到相交节点，要跳过多少个节点。

    以示例 1 为例：

    listA = [4,1,8,4,5], skipA = 2 → 从 A 头走 2 步到节点 8（相交点）
listB = [5,6,1,8,4,5], skipB = 3 → 从 B 头走 3 步到节点 8（相交点）

    A: 4 → 1 ↘
                8 → 4 → 5
    B: 5 → 6 → 1 ↗

"相交"指的是内存中同一个节点对象，不是"值相同"。
所以 A 里的 1 和 B 里的 1 虽然值都是 1，但是两个不同节点；
而 A 里的 8 和 B 里的 8 是同一个节点（内存地址相同）。

    你只需要知道：函数收到的就是两个链表头指针 headA、headB，如果两条链表在某个节点汇合成同一段后续，就返回那个汇合节点。

    一句话结论

    两个指针 a、b 分别从 headA、headB 出发，a 走到头就转去走 headB，b 走到头就转去走 headA。如果相交，两指针会在交点相遇；如果不相交，两指针会同时走到 null（各走 m+n 步）。

    解法（最优：双指针交叉走）

    class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None

def getIntersectionNode(headA: ListNode, headB: ListNode) -> ListNode:
    """
    时间 O(m+n)，空间 O(1)
    核心：指针 A 走完 A 链表转去走 B 链表，指针 B 反之
    若相交，两指针走的总路程相同，必在交点相遇
    """
    if not headA or not headB:
        return None

    pa, pb = headA, headB

    # 两指针走的总长度都是 m+n，要么在交点相遇，要么同时到 None
    while pa is not pb:
        pa = pa.next if pa else headB  # A 走到尾，转去 B
        pb = pb.next if pb else headA  # B 走到尾，转去 A

    return pa  # 相交则返回交点，不相交则两指针同时为 None

    推导过程

    情况 1：相交
  A 链表: 4 → 1 ↘
                  8 → 4 → 5   （A 链表长 5，交点前长 2）
  B 链表: 5 → 6 → 1 ↗           （B 链表长 6，交点前长 3）

  指针 a 路径：4→1→8→4→5→(转B)5→6→1→8  ← 在 8 相遇
  指针 b 路径：5→6→1→8→4→5→(转A)4→1→8  ← 在 8 相遇

  为什么相遇？
    指针 a 走了 (2 + 3) + (3) = ... 不对，换个数法：
    指针 a 走完 A 全长(5) + B 中交点前长度(3) = 8 步到交点
    指针 b 走完 B 全长(6) + A 中交点前长度(2) = 8 步到交点
    两者步数相同 → 同时到达交点 ✓

情况 2：不相交
  A 链表长 m，B 链表长 n，无交点
  指针 a 走 m 步到尾，转 B 走 n 步 → 共 m+n 步到 None
  指针 b 走 n 步到尾，转 A 走 m 步 → 共 n+m 步到 None
  同时到 None，pa is pb (都是 None) → 退出，返回 None ✓

    为什么这个技巧能成立

    关键：让两指针走的"总路程"相同。

A 链表交点前有 a 段，交点后有 c 段（A 总长 = a + c）
B 链表交点前有 b 段，交点后有 c 段（B 总长 = b + c）

指针 a 走 A 全程 + B 交点前 = (a+c) + b
指针 b 走 B 全程 + A 交点前 = (b+c) + a
两者都是 a + b + c，所以同时到达交点。

    其他解法对比

    解法时间空间思路
    哈希集合O(m+n)O(m)把 A 的节点存集合，B 遍历找第一个在集合里的
    双指针交叉（本解）O(m+n)O(1)A 走完转 B，B 走完转 A
    长度对齐法O(m+n)O(1)先算两链表长度差，长的先走差值步

    双指针交叉法最优雅，且满足进阶的 O(1) 空间。长度对齐法也 O(1)，但需要先遍历两次算长度。

    易错点

    1. 判空用 pa is not pb 而不是 pa != pb——Python 中也要用 is，因为题目要求返回同一个节点对象。
    2. 写成 pa = pa.next or headB → 当 pa.next 是 None 时返回 headB 是对的，但用 if pa else headB 更清晰，避免 val=0 的节点被误判。
    3. 忘记处理 headA or headB 为空的情况 → 直接返回 None。

    Q2: 206. 反转链表（迭代三指针）

    题目：给你单链表头节点 head，反转链表并返回新头。

    206. 反转链表 - 力扣 标签：递归 | 链表

    示例：

    输入：head = [1,2,3,4,5]
输出：[5,4,3,2,1]

    一句话结论

    迭代法用三个指针 prev/curr/next，每一步把 curr.next 指向 prev，然后三个指针整体右移一格。这是链表题的”基本功”，很多题都要用。

    解法一（最优：迭代法）

    class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def reverseList(head: ListNode) -> ListNode:
    """
    时间 O(n)，空间 O(1)
    核心：三指针 prev/curr/next，逐个改 next 指向
    """
    prev = None
    curr = head

    while curr:
        nxt = curr.next     # 先存下一个，否则改完就丢了
        curr.next = prev    # 反转指向：curr 指向前一个
        prev = curr         # prev 前进
        curr = nxt          # curr 前进

    return prev  # 循环结束时 curr=None，prev 是新头

    推导过程

    head = 1 → 2 → 3 → 4 → 5 → None

初始：prev=None, curr=1

第 1 步：
  nxt = 2
  curr.next = None  （1 指向 None）
  prev = 1, curr = 2
  链表：None ← 1    2 → 3 → 4 → 5 → None

第 2 步：
  nxt = 3
  curr.next = 1     （2 指向 1）
  prev = 2, curr = 3
  链表：None ← 1 ← 2    3 → 4 → 5 → None

第 3 步：
  nxt = 4
  curr.next = 2     （3 指向 2）
  prev = 3, curr = 4
  链表：None ← 1 ← 2 ← 3    4 → 5 → None

... 依此类推

第 5 步后：
  prev = 5, curr = None
  链表：None ← 1 ← 2 ← 3 ← 4 ← 5

返回 prev=5，即新头 ✓

    为什么必须先存 nxt

    curr.next = prev 这一句会"切断" curr 和原来下一个节点的联系。
如果没提前存 nxt，执行完后 curr.next 已经变了，
再写 curr = curr.next 就会回到 prev 那边，逻辑全乱。
所以必须：
  nxt = curr.next   # 先存
  curr.next = prev  # 再改
  prev, curr = curr, nxt  # 再移

    解法二（递归法）

    def reverseList_recursive(head: ListNode) -> ListNode:
    """
    时间 O(n)，空间 O(n)（递归栈）
    核心：递归反转后面的，再把当前接到尾部
    """
    # 1. 终止条件：空或单节点
    if not head or not head.next:
        return head

    # 2. 递归反转 head 之后的部分，new_head 是反转后的新头
    new_head = reverseList_recursive(head.next)

    # 3. head.next 现在是反转后链表的"尾节点"
    #    让它指向 head，head 指向 None（成为新尾）
    head.next.next = head
    head.next = None

    return new_head

    递归法推导

    head = 1 → 2 → 3 → None

reverseList(1):
  reverseList(2):
    reverseList(3):
      3.next=None → 返回 3（new_head=3）
    回到 reverseList(2)：
      head=2, head.next=3
      2.next.next = 2  →  3 指向 2
      2.next = None
      链表：3 → 2 → None，返回 new_head=3
    回到 reverseList(1)：
      head=1, head.next=2
      1.next.next = 1  →  2 指向 1
      1.next = None
      链表：3 → 2 → 1 → None，返回 new_head=3 ✓

    两种解法对比

    解法时间空间优点缺点
    迭代（推荐）O(n)O(1)省栈空间，不易爆栈代码略长
    递归O(n)O(n)代码短，思路清晰链表长时栈溢出

    面试推荐迭代法。递归法可以作为”你能想到递归吗”的加分项，但要主动说明递归栈空间的缺点。

    易错点

    1. 迭代时忘记存 nxt → 改完 curr.next 后丢失后续节点。
    2. 返回 curr 而不是 prev → 循环结束时 curr=None，返回错。
    3. 递归法里 head.next.next = head 这句最难理解，要多画图。
    4. 递归法忘记 head.next = None → 会形成环。

    Q3: 234. 回文链表（找中点 + 反转后半 + 比较）

    题目：判断单链表是否为回文（正读反读一样）。

    234. 回文链表 - 力扣 标签：栈 | 递归 | 链表 | 双指针

    示例：

    输入：head = [1,2,2,1]
输出：true

    一句话结论

    用快慢指针找中点 → 反转后半部分 → 双指针比较前后两半。三步都是 O(n) 时间 O(1) 空间，是这道题进阶要求的最优解。

    解法（最优：快慢指针 + 反转 + 比较）

    class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def isPalindrome(head: ListNode) -> bool:
    """
    时间 O(n)，空间 O(1)
    核心：快慢指针找中点 → 反转后半 → 双指针比较
    """
    if not head or not head.next:
        return True

    # 第 1 步：快慢指针找中点（slow 停在前半的最后一个）
    slow, fast = head, head
    while fast.next and fast.next.next:
        slow = slow.next
        fast = fast.next.next
    # slow 现在是前半最后一个节点（偶数）或正中间（奇数）

    # 第 2 步：反转 slow 之后的部分
    second_half = reverse(slow.next)

    # 第 3 步：双指针比较前半和反转后的后半
    p1, p2 = head, second_half
    result = True
    while p2:  # 只比后半长度，前半多一个节点（奇数时）不影响
        if p1.val != p2.val:
            result = False
            break
        p1 = p1.next
        p2 = p2.next

    # 第 4 步（可选）：恢复链表
    # reverse(second_half)

    return result


def reverse(head: ListNode) -> ListNode:
    """反转链表，返回新头"""
    prev = None
    curr = head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev

    推导过程

    情况 1：偶数 [1,2,2,1]

第 1 步：快慢指针找中点
  slow=1, fast=1
  fast.next=2 且 fast.next.next=2 → slow=2, fast=2
  fast.next=1 且 fast.next.next=None → 退出
  slow 停在 2（第 2 个节点，前半最后一个）

第 2 步：反转 slow 之后 [2,1] → [1,2]
  链表变成：1 → 2 → 1 → 2 → None（前半没动，后半反向）

第 3 步：比较
  p1 从 head=1 开始：1 → 2
  p2 从 second_half=1 开始：1 → 2
  1==1, 2==2 → 回文 ✓

情况 2：奇数 [1,2,3,2,1]

第 1 步：快慢指针
  slow=1, fast=1
  fast.next=2 且 fast.next.next=3 → slow=2, fast=3
  fast.next=2 且 fast.next.next=1 → slow=3, fast=1
  fast.next=None → 退出
  slow 停在 3（正中间）

第 2 步：反转 slow 之后 [2,1] → [1,2]
  链表：1 → 2 → 3 → 1 → 2 → None（中间节点 3 不参与比较）

第 3 步：比较
  p1 从 head=1：1 → 2 → 3（停）
  p2 从 second_half=1：1 → 2（停）
  1==1, 2==2 → 回文 ✓（中间的 3 被忽略，不影响判断）

    为什么快慢指针循环条件是 fast.next and fast.next.next

    目的是让 slow 停在"前半最后一个"（偶数）或"正中间"（奇数）。

如果用 fast and fast.next：
  偶数 [1,2,2,1]：slow 停在第 2 个 2（后半第一个），不是前半最后
  → 反转后会包含中间元素，比较时对不齐

用 fast.next and fast.next.next：
  偶数 [1,2,2,1]：slow 停在 2（前半最后）✓
  奇数 [1,2,3,2,1]：slow 停在 3（正中间）✓

这样后半长度 ≤ 前半，比较时只走后半长度，奇数中间节点自动忽略。

    其他解法对比

    解法时间空间思路
    复制到数组 + 双指针O(n)O(n)链表 → 数组，首尾指针比较
    栈O(n)O(n)前半入栈，后半和栈顶比
    快慢指针+反转（本解）O(n)O(1)原地操作，满足进阶
    递归O(n)O(n)利用递归栈”反向”遍历

    本解法满足进阶的 O(1) 空间。递归法虽然代码短，但栈空间 O(n)，不算 O(1)。

    易错点

    1. 快慢指针循环条件写错 → slow 停的位置不对，比较时错位。
    2. 忘记只比较后半长度 → 奇数时中间节点会多出来。
    3. 反转后没恢复链表 → 题目虽不强制，但面试最好恢复（展示工程素养）。
    4. 用 fast and fast.next 代替 fast.next and fast.next.next → 对齐错。

    Q4: 141. 环形链表（快慢指针追及）

    题目：判断链表中是否有环。环指尾节点连接到链表中某个节点，可无限遍历。

    141. 环形链表 - 力扣 标签：哈希 | 链表 | 双指针

    示例：

    输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点（索引 1）。

    注：pos 只是力扣用来描述环位置的参数，不传给你的函数。你只收到 head。

    一句话结论

    快指针每次走 2 步，慢指针每次走 1 步。如果有环，快指针一定会在环里追上慢指针（相遇）；如果无环，快指针先到 null。

    解法（最优：快慢指针 / Floyd 判圈法）

    class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None

def hasCycle(head: ListNode) -> bool:
    """
    时间 O(n)，空间 O(1)
    核心：快慢指针，快走2步慢走1步，有环必相遇
    """
    slow, fast = head, head

    while fast and fast.next:
        slow = slow.next         # 慢走 1 步
        fast = fast.next.next    # 快走 2 步

        if slow is fast:          # 注意用 is
            return True

    return False

    推导过程

    情况 1：有环  3 → 2 → 0 → 4 → (回到 2)

slow, fast 都从 3 出发
  第 1 步：slow=2, fast=0
  第 2 步：slow=0, fast=2（fast 进环了）
  第 3 步：slow=4, fast=4（fast 追上 slow）→ 返回 true ✓

情况 2：无环  3 → 2 → 0 → 4 → None

slow, fast 都从 3 出发
  第 1 步：slow=2, fast=0
  第 2 步：slow=0, fast=None（fast.next.next 越界前 fast.next=None）
  循环条件 fast and fast.next 不满足 → 退出，返回 false ✓

    为什么有环快慢指针必相遇（数学证明）

    设环外长度 a，环长度 b。慢指针进环时，快指针已在环内。

设慢指针进环时，快指针在慢指针前面 k 步（k < b）。
此后每一步，快指针比慢指针多走 1 步，距离每步缩小 1。
所以再走 b - k 步，距离缩到 0，相遇。

因为 k < b，所以 b - k > 0，一定能相遇。
关键是：相对速度是 1 步/次，而环长有限，所以必追上。

    为什么循环条件是 fast and fast.next

    快指针一次走 2 步：fast = fast.next.next
所以必须保证 fast 不是 None，且 fast.next 不是 None。

如果无环：
  fast 先到链表尾，fast.next = None
  → fast.next.next 会报错（None 没有 next）
  → 所以要先判断 fast and fast.next

    其他解法对比

    解法时间空间思路
    哈希集合O(n)O(n)访问过的节点存集合，重复即有环
    快慢指针（本解）O(n)O(1)Floyd 判圈法
    标记法O(n)O(1)改节点 val 打标记（会破坏数据，不推荐）

    快慢指针满足进阶的 O(1) 空间，且不破坏链表，是最优解。

    延伸：如何找到环的入口（142 题）

    def detectCycle(head: ListNode) -> ListNode:
    """找到环的入口节点"""
    slow, fast = head, head
    has_cycle = False

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            has_cycle = True
            break

    if not has_cycle:
        return None

    # 关键定理：相遇点到环入口的距离 = 头到环入口的距离
    # 让一个指针从头开始，另一个从相遇点开始，同速走，相遇即入口
    p1, p2 = head, slow
    while p1 is not p2:
        p1 = p1.next
        p2 = p2.next
    return p1

    记忆：相遇后，一头一相遇点，同速走必在环入口相遇。这是 Floyd 算法的完整版。

    易错点

    1. 判等用 == 而非 is → 节点 val 相同但不是同一对象会误判。必须用 is。
    2. 循环条件写错 → fast.next and fast 顺序错，会先访问空。
    3. 快慢指针都从 head 出发，第一次循环就 slow is fast → 误判。要先走再判，所以 if slow is fast 放在移动后。
    4. fast = fast.next.next 没判断 fast.next 是否为 None → 空指针异常。

    链表类题目通用思路

    模式典型题目关键技巧
    双指针交叉走160 相交链表A 走完转 B，B 走完转 A
    三指针反转206 反转链表prev/curr/next，改 next 指向
    快慢指针找中点234 回文链表快走2慢走1，slow 到中点
    快慢指针判环141 环形链表快追慢，有环必相遇

    核心认知：

    1. 画图是链表题的第一工具——谁指向谁、改完什么样，画出来一目了然。
    2. 快慢指针是链表的”瑞士军刀”：找中点、判环、找交点、找倒数第 k 个，都能用。
    3. 反转链表是基础组件——回文、K 个一组反转、加法链表都会用到。
    4. 注意空指针：fast.next.next 之前一定判断 fast.next，否则崩。
