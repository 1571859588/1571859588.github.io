    阶段五_大模型推理部署技术栈

    阶段五：大模型推理部署技术栈

    来源：《完整 AI-Infra 转行落地教程》阶段五 | 周期：3 周 笔记类型：学习笔记 + 实操记录 关联：求职核心，企业日常工作内容。这是 AI-Infra 的业务层，企业 90% 的工作就是部署、优化推理服务，4 个框架必须全部实操。

    一、阶段定位与目标

    1.1 为什么这是求职核心

    AI-Infra 的日常工作就是把大模型部署成可调用的 API 服务。这一阶段学的是企业每天在用的技术，面试时项目深挖几乎全围绕这里。

    1.2 阶段验收标准

    单机使用 vLLM 稳定部署 14B 量化大模型，实现 API 调用、RAG 问答。

    二、4 个主流推理框架（按优先级排序）

    2.1 框架对比总览

    框架定位适合场景优先级
    Ollama轻量化小模型快速部署入门首选（Qwen、Llama3）1（先学）
    TGIHuggingFace 官方推理框架HF 生态深度用户3
    vLLM工业界第一主流生产部署、高并发2（重中之重）
    Ray分布式多卡并行超大模型部署4（进阶）

    2.2 Ollama：轻量化入门首选

    特点：

    • 安装极简（一行命令）
    • 自带模型管理（ollama pull / ollama run）
    • 适合 7B 以下小模型本地测试
    • 不适合高并发生产场景

    安装与使用：

    # 安装
curl -fsSL https://ollama.com/install.sh | sh

# 拉取模型
ollama pull qwen:7b

# 命令行对话
ollama run qwen:7b
>>> 你好
你好！有什么可以帮助你的？

# 启动 API 服务（默认 11434 端口）
ollama serve

# 调用 API
curl http://localhost:11434/api/generate -d '{
  "model": "qwen:7b",
  "prompt": "你好",
  "stream": false
}'

    OpenAI 兼容 API：

    curl http://localhost:11434/v1/chat/completions -d '{
  "model": "qwen:7b",
  "messages": [{"role": "user", "content": "你好"}]
}'

    2.3 vLLM（重中之重，面试必考）

    为什么是工业界第一主流：

    • PagedAttention 技术：降低显存碎片，提升显存利用率
    • 高并发吞吐：比 TGI 高 2-4 倍吞吐量
    • OpenAI 兼容 API：开箱即用，业务系统无缝对接
    • 支持量化模型：AWQ / GPTQ 原生支持

    PagedAttention 原理（面试必考）

    传统推理的问题：

    KV Cache（模型推理时的键值缓存）按连续内存分配
不同请求长度不同 → 内存碎片严重 → 显存利用率低（可能只有 60%）

    PagedAttention 的解法：

    借鉴操作系统的虚拟内存分页机制
把 KV Cache 分成固定大小的 block（如 16 token 一块）
按需分配，不连续存储 → 显存利用率可达 90%+

    效果对比：

    维度传统推理vLLM (PagedAttention)
    显存利用率~60%~90%
    并发吞吐基准2-4 倍
    KV Cache 碎片严重几乎消除

    vLLM 部署 Qwen-14B AWQ

    # 安装
pip install vllm

# 启动服务（OpenAI 兼容 API）
vllm serve /models/Qwen-14B-AWQ \
  --quantization awq \
  --dtype float16 \
  --max-model-len 4096 \
  --port 8000 \
  --host 0.0.0.0

# 或用 Python 启动
python -m vllm.entrypoints.openai.api_server \
  --model /models/Qwen-14B-AWQ \
  --quantization awq \
  --port 8000

    调用 vLLM API

    # OpenAI 兼容格式
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "/models/Qwen-14B-AWQ",
    "messages": [{"role": "user", "content": "你好"}],
    "max_tokens": 100
  }'

# 流式输出
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "/models/Qwen-14B-AWQ",
    "messages": [{"role": "user", "content": "讲个故事"}],
    "stream": true
  }'

    并发压测

    # 用 vLLM 自带的 benchmark
python -m vllm.entrypoints.openai.api_server \
  --model /models/Qwen-14B-AWQ \

# 压测
benchmark_serving.py \
  --backend vllm \

    2.4 TGI（HuggingFace 官方）

    特点：

    • HuggingFace 官方推理框架
    • 和 HF 生态深度集成
    • 支持 Flash Attention、Continuous Batching
    • Docker 部署为主
    # Docker 部署
docker run --gpus all -p 8080:80 \
  -v /data/models:/models \
  ghcr.io/huggingface/text-generation-inference:latest \
  --model-id /models/Qwen-14B \
  --num-shard 1 \
  --quantize awq

    2.5 Ray（分布式多卡）

    特点：

    • 分布式多卡并行推理
    • 用于超大模型（70B+）部署
    • 支持自动缩放
    import ray
from vllm import LLM

ray.init()

llm = LLM(
    model="/models/Qwen-72B",
    tensor_parallel_size=4,         # 4 卡张量并行
    pipeline_parallel_size=2,       # 2 级流水线
)

    三、配套技术

    3.1 OpenAI 兼容 API

    为什么重要：把模型封装成标准接口，对接业务系统、前端页面。几乎所有推理框架都支持 OpenAI API 格式。

    标准格式：

    # Python 调用（任何 OpenAI 兼容服务都行）
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8000/v1",    # vLLM 地址
    api_key="dummy"                         # vLLM 默认不校验
)

response = client.chat.completions.create(
    model="/models/Qwen-14B-AWQ",
    messages=[{"role": "user", "content": "你好"}]
)
print(response.choices[0].message.content)

    3.2 Milvus / FAISS 向量数据库（RAG 标配）

    作用：搭建 RAG 知识库，实现文档问答（几乎所有私有化 AI 项目标配）。

    RAG 架构：

    用户提问
   ↓
1. Embedding 模型把问题转成向量
   ↓
2. 在 Milvus/FAISS 里搜索相似文档片段
   ↓
3. 把问题和检索到的文档片段一起发给大模型
   ↓
4. 大模型基于文档内容生成回答

    Milvus 部署：

    # Docker 部署
docker run -d --name milvus \
  -p 19530:19530 \
  -v /data/milvus:/var/lib/milvus \
  milvusdb/milvus:latest

    Python 使用：

    from pymilvus import MilvusClient

client = MilvusClient(uri="http://localhost:19530")

# 创建集合
client.create_collection(
    collection_name="docs",
    dimension=1536                    # embedding 维度
)

# 插入文档向量
client.insert(
    collection_name="docs",
    data=[
        {"id": 1, "vector": [0.1, ...], "text": "文档内容1"},
        {"id": 2, "vector": [0.2, ...], "text": "文档内容2"},
    ]
)

# 搜索相似文档
results = client.search(
    collection_name="docs",
    data=[query_vector],
    limit=3,
    output_fields=["text"]
)

    4.3 任务 3：搭建 RAG 文档问答系统

    from openai import OpenAI
from pymilvus import MilvusClient

# 1. 初始化
llm_client = OpenAI(base_url="http://localhost:8000/v1", api_key="dummy")
milvus = MilvusClient(uri="http://localhost:19530")

# 2. 用户提问
question = "公司的请假政策是什么？"

# 3. 用 embedding 模型把问题转成向量
question_vector = get_embedding(question)    # 调 embedding API

# 4. 在 Milvus 搜索相似文档
results = milvus.search(
    collection_name="docs",
    data=[question_vector],
    limit=3,
    output_fields=["text"]
)

# 5. 拼接上下文
context = "\n".join([r["entity"]["text"] for r in results[0]])

# 6. 发给大模型生成回答
response = llm_client.chat.completions.create(
    model="/models/Qwen-14B-AWQ",
    messages=[
        {"role": "system", "content": f"基于以下文档回答问题：\n{context}"},
        {"role": "user", "content": question}
    ]
)
print(response.choices[0].message.content)

    五、推理优化手段（面试深挖）

    优化手段原理效果
    量化（AWQ/GPTQ）4-bit 压缩权重显存降 60%+，速度提升
    PagedAttentionKV Cache 分页管理显存利用率 60%→90%
    Continuous Batching动态拼批，不等齐吞吐量 2-4 倍
    Flash Attention优化 attention 计算速度提升 20-30%
    Tensor Parallel多卡切分模型层大模型单卡装不下时用
    KV Cache 量化压缩 KV Cache进一步省显存
    Speculative Decoding小模型先猜+大模型验证推理速度提升 1.5-2 倍
